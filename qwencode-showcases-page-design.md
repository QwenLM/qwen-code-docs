# Qwen Code Showcase 页面设计方案

## 设计目标

让**任何人**进入 Showcase 页面后，都能在最短时间内：

1. 知道 Qwen Code 是什么、能做什么（30 秒内）
2. 找到与自己相关的使用场景（1 分钟内）
3. 跟着视频上手操作（3-5 分钟内）

**核心原则**：按「我想做什么」组织内容，而不是按「我是谁」分类。

## 信息架构：三层递进

```
 ┌─────────────────────────────────────────────┐
 │  🎬 Quick Start（30秒了解 Qwen Code）         │  ← Hero 视频
 ├─────────────────────────────────────────────┤
 │  ⚡ 功能演示                                  │  ← 按功能特性分
 │  Chat / Agent / MCP / Insight / Cowork ...  │
 ├─────────────────────────────────────────────┤
 │  🎯 场景实战                                  │  ← 按「我想做什么」分
 │  建网站 / 数据分析 / 写文档 / 自动化办公 /       │
 │  代码重构 / 调试排错 / 学编程 / ...             │
└─────────────────────────────────────────────┘
```

## 第一层：Quick Start

### 定位

页面最顶部的 **Hero 区域**，一个 30-60 秒的视频，快速传递核心价值。

### 内容

- 展示完整的「安装 → 对话 → 完成任务」流程
- 不深入任何单一功能，只展示 Qwen Code 的「魔法时刻」
- 配一句 slogan，如：「你的 AI 编程伙伴，从想法到实现只需一句话」

### 设计要点

- 视频自动播放（静音），鼠标悬停显示播放控件
- 视频下方放「立即安装」CTA 按钮
- 移动端适配：视频比例 16:9，保证小屏可看清

## 第二层：功能演示

### 定位

展示 Qwen Code 的**核心功能模块**，帮助用户了解「这个工具有哪些能力」。对应每次版本上新的功能操作演示，是入门基础。

### 内容规划

| 功能模块 | 视频时长 | 核心展示内容 |
|---------|---------|------------|
| **Chat** | 1-2 min | 自然语言对话编程，问答式解决问题 |
| **Inline Edit** | 1-2 min | 选中代码 → 用自然语言描述修改意图 → 即时生效 |
| **Agent Mode** | 2-3 min | 给一个任务，AI 自主规划、执行、验证 |
| **MCP** | 1-2 min | 连接外部工具（GitHub、数据库等） |
| **Insight** | 1-2 min | 查看个人 AI 使用报告，优化协作效率 |
| **Cowork** | 1-2 min | 多人协作场景 |
| **Rules / QWEN.md** | 1 min | 个性化配置，让 AI 更懂你的项目 |

### 卡片设计

每个功能一张卡片，包含：

- 📹 **视频缩略图**（点击播放）
- 🏷️ **功能名称**
- 📝 **一句话说明**（不超过 20 字）
- 🔗 **「查看详细功能说明」链接** → 跳转到对应的图文文档页

### 布局

- 桌面端：3 列网格
- 移动端：单列瀑布流

## 第三层：场景实战

### 定位

这是页面的**核心价值区**。用户在这里找到「和我一样的人在用 Qwen Code 做什么」。

### 为什么不按人群分，而按任务分？

| 按人群分的问题 | 按任务分的优势 |
|--------------|--------------|
| 身份边界模糊（会写 SQL 的产品经理算谁？） | 用户直觉匹配：「我想做 X」→ 找到 X |
| 用户自我设限（非程序员跳过「程序员」分类） | 一个场景自然覆盖多个人群 |
| 新增视频分类纠结 | 分类标准清晰：「这个视频教用户做什么？」 |

### 难度标签系统

在场景区域顶部加一排**可点击的筛选标签**，替代人群分类：

```
全部 | 🟢 零基础友好 | 🔵 需要编程基础 | 🟣 进阶技巧
```

- **🟢 零基础友好**：不需要任何编程经验，跟着视频就能做
- **🔵 需要编程基础**：需要了解基本的编程概念和工作流
- **🟣 进阶技巧**：面向有经验的开发者，展示高级用法

### 场景内容规划

| 场景 | 难度 | 视频内容 | 覆盖人群 |
|------|------|---------|---------|
| 🌐 **从零搭建一个网站** | 🟢 | 用 Agent 模式从需求描述到本地预览 | 学生、非技术人员、独立开发者 |
| 📊 **数据分析与可视化** | 🟢 | 用 Qwen Code 处理 Excel/CSV，生成图表 | 办公族、学生、数据分析师 |
| 📝 **自动生成文档** | 🟢 | 生成 README、API 文档、技术方案 | 所有人 |
| 🔄 **自动化日常工作** | 🟢 | 批量处理文件、写脚本替代重复操作 | 办公族、运营 |
| 📚 **用 AI 学编程** | 🟢 | AI 辅助学习新语言/框架，边学边练 | 学生、转行者 |
| 🐛 **调试和修复 Bug** | 🔵 | 定位问题、理解报错、修复代码 | 程序员 |
| 🏗️ **代码重构与优化** | 🔵 | 大规模重构、性能优化、代码质量提升 | 程序员 |
| 🎨 **设计稿转代码** | 🔵 | 从设计稿/截图生成前端组件 | 设计师、前端开发 |
| 🧪 **编写测试用例** | 🔵 | 自动生成单元测试、集成测试 | 程序员 |
| ⚙️ **MCP 工具集成** | 🟣 | 连接 GitHub/数据库/API，打造自动化工作流 | 高级开发者 |
| 🤖 **自定义 AI 行为** | 🟣 | 配置 QWEN.md、Rules，让 AI 适配团队规范 | 技术 Leader |

> 以上场景为初始规划，可根据实际视频产出和用户反馈持续扩展。

### 卡片设计

每个场景一张卡片，包含：

- 📹 **视频缩略图**（点击播放）
- 🏷️ **场景名称 + 难度标签**（🟢/🔵/🟣）
- 📝 **简短描述**（1-2 句话，说明这个场景解决什么问题）
- 🔗 **「查看详细图文教程」链接** → 跳转到对应的博客/文档页

### 布局

- 桌面端：3 列网格，标签筛选栏固定在区域顶部
- 移动端：标签筛选栏横向滚动，卡片单列排列

## 内容生产 SOP

### 功能演示视频

1. **触发时机**：每次版本发布新功能时同步产出
2. **视频规格**：1-2 分钟，1080p，无旁白（字幕 + 操作录屏）
3. **配套产出**：一篇图文详细说明（发布在 blog/docs）
4. **上线流程**：视频上传 → 更新 showcase 页面 → 图文发布 → 添加跳转链接

### 场景实战视频

1. **触发时机**：按内容规划排期产出，或根据用户反馈新增
2. **视频规格**：3-5 分钟，1080p，可配旁白讲解
3. **配套产出**：一篇完整的图文教程（含步骤截图、代码示例）
4. **上线流程**：同上
