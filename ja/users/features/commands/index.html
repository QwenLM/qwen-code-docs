ユーザーガイド機能コマンド

コマンド

このドキュメントでは、Qwen Code がサポートするすべてのコマンドについて詳しく説明します。これにより、セッションの管理、インターフェースのカスタマイズ、および動作の制御を効率的に行えるようになります。

Qwen Code のコマンドは特定の接頭辞で起動され、以下の 3 つのカテゴリに分類されます。

接頭辞の種類機能の説明主な使用例
スラッシュコマンド (/)Qwen Code 自体のメタレベル制御セッションの管理、設定の変更、ヘルプの表示など
アットコマンド (@)ローカルファイルの内容を会話に素早く挿入AI に指定したファイルやディレクトリ内のコードを解析させるなど
エクスクラメーションコマンド (!)システムシェルとの直接的な対話git status、ls などのシステムコマンドを実行するなど

1. スラッシュコマンド (/)

スラッシュコマンドは、Qwen Code セッション、インターフェース、および基本的な動作を管理するために使用されます。

1.1 セッションおよびプロジェクト管理

これらのコマンドは、作業の進捗状況を保存・復元・要約する際に役立ちます。

コマンド説明使用例
/init現在のディレクトリを分析し、初期コンテキストファイルを作成します/init
/summary会話履歴に基づいてプロジェクトの要約を生成します/summary
/compressチャット履歴を要約に置き換えてトークン使用量を節約します/compress
/resume前回の会話セッションを再開します/resume
/restoreツール実行前のファイル状態に復元します/restore（一覧表示）または /restore <ID>

1.2 インターフェースとワークスペースの制御

インターフェースの外観および作業環境を調整するためのコマンド。

コマンド説明使用例
/clearターミナル画面の内容をクリアします/clear（ショートカット: Ctrl+L）
/themeQwen Code のビジュアルテーマを変更します/theme
/vim入力エリアの Vim 編集モードをオン/オフに切り替えます/vim
/directoryマルチディレクトリ対応ワークスペースを管理します/dir add ./src,./tests
/editorサポートされているエディタを選択するダイアログを開きます/editor

1.3 言語設定

インターフェースおよび出力言語を制御するためのコマンドです。

コマンド説明使用例
/language言語設定の表示または変更/language
→ ui [言語]UI インターフェースの言語を設定/language ui zh-CN
→ output [言語]LLM の出力言語を設定/language output Chinese
  • 利用可能な組み込み UI 言語: zh-CN（簡体字中国語）、en-US（英語）、ru-RU（ロシア語）、de-DE（ドイツ語）
  • 出力言語の例: Chinese（中国語）、English（英語）、Japanese（日本語）など

1.4 ツールおよびモデルの管理

AI ツールおよびモデルを管理するためのコマンド。

コマンド説明使用例
/mcp設定済みの MCP サーバーおよびツールの一覧表示/mcp, /mcp desc
/tools現在利用可能なツールの一覧表示/tools, /tools desc
/skills利用可能なスキルの一覧表示および実行/skills, /skills <名前>
/approval-modeツール使用時の承認モードを変更/approval-mode <モード (auto-edit)> --project
→plan分析のみ実行（実行は行わない）セキュアなレビュー
→default編集操作には承認を要求日常的な利用
→auto-edit編集操作を自動的に承認信頼された環境
→yoloすべての操作を自動的に承認クイックプロトタイピング
/model現在のセッションで使用するモデルを切り替え/model
/extensions現在のセッションで有効なすべての拡張機能を一覧表示/extensions
/memoryAI の指示コンテキストを管理/memory add 重要な情報

1.5 情報、設定、およびヘルプ

利用可能な情報の取得およびシステム設定を行うためのコマンドです。

コマンド説明使用例
/help利用可能なコマンドに関するヘルプ情報を表示します/help または /?
/aboutバージョン情報を表示します/about
/stats現在のセッションに関する詳細な統計情報を表示します/stats
/settings設定エディターを開きます/settings
/auth認証方法を変更します/auth
/bugQwen Code に関する問題を報告します/bug ボタンクリックが応答しない
/copy最後の出力内容をクリップボードにコピーします/copy
/quitQwen Code を即座に終了します/quit または /exit

1.6 よく使うショートカット

ショートカット機能備考
Ctrl/cmd+L画面をクリア/clear と同等
Ctrl/cmd+Tツールの説明を切り替えMCP ツール管理
Ctrl/cmd+C×2終了確認安全な終了メカニズム
Ctrl/cmd+Z入力を元に戻すテキスト編集
Ctrl/cmd+Shift+Z入力をやり直すテキスト編集

2. @ コマンド（ファイルの挿入）

@ コマンドは、ローカルのファイルまたはディレクトリの内容を会話に素早く追加するためのものです。

コマンド形式説明例
@<ファイルパス>指定されたファイルの内容を挿入します@src/main.py このコードを説明してください
@<ディレクトリパス>ディレクトリ内のすべてのテキストファイルを再帰的に読み込みます@docs/ このドキュメントの内容を要約してください
単独の @@ 記号自体について議論する場合に使用します@ プログラミングでこの記号は何に使われますか？

注: パスにスペースが含まれる場合は、バックスラッシュでエスケープする必要があります（例：@My\ Documents/file.txt）。

3. 感嘆符コマンド (!) — シェルコマンドの実行

感嘆符コマンドを使用すると、Qwen Code 内で直接システムコマンドを実行できます。

コマンド形式説明例
!<シェルコマンド>サブシェル内でコマンドを実行!ls -la, !git status
単独の !シェルモードに切り替え。以降の入力はすべてシェルコマンドとして直接実行されます!(Enter) → コマンド入力 → !(終了)

環境変数: ! 経由で実行されたコマンドには、QWEN_CODE=1 という環境変数が設定されます。

4. カスタムコマンド

作業効率の向上と一貫性の確保のため、頻繁に使用するプロンプトをショートカットコマンドとして保存します。

Note

カスタムコマンドは、オプションの YAML フロントマターを含む Markdown 形式で記述するようになりました。TOML 形式は非推奨となりましたが、下位互換性のため引き続きサポートされています。TOML ファイルが検出された場合、自動移行のプロンプトが表示されます。

クイック概要

機能説明利点優先度適用されるシナリオ
名前空間サブディレクトリによりコロン区切りのコマンドを作成コマンドの整理がしやすくなる
グローバルコマンド~/.qwen/commands/すべてのプロジェクトで利用可能低個人で頻繁に使うコマンド、プロジェクト横断的な利用
プロジェクトコマンド<プロジェクトのルートディレクトリ>/.qwen/commands/プロジェクト固有、バージョン管理可能高チーム間での共有、プロジェクト固有のコマンド

優先順位ルール：プロジェクトコマンド > ユーザーコマンド（同名のコマンドが存在する場合、プロジェクトコマンドが使用される）

コマンド名付け規則

ファイルパスからコマンド名への対応表

ファイルの場所生成されるコマンド例（呼び出し方）
~/.qwen/commands/test.md/test/test パラメーター
<project>/.qwen/commands/git/commit.md/git:commit/git:commit メッセージ

命名規則：パス区切り文字（/ または \）はコロン（:）に変換されます。

Markdown ファイル形式仕様（推奨）

カスタムコマンドでは、任意の YAML フロントマターを含む Markdown ファイルを使用します。

---
description: オプションの説明（/help で表示されます）
---
 
ここにプロンプトの内容を記述します。
パラメーター挿入には `{{args}}` を使用します。
フィールド必須説明例
descriptionオプションコマンドの説明（/help で表示されます）description: コード解析ツール
プロンプト本文必須モデルに送信されるプロンプトの内容フロントマター以降の任意の Markdown コンテンツ

TOML ファイル形式（非推奨）

Warning

非推奨： TOML 形式は現時点ではサポートされていますが、今後のバージョンで削除されます。Markdown 形式への移行をお願いします。

フィールド必須説明例
prompt必須モデルに送信されるプロンプトの内容prompt = "以下のコードを解析してください: {{args}}"
description任意コマンドの説明（/help で表示されます）description = "コード解析ツール"

パラメーター処理メカニズム

処理方法構文適用されるシナリオセキュリティ機能
コンテキスト認識型インジェクション{{args}}パラメーターを正確に制御する必要がある場合自動シェルエスケープ
既定のパラメーター処理特殊なマークなし単純なコマンド、パラメーター追加そのまま追加
シェルコマンドインジェクション!{command}動的なコンテンツが必要な場合実行前に確認が必要

1. コンテキストを意識した挿入 ({{args}})

シナリオTOML 設定呼び出し方法実際の効果
生の挿入prompt = "修正: {{args}}"/fix "ボタンの問題"修正: "ボタンの問題"
シェルコマンド内prompt = "検索: !{grep {{args}} .}"/search "hello"grep "hello" . を実行

2. デフォルトのパラメーター処理

入力状況処理方法例
パラメーターありプロンプト末尾に追加（2 行の空行で区切り）/cmd パラメーター → 元のプロンプト + パラメーター
パラメーターなしプロンプトをそのまま送信/cmd → 元のプロンプト

🚀 動的コンテンツの挿入

挿入タイプ構文処理順序目的
ファイルの内容@{ファイルパス}最初に処理静的な参照ファイルを挿入
シェルコマンド!{コマンド}中間に処理動的な実行結果を挿入
パラメーター置換{{args}}最後に処理ユーザーが指定したパラメーターを挿入

3. シェルコマンドの実行（!{...}）

操作ユーザーとのインタラクション
1. コマンドとパラメーターの解析—
2. 自動シェルエスケープ—
3. 確認ダイアログの表示✅ ユーザーによる確認
4. コマンドの実行—
5. 出力をプロンプトに挿入—

例：Git コミットメッセージの生成

---
description: ステージング済みの変更に基づいてコミットメッセージを生成する
---
 
以下の差分に基づき、コミットメッセージを生成してください：
 
```diff
!{git diff --staged}
```

4. ファイル内容の挿入（@{...}）

ファイル種別サポート状況処理方法
テキストファイル✅ 完全サポート内容を直接挿入
画像／PDF✅ マルチモーダル対応エンコードして挿入
バイナリファイル⚠️ 制限付きサポート無視されるか、切り捨てられる場合あり
ディレクトリ✅ 再帰的挿入.gitignore ルールに従う

例：コードレビュー用コマンド

---
description: ベストプラクティスに基づくコードレビュー
---
 
{{args}} をレビューし、以下の基準を参照してください：
 
@{docs/code-standards.md}

「純粋関数リファクタリング」コマンド作成手順表

操作コマンド／コード
1. ディレクトリ構造の作成mkdir -p ~/.qwen/commands/refactor
2. コマンドファイルの作成touch ~/.qwen/commands/refactor/pure.md
3. コマンド内容の編集以下の完全なコードを参照してください。
4. コマンドのテスト@file.js → /refactor:pure
---
description: コードを純粋関数にリファクタリングする
---
 
現在のコンテキスト内のコードを分析し、純粋関数へとリファクタリングしてください。
要件：
 
1. リファクタリング後のコードを提示すること
2. 主な変更点および純粋関数の特性がどのように実装されたかを説明すること
3. 関数の振る舞いは変更しないこと
