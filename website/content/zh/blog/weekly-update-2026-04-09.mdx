本周发布 **v0.14.0**、**v0.14.2** 两个功能版本和 **v0.13.2**、**v0.14.1** 两个 bugfix 版本。

v0.14.0 是一个重量级更新：

- **Channels 系统让 Qwen Code 不再局限于终端**，可以通过 Telegram、微信、钉钉等平台远程操作；
- Cron 定时任务让 AI 按时间表自动执行重复工作；
- Qwen 3.6 Plus 模型正式上线。v0.14.1 带来了跟进建议、详略模式切换等体验优化。
- v0.14.2 新增 `/plan` 规划模式命令、thinking block 跨轮保留、自适应输出 Token 升级（默认 8K + 自动升级到 64K），以及内置 bugfix 工作流和调试技能。

## ✨ 新增特性

### Channels：通过钉钉、Telegram、微信远程操作 Qwen Code

Qwen Code 不再只能在终端里用了。Channels 系统支持插件化接入多个平台——目前已支持 Telegram、微信和钉钉。你可以在手机上给 Bot 发一条消息，让它帮你在服务器上执行任务。

**你能用它做什么：**

- 出门在外，用 Telegram 给 Qwen Code 发消息，让它帮你跑个脚本、查个日志
- 在钉钉群里 @机器人，让它帮团队处理代码相关的任务
- 微信里随手发一句话，远程操作你的开发环境

详见 PR [#2628](https://github.com/QwenLM/qwen-code/pull/2628)

![](https://gw.alicdn.com/imgextra/i4/O1CN01fWI9Ow1NUwXa0kIJR_!!6000000001574-0-tps-4032-2565.jpg)

### Cron 定时任务：让 AI 按时间表自动干活

新增 Cron 工具，可以在当前会话中设置定时循环任务。AI 会按你设定的时间表自动执行，不需要你盯着。

**你能用它做什么：**

- 每隔 30 分钟自动检查一次测试是否通过
- 每天早上自动拉取最新代码并跑一遍构建
- 定时监控日志文件，发现异常自动通知你

详见 PR [#2731](https://github.com/QwenLM/qwen-code/pull/2731)

<video class="mt-5" poster="https://img.alicdn.com/imgextra/i2/6000000002950/O1CN018Sepbk1Xf9Wjb2uR5_!!6000000002950-0-tbvideo.jpg" src="https://cloud.video.taobao.com/vod/VN0xFz-zPV1vvFzIuT6S3HUTFUX36x_JtySMTzAJYpE.mp4" controls></video>

**如何开启使用**？

在 `~/.qwen/settings.json` 中添加：

```json
{
	"experimental":{
		"cron": true
	}
}
```

### Qwen 3.6 Plus 模型上线

Qwen 3.6 Plus 正式上线 Qwen Code，免费使用。在编程场景的表现对标 GPT-5 和 Claude 3.7 Sonnet，尤其在中文理解、长上下文处理方面有明显优势。阿里云百炼 Coding Plan 也已上线。

**你能用它做什么：**

- 直接在 Qwen Code 中使用最新的 Qwen 3.6 Plus 模型
- 免费 1000 次/日调用，100 万 token 上下文
- 中文编程场景体验更好

详见 PR [#2820](https://github.com/QwenLM/qwen-code/pull/2820)

![](https://gw.alicdn.com/imgextra/i1/O1CN01uhK7oJ1tMlAV50yRo_!!6000000005888-2-tps-1694-950.png)

### 跟进建议：AI 做完事会告诉你下一步可以做什么

AI 完成任务后，会自动给出 2-3 个跟进建议。不用你自己想 " 接下来该干嘛 "，直接点选就能继续。

**你能用它做什么：**

- 写完一个组件后，AI 建议 " 要不要加单元测试？"——直接点就开始
- 修完一个 Bug 后，AI 建议 " 要不要检查类似的地方？"——省得你自己想
- 新手不知道下一步做什么时，跟进建议就是你的导航

详见 PR [#2525](https://github.com/QwenLM/qwen-code/pull/2525)

![](https://gw.alicdn.com/imgextra/i3/O1CN01bduOSo1HBoIgKUkbq_!!6000000000720-2-tps-2208-1202.png)

**如何开启使用**？

在 `~/.qwen/settings.json` 中添加：

```json
{
	"ui":{
		"enableFollowupSuggestions": true
	}
}
```

为了更快的建议响应速度，可以通过 `/model --fast` 指定一个轻量模型专门用于后台建议。

![](https://gw.alicdn.com/imgextra/i3/O1CN01RGAS8t22o5LA9v4OK_!!6000000007166-2-tps-1694-948.png)

### 子代理跨模型选择：不同任务用不同模型

子代理现在可以选择和主代理不同的模型。主任务用强模型保证质量，子任务用轻量模型加速——灵活搭配，省 Token 不降质。

**你能用它做什么：**

- 主任务用 Qwen 3.6 Plus，子任务用轻量模型，Token 消耗降低但关键任务质量不打折
- 不同类型的子任务自动分配最合适的模型

详见 PR [#2698](https://github.com/QwenLM/qwen-code/pull/2698)

<video class="mt-5" poster="https://img.alicdn.com/imgextra/i4/6000000000621/O1CN01uFmNW81GST5dMLGJA_!!6000000000621-0-tbvideo.jpg" src="https://cloud.video.taobao.com/vod/gWur3ruz89gz8-LdUper69IQkHXcQNSgFs1HgKpllNs.mp4" controls></video>

**如何开始使用**？

创建一个子代理 Skill 文件，指定不同的模型：

```markdown
---
name: test-cross-provider
description: 测试跨 Provider 的 Agent
model: openai:gpt-4o
---

你是一个测试助手，请简单介绍一下你自己，包括你使用的模型名称。
```

然后在对话中告诉 Qwen Code 调用这个子代理即可。
### Ctrl+O 详略模式切换：一键切换输出详细程度

按 Ctrl+O 在 verbose（详细）和 compact（精简）模式之间切换。调试时看详细输出，日常使用看精简版，不用改配置。

**你能用它做什么：**

- 调试问题时切到 verbose 模式，看完整的执行过程
- 日常使用切到 compact 模式，界面更清爽
- 随时切换，不用重启或改配置文件

详见 PR [#2770](https://github.com/QwenLM/qwen-code/pull/2770)

<video class="mt-5" poster="https://img.alicdn.com/imgextra/i4/6000000002951/O1CN01BRtItT1XfbvIq2YyI_!!6000000002951-0-tbvideo.jpg" src="https://cloud.video.taobao.com/vod/7njhFZp_lcoGIThTivda3Q9rLXCdkf19MZo8GEZIm1I.mp4" controls></video>

### /plan 命令：先规划再动手

新增 `/plan` 命令，一键进入规划模式。AI 会先给出完整的执行方案，你确认后再开始执行。适合复杂任务——先看清全貌，再逐步推进。

**你能用它做什么：**

- 重构大型模块前，先让 AI 列出所有需要改的文件和步骤
- 多步骤任务先看方案，避免 AI 跑偏后推倒重来
- 团队协作时，先生成方案给同事确认，再执行

详见 PR [#2921](https://github.com/QwenLM/qwen-code/pull/2921)

![](https://gw.alicdn.com/imgextra/i4/O1CN01wm82SI1tw1fL4l7DE_!!6000000005965-2-tps-2840-1310.png)

### 自适应输出 Token：回复不再被截断

输出 Token 默认 8K，当 AI 发现回复被截断时自动升级到 64K 重试。你不用手动调参数，AI 自己判断需要多长的回复空间。

**你能用它做什么：**

- 生成长文件时不再被截断，一次输出完整内容
- 复杂代码生成不用分段拼接
- 无需手动配置 max_tokens，AI 自动适配

详见 PR [#2898](https://github.com/QwenLM/qwen-code/pull/2898)

### Thinking Block 跨轮保留

AI 的思考过程（thinking block）现在可以跨对话轮次保留，空闲时自动清理。多轮对话中 AI 能更好地延续之前的推理链路，回答更连贯。

详见 PR [#2897](https://github.com/QwenLM/qwen-code/pull/2897)
