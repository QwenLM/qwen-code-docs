本周我们发布了 v0.15.7 主要功能版本和 6 个迭代版本（v0.15.1–v0.15.6）。

DeepSeek V4 是本周 AI 社区最热的话题之一——从 HN 上 antirez 的本地推理引擎到各家竞速适配，我们第一时间完成了全面支持：1M 上下文窗口、384K 输出上限、reasoning effort "max" 等级，thinking blocks 兼容问题也已修复。

Agent 多任务是整个行业的共识方向（OpenAI Codex、Google AlphaEvolve 都在往这个方向走），我们把后台 Agent 系统从基础设施搭到了完整的任务面板——后台任务可以统一查看状态，随时取消；中断后自动暂停，也可以恢复继续。GitHub 刚发布了 Agent PR 审查指南，`qwen review` 同步新增了多个子命令和 agent，直接在终端里做代码审查。

另外，我们还围绕对话交互、效率提升继续完善：对话方向走偏时，双击 Esc 就能回到前面轮次重来；长任务跑完或需要确认权限时，终端和 VS Code 会主动提醒你；`/stats` 在配置价格后可以直接估算费用；模型切换方式也更简单了。

## ✨ 新增特性

### DeepSeek V4 深度支持

DeepSeek V4 发布后，Qwen Code 第一时间完成了全面适配。上下文窗口设为 1M，输出上限 384K——这意味着 Agent 可以一次性读取大型代码库并生成超长输出。同时支持 reasoning effort "max" 等级，让 DeepSeek 在复杂推理任务上全力以赴。多个 thinking blocks 兼容问题也已修复，确保 DeepSeek 的思维过程在各种使用场景下完整显示。

**你能用它做什么：**

- 用 DeepSeek V4 处理超大文件和整个代码仓库，1M 上下文不再担心截断
- 设置 reasoning effort "max"，让 DeepSeek 对复杂架构设计、长链推理全力思考
- 恢复会话、对话回退、上下文压缩后 DeepSeek 的思维链依然完整保留
- anthropic 兼容模式下 thinking blocks 正确注入，第三方部署（sglang/vllm）也能正常使用

### 后台任务统一查看、取消和恢复

之前后台 shell 命令只是被系统放到后台运行，用户很难知道它是否还在跑、输出在哪里、该怎么停止。现在 background agent 和后台 shell 都会进入统一的后台任务视图，可以查看状态、输出和详情。后台任务被中断后会自动暂停，用户可以恢复或取消。

**你能用它做什么：**

- 跑长任务不卡对话：`npm run dev`、测试、文件监听等命令可以后台运行，当前对话继续推进
- 随时查看任务状态并控制：通过 `/tasks` 或任务面板查看后台 shell、background agent 的运行状态和输出路径，也可以取消正在运行的任务
- 中断任务安全恢复：后台任务意外中断后不会丢失，可以恢复继续或放弃

详见 PR [#3642](https://github.com/QwenLM/qwen-code/pull/3642)、[#3739](https://github.com/QwenLM/qwen-code/pull/3739)

### 对话可以回退到前面重新开始

之前对话方向偏了，通常只能继续纠正或重新开一个会话。现在可以按两次 `Esc` 或运行 `/rewind`，选择前面的用户轮次，把对话历史回退到那一刻并从原问题继续改。

**你能用它做什么：**

- 撤回错误方向：AI 走偏后回到关键问题，换一个要求重新执行
- 保留上下文再试一次：不用新开会话，也不用复制粘贴前面的背景信息
- 更适合探索式开发：尝试多个实现方案时，可以回到分叉点重新推进

### `/review` 代码审查流程升级

`/review` 这次做了一轮完整升级：从 5 个 agent 扩展到 9 个 agent，并把原来分散在 prompt 里的审查步骤收进 6 个跨平台 CLI 子命令，输出也改成结构化 JSON。用户只需要输入 `/review <PR 链接或编号>`，AI 自动完成拉代码、加载项目规则、跑 lint、9 角色并行审查、自动去重、检查 CI 状态、发布内联评论全流程。

**你能用它做什么：**

- 一键审查 PR：输入 `/review https://github.com/xxx/pull/123`，自动完成从拉代码到写评论的全流程
- 9 角色同时审查：在正确性和安全性之外，新增“攻击者、深夜值班、维护者”等视角，不漏盲区
- CI 不通过不会 approve：自动检测 CI 状态和是否自 PR，不通过时自动降级为 comment
- 不确定的建议不在 PR 评论：低置信度问题只在终端提示，不在 PR 里留言
- 已有评论自动去重：识别已有 Qwen 评论，不重复提交相同建议

### 任务完成或需要确认时主动提醒你

之前终端里的提醒主要依赖不明显的 terminal bell，VS Code 插件里也缺少足够醒目的状态提示。现在在 iTerm2、Kitty、Ghostty 终端里，任务完成会自动弹出桌面通知；VS Code 里也会通过标签页圆点、通知气泡和声音提醒你。

**你能用它做什么：**

- 长任务不用一直盯着终端：可以先去处理其它事情，任务跑完系统会通知和提醒你
- 等待权限确认时及时发现：AI 需要你批准工具调用或回答问题时，更容易被看到
- 在 VS Code 多标签工作时不漏消息：切到其他文件后，聊天标签页会显示状态提示

### `/stats` 显示模型费用预估

现在 `/stats` 命令新增了费用预估功能。你只需要在 `settings.json` 里配置 `modelPricing`（给模型设好输入/输出每百万 Token 的价格），`/stats` 就会自动根据 Token 消耗计算出大概费用，不用自己去算。不配置的话和之前一样只显示 Token 数。

**你能用它做什么：**

- 配置常用模型价格：在 `settings.json` 里配置好价格，之后 `/stats` 自动算费用
- 对比不同模型开销：切换模型后看费用差异，找到性价比最高的方案
- 控制长期运行成本：自动化流程里定期检查费用，防止超支

详见 PR [#3780](https://github.com/QwenLM/qwen-code/pull/3780)

### `/model` 快速切换模型

之前切模型要先 `/model` 打开选择器再翻找，现在直接输入 `/model 模型名` 就切好了。

**你能用它做什么：**

- 不用翻选择器：输入模型名，如 `/model qwen3.6-plus` 直接切，省掉在选择器翻找的步骤
- 快速对比模型：`/model A` 问一次，再 `/model B` 重新问，多个模型对比效果
- 上游模型即时用：base URL 配好了但没在本地注册的模型也能直接切过去

## 📊 优化改进

- **OpenRouter 接入改为浏览器一键授权**：之前需要手动配置 API Key 和模型列表，现在走 `/auth` 浏览器授权流程，自动保存 Key 并拉取模型目录；新增 `/manage-models` 支持搜索、筛选和启用模型（[#3576](https://github.com/QwenLM/qwen-code/pull/3576)）
- **Todo 列表固定显示**：最新任务列表会固定在输入框上方，任务状态变化时同步更新，不用向上翻历史记录找进度（[#3507](https://github.com/QwenLM/qwen-code/pull/3507)、[#3647](https://github.com/QwenLM/qwen-code/pull/3647)）
- **文件读取更快来回不重复读**：新增文件读取缓存（FileReadCache），相同内容不再重复读取，多轮对话和工具调用响应更稳定（[#3717](https://github.com/QwenLM/qwen-code/pull/3717)）
- **网页搜索改为通过 MCP 接入**：内置 `web_search` provider 已迁移为 MCP-based 方案，你可以按需要配置 Bailian、Tavily、GLM WebSearch Prime 等搜索服务（[#3502](https://github.com/QwenLM/qwen-code/pull/3502)）
- **首次模型请求更快**：启动时提前预连接默认 API 端点，首个请求可减少一部分 TCP 和 TLS 建连耗时（[#3318](https://github.com/QwenLM/qwen-code/pull/3318)）
- **工具批量调用更容易读懂**：多个工具并行执行后会生成简短语义标签，不只显示机械的工具数量，方便快速理解 AI 这一批操作在做什么（[#3538](https://github.com/QwenLM/qwen-code/pull/3538)）
- **工具调用热路径更快**：运行时同步 I/O 减少，工具调用链路更轻，长任务和多工具调用时响应更稳定（[#3581](https://github.com/QwenLM/qwen-code/pull/3581)）
- **会话标题可手动重新生成**：自动标题能力增加 `/rename --auto`，标题不准确时可以让 fast model 重新生成（[#3540](https://github.com/QwenLM/qwen-code/pull/3540)）
- **前台 Subagent 也进入统一任务面板**：之前只有后台任务可见，现在前台 subagent 也在 /tasks 面板里统一管理（[#3768](https://github.com/QwenLM/qwen-code/pull/3768)）
- **Skill 加载更快、支持路径条件激活**：多个 skill 并行加载，启动更快；支持按目录条件自动激活，不用每次都手动指定（[#3604](https://github.com/QwenLM/qwen-code/pull/3604)）
- **底部状态栏显示 MCP 服务健康状态**：一眼看出 MCP server 是否在线，连接异常时更容易排查（[#3741](https://github.com/QwenLM/qwen-code/pull/3741)）
- **Shell 执行时间显示更清楚**：shell 状态同时展示已运行时间和超时信息，长命令执行时更容易判断进度（[#3512](https://github.com/QwenLM/qwen-code/pull/3512)）
- **长命令运行提示可转后台**：AI 检测到耗时命令时会主动建议转后台运行，避免卡住当前对话（[#3809](https://github.com/QwenLM/qwen-code/pull/3809)）
- **VS Code 支持 `/skills` 和 `/export`**：在 VS Code Companion 中可以更方便地打开 skills 选择器，也可以导出当前会话（[#2548](https://github.com/QwenLM/qwen-code/pull/2548)、[#2592](https://github.com/QwenLM/qwen-code/pull/2592)）
- **MCP 配置支持命令行传入**：SDK 和脚本化场景可以通过 CLI flag 传入 MCP server 配置，不用再手动编辑配置文件（[#1279](https://github.com/QwenLM/qwen-code/pull/1279)）
- **MCP 服务重复探测更智能**：合并重复的 MCP server 发现请求，减少启动时的网络开销（[#3818](https://github.com/QwenLM/qwen-code/pull/3818)）
- **Slash 命令显示参数提示**：命令补全后会用灰色提示展示参数格式，写自定义命令或 skill 时更容易告诉用户下一步该输入什么（[#3593](https://github.com/QwenLM/qwen-code/pull/3593)）
- **新增繁体中文界面语言**：现在可以通过 `/language ui zh-TW` 选择繁体中文界面（[#3569](https://github.com/QwenLM/qwen-code/pull/3569)）
- **VS Code Webview 复制更顺手**：聊天 Webview 增加原生右键复制操作，不用绕到其他方式复制内容（[#3477](https://github.com/QwenLM/qwen-code/pull/3477)）
- **ACP 模式下 MCP 工具调用更完整**：ACP Agent 支持 SSE/HTTP MCP server，并能并发执行工具调用，外部工具接入更稳（[#3574](https://github.com/QwenLM/qwen-code/pull/3574)、[#3463](https://github.com/QwenLM/qwen-code/pull/3463)）
