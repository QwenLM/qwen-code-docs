这篇文章用一个可复用的工作流，把 **Qwen Code** 接入 **Obsidian**：在笔记里直接打开 Terminal，完成安装与认证，再通过 Skills / MCP 把「润色写作、生成配图、抓取网页素材」这些能力接进你的知识库日常。

## 为什么是 Obsidian + Qwen Code？

Obsidian 的核心优势在于**本地 Markdown、双向链接和知识图谱**——你的笔记就是一堆 `.md` 文件，天然适合 Agent 读写。而 Qwen Code 是一个运行在终端里的 AI Agent，不挑编辑器，只要有 Terminal 就能工作。

把两者接在一起，你的知识库就不再只是"存放笔记的地方"，而是一个**可以被 AI 主动操作的工作空间**：Agent 能直接读取你的笔记、修改内容、生成配图、自动打标签，甚至帮你从外部网站抓取素材写进来。这是 VS Code / Cursor 这类代码编辑器难以替代的——因为 Obsidian 的插件生态、双链图谱和知识管理工作流本身就是为"写作与思考"设计的。

## 环境准备

开始之前，请确保本地已安装以下工具：

- **Obsidian**：[官网下载](https://obsidian.md/)
- **Node.js 18+**：Skills 和 MCP 的安装依赖 `npx`，[下载地址](https://nodejs.org/)
- 一个已有的 Obsidian Vault（笔记库），或新建一个空 Vault 用于测试

准备就绪后，跟着下面的步骤走就行。

## 安装 Obsidian Terminal 插件

下载 Obsidian 后，打开「设置」→「第三方插件」→「社区插件市场」，搜索「Terminal」并安装。

![](https://img.alicdn.com/imgextra/i4/O1CN01z4iGql1i9yKcmbRIr_!!6000000004371-2-tps-2160-782.png)

安装完成后，点击侧边栏的 Terminal 按钮，选择「整合式」集成。建议把 Terminal 拖拽到右侧顶部边栏，操作起来更顺手。

![](https://img.alicdn.com/imgextra/i4/O1CN0123qrd31pWbD1pwfLh_!!6000000005368-2-tps-3840-1214.png)

## 安装 Qwen Code

在 Terminal 中输入：

```bash
qwen
```

如果本地尚未安装 Qwen Code，可参考 [GitHub 安装说明](https://github.com/QwenLM/qwen-code)，或直接运行以下命令一键安装：

**macOS / Linux：**
```bash
bash -c "$(curl -fsSL https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen.sh)"
```

**Windows：**
```cmd
curl -fsSL -o %TEMP%\install-qwen.bat https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen.bat && %TEMP%\install-qwen.bat
```

## 认证 Qwen

安装成功后，可选择以下认证方式：

- **QwenOAuth**：每日 1000 次免费调用 `Qwen3.5-plus`
- **阿里云 Coding Plan**：购买 API Key，适合高频使用场景

![](https://img.alicdn.com/imgextra/i4/O1CN01TqSsrj1a35S3TsUjE_!!6000000003273-2-tps-1908-792.png)

认证完成后，即可开启 Qwen Code 知识管理之旅。

## 实用技巧

### 技巧一：引用文件 —— 让 Agent 直接操作你的笔记

找到需要编辑的文件，右键选择「复制路径」→「绝对路径」，然后在 Qwen 命令行里粘贴（⌘+V / Ctrl+V）：

![](https://img.alicdn.com/imgextra/i3/O1CN01XSMGVX1Whc29M5oYd_!!6000000002820-2-tps-3814-1226.png)

按下回车，Qwen 即可启动：

![](https://img.alicdn.com/imgextra/i3/O1CN01GVh2HP1P2fJUKyynT_!!6000000001783-2-tps-1954-1092.png)

引用文件后，你可以做的事情远不止"让它看一眼"。试试这些 prompt：

```
帮我把这篇会议纪要提炼成 5 条关键结论，并在末尾附上待办事项清单
```

```
把这篇英文论文笔记翻译成中文，保留所有 Markdown 格式和链接
```

```
阅读这三份周报，合并成一份本月总结，按项目维度归类
```

你也可以一次引用多个文件路径，Agent 会自动理解它们之间的关系。这在"跨笔记汇总"场景下尤其好用。

### 技巧二：修改输出语言与主题

如果需要模型以特定语言输出（如法语），可通过以下命令设置：

```bash
/language output 法语
```

![](https://img.alicdn.com/imgextra/i2/O1CN01qDNKSy1KfWqIn66U2_!!6000000001191-2-tps-3142-804.png)

同样，如果你希望 Terminal 界面使用深色主题，也可以直接这么告诉 Qwen Code：

```
请将 Obsidian 的 Terminal 主题改为深色
```

然后重启使其生效：

![](https://img.alicdn.com/imgextra/i4/O1CN01MHVw5E1tiHmKqiew0_!!6000000005935-2-tps-2536-906.png)

### 技巧三：添加写作润色 Skills

在 Qwen Code 中输入以下提示词：

```md
先检查 `find-skills` 是否存在；如果不存在，请使用 `npx skills add https://github.com/vercel-labs/skills --skill find-skills` 安装该 Skill。

然后帮我查找与「写作润色」相关的 Skills 并安装。
```

![](https://img.alicdn.com/imgextra/i4/O1CN01IuGbnF1fJBRtgHM9d_!!6000000003985-2-tps-3138-1508.png)

选择 `Option 1` 和 `Option 3`，Qwen Code 将自动安装写作相关 Skills：

![](https://img.alicdn.com/imgextra/i3/O1CN01ahEBnN1qaYF862TJz_!!6000000005512-2-tps-3150-980.png)

我简单试了下效果，还不错——Skill 帮我做了这些事情：

- 统一了中英文之间的空格（半角 / 全角混用一扫而空）
- 把过长的段落拆成更易读的短段
- 去掉了口语化的表达，换成更书面、更紧凑的措辞
- 在段落之间补上了过渡句，让全文读起来更连贯

![](https://img.alicdn.com/imgextra/i3/O1CN01ALc1Pu25EJFG8pv2i_!!6000000007494-2-tps-3086-1490.png)

如果你对润色风格有特定要求（比如"更口语化"或"更学术"），可以在 prompt 里明确指定，Skill 会按你的要求调整。

### 技巧四：添加生图 Skills

为文章配图时，可以使用 Qwen Image MCP 来实现：

> 下面依然是给 Qwen Code 的提示词：不需要自己手动折腾安装流程，让 Agent 去做更省心。

请先修改 `~/.qwen/settings.json` 文件，手动增加 `mcpServers` 字段，并通过 `qwen mcp list` 验证是否安装成功：

```json
"mcpServers": {
  "playwright": {
    "command": "npx",
    "args": [
      "@playwright/mcp@latest",
      "--user-data-dir",
      ".playwright-mcp-profile"
    ]
  },
  "qwenimage": {
    "url": "https://dashscope.aliyuncs.com/api/v1/mcps/QwenImage/sse",
    "headers": {
      "Authorization": "Bearer ${DASHSCOPE_API_KEY}"
    }
  }
}
```

![](https://img.alicdn.com/imgextra/i4/O1CN01WIZcuL1N4NZ1Fj7lx_!!6000000001516-2-tps-2506-1016.png)

几秒钟后，MCP 服务就配置完成了：

![](https://img.alicdn.com/imgextra/i1/O1CN01XWtwg81qvcSgTW8Z5_!!6000000005558-2-tps-1446-624.png)

开始创建第一个配图：

```
使用 qwen image mcp 帮我创建一个杂志风格的手绘插画并下载，表达内容：Qwen Code + Obsidian：让知识管理触手可及
```

![](https://img.alicdn.com/imgextra/i4/O1CN01YZmcSO1Oo0dhQ85CS_!!6000000001751-2-tps-3024-908.png)

### 技巧五：使用 Playwright MCP 获取需要授权的素材

写作时，素材往往存放在需要登录的内部文档产品里。这时可以让 Agent 操作浏览器，代替我们完成登录并抓取文档内容。

在上一步，我们已经配置了 Playwright MCP 服务，直接开始使用：

```
请使用 playwright mcp 帮我获取语雀的文档内容，并总结成一段 50 字的中心大意给到我：https://aliyuque.antfin.com/pomelo.lcw/hwbywq/wrt6wn7u6p34t17u
```

Agent 会自动打开浏览器、完成登录（如果你之前已在 `.playwright-mcp-profile` 中保存过登录态）、抓取页面正文，最后返回一段精炼的摘要。整个过程你只需要等几秒钟，不需要手动复制粘贴任何内容。

![](https://img.alicdn.com/imgextra/i3/O1CN01tod9Va1ZkJDFeeRhz_!!6000000003232-2-tps-1450-652.png)

> **提示**：第一次使用时，Playwright 会弹出浏览器窗口要求你手动登录一次。登录态会保存在 `.playwright-mcp-profile` 目录中，后续调用无需重复登录。

### 技巧六：让 Agent 帮你整理知识库

前面的技巧都聚焦在"单篇内容的生产"。但 Obsidian 用户真正关心的往往是**知识库层面的组织**——标签、双链、MOC（Map of Content）。这恰好是 Qwen Code 擅长的批量处理场景。

**自动打标签**：让 Agent 扫描指定目录下的所有笔记，根据内容生成标签建议并写入 frontmatter：

```
请扫描 /Notes/Reading 目录下的所有 .md 文件，根据每篇笔记的内容为其在 frontmatter 中添加 tags 字段（如果已有则合并），标签用中文，每篇不超过 5 个。
```

**生成 MOC 索引**：当某个主题下的笔记积累到一定数量时，手动维护索引页很累。交给 Agent：

```
请阅读 /Notes/Projects 目录下的所有笔记，按项目名称分组，生成一份 MOC 索引页（Markdown 格式，使用 [[双链]] 语法），保存为 /Notes/Projects/MOC.md。
```

**补全双向链接**：笔记之间明明有关联，但写的时候忘了加链接？

```
请扫描 /Notes 目录下的所有笔记，找出内容中提到了其他笔记标题但没有用 [[]] 链接的地方，帮我自动补上双链。修改前先列出变更清单让我确认。
```

这类"批量读 → 分析 → 批量写"的任务，正是 Agent 比人类高效得多的地方。你只需要在 prompt 里描述规则，剩下的让 Qwen Code 自动遍历、自动修改。

## 小结

到这里，Qwen Code 已经可以在 Obsidian 里顺滑跑起来了。回顾一下我们搭建的完整工作流：

1. **写作阶段**：用 Skills 做润色 / 扩写 / 翻译，一条 prompt 搞定
2. **配图阶段**：用 Qwen Image MCP 按描述生成插画，不用再找素材网站
3. **素材收集**：用 Playwright MCP 操作浏览器，抓取需要登录的内部文档
4. **知识整理**：让 Agent 批量打标签、补双链、生成 MOC 索引

最后再用 `essay-polish` 这类写作润色 Skill 做一次通篇打磨，就能把「资料收集 → 内容成稿 → 知识归档」的链路完整收束到同一个 Vault 里。

Obsidian 负责组织，Qwen Code 负责执行——这大概是目前最轻量的「AI 知识管理」方案了。希望对你有帮助。
