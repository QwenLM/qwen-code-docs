本周发布 **v0.12.4** 功能版本和 2 个 bugfix 版本。

v0.12.0 上线后收到了不少用户反馈，这周我们重点处理了影响体验的问题：Windows 中文输出乱码、交互式 Shell 输出丢失、API 重试报错。新功能方面，Token 限制翻倍到 16K，AI 能读取更多上下文了；同时支持了 JetBrains 和 Zed 编辑器。

## ✨ 新增特性

### Zed 和 JetBrains 编辑器集成

除了 VS Code，现在 Qwen Code 也支持 Zed 和 JetBrains 系列编辑器（IntelliJ IDEA、PyCharm、WebStorm 等）了。详见 PR [#2372](https://github.com/QwenLM/qwen-code/pull/2372) 。

**你能用它做什么：**

- 在 JetBrains IDE 里直接用 Qwen Code，不用切换窗口
- Zed 编辑器用户也能享受 AI 编程助手
- 不同编辑器统一体验，换工具不用重新适应

文档查看：

- [JetBrains IDE \| Qwen Code Docs](https://qwenlm.github.io/qwen-code-docs/zh/users/integration-jetbrains/)
- [Zed 编辑器 \| Qwen Code Docs](https://qwenlm.github.io/qwen-code-docs/zh/users/integration-zed/)

![](https://gw.alicdn.com/imgextra/i3/O1CN01rTE0X61tikAwLN7Zz_!!6000000005936-2-tps-2032-1386.png)

### Token 限制翻倍：读取更多上下文，得到更多信息

输出 Token 限制从 8K 提升到 16K，AI 可以读取更多上下文、生成更完整的回复。同时自动检测模型的 max_tokens 设置，不用手动配置。

**你能用它做什么：**

- AI 可以读取更多文件内容，理解更大的项目结构
- 生成更完整的代码和文档，不用分段拼接
- 自动适配不同模型的 Token 限制，不用担心超出限制报错

详见 PR [#2411](https://github.com/QwenLM/qwen-code/pull/2411)、[#2356](https://github.com/QwenLM/qwen-code/pull/2356)、[#2362](https://github.com/QwenLM/qwen-code/pull/2362)

### Plan Mode：被拒绝的方案也能看到了

在 Plan Mode 下，AI 给出的方案被你拒绝后，内容不会消失。你可以对比不同方案，选最合适的那个。

**你能用它做什么：**

- 对比 AI 给出的多个方案，选最优解
- 拒绝后还能回看之前的内容，不用让 AI 重新生成
- 方案对比更直观，决策更从容

详见 PR [#2157](https://github.com/QwenLM/qwen-code/pull/2157)

## 📊 优化改进

- **Shell 输出不再刷屏**：大输出自动截断，工具输出处理也更简洁了，不会因为输出太长导致响应变慢（[#2388](https://github.com/QwenLM/qwen-code/pull/2388)）
- **斜杠命令描述本地化**：`/help` 等命令的描述支持多语言显示（[#2333](https://github.com/QwenLM/qwen-code/pull/2333)）
