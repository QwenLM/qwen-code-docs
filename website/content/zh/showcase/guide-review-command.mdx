## 概述

提交代码前总想找人帮忙看一眼，但同事忙得顾不上？现在直接用 `/review`，AI 会帮你检查代码质量、发现潜在问题、提出改进建议。不是简单的 lint 检查，而是像一个有经验的同事那样审查你的逻辑、命名、边界处理。

## 你能用它做什么

- **提交前审查**：让 AI 过一遍代码，发现问题早修复，不用等 Code Review 时才被指出来
- **独立开发的第二双眼睛**：减少低级错误，提升代码质量
- **学习最佳实践**：AI 会解释为什么这样写更好，不只是告诉你"这里有问题"

## 操作步骤

<Steps>

### 打开需要审查的文件

在 Qwen Code 中打开你想要审查的代码文件或项目目录。确保你的代码已经保存。

### 输入 /review 命令

在对话框中输入 `/review`，Qwen Code 会自动分析当前文件或最近的代码变更。

```bash
/review
```

### 查看审查结果

AI 会从代码质量、逻辑正确性、命名规范、边界处理等多个维度给出审查意见，并提供具体的改进建议。

</Steps>

<Callout type="info">
`/review` 不同于简单的 lint 检查，它会深入理解代码逻辑，识别潜在的 bug、性能问题和设计缺陷。
</Callout>
