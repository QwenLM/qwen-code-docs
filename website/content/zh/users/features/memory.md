# 记忆

每次 Qwen Code 会话都从一个全新的上下文窗口开始。有两种机制可以在不同会话之间传递知识，让你无需每次都重新解释：

- **QWEN.md** — 你_一次性_编写好指令，Qwen 每次会话都会读取
- **自动记忆** — Qwen 根据从你这里学到的东西自行记录的笔记

---

## QWEN.md：你给 Qwen 的指令

QWEN.md 是一个纯文本文件，你可以在其中写下 Qwen 应该始终了解的项目信息或个人偏好。可以将其视为一份永久性的简报，每次对话开始时都会加载。

### QWEN.md 中可以放什么

添加那些你原本需要每次会话都重复说明的内容：

- 构建和测试命令（`npm run test`、`make build`）
- 团队遵循的编码规范（“所有新文件必须包含 JSDoc 注释”）
- 架构决策（“我们使用仓库模式，永远不要直接从控制器调用数据库”）
- 个人偏好（“始终使用 pnpm，不要用 npm”）

不要包含 Qwen 可以通过阅读你的代码自行推断出的内容。QWEN.md 在简短且具体时效果最佳——内容越长，Qwen 遵循的可靠性就越低。

### 在哪里创建 QWEN.md

| 文件                          | 适用范围                                    |
| ----------------------------- | ------------------------------------------- |
| `~/.qwen/QWEN.md`             | 你个人，所有项目通用                        |
| 项目根目录下的 `QWEN.md`      | 整个团队（提交到版本控制）                  |
| `.qwen/QWEN.local.md`         | 仅你个人，仅限该项目（不要加入 Git）        |

你可以组合使用上述任意文件。Qwen 会在会话开始时加载所有这些文件。

如果你的仓库中已有供其他 AI 工具使用的 `AGENTS.md` 文件，Qwen 也会读取它。无需重复编写指令。

#### 何时使用 `.qwen/QWEN.local.md`

用于存放**项目特定但个人化**的指令——这些内容属于该项目，但不适合与团队共享：

- 你自己的集群 ID、容器仓库命名空间或云账号
- 一个硬编码了你本地环境个人调试命令
- 你希望 Qwen 了解的进行中工作笔记，但不想提交

该文件在**项目共享的 QWEN.md 之后**加载，因此你的个人指令可以补充或覆盖团队的设置。

**你必须自己将其加入 .gitignore。** 尽管 `.qwen/` 通常被视为本地目录，但 qwen-code 不会自动为你生成 `.gitignore`，有些项目会提交 `.qwen/settings.json`。请将以下行添加到你的 `.gitignore`（或全局 git ignore）中：

```
.qwen/QWEN.local.md
```

### 通过 `/init` 自动生成

运行 `/init`，Qwen 会分析你的代码库，根据找到的构建命令、测试指令和规范来创建一份初始 QWEN.md。如果文件已存在，它会建议补充内容而非覆盖。

### 引用其他文件

你可以在 QWEN.md 中指向其他文件，以便 Qwen 也读取它们：

```markdown
参考 @README.md 了解项目概览。

# 规范

- Git 工作流：@docs/git-workflow.md
```

在 QWEN.md 的任何地方使用 `@path/to/file`。相对路径会基于 QWEN.md 文件自身的位置解析。

## 自动记忆：Qwen 对你的了解

自动记忆在后台运行。每次对话结束后，Qwen 会默默保存它学到有用的内容——你的偏好、你给出的反馈、项目上下文——以便在未来的会话中使用，无需你重复说明。

这与 QWEN.md 不同：不是你编写的，而是 Qwen 自行完成的。

### Qwen 保存的内容

Qwen 会寻找四类值得记住的信息：

| 类别                    | 示例                                                         |
| ----------------------- | ------------------------------------------------------------ |
| **关于你**              | 你的角色、背景、你喜欢的工作方式                             |
| **你的反馈**            | 你做出的更正、你确认过的方法                                 |
| **项目上下文**          | 进行中的工作、决策、代码中不明显的目标                       |
| **外部引用**            | 你提到的仪表盘、工单跟踪系统、文档链接                       |

Qwen 不会保存所有内容——只保存那些下次确实有用的信息。

### 存储位置

自动记忆文件位于 `~/.qwen/projects/<project>/memory/`。同一仓库的所有分支和工作树共享同一个记忆文件夹，因此 Qwen 在一个分支学到的东西在其他分支也可用。

所有保存的内容都是纯 Markdown 格式——你可以随时打开、编辑或删除任何文件。

### 定期清理

Qwen 会定期检查已保存的记忆，移除重复项并清理过时的条目。当积累的会话足够多后，每天会在后台自动运行一次。如果你希望立即执行，可以手动触发 `/dream`。

清理运行时，屏幕角落会显示 **✦ 正在做梦**。你的会话正常进行。

### 开启或关闭

自动记忆默认开启。要切换状态，打开 `/memory` 并使用顶部的开关。你可以单独关闭自动保存、定期清理，或两者都关闭。

你也可以在 `~/.qwen/settings.json`（适用于所有项目）或 `.qwen/settings.json`（仅限当前项目）中设置：

```json
{
  "memory": {
    "enableManagedAutoMemory": true,
    "enableManagedAutoDream": true
  }
}
```

## 命令

### `/memory`

打开记忆面板。在这里你可以：

- 打开或关闭自动记忆保存
- 打开或关闭定期清理（做梦）
- 打开你的个人 QWEN.md（`~/.qwen/QWEN.md`）
- 打开项目 QWEN.md
- 浏览自动记忆文件夹

### `/init`

为你的项目生成一份初始 QWEN.md。Qwen 会读取你的代码库，并填入它发现的构建命令、测试指令和规范。

### `/remember <文本>`

立即将某些内容保存到自动记忆，无需等待 Qwen 自动拾取：

```
/remember Python 变量名始终使用 snake_case
/remember 预发环境地址是 staging.example.com
```

### `/forget <文本>`

移除与描述匹配的自动记忆条目：

```
/forget 登录 bug 的旧变通方案
```

### `/dream`

立即运行记忆清理，而不等待自动调度：

```
/dream
```

## 故障排除

### Qwen 没有遵循我的 QWEN.md

打开 `/memory` 查看已加载的文件。如果你的文件没有列出，Qwen 就无法看到它——请确保它位于项目根目录或 `~/.qwen/` 中。

指令在具体时效果更好：

- ✓ `TypeScript 文件使用 2 空格缩进`
- ✗ `代码格式化好一点`

如果你有多个 QWEN.md 文件且指令冲突，Qwen 可能行为不一致。请检查并移除任何矛盾之处。

### 我想查看 Qwen 保存了什么

运行 `/memory` 并选择 **打开自动记忆文件夹**。所有保存的记忆都是可读的 Markdown 文件，你可以浏览、编辑或删除。

### Qwen 总是忘记事情

如果自动记忆已开启但 Qwen 似乎跨会话不记得事情，请尝试运行 `/dream` 强制执行一次清理。同时检查 `/memory` 确认两个开关都已启用。

对于你希望 Qwen 始终记住的事情，请将其添加到 QWEN.md 中——自动记忆尽力而为，而 QWEN.md 是保证有效的。
