# 按计划运行提示词

> 使用 `/loop` 和 cron 调度工具在 Qwen Code 会话中重复运行提示词、轮询状态或设置一次性提醒。

计划任务让 Qwen Code 按间隔自动重新运行提示词。你可以用它来轮询部署状态、关注 PR、检查长时间运行的构建进度，或者在会话中稍后提醒自己做某事。

任务基于会话作用域：它们存在于当前 Qwen Code 进程中，退出后即消失。不会写入磁盘。

> **提示：** 计划任务默认启用。要禁用，请在[设置](../configuration/settings.md)中设置 `experimental.cron: false`，或在环境中设置 `QWEN_CODE_DISABLE_CRON=1`。

## 使用 /loop 安排重复提示词

`/loop` [内置技能](skills.md) 是安排重复提示词的最快方式。传入一个可选的间隔和提示词，Qwen Code 会设置一个在会话保持打开期间后台触发的 cron 任务。

```text
/loop 5m check if the deployment finished and tell me what happened
```

Qwen Code 解析间隔，将其转换为 cron 表达式，安排任务，并确认频率和任务 ID。然后它会立即执行一次提示词——你无需等待第一次 cron 触发。

### 间隔语法

间隔是可选的。你可以将间隔放在开头、末尾，或者完全省略。

| 形式                     | 示例                                   | 解析后的间隔              |
| :----------------------- | :------------------------------------- | :------------------------ |
| 开头标记                 | `/loop 30m check the build`            | 每 30 分钟                |
| 末尾 `every` 子句        | `/loop check the build every 2 hours`   | 每 2 小时                 |
| 无间隔                   | `/loop check the build`                | 默认为每 10 分钟          |

支持的单位有 `s`（秒）、`m`（分钟）、`h`（小时）、`d`（天）。秒会被向上舍入到最近的分钟，因为 cron 的粒度是分钟。间隔不能整除其单位时（如 `7m` 或 `90m`），会舍入到最近的整齐间隔，Qwen Code 会告诉你它选择了什么。

### 循环执行另一个命令

计划执行的提示词本身也可以是一个命令或技能调用。这对于重复运行你已经打包的工作流很有用。

```text
/loop 20m /review-pr 1234
```

每次任务触发时，Qwen Code 会像你亲自输入一样执行 `/review-pr 1234`。

### 管理循环

`/loop` 还支持两个子命令来管理现有任务：

```text
/loop list
```

列出所有已安排的任务及其 ID 和 cron 表达式。

```text
/loop clear
```

一次取消所有已安排的任务。

## 设置一次性提醒

对于一次性提醒，用自然语言描述你的需求，而不是使用 `/loop`。Qwen Code 会安排一个只触发一次的任务，运行后自动删除。

```text
remind me at 3pm to push the release branch
```

```text
in 45 minutes, check whether the integration tests passed
```

Qwen Code 使用 cron 表达式将触发时间固定到具体的分钟和小时，并确认何时触发。

## 管理计划任务

用自然语言让 Qwen Code 列出或取消任务，或者直接引用底层工具。

```text
what scheduled tasks do I have?
```

```text
cancel the deploy check job
```

在底层，Qwen Code 使用以下工具：

| 工具          | 用途                                                                                         |
| :------------ | :------------------------------------------------------------------------------------------- |
| `CronCreate`  | 安排新任务。接受一个 5 字段 cron 表达式、要运行的提示词，以及是重复触发还是只触发一次。        |
| `CronList`    | 列出所有计划任务及其 ID、调度信息和提示词。                                                  |
| `CronDelete`  | 按 ID 取消任务。                                                                            |

每个计划任务都有一个 8 字符的 ID，你可以将其传递给 `CronDelete`。一个会话最多可同时保存 50 个计划任务。

## 计划任务的运行方式

调度器每秒检查是否有到期的任务，并在会话空闲时将其加入队列。计划提示词在你的操作间隙触发，而不是在 Qwen Code 响应过程中触发。如果任务到期时 Qwen Code 正忙，提示词会等到当前轮次结束后再执行。

所有时间都按你的本地时区解释。像 `0 9 * * *` 这样的 cron 表达式表示运行 Qwen Code 的地方的上午 9 点，而不是 UTC。

### 抖动

为避免每个会话在同一墙钟时刻命中 API，调度器会为触发时间添加一个小的确定性偏移：

- **重复任务** 最多延迟其周期的 10% 触发，上限为 15 分钟。例如，每小时执行的任务可能在 `:00` 到 `:06` 之间的任何时刻触发。
- **一次性任务** 如果安排在整点或半点（分钟 `:00` 或 `:30`），则会提前最多 90 秒触发。

偏移量由任务 ID 派生，因此同一个任务总是获得相同的偏移量。如果精确时间很重要，请选择一个不是 `:00` 或 `:30` 的分钟，例如使用 `3 9 * * *` 而不是 `0 9 * * *`，这样一次性任务的抖动就不会生效。

### 三天过期

重复任务在创建 3 天后自动过期。任务会再触发一次，然后自行删除。这限制了被遗忘的循环可以运行多长时间。如果需要重复任务运行更久，请在过期前取消并重新创建。

一次性任务不会在计时器上过期——它们触发一次后即自行删除。

## Cron 表达式参考

`CronCreate` 接受标准的 5 字段 cron 表达式：`分钟 小时 日 月 星期`。所有字段都支持通配符（`*`）、单个值（`5`）、步长（`*/15`）、范围（`1-5`）和逗号分隔的列表（`1,15,30`）。

| 示例            | 含义                            |
| :-------------- | :------------------------------ |
| `*/5 * * * *`   | 每 5 分钟                       |
| `0 * * * *`     | 每小时整点                      |
| `7 * * * *`     | 每小时过 7 分钟                 |
| `0 9 * * *`     | 每天本地时间上午 9 点           |
| `0 9 * * 1-5`   | 工作日本地时间上午 9 点         |
| `30 14 15 3 *`  | 3 月 15 日本地时间下午 2:30     |

星期几使用 `0` 或 `7` 表示星期日，`6` 表示星期六。当日和星期都受限时（两者都不是 `*`），只要其中一个字段匹配就算匹配——这遵循标准的 vixie-cron 语义。

扩展语法如 `L`、`W`、`?` 以及名称别名（如 `MON` 或 `JAN`）不受支持。

## 限制

基于会话的调度有其固有的限制：

- 任务仅在 Qwen Code 运行且空闲时触发。关闭终端或让会话退出会取消所有任务。
- 不会补发错过的触发。如果任务计划时间在 Qwen Code 忙于长时间请求时过去，它会在 Qwen Code 空闲时触发一次，而不会为每个错过的间隔触发一次。
- 重启后不持久。重启 Qwen Code 会清除所有基于会话的任务。
