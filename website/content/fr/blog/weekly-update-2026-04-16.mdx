Cette semaine, nous avons publié trois versions **v0.14.3**, **v0.14.4** et **v0.14.5**.

- v0.14.3 est une version d'optimisation de l'expérience principale : La commande `/review` a été entièrement réécrite avec analyse déterministe, correction automatique et renforcement de la sécurité ; la commande `/statusline` prend en charge l'affichage personnalisable de la barre d'état ; le dialogue `/settings` ajoute un sélecteur de modèle et des descriptions de paramètres ; l'optimisation de la saisie CJK prend en charge le déplacement du curseur par mot ; la localisation French est en ligne.
- v0.14.4 apporte des fonctionnalités majeures incluant le profileur de démarrage, le fork de sous-agents, l'exposition des données d'utilisation du contexte, le middleware de réécriture de messages ACP, ainsi que l'affichage du compte dans l'extension VSCode et d'autres améliorations d'expérience.
- v0.14.5 se concentre sur les corrections de stabilité : résolution de plus de 20 problèmes incluant la détection de limite de débit API, le débordement des onglets de session VSCode, le baseUrl Gemini personnalisé et la détection d'erreurs de limite de débit.

## ✨ Nouvelles fonctionnalités

### /review Revue de code : Analyse déterministe + Correction automatique + Renforcement de la sécurité

La commande `/review` a été entièrement réécrite avec prise en charge de l'analyse déterministe, des suggestions de correction automatique et de la détection des vulnérabilités de sécurité.

**Ce que vous pouvez faire :**

- Exécuter des revues de code structurées sur les modifications actuelles, identifiant automatiquement les bugs potentiels
- Générer des suggestions de correction en un clic, les appliquant directement aux PRs
- Scanner les risques de sécurité courants (clés codées en dur, risques d'injection, etc.)

Voir PR [#2932](https://github.com/QwenLM/qwen-code/pull/2932)

### Barre d'état personnalisable : commande /statusline pour un affichage flexible

La commande `/statusline` vous permet de personnaliser le contenu de la barre d'état du terminal, prenant en charge le nom du modèle, l'utilisation des tokens, l'état du sandbox, et plus encore.

**Ce que vous pouvez faire :**

- Voir le modèle actuel et la consommation de tokens en temps réel
- Afficher l'état du sandbox, le mode de permissions et d'autres informations clés
- Simplifier ou étendre le contenu affiché selon vos préférences

Voir PR [#2923](https://github.com/QwenLM/qwen-code/pull/2923)

### Interface /settings : Sélecteur Fast Model + Descriptions des paramètres

Le Fast Model dans la boîte de dialogue `/settings` est passé d'un champ de saisie texte à un sélecteur déroulant ; chaque paramètre actif affiche désormais une description en dessous.

**Ce que vous pouvez faire :**

- Sélectionner le Fast Model directement dans une liste déroulante sans saisir manuellement le nom du modèle
- Consulter la description de chaque paramètre pour réduire la confusion de configuration

Voir PR [#3120](https://github.com/QwenLM/qwen-code/pull/3120)

- Déplacer le curseur par mot (et non caractère par caractère), rendant l'édition de longues phrases plus efficace
- Utiliser Ctrl+Flèches pour naviguer rapidement dans la zone de saisie, réduisant les frappes

### Édition de la file d'attente : ↑ pour modifier les messages en attente

Les messages dans la file d'attente de saisie peuvent être extraits pour modification avec ↑ ou ESC, sans avoir à les retaper.

**Ce que vous pouvez faire :**

- Repéré une erreur avant l'envoi ? Appuyez sur ↑ pour modifier directement le message en file d'attente
- Lorsque plusieurs messages sont en file d'attente, les extraire et modifier un par un pour une meilleure efficacité

Voir PR [#2871](https://github.com/QwenLM/qwen-code/pull/2871)
