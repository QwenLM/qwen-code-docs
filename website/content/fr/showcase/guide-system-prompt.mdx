## Aperçu

Configurez des prompts système personnalisés via SDK et CLI pour contrôler le style de réponse et le comportement de l'IA. Par exemple, faites-la toujours répondre en chinois, écrire les commentaires de code en anglais, ou suivre les conventions de nommage de votre équipe. Plus besoin de répéter « veuillez répondre en chinois » à chaque conversation.

## Ce que vous pouvez faire avec

- **Unifier la langue de réponse** : Faire répondre l'IA toujours dans votre langue préférée avec les commentaires de code en anglais
- **Suivre les conventions d'équipe** : Faire suivre à l'IA les conventions de nommage et le style de code de votre équipe
- **Personnaliser le comportement** : Contrôler le niveau de détail des réponses de l'IA, si elle pose des questions proactivement, etc.

## Étapes

<Steps>

### Créer un fichier de prompt système

Créez un fichier `AGENTS.md` (anciennement `QWEN.md`) à la racine de votre projet avec votre prompt système personnalisé :

```markdown
# Conventions du projet

- Toujours répondre en chinois
- Commentaires de code en anglais
- Utiliser camelCase pour le nommage des variables
- Utiliser les composants fonctionnels + Hooks pour React
- Préférer TypeScript
```

### Configurer les prompts système globaux

Pour appliquer des conventions à tous les projets, configurez les prompts système globaux dans `~/.qwen/AGENTS.md`.

Vous pouvez aussi utiliser :

```bash
qwen --system-prompt "Quand je pose une question en chinois, réponds en chinois ; quand je pose une question en anglais, réponds en anglais."
```

### Vérifier l'effet

Lancez Qwen Code, commencez une conversation et observez si l'IA répond selon vos conventions de prompt système.

</Steps>

<Callout type="info">
Le fichier `AGENTS.md` au niveau du projet remplace la configuration globale. Vous pouvez définir des conventions différentes pour différents projets, et les membres de l'équipe peuvent partager le même fichier de configuration pour rester cohérents.
</Callout>
