## Überblick

Konfigurieren Sie benutzerdefinierte System-Prompts über SDK und CLI, um den Antwortstil und das Verhalten der KI zu steuern. Zum Beispiel, dass sie immer auf Chinesisch antwortet, Code-Kommentare auf Englisch schreibt oder die Namenskonventionen Ihres Teams befolgt. Sie müssen nicht bei jedem Gespräch wiederholen: „Bitte antworten Sie auf Chinesisch".

## Was Sie damit tun können

- **Antwortsprache vereinheitlichen**: KI immer in Ihrer bevorzugten Sprache antworten lassen, Code-Kommentare auf Englisch
- **Teamkonventionen befolgen**: KI die Namenskonventionen und den Code-Stil Ihres Teams befolgen lassen
- **Verhalten anpassen**: Detailgrad der KI-Antworten steuern, ob sie proaktiv Fragen stellt usw.

## Schritte

<Steps>

### System-Prompt-Datei erstellen

Erstellen Sie eine `AGENTS.md`-Datei (ehemals `QWEN.md`) im Projektstammverzeichnis mit Ihrem benutzerdefinierten System-Prompt:

```markdown
# Projektkonventionen

- Immer auf Chinesisch antworten
- Code-Kommentare auf Englisch
- camelCase für Variablennamen verwenden
- Funktionale Komponenten + Hooks für React
- TypeScript bevorzugen
```

### Globale System-Prompts konfigurieren

Um Konventionen projektübergreifend anzuwenden, konfigurieren Sie globale System-Prompts in `~/.qwen/AGENTS.md`.

Sie können auch verwenden:

```bash
qwen --system-prompt "Wenn ich auf Chinesisch frage, antworte auf Chinesisch; wenn ich auf Englisch frage, antworte auf Englisch."
```

### Wirkung überprüfen

Starten Sie Qwen Code, beginnen Sie ein Gespräch und beobachten Sie, ob die KI gemäß Ihren System-Prompt-Konventionen antwortet.

</Steps>

<Callout type="info">
Die projektspezifische `AGENTS.md` überschreibt die globale Konfiguration. Sie können für verschiedene Projekte unterschiedliche Konventionen festlegen, und Teammitglieder können dieselbe Konfigurationsdatei teilen, um konsistent zu bleiben.
</Callout>
