Diese Woche wurden die offizielle Version **v0.13.0** und die Preview-Version v0.13.1 veröffentlicht.

Seit dem Release von v0.12.x haben viele Benutzer Feedback gegeben, dass Qwen Code "programmierbarer" sein sollte – nicht nur für Gespräche, sondern auch zur Integration in eigene Workflows. Die v0.13.0 dieser Woche konzentriert sich genau darauf: Das Hooks-System ermöglicht das Einfügen benutzerdefinierter Skripte an kritischen Punkten, das Skills-Verzeichnis ermöglicht die gemeinsame Nutzung benutzerdefinierter Fähigkeiten im Team, `/btw` löst die Unannehmlichkeit von "ich mache gerade A und möchte plötzlich B fragen". Zusätzlich lässt `/review` KI Code überprüfen, und die Agent-Kollaborations-Arena ermöglicht das gleichzeitige Ausführen mehrerer Modelle und die automatische Auswahl des besten Ergebnisses.

Vielen Dank an die neuen Mitwirkenden dieser Woche [**@simon100500**](https://github.com/simon100500) und [**@Br1an67**](https://github.com/Br1an67) 🎉

## ✨ Neue Funktionen

### /review: Lassen Sie KI Ihren Code prüfen

Bevor Sie Code committen, möchten Sie immer jemanden haben, der einen Blick darauf wirft, aber die Kollegen sind zu beschäftigt? Nutzen Sie jetzt einfach `/review`, KI wird Ihre Codequalität überprüfen, potenzielle Probleme entdecken und Verbesserungsvorschläge machen. Es ist keine einfache lint-Prüfung, sondern eine Überprüfung Ihrer Logik, Benennung und Randbedingungen wie ein erfahrener Kollege.

**Wofür können Sie es verwenden:**

- Lassen Sie KI Code vor dem Commit prüfen, um Probleme frühzeitig zu beheben, ohne erst beim Code Review darauf hingewiesen zu werden
- Als "zweites Paar Augen" bei der unabhängigen Entwicklung, um grundlegende Fehler zu reduzieren
- Lernen Sie Best Practices für Code – KI erklärt, warum es besser so geschrieben wird, und sagt nicht nur "hier gibt es ein Problem"

### /btw Seitfrage: Stellen Sie beim Schreiben von Code spontan eine Frage, ohne den Fokus zu verlieren

Sie schreiben Code zur Hälfte und vergessen plötzlich die Parameterreihenfolge einer API. Früher mussten Sie eine neue Unterhaltung öffnen, um nachzuschlagen, und dann zurückwechseln, wodurch der Kontext unterbrochen wurde.

Geben Sie jetzt `/btw` ein und fügen Sie direkt in der aktuellen Unterhaltung eine beiläufige Bemerkung ein. Qwen Code kehrt nach der Antwort automatisch zum vorherigen Kontext zurück, als wäre nichts passiert. Ihre Hauptunterhaltung wird nicht verunreinigt, und KI behandelt Ihre beiläufige Bemerkung nicht als Teil der aktuellen Aufgabe.

**Wofür können Sie es verwenden:**

- Überprüfen Sie beim Schreiben von React-Komponenten spontan die Verwendung eines Hooks und fahren Sie dann fort
- Bestätigen Sie während des Debuggens vorübergehend die Bedeutung einer Konfigurationsoption, ohne ein neues Fenster zu öffnen
- Stellen Sie eine nicht verwandte Frage, vergessen Sie sie danach, ohne die aktuelle Arbeit zu beeinflussen

### Hooks-Erweiterungsmechanismus: Lassen Sie Qwen Code nach Ihren Regeln arbeiten

Sind Sie schon einmal in diese Situation geraten – KI hat Code generiert, aber das Format ist falsch, und Sie müssen jedes Mal manuell prettier ausführen? Oder haben Sie vergessen, Tests vor dem Commit auszuführen, und CI ist fehlgeschlagen? Das Hooks-System ist genau dafür da. Sie können Ihre eigenen Skripte an 10 Schlüsselpunkten von Qwen Code anhängen, damit sie automatisch zu bestimmten Zeiten ausgeführt werden.

Wofür können Sie es verwenden:

- Führen Sie vor dem Commit automatisch Tests aus – vermeiden Sie das Hochladen von Code, der nicht durchläuft, CI wird nicht mehr wegen Ihnen rot
- Formatieren Sie automatisch nach der Codegenerierung – nachdem KI Code geschrieben hat, führt prettier/eslint automatisch einmal aus, Sie müssen es nicht manuell verarbeiten
- Speichern Sie automatisch Unterhaltungsprotokolle am Ende der Sitzung – haben Sie Beweise für die Rückverfolgung von Problemen
- Führen Sie Berechtigungsprüfungen vor der Toolausführung durch – eine zusätzliche Schutzebene für sensible Vorgänge

Die Konfiguration ist auch sehr einfach, legen Sie einfach Skriptdateien im Verzeichnis `.agents/hooks` im Projektstamm ab. Siehe PR [#2352](https://github.com/QwenLM/qwen-code/pull/2352) 、 [#2203](https://github.com/QwenLM/qwen-code/pull/2203)

### Agent-Kollaborations-Arena: Mehrere Modelle lösen gleichzeitig Aufgaben, wählen das beste Ergebnis

Für wichtigen Code lassen Sie Kollegen review, jetzt können Sie auch mehrere KI-Modelle gleichzeitig dieselbe Aufgabe bearbeiten lassen. Um die Fähigkeiten der Modelle zu vergleichen, können Sie direkt den Befehl `/arena` aufrufen, Modelle auswählen und die Aufgabe eingeben.

**Wofür können Sie es verwenden:**

- Lassen Sie beim Refactoring von Kernmodulen mehrere Modelle gleichzeitig Lösungen geben und wählen Sie die beste
- Vergleichen Sie beim Schreiben von Algorithmusaufgaben die Lösungen verschiedener Modelle und lernen Sie verschiedene Ansätze
- Bewerten Sie, wie neue Modelle in Ihrem Projekt abschneiden, lassen Sie tatsächliche Aufgaben sprechen

### VS Code Bild einfügen: Screenshots direkt an KI senden

Fügen Sie in VS Code direkt mit Ctrl+V Screenshots ein und senden Sie sie an KI. Haben Sie einen UI-Bug gesehen? Schicken Sie den Screenshot, damit KI ihn analysiert. Möchten Sie einen Entwurf reproduzieren? Schicken Sie den Screenshot, damit KI Code schreibt. Kein Speichern von Bildern, Suchen nach Pfaden oder manuelles Hochladen mehr.

**Wofür können Sie es verwenden:**

- Machen Sie einen Screenshot eines UI-Bugs und lassen Sie KI das Problem lokalisieren
- Fügen Sie einen Screenshot eines Entwurfs ein und lassen Sie KI direkt den entsprechenden Komponentencode schreiben
- Ist die Fehlermeldung zu lang, um sie einzutippen? Schicken Sie den Screenshot

### System-Prompt-Anpassung: Lassen Sie KI in Ihrem Stil antworten

Konfigurieren Sie benutzerdefinierte System-Prompts über SDK und CLI, um den Antwortstil und das Verhalten von KI zu steuern. Zum Beispiel immer auf Chinesisch antworten, Code-Kommentare auf Englisch, die Namenskonventionen Ihres Teams befolgen. Sie müssen nicht bei jeder Unterhaltung wiederholen "bitte auf Chinesisch antworten".

### Explore Agent: Lassen Sie KI zuerst recherchieren, dann handeln

Neuer Explore Agent, speziell für Code-Recherche. Bevor Sie Code ändern, lassen Sie Explore Agent zuerst die Codestruktur erkunden, Abhängigkeiten verstehen und die wichtigsten Einstiegspunkte finden. Nach der Recherche lassen Sie den Haupt-Agenten handeln, die Richtung ist genauer.

## 📊 Optimierungen und Verbesserungen

- **Echtzeit-Anzeige des Token-Verbrauchs**: Zeigt den aktuellen Token-Verbrauch direkt während des KI-Denkprozesses an, ohne bis zum Ende der Antwort warten zu müssen, um zu wissen, wie viel verwendet wurde ([#2445](https://github.com/QwenLM/qwen-code/pull/2445))
- **Gleichzeitige Ausführung von Sub-Agenten**: Früher liefen mehrere Unteraufgaben in der Warteschlange, jetzt laufen sie gleichzeitig. Die Verarbeitungsgeschwindigkeit komplexer Aufgaben hat sich deutlich verbessert, Sie warten weniger ([#2434](https://github.com/QwenLM/qwen-code/pull/2434))
- **Klarere Berechtigungshinweise**: Das Berechtigungs-Popup sagt Ihnen jetzt in Klartext "Datei xxx wird gelesen" statt einer Reihe technischer Parameter, und wenn Sie eine Operation ablehnen, wird Ihnen auch gesagt, warum sie abgelehnt wurde ([#2637](https://github.com/QwenLM/qwen-code/pull/2637), [#2283](https://github.com/QwenLM/qwen-code/pull/2283))
- **Intelligenteres Datei-Completion in VS Code**: Verwendet Fuzzy-Suche beim Eingeben von Dateinamen, geben Sie `comp` ein und finden Sie `src/components/Button.tsx`, ohne den vollständigen Pfad zu kennen ([#2437](https://github.com/QwenLM/qwen-code/pull/2437))
- **Intuitiveres Tab-Tasten-Verhalten**: In VS Code fügt Tab nur den Completion-Inhalt ein, löst nicht mehr versehentlich Einrückung oder andere Operationen aus ([#2431](https://github.com/QwenLM/qwen-code/pull/2431))
- **Automatische Kürzung der MCP-Tool-Ausgabe**: MCP-Tools werden automatisch gekürzt, wenn sie zu lange Inhalte zurückgeben, was nicht dazu führt, dass der Kontext aufgebläht wird und die Qualität der nachfolgenden Unterhaltung abnimmt ([#2446](https://github.com/QwenLM/qwen-code/pull/2446))
- **Ctrl+R Verlaufssuche**: Suchergebnisse werden nach Neuestem sortiert, die Befehle, die Sie kürzlich verwendet haben, stehen ganz oben ([#2425](https://github.com/QwenLM/qwen-code/pull/2425))
- **Optimierung der Fehlerbehandlung und Kontingenterkennung**: Bei API-Ratenbegrenzung oder erschöpftem Kontingent sind die Hinweise klarer, nicht mehr nur ein verschwommener Fehlercode ([#2458](https://github.com/QwenLM/qwen-code/pull/2458))
