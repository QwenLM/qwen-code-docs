## Overview

Want someone to review your code before committing, but your colleagues are too busy? Just use `/review` — AI will check code quality, find potential issues, and suggest improvements. It's not a simple lint check, but a thorough review of your logic, naming, and edge case handling, like an experienced colleague.

## What You Can Do With It

- **Pre-commit review**: Have AI go through your code, catch issues early — no more surprises during Code Review
- **A second pair of eyes for solo development**: Reduce low-level errors, improve code quality
- **Learn best practices**: AI explains why a certain approach is better, not just "there's a problem here"

## Steps

<Steps>

### Open the file to review

In Qwen Code, open the code file or project directory you want to review. Make sure your code is saved.

### Enter the /review command

Type `/review` in the chat. Qwen Code will automatically analyze the current file or recent code changes.

```bash
/review
```

### View the review results

AI will provide review feedback from multiple dimensions including code quality, logical correctness, naming conventions, and edge case handling, along with specific improvement suggestions.

</Steps>

<Callout type="info">
`/review` is different from simple lint checks — it deeply understands code logic, identifying potential bugs, performance issues, and design flaws.
</Callout>
