## Overview

Configure custom system prompts via SDK and CLI to control AI's response style and behavior. For example, make it always respond in Chinese, write code comments in English, or follow your team's naming conventions. No need to repeat "please respond in Chinese" every conversation.

## What You Can Do With It

- **Unify response language**: Make AI always respond in your preferred language with code comments in English
- **Follow team conventions**: Make AI follow your team's naming conventions and code style
- **Customize behavior**: Control AI's response detail level, whether it proactively asks questions, etc.

## Steps

<Steps>

### Create a system prompt file

Create an `AGENTS.md` file (formerly `QWEN.md`) in your project root with your custom system prompt:

```markdown
# Project Conventions

- Always respond in Chinese
- Code comments in English
- Use camelCase for variable naming
- Use functional components + Hooks for React
- Prefer TypeScript
```

### Configure global system prompts

To apply conventions across all projects, configure global system prompts in `~/.qwen/AGENTS.md`.

You can also use:

```bash
qwen --system-prompt "When I ask in Chinese, respond in Chinese; when I ask in English, respond in English."
```

### Verify the effect

Start Qwen Code, begin a conversation, and observe whether AI responds according to your system prompt conventions.

</Steps>

<Callout type="info">
Project-level `AGENTS.md` overrides global configuration. You can set different conventions for different projects, and team members can share the same config file to stay consistent.
</Callout>
