## Overview

Before modifying code, let Explore Agent help you understand the code structure, dependencies, and key entry points. Research first, then let the main Agent take action — more accurate direction, less rework from blind modifications.

## What You Can Do With It

- **Understand code structure**: Quickly grasp the architecture and module layout of an unfamiliar project
- **Map dependencies**: Find out which files and modules a feature involves
- **Locate key entry points**: Quickly find the code locations that need modification in large projects

## Steps

<Steps>

### Switch to Explore Agent

In Qwen Code, switch to Explore Agent mode to focus on code research rather than modification.

### Describe your research goal

Tell Explore Agent what you want to understand, for example:

```
Analyze the authentication module of this project, find all related files and dependencies
```

### Get the research report

Explore Agent will analyze the code structure and output a detailed research report, including file lists, dependency graphs, key entry points, etc.

### Switch back to the main Agent to start development

After research is complete, switch back to the main Agent and start actual code modifications based on the research results. The research report will be passed as context to the main Agent.

</Steps>

<Callout type="info">
Explore Agent only does research — it won't modify any code. Its output can be directly used as input for the main Agent, forming an efficient "research first, develop later" workflow.
</Callout>
