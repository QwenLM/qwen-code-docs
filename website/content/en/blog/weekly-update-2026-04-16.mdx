This week we released three versions **v0.14.3**, **v0.14.4**, and **v0.14.5**.

- v0.14.3 is a core experience optimization release: The `/review` command has been completely rewritten with deterministic analysis, auto-fix, and security hardening; `/statusline` command supports customizable status bar display; `/settings` dialog adds model selector and setting descriptions; CJK input optimization supports word-granularity cursor movement; French localization goes live.
- v0.14.4 brings heavyweight features including startup profiler, fork sub-agents, context usage data exposure, ACP message rewrite middleware, plus VSCode extension account display and other experience improvements.
- v0.14.5 focuses on stability fixes: resolving 20+ issues including API rate limit detection, VSCode session tab overflow, custom Gemini baseUrl, and rate limit error detection.

## ✨ New Features

### /review Code Review: Deterministic Analysis + Auto-Fix + Security Hardening

The `/review` command has been completely rewritten with support for deterministic analysis, auto-fix suggestions, and security vulnerability detection.

**What You Can Do With It:**

- Execute structured code reviews on current changes, automatically identifying potential bugs
- Generate fix suggestions with one click, applying them directly to PRs
- Scan for common security risks (e.g., hardcoded keys, injection vulnerabilities)

See PR [#2932](https://github.com/QwenLM/qwen-code/pull/2932)

### Fork Sub-Agents

Implemented fork sub-agent functionality with shared context for subtask processing.

**What You Can Do With It:**

- Spawn subtasks from the main task to handle independent modules
- Sub-agents inherit the main task's context without redundant information passing
- Process multiple related tasks in parallel for improved efficiency in complex scenarios
