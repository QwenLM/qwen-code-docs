## Visão Geral

Quer que alguém revise seu código antes do commit, mas seus colegas estão ocupados demais? Basta usar `/review` — a IA verificará a qualidade do código, encontrará problemas potenciais e sugerirá melhorias. Não é uma simples verificação de lint, mas uma revisão completa da sua lógica, nomenclatura e tratamento de casos extremos, como um colega experiente.

## O que você pode fazer com isso

- **Revisão antes do commit**: Faça a IA passar pelo seu código, detectar problemas cedo — sem mais surpresas durante o Code Review
- **Um segundo par de olhos para desenvolvimento solo**: Reduzir erros básicos, melhorar a qualidade do código
- **Aprender melhores práticas**: A IA explica por que uma certa abordagem é melhor, não apenas "há um problema aqui"

## Passos

<Steps>

### Abrir o arquivo para revisão

No Qwen Code, abra o arquivo de código ou diretório do projeto que deseja revisar. Certifique-se de que seu código está salvo.

### Inserir o comando /review

Digite `/review` no chat. O Qwen Code analisará automaticamente o arquivo atual ou as alterações de código recentes.

```bash
/review
```

### Ver os resultados da revisão

A IA fornecerá feedback de revisão em múltiplas dimensões, incluindo qualidade do código, correção lógica, convenções de nomenclatura e tratamento de casos extremos, junto com sugestões específicas de melhoria.

</Steps>

<Callout type="info">
`/review` é diferente de simples verificações de lint — ele compreende profundamente a lógica do código, identificando bugs potenciais, problemas de desempenho e falhas de design.
</Callout>
