## Visão Geral

Configure prompts de sistema personalizados via SDK e CLI para controlar o estilo de resposta e comportamento da IA. Por exemplo, faça-a sempre responder em chinês, escrever comentários de código em inglês ou seguir as convenções de nomenclatura da sua equipe. Sem necessidade de repetir "por favor, responda em chinês" a cada conversa.

## O que você pode fazer com isso

- **Unificar o idioma de resposta**: Fazer a IA sempre responder no seu idioma preferido com comentários de código em inglês
- **Seguir convenções da equipe**: Fazer a IA seguir as convenções de nomenclatura e estilo de código da sua equipe
- **Personalizar comportamento**: Controlar o nível de detalhe das respostas da IA, se ela faz perguntas proativamente, etc.

## Passos

<Steps>

### Criar um arquivo de prompt de sistema

Crie um arquivo `AGENTS.md` (anteriormente `QWEN.md`) na raiz do seu projeto com seu prompt de sistema personalizado:

```markdown
# Convenções do Projeto

- Sempre responder em chinês
- Comentários de código em inglês
- Usar camelCase para nomenclatura de variáveis
- Usar componentes funcionais + Hooks para React
- Preferir TypeScript
```

### Configurar prompts de sistema globais

Para aplicar convenções em todos os projetos, configure prompts de sistema globais em `~/.qwen/AGENTS.md`.

Você também pode usar:

```bash
qwen --system-prompt "Quando eu perguntar em chinês, responda em chinês; quando eu perguntar em inglês, responda em inglês."
```

### Verificar o efeito

Inicie o Qwen Code, comece uma conversa e observe se a IA responde de acordo com as convenções do seu prompt de sistema.

</Steps>

<Callout type="info">
O `AGENTS.md` no nível do projeto substitui a configuração global. Você pode definir convenções diferentes para projetos diferentes, e os membros da equipe podem compartilhar o mesmo arquivo de configuração para manter a consistência.
</Callout>
