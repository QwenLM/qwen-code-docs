На этой неделе мы выпустили функциональную версию **v0.12.4** и 2 версии с исправлениями ошибок, а также предварительную версию **v0.13.0-preview**.

Хотите попробовать новые функции предварительной версии? Выполните `npm i @qwen-code/qwen-code@v0.13.0-preview.1 -g` для установки.

После запуска v0.12.0 мы получили много отзывов от пользователей. На этой неделе мы сосредоточились на исправлении проблем, влияющих на пользовательский опыт: проблемы с кодировкой китайского вывода в Windows, потеря вывода интерактивной оболочки и ошибки повторных попыток API. В плане новых функций мы внесли комплексные улучшения в токены: лимит удвоен до 16K, отображение потребления в реальном времени и новая команда `/context` для детализации. Мы также добавили поддержку редакторов JetBrains и Zed.

## ✨ Новые функции

### Лимит токенов удвоен: Читайте больше контекста, получайте больше информации

Лимит выходных токенов увеличен с 8K до 16K, что позволяет ИИ читать больше контекста и генерировать более полные ответы. Также автоматически определяется настройка max_tokens модели - ручная настройка не требуется.

**Варианты использования:**

- ИИ может читать больше содержимого файлов и понимать более крупные структуры проектов
- Генерировать более полный код и документы без склеивания сегментов
- Автоматически адаптироваться к лимитам токенов разных моделей, не беспокоясь о превышении

См. PR [#2411](https://github.com/QwenLM/qwen-code/pull/2411), [#2356](https://github.com/QwenLM/qwen-code/pull/2356), [#2362](https://github.com/QwenLM/qwen-code/pull/2362)

### Отображение потребления токенов в реальном времени (Preview)

Во время размышлений и генерации ИИ интерфейс показывает потребление токенов в реальном времени. Не нужно ждать окончания разговора - узнавайте потребление токенов в любой момент.

**Варианты использования:**

- Видеть потребление токенов в реальном времени во время разговоров
- Своевременно корректировать при аномальном потреблении, чтобы избежать потерь
- Сравнивать стоимость токенов разных задач для оптимизации использования

См. PR [#2445](https://github.com/QwenLM/qwen-code/pull/2445)

![](https://gw.alicdn.com/imgextra/i2/O1CN01J3OhN228vlftWyHmx_!!6000000007995-1-tps-1280-720.gif)

### Команда /context: Просмотр детализации потребления токенов (Preview)

Используйте команду `/context`, чтобы увидеть детализацию потребления токенов в окне контекста: какие файлы используют сколько токенов, сколько места осталось - всё с одного взгляда.

**Варианты использования:**

- Просматривать текущее потребление токенов разговора, знать сколько ещё контента можно добавить
- Обнаруживать, какие файлы используют слишком много токенов, и решать, нужна ли очистка
- Оптимизировать использование контекста, чтобы ИИ фокусировался на ключевом содержимом

См. PR [#1835](https://github.com/QwenLM/qwen-code/pull/1835)

![](https://gw.alicdn.com/imgextra/i3/O1CN01cekEKq1mRc3CM2AQd_!!6000000004951-1-tps-1280-720.gif)

### Интеграция с редакторами Zed и JetBrains

Помимо VS Code, Qwen Code теперь поддерживает Zed и серию редакторов JetBrains (IntelliJ IDEA, PyCharm, WebStorm и др.). См. PR [#2372](https://github.com/QwenLM/qwen-code/pull/2372).

**Варианты использования:**

- Использовать Qwen Code напрямую в IDE JetBrains без переключения окон
- Пользователи редактора Zed также могут наслаждаться ИИ-помощником в кодировании
- Единый опыт работы с разными редакторами, не нужно переучиваться при смене инструмента

Документация:

- [JetBrains IDE | Qwen Code Docs](https://qwenlm.github.io/qwen-code-docs/ru/users/integration-jetbrains/)
- [Редактор Zed | Qwen Code Docs](https://qwenlm.github.io/qwen-code-docs/ru/users/integration-zed/)

![](https://gw.alicdn.com/imgextra/i3/O1CN01rTE0X61tikAwLN7Zz_!!6000000005936-2-tps-2032-1386.png)

### Plan Mode: Отклонённые предложения теперь видны

В Plan Mode, когда предложение ИИ отклоняется вами, содержимое не исчезает. Вы можете сравнивать разные предложения и выбирать наиболее подходящее.

**Варианты использования:**

- Сравнивать несколько предложений ИИ и выбирать лучшее решение
- Просматривать предыдущее содержимое после отклонения без просьбы к ИИ регенерировать
- Более интуитивное сравнение предложений для более уверенных решений

См. PR [#2157](https://github.com/QwenLM/qwen-code/pull/2157)

![](https://gw.alicdn.com/imgextra/i4/O1CN01rGEuoD1XpgdIQ8X2V_!!6000000002973-1-tps-860-584.gif)

### Поддержка директории .agents (Preview)

Файлы навыков теперь можно размещать в директории `.agents` проекта, управляя ими вместе с кодом проекта. Члены команды могут использовать ту же конфигурацию навыков после клонирования проекта.

**Варианты использования:**

- Размещать навыки, связанные с проектом, в директории `.agents` для контроля версий вместе с кодом
- Члены команды делятся конфигурацией навыков без индивидуальной ручной настройки
- Разные проекты могут иметь независимые конфигурации навыков без взаимных помех

См. PR [#2202](https://github.com/QwenLM/qwen-code/pull/2202)

![](https://gw.alicdn.com/imgextra/i3/O1CN01S3mZO01RMw4i66XNH_!!6000000002098-1-tps-1280-720.gif)

### Экспорт сессий с метаданными и статистикой (Preview)

При экспорте сессий теперь включаются метаданные и статистика: время разговора, потребление токенов, количество сообщений и т.д. Удобно для архивирования и анализа использования.

**Варианты использования:**

- Автоматически записывать время разговора и потребление токенов при экспорте для удобного отслеживания
- Статистически анализировать паттерны использования для понимания своих привычек
- Архивировать важные разговоры с полной информацией

См. PR [#2328](https://github.com/QwenLM/qwen-code/pull/2328)

![](https://gw.alicdn.com/imgextra/i3/O1CN010xCLBN1txrFHqCjhG_!!6000000005969-2-tps-1264-589.png)
