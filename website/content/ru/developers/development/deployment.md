# Запуск и развертывание Qwen Code

В этом документе описывается, как запустить Qwen Code, и объясняется архитектура развертывания, которую использует Qwen Code.

## Запуск Qwen Code

Существует несколько способов запуска Qwen Code. Выбор зависит от ваших задач.

---

### 1. Стандартная установка (рекомендуется для большинства пользователей)

Это рекомендуемый способ установки Qwen Code для конечных пользователей. Он предполагает загрузку пакета Qwen Code из реестра NPM.

- **Глобальная установка:**

  ```bash
  npm install -g @qwen-code/qwen-code
  ```

  Затем запустите CLI из любой директории:

  ```bash
  qwen
  ```

- **Запуск через NPX:**

  ```bash
  # Execute the latest version from NPM without a global install
  npx @qwen-code/qwen-code
  ```

---

### 2. Запуск в песочнице (Docker/Podman)

Для обеспечения безопасности и изоляции Qwen Code можно запускать внутри контейнера. Это способ по умолчанию, который CLI использует для выполнения инструментов, способных вызывать побочные эффекты.

- **Напрямую из реестра:**
  Вы можете запустить опубликованный образ песочницы напрямую. Это удобно для сред, где установлен только Docker и требуется запустить CLI.
  ```bash
  # Run the published sandbox image
  docker run --rm -it ghcr.io/qwenlm/qwen-code:0.0.11
  ```
- **С использованием флага `--sandbox`:**
  Если Qwen Code установлен локально (с помощью стандартной установки, описанной выше), вы можете указать ему запускаться внутри контейнера песочницы.
  ```bash
  qwen --sandbox -y -p "your prompt here"
  ```

---

### 3. Запуск из исходного кода (рекомендуется для контрибьюторов Qwen Code)

Участникам проекта потребуется запускать CLI напрямую из исходного кода.

- **Режим разработки:**
  Этот метод обеспечивает hot-reloading и удобен для активной разработки.
  ```bash
  # From the root of the repository
  npm run start
  ```
- **Режим, близкий к production (связанный пакет):**
  Этот метод имитирует глобальную установку путем линковки локального пакета. Он удобен для тестирования локальной сборки в рабочем процессе, близком к production.

  ```bash
  # Link the local cli package to your global node_modules
  npm link packages/cli

  # Now you can run your local version using the `qwen` command
  qwen
  ```

---

### 4. Запуск последнего коммита Qwen Code из GitHub

Вы можете запустить самую свежую версию Qwen Code напрямую из репозитория GitHub. Это удобно для тестирования функций, находящихся в разработке.

```bash
# Execute the CLI directly from the main branch on GitHub
npx https://github.com/QwenLM/qwen-code
```

## Архитектура развертывания

Описанные выше способы запуска реализованы благодаря следующим архитектурным компонентам и процессам:

**Пакеты NPM**

Проект Qwen Code представляет собой монорепозиторий, который публикует основные пакеты в реестре NPM:

- `@qwen-code/qwen-code-core`: бэкенд, отвечающий за логику и выполнение инструментов.
- `@qwen-code/qwen-code`: пользовательский интерфейс (фронтенд).

Эти пакеты используются при стандартной установке и при запуске Qwen Code из исходного кода.

**Процессы сборки и упаковки**

В зависимости от канала распространения используются два различных процесса сборки:

- **Публикация в NPM:** Для публикации в реестре NPM исходный код TypeScript в `@qwen-code/qwen-code-core` и `@qwen-code/qwen-code` транслируется в стандартный JavaScript с помощью компилятора TypeScript (`tsc`). В пакет NPM публикуется полученная директория `dist/`. Это стандартный подход для библиотек на TypeScript.

- **Запуск через `npx` из GitHub:** При запуске последней версии Qwen Code напрямую из GitHub срабатывает другой процесс, запускаемый скриптом `prepare` в `package.json`. Этот скрипт использует `esbuild` для сборки всего приложения и его зависимостей в один самодостаточный JavaScript-файл. Этот бандл создается на лету на машине пользователя и не коммитится в репозиторий.

**Образ песочницы Docker**

Метод запуска на базе Docker поддерживается контейнерным образом `qwen-code-sandbox`. Этот образ публикуется в реестре контейнеров и содержит предустановленную глобальную версию Qwen Code.

## Процесс релиза

Процесс релиза автоматизирован с помощью GitHub Actions. Рабочий процесс релиза выполняет следующие действия:

1.  Сборка пакетов NPM с помощью `tsc`.
2.  Публикация пакетов NPM в реестре артефактов.
3.  Создание релизов GitHub с собранными артефактами.
