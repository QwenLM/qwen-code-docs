# Выполнение и развертывание Qwen Code

В этом документе описывается, как запускать Qwen Code, а также объясняется архитектура его развертывания.

## Запуск Qwen Code

Существует несколько способов запуска Qwen Code. Выбираемый вами вариант зависит от того, как вы планируете его использовать.

---

### 1. Стандартная установка (рекомендуется для типовых пользователей)

Это рекомендуемый способ установки Qwen Code для конечных пользователей. Он предполагает загрузку пакета Qwen Code из реестра NPM.

- **Глобальная установка:**

  ```bash
  npm install -g @qwen-code/qwen-code
  ```

  После этого CLI можно запускать из любого места:

  ```bash
  qwen
  ```

- **Запуск через NPX:**

  ```bash
  # Запуск последней версии из NPM без глобальной установки
  npx @qwen-code/qwen-code
  ```

### 2. Запуск в песочнице (Docker/Podman)

Для обеспечения безопасности и изоляции Qwen Code можно запускать внутри контейнера. Это способ выполнения инструментов с побочными эффектами по умолчанию в CLI.

- **Непосредственно из реестра:**
  Вы можете запустить опубликованный образ песочницы напрямую. Это полезно в средах, где доступен только Docker, и вы хотите запустить CLI.
  ```bash
  # Запуск опубликованного образа песочницы
  docker run --rm -it ghcr.io/qwenlm/qwen-code:0.0.11
  ```
- **С использованием флага `--sandbox`:**
  Если Qwen Code установлен локально (с помощью стандартной установки, описанной выше), вы можете указать ему выполнять работу внутри контейнера песочницы.
  ```bash
  qwen --sandbox -y -p "ваш запрос здесь"
  ```

### 3. Запуск из исходного кода (рекомендуется для участников проекта Qwen Code)

Участникам проекта следует запускать CLI непосредственно из исходного кода.

- **Режим разработки:**  
  Этот метод обеспечивает горячую перезагрузку и удобен при активной разработке.  
  ```bash
  # Из корня репозитория
  npm run start
  ```
- **Режим, имитирующий рабочее окружение (связанный пакет):**  
  Этот метод имитирует глобальную установку путём связывания локального пакета. Он полезен для тестирования локальной сборки в условиях, близких к продакшену.  

  ```bash
  # Свяжите локальный пакет CLI с глобальным каталогом node_modules
  npm link packages/cli

  # Теперь вы можете запустить локальную версию командой `qwen`
  qwen
  ```

---

### 4. Запуск последнего коммита Qwen Code из GitHub

Вы можете запустить самую свежую версию Qwen Code напрямую из репозитория на GitHub. Это полезно для тестирования функций, находящихся в стадии разработки.

```bash

# Запуск CLI напрямую из основной ветки на GitHub
npx https://github.com/QwenLM/qwen-code

## Архитектура развертывания

Описанные выше методы выполнения реализуются благодаря следующим архитектурным компонентам и процессам:

**Пакеты NPM**

Проект Qwen Code представляет собой монорепозиторий, публикующий основные пакеты в реестр NPM:

- `@qwen-code/qwen-code-core`: бэкенд, отвечающий за логику и выполнение инструментов;
- `@qwen-code/qwen-code`: фронтенд, предназначенный для взаимодействия с пользователем.

Эти пакеты используются при стандартной установке и при запуске Qwen Code из исходного кода.

**Процессы сборки и упаковки**

Используются два различных процесса сборки в зависимости от канала распространения:

- **Публикация в NPM:** При публикации в реестр NPM исходный код на TypeScript из пакетов `@qwen-code/qwen-code-core` и `@qwen-code/qwen-code` транспилируется в стандартный JavaScript с помощью компилятора TypeScript (`tsc`). В результате получается каталог `dist/`, который и публикуется в виде пакета NPM. Это стандартный подход для библиотек на TypeScript.

- **Выполнение через `npx` из GitHub:** При запуске последней версии Qwen Code напрямую из GitHub срабатывает другой процесс, инициируемый скриптом `prepare` в файле `package.json`. Этот скрипт использует `esbuild` для создания единого самодостаточного JavaScript-файла, содержащего всё приложение и его зависимости. Такой файл собирается динамически на машине пользователя и не сохраняется в репозитории.

**Docker-образ песочницы**

Метод выполнения на основе Docker поддерживается контейнерным образом `qwen-code-sandbox`. Этот образ публикуется в реестре контейнеров и содержит предустановленную глобальную версию Qwen Code.

## Процесс выпуска

Процесс выпуска автоматизирован с помощью GitHub Actions. Рабочий процесс выпуска выполняет следующие действия:

1.  Сборка пакетов NPM с использованием `tsc`.
2.  Публикация пакетов NPM в реестр артефактов.
3.  Создание релизов GitHub с упакованными ресурсами.
