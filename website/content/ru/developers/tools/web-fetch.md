# Инструмент получения веб-контента (`web_fetch`)

В этом документе описывается инструмент `web_fetch` для Qwen Code.

## Описание

Используйте `web_fetch`, чтобы получить контент по указанному URL и обработать его с помощью ИИ-модели. Инструмент принимает URL и промпт в качестве входных данных, загружает содержимое по указанному URL, преобразует HTML в Markdown и обрабатывает полученный контент с помощью промпта, используя небольшую и быструю модель.

### Аргументы

`web_fetch` принимает два аргумента:

- `url` (строка, обязательный): URL, с которого требуется получить контент. Должен быть корректным полным URL, начинающимся с `http://` или `https://`.
- `prompt` (строка, обязательный): Промпт, описывающий информацию, которую необходимо извлечь из содержимого страницы.

## Как использовать `web_fetch` с Qwen Code

Чтобы использовать `web_fetch` с Qwen Code, укажите URL и запрос, описывающий, какие данные нужно извлечь со страницы по этому URL. Перед загрузкой содержимого инструмент запросит подтверждение. После подтверждения он напрямую загрузит содержимое и обработает его с помощью ИИ-модели.

Инструмент автоматически преобразует HTML в текст, корректно обрабатывает URL-адреса GitHub blob (преобразуя их в raw-URL), а также обновляет HTTP-ссылки до HTTPS для обеспечения безопасности.

Пример использования:

```
web_fetch(url="https://example.com", prompt="Кратко изложи основные тезисы этой статьи")
```

## Примеры использования `web_fetch`

Краткое содержание одной статьи:

```
web_fetch(url="https://example.com/news/latest", prompt="Можете ли вы кратко изложить основные тезисы этой статьи?")
```

Извлечение конкретной информации:

```
web_fetch(url="https://arxiv.org/abs/2401.0001", prompt="Каковы ключевые выводы и методология, описанные в этой статье?")
```

Анализ документации GitHub:

```
web_fetch(url="https://github.com/QwenLM/Qwen/blob/main/README.md", prompt="Какие шаги необходимы для установки и какие основные функции представлены?")
```

## Важные замечания

- **Обработка одного URL:** Средство `web_fetch` обрабатывает по одному URL за раз. Чтобы проанализировать несколько URL, выполните отдельные вызовы этого средства.
- **Формат URL:** Средство автоматически обновляет HTTP-URL до HTTPS и преобразует URL GitHub Blob в формат «raw» для улучшения доступа к содержимому.
- **Обработка содержимого:** Средство напрямую извлекает содержимое и обрабатывает его с помощью ИИ-модели, преобразуя HTML в читаемый текстовый формат.
- **Качество вывода:** Качество результата зависит от ясности инструкций в запросе.
- **Средства MCP:** Если доступно средство получения веб-контента от MCP (начинающееся с `mcp__`), предпочтительно использовать именно его, поскольку оно может иметь меньше ограничений.
