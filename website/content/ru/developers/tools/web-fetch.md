# Инструмент веб-запросов (`web_fetch`)

В этом документе описывается инструмент `web_fetch` для Qwen Code.

## Описание

Используйте `web_fetch` для получения содержимого по указанному URL и его обработки с помощью ИИ-модели. Инструмент принимает URL и подсказку в качестве входных данных, извлекает содержимое по URL, преобразует HTML в markdown и обрабатывает содержимое с помощью подсказки, используя небольшую и быструю модель.

### Аргументы

`web_fetch` принимает два аргумента:

- `url` (строка, обязательный): URL, по которому нужно получить содержимое. Должен быть полностью сформированным действительным URL, начинающимся с `http://` или `https://`.
- `prompt` (строка, обязательный): Подсказка, описывающая, какую информацию вы хотите извлечь из содержимого страницы.

## Как использовать `web_fetch` с Qwen Code

Чтобы использовать `web_fetch` с Qwen Code, укажите URL и подсказку, описывающую, что вы хотите извлечь по этому адресу. Инструмент запросит подтверждение перед получением данных по URL. После подтверждения инструмент получит содержимое напрямую и обработает его с помощью модели ИИ.

Инструмент автоматически преобразует HTML в текст, обрабатывает URL-адреса GitHub blob (преобразуя их в сырые URL) и обновляет HTTP URL до HTTPS для обеспечения безопасности.

Пример использования:

```
web_fetch(url="https://example.com", prompt="Summarize the main points of this article")
```

## Примеры использования `web_fetch`

Краткое содержание одной статьи:

```
web_fetch(url="https://example.com/news/latest", prompt="Можете ли вы кратко изложить основные моменты этой статьи?")
```

Извлечение конкретной информации:

```
web_fetch(url="https://arxiv.org/abs/2401.0001", prompt="Каковы ключевые выводы и методология, описанные в этом документе?")
```

Анализ документации на GitHub:

```
web_fetch(url="https://github.com/QwenLM/Qwen/blob/main/README.md", prompt="Каковы шаги по установке и основные функции?")
```

## Важные замечания

- **Обработка одного URL:** `web_fetch` обрабатывает только один URL за раз. Чтобы проанализировать несколько URL, сделайте отдельные вызовы инструмента.
- **Формат URL:** Инструмент автоматически обновляет HTTP URL до HTTPS и преобразует URL-адреса GitHub blob в необработанный формат для лучшего доступа к содержимому.
- **Обработка контента:** Инструмент получает контент напрямую и обрабатывает его с помощью модели ИИ, преобразуя HTML в читаемый текстовый формат.
- **Качество вывода:** Качество вывода будет зависеть от ясности инструкций в запросе.
- **Инструменты MCP:** Если доступен предоставленный MCP инструмент веб-запроса (начинающийся с "mcp\_\_"), предпочтительно использовать именно его, поскольку он может иметь меньше ограничений.
