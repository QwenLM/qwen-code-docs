# Настройка дизайна области баннера

> Позволяет пользователям заменять ASCII-арт QWEN, заменять название бренда и
> полностью скрывать баннер — не давая им скрывать операционные
> данные (версию, аутентификацию, модель, рабочую директорию), которые делают Qwen Code
> отлаживаемым и заслуживающим доверия.

## Обзор

CLI Qwen Code выводит баннер при запуске, содержащий ASCII-логотип QWEN
и панель информации с рамкой. В нескольких реальных сценариях требуется некоторый контроль
над этой областью:

- **Белый лейбл / интеграция стороннего бренда**: компании и команды,
  встраивающие Qwen Code в свои продукты, хотят отображать свой фирменный стиль,
  а не стандартный «Qwen Code».
- **Персонализация**: пользователи хотят настроить баннер терминала под стандарты своей команды
  или собственные предпочтения.
- **Различие между мультитенантными / мультиэкземпляровыми средами**: в общих окружениях
  разные команды хотят иметь быстрый визуальный сигнал, в каком экземпляре они находятся.

Подход к дизайну прост: **фирменное оформление можно заменить; операционные
данные — нет**. Настройка должна позволять пользователям размещать собственный бренд
поверх, а не скрывать информацию, которая делает сессию отлаживаемой.
Этот подход определяет каждое решение «что можно изменять, а что заблокировано»
в остальной части документа.

Отслеживается в [issue #3005](https://github.com/QwenLM/qwen-code/issues/3005).

## Таксономия областей баннера

Сейчас баннер отображается компонентом `Header` (монтируется из `AppHeader`) и
делится на следующие области:

```
  marginX=2                                                           marginX=2
  │                                                                          │
  ▼                                                                          ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ┌──── Колонка логотипа ─┐  gap=2  ┌──── Панель информации (с рамкой) ──┐ │
│   │                      │         │                                     │ │
│   │  ███ ASCII QWEN ███  │         │  ① Заголовок: >_ Qwen Code (vX.Y.Z)  │ │
│   │  ███   ART ART  ███  │         │  ② Подзаголовок: «пусто, или переопределение»│
│   │  ███ ASCII QWEN ███  │         │  ③ Статус: Qwen OAuth | qwen-…    │ │
│   │                      │         │  ④ Путь: ~/projects/example        │ │
│   └──────── A ───────────┘         └──────────────── B ──────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                              область: AppHeader
                          │ Компонент Tips отображается ниже (управляется ui.hideTips) │
```

Два верхних блока:

- **A. Колонка логотипа** — один блок ASCII-арта с градиентом. Исходный код
  находится в `shortAsciiLogo` из
  `packages/cli/src/ui/components/AsciiArt.ts`.
- **B. Панель информации** — блок с рамкой, содержащий четыре строки. Вторая
  строка по умолчанию является пустым визуальным разделителем, который может быть заменён на
  подзаголовок, переданный вызывающим кодом:
  - **B①** Заголовок: `>_ Qwen Code (vX.Y.Z)` — текст бренда + суффикс версии.
  - **B②** Подзаголовок / разделитель: по умолчанию пустая строка с одним пробелом. Когда
    задан `ui.customBannerSubtitle`, эта строка отображается (например,
    форк может использовать `Built-in DataWorks Official Skills`).
  - **B③** Статус: `<тип отображения аутентификации> | <модель> ( /model to change)`.
  - **B④** Путь: сокращённая, с тильдой, рабочая директория.

Всё это обёрнуто компонентом `<AppHeader>`, который уже управляет отображением
баннера через `showBanner = !config.getScreenReader()` (режим экранного диктора
возвращается к простому выводу).

## Правила настройки — что можно изменять, что заблокировано

| Область                                      | Текущий источник                    | Категория настройки               | Обоснование                                                                                                                                                                                              |
| --------------------------------------------- | ----------------------------------- | --------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A. Колонка логотипа**                       | `shortAsciiLogo` (`AsciiArt.ts`)    | **Заменяемо + автоматически скрываемо** | Чисто фирменная поверхность. Белый лейбл требует полного контроля над визуальным оформлением. Существующий механизм «авто-скрытие на узких терминалах» сохраняется.                                               |
| **B①. Заголовок — текст бренда** (`>_ Qwen Code`) | Жёстко задан в `Header.tsx`         | **Заменяемо**                     | Фирменная поверхность. Символ `>_` является частью существующего бренда; если пользователь хочет его убрать, достаточно опустить его в `customBannerTitle`.                                                 |
| **B①. Заголовок — суффикс версии** (`(vX.Y.Z)`) | Свойство `version`                  | **Заблокировано**                 | Критично для отчётов об ошибках. Скрытие делает вопрос «какая версия у вас?» отвечаемым только через `--version`, что является реальной затратой в рабочих процессах поддержки. Мы жертвуем небольшим удобством белого лейбла ради отслеживаемости поддержки. |
| **B②. Строка подзаголовка / разделителя**       | По умолчанию пустая                 | **Заменяемо**                     | Чисто фирменная / контекстная поверхность. Используется форками белого лейбла для обозначения сборки (например, «Built-in DataWorks Official Skills»). Очищается так же, как заголовок; только одна строка — никаких разрывов строк, нарушающих макет. |
| **B③. Строка статуса** (аутентификация + модель) | Свойства `formattedAuthType`, `model` | **Заблокировано**                 | Операционный сигнал и сигнал безопасности. Пользователи всегда должны видеть, какие учётные данные используются и какая модель будет тратить их токены. Подавление — это выстрел себе в ногу даже для сценариев белого лейбла. |
| **B④. Строка пути** (рабочая директория)       | Свойство `workingDirectory`        | **Заблокировано**                 | Операционная информация. «В какой директории я нахожусь?» — постоянный вопрос; баннер — его канонический ответ.                                                                                              |
| **Весь баннер** (A + B)                        | Монтаж `<Header>` в `AppHeader.tsx` | **Скрываемо**                     | Одна настройка `ui.hideBanner: true` пропускает обе области — аналогично существующему шлюзу для экранного диктора. `<Tips>` по-прежнему управляется независимо через `ui.hideTips`.                         |

Матрица сводится к четырём настройкам, не более:

| Настройка                   | По умолчанию | Эффект                                                                                                                            | Затрагиваемая область |
| --------------------------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------- | --------------------- |
| `ui.hideBanner`             | `false`      | Скрывает весь баннер (области A + B).                                                                                             | A + B                 |
| `ui.customBannerTitle`      | не задано    | Заменяет текст бренда в B①. Суффикс версии всё равно добавляется. Обрезается; пустая строка означает «использовать значение по умолчанию». | B① текст бренда        |
| `ui.customBannerSubtitle`   | не задано    | Заменяет пустую строку-разделитель B② на однострочный подзаголовок. Очищается; максимум 160 символов; пустая строка означает «оставить пустой разделитель». | B② разделитель         |
| `ui.customAsciiArt`         | не задано    | Заменяет область A. Три принимаемые формы (см. ниже). Возвращается к значению по умолчанию при любой ошибке.                     | A                     |

Что **не** предлагается по дизайну:

- Нет настройки, скрывающей только суффикс версии.
- Нет настройки, скрывающей только строку аутентификации/модели.
- Нет настройки, скрывающей только строку пути.
- Нет настройки, изменяющей цвета градиента логотипа (этим управляет тема).
- Нет настройки, изменяющей порядок или структуру панели информации.

Если в дальнейшей реализации потребуется открыть что-либо из этого, это должны быть
новые поля с собственным обоснованием — а не производные от трёх указанных выше полей.

## Руководство по настройке для пользователя — как изменить

### Ограничения с первого взгляда

К каждой настройке баннера применяется несколько ограничений. Помните о них
перед тем, как создавать изображение вручную, чтобы обработчик не обрезал или не отклонил
ваш ввод.

| Что                             | Ограничение                                                                                                                                                                 |
| -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Количество символов в заголовке** | **Максимум 80 символов** (после очистки). Более длинные обрезаются, и в лог записывается предупреждение `[BANNER]`. Символы новой строки и управляющие символы удаляются до подсчёта длины. |
| **Количество символов в подзаголовке** | **Максимум 160 символов** (после очистки). Та же процедура очистки, что и для заголовка; то же предупреждение `[BANNER]` при обрезании. |
| **Размер блока ASCII-арта**         | **Максимум 200 строк × 200 колонок** для каждого размера. Всё, что больше, обрезается, и в лог записывается предупреждение `[BANNER]`. |
| **Размер файла ASCII-арта на диске** | **Максимум 64 КБ**. Файлы большего размера читаются до этого предела; остальное игнорируется.                                                                             |
| **Отображаемая ширина ASCII-арта** | Определяется количеством колонок терминала при запуске, **а не фиксированным числом символов**. См. «Насколько широким может быть логотип?» ниже для формулы и чисел для разных терминалов. |

**Фиксированного ограничения по количеству символов для ASCII-арта нет** — есть только
указанные выше ограничения по колонкам / строкам и выделенный при запуске бюджет ширины.
Название бренда из 17 символов, которое будет хорошо смотреться в одном шрифте, может потребовать
наложения или более плотного шрифта в другом; ограничивающим фактором является визуальная ширина, а не количество букв.

### Где находятся настройки

Все четыре настройки находятся в разделе `ui` файла `settings.json`. Поддерживаются как
пользовательский уровень (`~/.qwen/settings.json`), так и уровень рабочей области (`.qwen/settings.json` в корне проекта) со стандартным приоритетом объединения
(рабочая область переопределяет пользователя, системный переопределяет рабочую область).

`customAsciiArt` обрабатывается особым образом: вместо того чтобы рассматривать весь объект
как одно значение, которое заменяется в области с более высоким приоритетом,
обработчик проходит по областям для каждого размера. Если пользовательские настройки определяют `{ small }`, а настройки рабочей области определяют `{ large }`, то оба учитываются — `small` от пользователя,
`large` от рабочей области. Это позволяет одновременно работать двум вещам:

1. Каждая запись `{ path }` разрешается относительно файла, объявившего её
   (рабочая область `.qwen/` против пользовательской `~/.qwen/`); только объединённое представление потеряло бы эту информацию об области.
2. Пользователи могут сохранить размер `large` по умолчанию в своих личных настройках и переопределить
   только `small` для каждой рабочей области, не повторяя весь объект.

Когда один и тот же размер определён в нескольких областях, применяется обычный приоритет
(системный > рабочая область > пользователь). Установка `customAsciiArt` в виде простой строки
или `{ path }` в любой области по-прежнему заполняет оба размера в этой области.

### Полностью скрыть баннер

```jsonc
{
  "ui": {
    "hideBanner": true,
  },
}
```

Вывод при запуске пропускает как колонку логотипа, так и панель информации. Советы
по-прежнему отображаются, если только `ui.hideTips` не установлен в `true`.

### Заменить заголовок бренда

```jsonc
{
  "ui": {
    "customBannerTitle": "Acme CLI",
  },
}
```

Отображается как `Acme CLI (vX.Y.Z)` на панели информации. Символ `>_`
удаляется при установке пользовательского заголовка; если вы хотите его вернуть, укажите
его самостоятельно: `"customBannerTitle": ">_ Acme CLI"`.

### Добавить подзаголовок бренда

```jsonc
{
  "ui": {
    "customBannerSubtitle": "Built-in DataWorks Official Skills",
  },
}
```

Отображает подзаголовок на отдельной строке, цветом второстепенного текста, вместо
пустого разделителя, который обычно находится между заголовком и строкой аутентификации/модели:

```
┌─────────────────────────────────────────────────────────┐
│ DataWorks DataAgent (vX.Y.Z)                            │  ← B① заголовок
│ Built-in DataWorks Official Skills                      │  ← B② подзаголовок
│ Qwen OAuth | qwen-coder ( /model to change)             │  ← B③ статус
│ ~/projects/example                                      │  ← B④ путь
└─────────────────────────────────────────────────────────┘
```

Ограничения:

- Только одна строка. Символы новой строки и другие управляющие байты удаляются /
  заменяются на пробелы, чтобы случайная вставка не нарушила макет панели информации.
- Очищается до максимум 160 символов (более мягкое ограничение, чем для заголовка, потому что
  слоганы / строки «на базе» часто бывают немного длиннее).
- Оставьте поле не заданным (или установите пустую строку / пробелы), чтобы
  сохранить существующую пустую строку-разделитель — обратная совместимость по умолчанию.
- Подзаголовок не меняет, какие строки заблокированы; аутентификация, модель
  и рабочая директория всегда видны независимо от состояния подзаголовка.

### Заменить ASCII-арт — встроенная строка

```jsonc
{
  "ui": {
    "customAsciiArt": "  ___  _    _  ____ \n / _ \\| |  / |/ _\\\n| |_| | |__| | __/\n \\___/|____|_|___|",
  },
}
```

Используйте `\n` для вставки переводов строки внутри JSON-строки. Арт отображается
с активной градиентной темой, как и стандартный логотип.

> **Нет под рукой ASCII-арта?** Используйте любой внешний генератор и вставьте
> результат. Самый простой путь — `figlet`:
> `npx figlet -f "ANSI Shadow" "xxxCode" > brand.txt` и затем укажите
> `customAsciiArt: { "path": "./brand.txt" }`. CLI не выполняет
> преобразование текста в арт во время выполнения — см. раздел _Вне области действия_
> о причинах.

### Заменить ASCII-арт — внешний файл

```jsonc
{
  "ui": {
    "customAsciiArt": { "path": "./brand.txt" },
  },
}
```

Позволяет избежать JSON-экранирования многострочной строки. Правила разрешения путей:

- **Настройки рабочей области**: относительные пути разрешаются относительно каталога `.qwen/`
  рабочей области.
- **Пользовательские настройки**: относительные пути разрешаются относительно `~/.qwen/`.
- Абсолютные пути используются как есть.
- Файл читается **один раз при запуске**, очищается и кэшируется. Изменение файла
  в середине сессии не приводит к повторному отображению баннера — перезапустите CLI.

### Заменить ASCII-арт — с учётом ширины

```jsonc
{
  "ui": {
    "customAsciiArt": {
      "small": "  ACME\n  ----",
      "large": { "path": "./brand-wide.txt" },
    },
  },
}
```

`large` используется, когда терминал достаточно широк; в противном случае используется `small`;
в противном случае колонка логотипа скрывается (существующий запасной вариант с двумя колонками).
Любой размер может быть строкой или `{ path }`. Любой размер может быть опущен:
отсутствующий размер просто переходит к следующему шагу.

### Насколько широким может быть логотип? — бюджет ширины

Нет жёсткого ограничения по количеству символов для заголовка или арта. Есть
**бюджет ширины**, определяемый колонками терминала, и абсолютное жёсткое ограничение, чтобы некорректный файл не заморозил макет:

| Параметр                                         | Ограничение                                                             |
| ------------------------------------------------ | ----------------------------------------------------------------------- |
| Колонки терминала при запуске                    | Сколько сообщает терминал пользователя.                                 |
| Внешний отступ контейнера                        | 4 колонки (2 слева + 2 справа).                                        |
| Промежуток между логотипом и панелью информации  | 2 колонки.                                                              |
| Минимальная ширина панели информации             | 44 колонки (40 путь + рамка + отступ).                                 |
| **Доступная ширина логотипа** (для каждого размера, во время рендеринга) | `terminalCols − 4 − 2 − 44 = terminalCols − 50`.                       |
| Жёсткое ограничение на каждый размер арта (после очистки) | 200 колонок × 200 строк. Всё, что больше, обрезается + предупреждение `[BANNER]`. |
| Жёсткое ограничение на `customBannerTitle` (после очистки) | 80 символов. Всё, что больше, обрезается + предупреждение `[BANNER]`.  |

Чтение бюджета при типичной ширине терминала:

| Колонки терминала | Максимальная ширина логотипа при рендеринге | Что это означает на практике                                                 |
| ----------------- | ------------------------------------------- | ----------------------------------------------------------------------------- |
| 80                | 30                                          | Большинство букв figlet «ANSI Shadow» занимают ~7–11 колонок — максимум 3 буквы. |
| 100               | 50                                          | Короткое слово в ANSI Shadow (~6 букв), или два коротких слова друг под другом. |
| 120               | 70                                          | Многострочный словесный арт помещается с запасом.                             |
| 200               | 150                                         | Длинные встроенные строки, такие как полные названия продуктов в ANSI Shadow, помещаются. |

Два практических вывода при создании своего арта:

1. **Многословный бренд чаще всего не отобразится как одна строка ANSI Shadow на большинстве терминалов.** При ~7–9 колонках на букву ANSI Shadow, даже бренд из 12 символов,
   например `Custom Agent`, занимает примерно 95 колонок арта в одной строке — уже больше,
   чем может выделить терминал в 100 колонок рядом с панелью информации. Размещайте слова
   на нескольких строках, выбирайте более плотный шрифт figlet или используйте компактное
   однострочное текстовое оформление, например `▶ Custom Agent ◀`.
2. **Используйте форму с учётом ширины `{ small, large }`**, когда один размер
   вынуждает выбирать между «отлично выглядит на широком / умирает на узком»
   и «нормально выглядит на узком / тратит место впустую на широком». В примере ниже
   слова размещаются друг под другом для терминала шириной ≥104 колонки в `large` и
   переходят к однострочному оформлению из 16 колонок в `small`.

```jsonc
{
  "ui": {
    "customBannerTitle": "Custom Agent",
    "customAsciiArt": {
      "small": "▶ Custom Agent ◀",
      "large": { "path": "./banner-large.txt" },
    },
  },
}
```

Где `banner-large.txt` содержит вывод ANSI Shadow со словами друг под другом
(~54 колонки × 12 строк), например, сгенерированный:

```bash
( npx figlet -f "ANSI Shadow" CUSTOM
  npx figlet -f "ANSI Shadow" AGENT ) > banner-large.txt
```

### Объединить всё

```jsonc
{
  "ui": {
    "hideBanner": false,
    "customBannerTitle": "Acme CLI",
    "customAsciiArt": {
      "small": "  ACME\n  ----",
      "large": { "path": "./brand-wide.txt" },
    },
  },
}
```

### Как проверить изменения

1. Сохраните `settings.json` и запустите новую сессию `qwen` — разрешение баннера
   выполняется один раз при запуске.
2. Измените размер терминала, чтобы убедиться, что размеры `small` / `large` меняются
   как ожидается, и что колонка логотипа исчезает при очень узкой ширине.
3. Если что-то отображается не так, как ожидалось, посмотрите
   `~/.qwen/debug/<sessionId>.txt` (символическая ссылка `latest.txt` указывает
   на текущую сессию) и найдите строки с `[BANNER]` — каждый «мягкий» сбой
   записывает предупреждение с основной причиной.

```
        │                                workingDirectory=… />

   resolveCustomBanner(settings)          ▼
   ┌─────────────────────────┐  packages/cli/src/ui/components/

   │ 5. мемоизация по        │
   │    источнику            │
   └─────────────────────────┘
```
Пятишаговый алгоритм разрешения запускается один раз при загрузке настроек и повторно только при событиях перезагрузки настроек:

1. **Нормализация**. Простая `строка` или `{ path }` преобразуется в `{ small: x, large: x }`. Объект `{ small, large }` проходит без изменений.
2. **Разрешение каждого уровня**. Для каждого `AsciiArtSource`:
   - Если это строка, используется как есть.
   - Если это `{ path }`, файл читается синхронно с защитой `O_NOFOLLOW` (на Windows: просто чтение — константа не раскрыта), с ограничением в 64 КБ. Относительные пути разрешаются относительно _каталога владельца файла настроек_ — настройки рабочей области относительно `.qwen/` рабочей области, настройки пользователя — относительно `~/.qwen/`. Ошибка чтения логирует предупреждение `[BANNER]` и откатывается к значению по умолчанию для этого уровня.
3. **Санитизация**. Баннерный стриппер удаляет лидеры OSC/CSI/SS2/SS3 и заменяет каждый другой управляющий байт C0/C1 (и DEL) на пробел, сохраняя `\n`, чтобы многострочное искусство выжило. Обрезаются завершающие пробелы в каждой строке, затем ограничение 200 строк × 200 столбцов. Всё, что выходит за пределы, обрезается, и логируется предупреждение `[BANNER]`.
4. **Выбор уровня во время рендеринга**. В `Header.tsx`, имея разрешённые `small` и `large`, оценивается существующий бюджет ширины (`availableTerminalWidth ≥ logoWidth + logoGap + minInfoPanelWidth`):
   - Предпочтение отдаётся `large`, если он помещается.
   - Иначе откат к `small`, если он помещается.
   - Иначе, **если пользователь предоставил какое-либо пользовательское искусство**, колонка логотипа полностью скрывается (существующая ветвь `showLogo = false`) — откат к встроенному логотипу QWEN здесь молча отменил бы white-label развёртывание на узких терминалах. Информационная панель всё ещё отображается.
   - Иначе (пользовательское искусство вообще не предоставлено) выполняется проход к `shortAsciiLogo`, и существующий шлюз ширины решает, показывать ли логотип по умолчанию или нет.
5. **Запасной вариант**. Если оба уровня оказались пустыми или недействительными из-за мягких сбоев (отсутствующий файл, санитизация отклонила всё, некорректная конфигурация), вести себя так, как если бы настройки не были изменены: отобразить `shortAsciiLogo` и следовать шлюзу ширины логотипа по умолчанию. CLI никогда не должен падать из-за ошибки конфигурации баннера.

Псевдокод для выбора уровня:

```ts
function pickTier(
  small: string | undefined,
  large: string | undefined,
  availableWidth: number,
  logoGap: number,
  minInfoPanelWidth: number,
): string | undefined {
  for (const candidate of [large, small]) {
    if (!candidate) continue;
    const w = getAsciiArtWidth(candidate);
    if (availableWidth >= w + logoGap + minInfoPanelWidth) {
      return candidate;
    }
  }
  return undefined; // колонка логотипа скрыта
}
```

когда `settings.isTrusted` равно false.

## Рассмотренные альтернативные подходы

Было рассмотрено пять форм этой функции. Они перечислены здесь, чтобы
будущие участники понимали пространство дизайна и могли пересмотреть
выбор, если ограничения изменятся.

### Вариант 1 — Три плоских настройки (РЕКОМЕНДУЕТСЯ, соответствует задаче)

```jsonc
{
  "ui": {
    "customAsciiArt": "...", // string | {path} | {small,large}
    "customBannerTitle": "Acme CLI",
    "hideBanner": false,
  },
}
```

- **Результат**: минимальная пользовательская поверхность; именно то, что
  требуется в задаче.
- **Плюсы**: нулевой порог входа; тривиальная документация; согласовано с
  существующими плоскими свойствами `ui.*` (`hideTips`, `customWittyPhrases`,
  и т.д.).
- **Минусы**: три ключа верхнего уровня, которые концептуально принадлежат
  друг другу, не сгруппированы; будущие баннерные настройки (градиент, подзаголовок)
  добавят больше соседних свойств в `ui` вместо чистого вложения.

### Вариант 2 — Вложенное пространство имён `ui.banner`

```jsonc
{
  "ui": {
    "banner": {
      "hide": false,
      "title": "Acme CLI",
      "asciiArt": { "path": "./brand.txt" },
    },
  },
}
```

- **Результат**: те же возможности, что и в Варианте 1, но сгруппированы по функции.
- **Плюсы**: чистое пространство имён для будущих баннерных настроек; более лёгкое
  обнаружение через `/settings`.
- **Минусы**: расходится с точной формулировкой задачи; существующие настройки UI
  в основном плоские (только `ui.accessibility` и `ui.statusLine`
  вложены), поэтому согласованность неоднозначна; добавляет один уровень вложенности для запоминания.

### Вариант 3 — Пресеты профилей баннера + переопределения слотов

```jsonc
{
  "ui": {
    "bannerProfile": "minimal" | "default" | "branded" | "hidden",
    "banner": { /* переопределения слотов для 'branded' */ }
  }
}
```

- **Результат**: пользователи выбирают из именованных пресетов; продвинутые пользователи
  переопределяют слоты внутри выбранного профиля.
- **Плюсы**: хороший UX для начала работы; пресеты поставляются с CLI.
- **Минусы**: значительная сложность; пресеты требуют поддержки; задача просит
  прямую настройку, а не курацию.

### Вариант 4 — Полное переопределение баннера (один строковый шаблон)

```jsonc
{
  "ui": {
    "bannerTemplate": "{{logo}}\n>_ {{title}} ({{version}})\n{{auth}} | {{model}}\n{{path}}",
  },
}
```

- **Результат**: один свободный шаблон с фиксированными переменными, которые заполняются.
- **Плюсы**: максимальная гибкость для нестандартных макетов.
- **Минусы**: переопределяет макет в пользовательском пространстве; теряет двухколоночную
  устойчивость Ink к ширине терминала; очень легко написать шаблон, который
  ломается на узких терминалах; большая площадь воздействия для небольшой функции.

### Вариант 5 — Плагин / API-хук

Предоставить хук рендеринга баннера через систему расширений.

- **Результат**: настройка на уровне кода; расширения могут рендерить что угодно.
- **Плюсы**: максимальная мощность; позволяет предприятиям поставлять закрытый
  брендирующий плагин.
- **Минусы**: большая площадь API; требуется проверка безопасности для произвольного
  рендеринга терминала; значительно выходит за рамки задачи.

### Рекомендация

**Вариант 1** рекомендуется. Он буквально соответствует задаче,
вписывается в существующий стиль `ui.*` и избегает принятия решения
о вложенном пространстве имён до того, как мы узнаем, как на самом деле будут выглядеть
другие баннерные настройки. Если в будущем начнут накапливаться соседние свойства, миграция
на Вариант 2 является аддитивной — `ui.banner.title` и `ui.customBannerTitle` могут
сосуществовать в течение периода устаревания.

## Безопасность и обработка сбоев

Пользовательское содержимое баннера отображается буквально в терминале И, в
форме пути, читается с диска. Обе поверхности достижимы для атаки, если
загружен враждебный или скомпрометированный файл настроек. Та же модель угроз,
которая применяется к функции заголовка сессии, действует и здесь.

| Проблема                                                    | Защита                                                                                                                                                                                                                                                                         |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Внедрение ANSI/OSC-8/CSI в искусство, заголовок или подзаголовок | Баннерный стриппер (`sanitizeArt` / `sanitizeSingleLine`): удаляет лидеры OSC/CSI/SS2/SS3 и заменяет каждый другой управляющий байт C0/C1 (и DEL) на пробел. Применяется до рендеринга и записи кэша.                                                                       |
| Чрезмерно большой файл замораживает запуск                   | Жёсткое ограничение 64 КБ на чтение файлов.                                                                                                                                                                                                                                  |
| Патологическое искусство замораживает макет                  | Ограничение 200 строк × 200 столбцов для каждой разрешённой строки. Превышение обрезается; логируется предупреждение `[BANNER]`.                                                                                                                                              |
| Перенаправление симлинка в форме пути                       | `O_NOFOLLOW` при чтении файлов (на Windows: просто чтение; константа не раскрыта).                                                                                                                                                                                             |
| Отсутствующий или нечитаемый файл                           | Перехват, логирование предупреждения `[BANNER]`, откат к значению по умолчанию. Никогда не выбрасывать ошибку в UI.                                                                                                                                                           |
| Заголовок или подзаголовок с переносами строк / избыточной длиной | Переносы строк преобразуются в пробелы; ограничение 80 (заголовок) / 160 (подзаголовок) символов.                                                                                                                                                                           |
| Ненадёжная рабочая область влияет на рендеринг или чтение файлов | Когда `settings.isTrusted` равно false, резолвер полностью пропускает `settings.workspace` (повторяет шлюз доверия, который применяет `settings.merged`).                                                                                                                      |
| Состояние гонки при перезагрузке настроек                    | Разрешение мемоизируется по источнику (путь или хеш строки) для каждого вызова. Перезагрузки повторно запускают резолвер и перечитывают затронутые файлы.                                                                                                                     |

Сводка сценариев сбоев: каждый мягкий сбой заканчивается `shortAsciiLogo` (или
заблокированным заголовком по умолчанию) плюс предупреждение в отладочном логе. Жёсткие сбои
(выброшенные ошибки) не допускаются ни в одной ветви резолвера.

## Вне области

Эти пункты были рассмотрены и намеренно отложены. Каждый может быть отдельным
последующим улучшением, если возникнет пользовательский спрос.

| Пункт                                                              | Почему нет                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

## План верификации

Для финального PR с реализацией должны проходить следующие сквозные проверки.

1. `~/.qwen/settings.json` с `customBannerTitle: "Acme CLI"` и строчным `customAsciiArt` → `qwen` показывает новый заголовок и арт; суффикс версии всё ещё присутствует.
2. `customBannerSubtitle: "Built-in Acme Skills"` → строка подзаголовка отображается между заголовком и строкой аутентификации/модели цветом второстепенного текста; аутентификация, модель и путь всё ещё видны. Сброс этого параметра восстанавливает пустую строку-разделитель (обратная совместимость).
3. `hideBanner: true` → `qwen` запускается без баннера; советы и чат отображаются нормально.
4. `customAsciiArt: { "path": "./brand.txt" }` в рабочем пространстве `settings.json`, где `brand.txt` находится рядом в `.qwen/` → загружается с диска при открытии рабочего пространства.
5. `customAsciiArt: { "small": "...", "large": "..." }` → измените размер терминала между широким/средним/узким; large при широких размерах, small при средних, столбец логотипа скрыт при узких, информационная панель всегда видна.
6. Inject `\x1b[31mhostile` в `customBannerTitle` _и_ `customBannerSubtitle` → оба отображаются как буквальный текст, не интерпретируются как красный.
7. Указать `path` на отсутствующий файл → CLI запускается; предупреждение `[BANNER]` появляется в `~/.qwen/debug/<sessionId>.txt`; отображается стандартный арт.
8. Открыть рабочее дерево с отключённым доверием к рабочему пространству → определённый в рабочем пространстве `customAsciiArt` (включая записи `{ path }`) молча игнорируется; настройки области пользователя всё ещё применяются.
9. `npm test` и `npm run typecheck` проходят для пакета CLI; модульные тесты в `customBanner.test.ts` покрывают каждую допустимую форму и каждый путь отказа (отсутствующий файл, слишком большой файл, внедрение ANSI, некорректный объект).
