# Начало работы с расширениями Qwen Code

В этом руководстве вы создадите своё первое расширение Qwen Code. Вы научитесь настраивать новое расширение, добавлять пользовательский инструмент через сервер MCP, создавать пользовательскую команду и предоставлять контекст модели с помощью файла `QWEN.md`.

## Предварительные требования

Прежде чем начать, убедитесь, что у вас установлен Qwen Code и есть базовое понимание Node.js и TypeScript.

## Шаг 1. Создание нового расширения

Самый простой способ начать — использовать один из встроенных шаблонов. В качестве основы мы возьмём пример `mcp-server`.

Выполните следующую команду, чтобы создать новый каталог `my-first-extension` с файлами шаблона:

```bash
qwen extensions new my-first-extension mcp-server
```

Будет создан новый каталог со следующей структурой:

```
my-first-extension/
├── example.ts
├── qwen-extension.json
├── package.json
└── tsconfig.json
```

## Шаг 2. Изучите файлы расширения

Рассмотрим ключевые файлы в вашем новом расширении.

### `qwen-extension.json`

Это манифест-файл вашего расширения. Он сообщает Qwen Code, как загружать и использовать ваше расширение.

```json
{
  "name": "my-first-extension",
  "version": "1.0.0",
  "mcpServers": {
    "nodeServer": {
      "command": "node",
      "args": ["${extensionPath}${/}dist${/}example.js"],
      "cwd": "${extensionPath}"
    }
  }
}
```

- `name`: Уникальное имя вашего расширения.
- `version`: Версия вашего расширения.
- `mcpServers`: В этом разделе определяется один или несколько серверов протокола контекста моделей (Model Context Protocol, MCP). Серверы MCP позволяют добавлять в модель новые инструменты.
  - `command`, `args`, `cwd`: Эти поля задают способ запуска вашего сервера. Обратите внимание на использование переменной `${extensionPath}`, которую Qwen Code заменяет абсолютным путём к каталогу установки вашего расширения. Это обеспечивает работоспособность расширения независимо от места его установки.

### `example.ts`

Этот файл содержит исходный код вашего сервера MCP. Это простой сервер Node.js, использующий пакет `@modelcontextprotocol/sdk`.

```typescript
/**
 * @license
 * Copyright 2025 Google LLC
 * SPDX-License-Identifier: Apache-2.0
 */

import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { z } from 'zod';

const server = new McpServer({
  name: 'prompt-server',
  version: '1.0.0',
});

// Регистрирует новый инструмент с именем 'fetch_posts'
server.registerTool(
  'fetch_posts',
  {
    description: 'Получает список постов из публичного API.',
    inputSchema: z.object({}).shape,
  },
  async () => {
    const apiResponse = await fetch(
      'https://jsonplaceholder.typicode.com/posts',
    );
    const posts = await apiResponse.json();
    const response = { posts: posts.slice(0, 5) };
    return {
      content: [
        {
          type: 'text',
          text: JSON.stringify(response),
        },
      ],
    };
  },
);

// ... (регистрация шаблонов пропущена для краткости)

const transport = new StdioServerTransport();
await server.connect(transport);
```

Этот сервер определяет один инструмент — `fetch_posts`, который получает данные из публичного API.

### `package.json` и `tsconfig.json`

Это стандартные файлы конфигурации для проекта на TypeScript. В файле `package.json` определяются зависимости и скрипт сборки `build`, а в файле `tsconfig.json` настраивается компилятор TypeScript.

## Шаг 3. Сборка и привязка расширения

Прежде чем использовать расширение, необходимо скомпилировать код на TypeScript и привязать расширение к вашей локальной установке Qwen Code для разработки.

1.  **Установите зависимости:**

    ```bash
    cd my-first-extension
    npm install
    ```

2.  **Соберите сервер:**

    ```bash
    npm run build
    ```

    В результате файл `example.ts` будет скомпилирован в `dist/example.js` — именно этот файл указан в вашем файле `qwen-extension.json`.

3.  **Привяжите расширение:**

    Команда `link` создаёт символьную ссылку из каталога расширений Qwen Code в ваш каталог разработки. Это означает, что любые изменения, внесённые вами, будут отражаться немедленно без необходимости повторной установки.

    ```bash
    qwen extensions link .
    ```

Теперь перезапустите сеанс Qwen Code. Новый инструмент `fetch_posts` станет доступен. Вы можете протестировать его, задав вопрос: «fetch posts».

## Шаг 4. Добавление пользовательской команды

Пользовательские команды позволяют создавать ярлыки для сложных запросов. Добавим команду, выполняющую поиск шаблона в вашем коде.

1.  Создайте каталог `commands` и подкаталог для группы команд:

    ```bash
    mkdir -p commands/fs
    ```

2.  Создайте файл `commands/fs/grep-code.md`:

    ```markdown
    ---
    description: Поиск шаблона в коде и краткое резюме результатов
    ---

    Пожалуйста, кратко резюмируйте результаты поиска шаблона `{{args}}`.

    Результаты поиска:
    !{grep -r {{args}} .}
    ```

    Эта команда `/fs:grep-code` примет аргумент, выполнит в оболочке команду `grep` с этим аргументом и передаст результаты в запрос для краткого резюмирования.

> **Примечание:** Команды используют формат Markdown с необязательным YAML-фронтматтером. Формат TOML устарел, но по-прежнему поддерживается для обратной совместимости.

После сохранения файла перезапустите Qwen Code. Теперь вы можете выполнить `/fs:grep-code "некоторый шаблон"`, чтобы воспользоваться новой командой.

## Шаг 5. Добавление пользовательских навыков и подагентов (необязательно)

Расширения также могут предоставлять пользовательские навыки и подагенты для расширения возможностей Qwen Code.

### Добавление пользовательского навыка

Навыки — это функции, вызываемые моделью, которые ИИ может автоматически использовать при необходимости.

1.  Создайте каталог `skills` с подкаталогом для навыка:

    ```bash
    mkdir -p skills/code-analyzer
    ```

2.  Создайте файл `skills/code-analyzer/SKILL.md`:

    ```markdown
    ---
    name: code-analyzer
    description: Анализирует структуру кода и предоставляет информацию о его сложности, зависимостях и потенциальных улучшениях
    ---

    # Анализатор кода

    ## Инструкции

    При анализе кода сосредоточьтесь на следующем:

    - Сложность кода и удобство его сопровождения
    - Зависимости и связанность компонентов
    - Потенциальные проблемы производительности
    - Рекомендации по улучшению кода

    ## Примеры

    - «Проанализируй сложность этой функции»
    - «Какие зависимости у этого модуля?»
    ```

### Добавление пользовательского субагента

Субагенты — это специализированные ИИ-ассистенты для выполнения конкретных задач.

1.  Создайте каталог `agents`:

    ```bash
    mkdir -p agents
    ```

2.  Создайте файл `agents/refactoring-expert.md`:

    ```markdown
    ---
    name: refactoring-expert
    description: Специализируется на рефакторинге кода, улучшении его структуры и поддерживаемости
    tools:
      - read_file
      - write_file
      - read_many_files
    ---

    Вы — специалист по рефакторингу, ориентированный на повышение качества кода.

    Ваша экспертиза включает:

    - выявление «запахов кода» (code smells) и антипаттернов,
    - применение принципов SOLID,
    - повышение читаемости и поддерживаемости кода,
    - безопасный рефакторинг с минимальным риском.

    Для каждой задачи рефакторинга:

    1. Проанализируйте текущую структуру кода.
    2. Определите области, требующие улучшения.
    3. Предложите шаги по рефакторингу.
    4. Постепенно внедрите изменения.
    5. Убедитесь, что функциональность сохранена.
    ```

После перезапуска Qwen Code ваши пользовательские навыки станут доступны через `/skills`, а субагенты — через `/agents manage`.

## Шаг 6. Добавьте пользовательский файл `QWEN.md`

Вы можете предоставить модели постоянный контекст, добавив в расширение файл `QWEN.md`. Это полезно для передачи модели инструкций о том, как ей следует вести себя, или информации об инструментах вашего расширения. Обратите внимание, что для расширений, созданных исключительно для предоставления команд и подсказок, этот файл может быть необязателен.

1.  Создайте файл с именем `QWEN.md` в корневой директории вашего расширения:

    ```markdown
    # Инструкции для моего первого расширения

    Вы — эксперт-ассистент разработчика. Когда пользователь просит получить посты, используйте инструмент `fetch_posts`. Отвечайте кратко.
    ```

2.  Обновите файл `qwen-extension.json`, чтобы CLI загружал этот файл:

    ```json
    {
      "name": "my-first-extension",
      "version": "1.0.0",
      "contextFileName": "QWEN.md",
      "mcpServers": {
        "nodeServer": {
          "command": "node",
          "args": ["${extensionPath}${/}dist${/}example.js"],
          "cwd": "${extensionPath}"
        }
      }
    }
    ```

Снова перезапустите CLI. Теперь модель будет использовать контекст из вашего файла `QWEN.md` в каждой сессии, где активно это расширение.

## Шаг 7. Публикация расширения

Когда вы останетесь довольны своим расширением, его можно опубликовать и поделиться им с другими. Основные способы публикации расширений — через Git-репозиторий или через GitHub Releases. Использование публичного Git-репозитория — самый простой способ.

Подробные инструкции по обоим методам см. в [Руководстве по публикации расширений](extension-releasing.md).

Вы успешно создали расширение для Qwen Code! Вы узнали, как:

- Инициализировать новое расширение на основе шаблона.
- Добавлять пользовательские инструменты с помощью сервера MCP.
- Создавать удобные пользовательские команды.
- Добавлять пользовательские навыки и подагентов.
- Предоставлять модели постоянный контекст.
- Подключать своё расширение для локальной разработки.

Далее вы можете изучить более продвинутые возможности и добавить в Qwen Code мощные новые функции.
