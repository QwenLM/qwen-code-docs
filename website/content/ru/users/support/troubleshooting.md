# Устранение неполадок

В этом руководстве приведены решения распространённых проблем и советы по отладке, в том числе:

- Ошибки аутентификации или входа в систему
- Часто задаваемые вопросы (ЧАВО)
- Советы по отладке
- Поиск похожих проблем в GitHub Issues или создание новых Issues

## Ошибки аутентификации или входа в систему

- **Ошибка: `Qwen OAuth free tier was discontinued on 2026-04-15`**
  - **Причина:** Бесплатный уровень Qwen OAuth больше не доступен с 15 апреля 2026 года.
  - **Решение:** Переключитесь на другой способ аутентификации. Выполните `qwen` → `/auth` и выберите один из вариантов:
    - **API-ключ**: Используйте API-ключ от Alibaba Cloud Model Studio ([Beijing](https://bailian.console.aliyun.com/) / [intl](https://modelstudio.console.alibabacloud.com/)). См. руководство по настройке API ([Beijing](https://bailian.console.aliyun.com/cn-beijing/?tab=doc#/doc/?type=model&url=3023091) / [intl](https://modelstudio.console.alibabacloud.com/ap-southeast-1?tab=doc#/doc/?type=model&url=2974721)).
    - **Alibaba Cloud Coding Plan**: Подпишитесь на ежемесячную фиксированную плату с более высокими квотами. См. руководство по Coding Plan ([Beijing](https://bailian.console.aliyun.com/cn-beijing/?tab=coding-plan#/efm/coding-plan-index) / [intl](https://modelstudio.console.alibabacloud.com/?tab=coding-plan#/efm/coding-plan-index)).

- **Ошибка: `UNABLE_TO_GET_ISSUER_CERT_LOCALLY`, `UNABLE_TO_VERIFY_LEAF_SIGNATURE` или `unable to get local issuer certificate`**
  - **Причина:** Возможно, вы находитесь в корпоративной сети с файрволом, который перехватывает и проверяет SSL/TLS-трафик. Для этого часто требуется, чтобы Node.js доверял пользовательскому корневому сертификату CA.
  - **Решение:** Установите переменную окружения `NODE_EXTRA_CA_CERTS` с абсолютным путём к файлу корневого сертификата CA вашей корпоративной сети.
    - Пример: `export NODE_EXTRA_CA_CERTS=/path/to/your/corporate-ca.crt`

- **Ошибка: `Device authorization flow failed: fetch failed`**
  - **Причина:** Node.js не смог связаться с конечными точками Qwen OAuth (часто проблема с прокси или доверием SSL/TLS). Если доступно, Qwen Code также выводит основную причину ошибки (например: `UNABLE_TO_VERIFY_LEAF_SIGNATURE`). Примечание: эта ошибка характерна для устаревшего потока Qwen OAuth.
  - **Решение:**
    - Если вы всё ещё используете Qwen OAuth, переключитесь на API-ключ или Coding Plan через `/auth`.
    - Если вы работаете через прокси, настройте его с помощью `qwen --proxy <url>` (или настройки `proxy` в `settings.json`).
    - Если в вашей сети используется корпоративный сертификат TLS-проверки, задайте `NODE_EXTRA_CA_CERTS`, как описано выше.

- **Проблема: Невозможно отобразить UI после сбоя аутентификации**
  - **Причина:** Если аутентификация не удалась после выбора типа аутентификации, настройка `security.auth.selectedType` может сохраниться в `settings.json`. При перезапуске CLI может застрять, пытаясь аутентифицироваться с неудачным типом, и не сможет отобразить UI.
  - **Решение:** Удалите поле `security.auth.selectedType` в вашем файле `settings.json`:
    - Откройте `~/.qwen/settings.json` (или `./.qwen/settings.json` для настроек проекта)
    - Удалите поле `security.auth.selectedType`
    - Перезапустите CLI, чтобы он снова предложил выбрать способ аутентификации

## Часто задаваемые вопросы (ЧАВО)

- **В: Как обновить Qwen Code до последней версии?**
  - **О:** Если вы установили Qwen Code с помощью автономного установщика, выполните команду установки заново. Если вы установили его глобально через `npm`, обновите его командой `npm install -g @qwen-code/qwen-code@latest`. Если вы скомпилировали его из исходного кода, загрузите последние изменения из репозитория и пересоберите командой `npm run build`.

- **В: Где хранятся файлы конфигурации или настроек Qwen Code?**
  - **О:** Конфигурация Qwen Code хранится в двух файлах `settings.json`:
    1. В вашем домашнем каталоге: `~/.qwen/settings.json`.
    2. В корневом каталоге вашего проекта: `./.qwen/settings.json`.

    Дополнительные сведения см. в [Конфигурации Qwen Code](../configuration/settings).

- **В: Почему в выводе статистики не отображается количество кэшированных токенов?**
  - **О:** Информация о кэшированных токенах отображается только при их использовании. Эта функция доступна для пользователей API-ключей (например, API-ключ Alibaba Cloud Model Studio или Google Cloud Vertex AI). Вы по-прежнему можете просматривать общее использование токенов с помощью команды `/stats`.

## Распространённые сообщения об ошибках и их решения

- **Ошибка: `EADDRINUSE` (адрес уже используется) при запуске MCP-сервера.**
  - **Причина:** Другой процесс уже использует порт, к которому пытается привязаться MCP-сервер.
  - **Решение:**
    Остановите другой процесс, использующий порт, или настройте MCP-сервер на другой порт.

- **Ошибка: Команда не найдена (при попытке запустить Qwen Code с помощью `qwen`).**
  - **Причина:** CLI не установлен корректно или не находится в вашем `PATH`.
  - **Решение:**
    Обновление зависит от способа установки:
    - Если вы установили `qwen` с помощью автономного установщика, выполните команду установки заново и откройте новое окно терминала.
    - Если вы установили `qwen` глобально, проверьте, что каталог глобальных бинарных файлов `npm` находится в вашем `PATH`. Вы можете обновить командой `npm install -g @qwen-code/qwen-code@latest`.
    - Если вы запускаете `qwen` из исходного кода, убедитесь, что используете правильную команду для его вызова (например, `node packages/cli/dist/index.js ...`). Для обновления загрузите последние изменения из репозитория и пересоберите командой `npm run build`.

- **Ошибка: `MODULE_NOT_FOUND` или ошибки импорта.**
  - **Причина:** Зависимости установлены некорректно или проект не собран.
  - **Решение:**
    1.  Выполните `npm install`, чтобы убедиться, что все зависимости присутствуют.
    2.  Выполните `npm run build` для компиляции проекта.
    3.  Проверьте, что сборка прошла успешно, с помощью `npm run start`.

- **Ошибка: "Operation not permitted", "Permission denied" или подобные.**
  - **Причина:** Если изоляция (sandbox) включена, Qwen Code может пытаться выполнить операции, запрещённые вашей конфигурацией изоляции, например запись за пределами каталога проекта или системного временного каталога.
  - **Решение:** Обратитесь к документации [Конфигурация: Изоляция](../features/sandbox) для получения дополнительной информации, включая настройку конфигурации изоляции.

- **Qwen Code не запускается в интерактивном режиме в средах "CI"**
  - **Проблема:** Qwen Code не входит в интерактивный режим (приглашение не появляется), если задана переменная окружения, начинающаяся с `CI_` (например, `CI_TOKEN`). Это происходит потому, что пакет `is-in-ci`, используемый базовым UI-фреймворком, обнаруживает эти переменные и предполагает неинтерактивную среду CI.
  - **Причина:** Пакет `is-in-ci` проверяет наличие `CI`, `CONTINUOUS_INTEGRATION` или любой переменной окружения с префиксом `CI_`. При обнаружении любой из них считается, что среда неинтерактивна, что предотвращает запуск CLI в интерактивном режиме.
  - **Решение:** Если переменная с префиксом `CI_` не нужна для работы CLI, вы можете временно удалить её для команды. Например: `env -u CI_TOKEN qwen`

- **Режим DEBUG не работает из файла .env проекта**
  - **Проблема:** Установка `DEBUG=true` в файле `.env` проекта не включает режим отладки для CLI.
  - **Причина:** Переменные `DEBUG` и `DEBUG_MODE` автоматически исключаются из файлов `.env` проекта, чтобы предотвратить влияние на поведение CLI.
  - **Решение:** Используйте вместо этого файл `.qwen/.env` или настройте параметр `advanced.excludedEnvVars` в вашем `settings.json`, чтобы исключать меньше переменных.

- **Прокрутка трекпадом в tmux изменяет историю команд вместо прокрутки диалога**
  - **Проблема:** В сессии tmux прокрутка трекпадом или колёсиком мыши может переключать предыдущие команды, как при нажатии `Up Arrow` или `Down Arrow`.
  - **Причина:** tmux может преобразовывать жесты колёсика в обычные последовательности клавиш со стрелками. Эти последовательности неотличимы от реальных нажатий клавиш со стрелками на момент получения их Qwen Code.
  - **Решение:** Включите `ui.useTerminalBuffer`; затем используйте `Shift+Up` / `Shift+Down` или колёсико мыши, если tmux перенаправляет события колёсика в приложение. Если вы предпочитаете прокрутку в терминале хоста, настройте привязки мыши tmux для событий колёсика.

## Компаньон IDE не подключается

- Убедитесь, что в VS Code открыта одна рабочая папка.
- Перезапустите встроенный терминал после установки расширения, чтобы он унаследовал:
  - `QWEN_CODE_IDE_WORKSPACE_PATH`
  - `QWEN_CODE_IDE_SERVER_PORT`
- Если работаете в контейнере, проверьте, что `host.docker.internal` разрешается. В противном случае укажите соответствующий хост.
- Переустановите компаньон с помощью `/ide install` и используйте "Qwen Code: Run" в палитре команд, чтобы проверить его запуск.

## Коды завершения

Qwen Code использует определённые коды завершения, указывающие на причину завершения. Это особенно полезно для скриптов и автоматизации.

| Код завершения | Тип ошибки                 | Описание                                                                                         |
| -------------- | -------------------------- | ------------------------------------------------------------------------------------------------ |
| 41             | `FatalAuthenticationError` | Произошла ошибка в процессе аутентификации.                                                      |
| 42             | `FatalInputError`          | CLI были переданы неверные или отсутствующие входные данные (только неинтерактивный режим).         |
| 44             | `FatalSandboxError`        | Произошла ошибка в среде изоляции (например, Docker, Podman или Seatbelt).                        |
| 52             | `FatalConfigError`         | Файл конфигурации (`settings.json`) недействителен или содержит ошибки.                           |
| 53             | `FatalTurnLimitedError`    | Достигнуто максимальное количество оборотов диалога для сессии (только неинтерактивный режим).    |

## Советы по отладке

- **Отладка CLI:**
  - Используйте флаг `--verbose` (если доступен) с командами CLI для более подробного вывода.
  - Проверьте логи CLI, которые часто находятся в каталоге конфигурации или кэша пользователя.

- **Отладка ядра:**
  - Проверьте вывод консоли сервера на наличие сообщений об ошибках или стеков вызовов.
  - Увеличьте уровень детализации логов, если это настраивается.
  - Используйте инструменты отладки Node.js (например, `node --inspect`), если необходимо пройтись по серверному коду.

- **Проблемы с инструментами:**
  - Если конкретный инструмент не работает, попробуйте изолировать проблему, выполнив простейшую версию команды или операции, которую он выполняет.
  - Для `run_shell_command` сначала проверьте, что команда работает напрямую в вашей оболочке.
  - Для _инструментов файловой системы_ убедитесь, что пути верны, и проверьте права доступа.

- **Предварительные проверки:**
  - Всегда запускайте `npm run preflight` перед фиксацией кода. Это может выявить множество распространённых проблем, связанных с форматированием, линтингом и ошибками типов.
