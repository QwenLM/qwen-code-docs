# Запуск запросов по расписанию

> Используйте `/loop` и инструменты планирования cron для многократного выполнения запросов, опроса статуса или установки одноразовых напоминаний в сессии Qwen Code.

Запланированные задачи позволяют Qwen Code автоматически повторно выполнять запрос через заданный интервал. Используйте их для опроса развёртывания, отслеживания PR, проверки длительной сборки или чтобы напомнить себе о чём-то позже в ходе сессии.

Задачи ограничены сессией: они живут в текущем процессе Qwen Code и исчезают при выходе. Ничего не записывается на диск.

> **Совет:** Запланированные задачи включены по умолчанию. Чтобы отключить их, установите `experimental.cron: false` в ваших [настройках](../configuration/settings.md) или задайте переменную окружения `QWEN_CODE_DISABLE_CRON=1`.

## Запланировать повторяющийся запрос с помощью /loop

Встроенный [скилл](skills.md) `/loop` — самый быстрый способ запланировать повторяющийся запрос. Укажите опциональный интервал и запрос, и Qwen Code настроит задачу cron, которая будет выполняться в фоне, пока сессия открыта.

```text
/loop 5m проверь, завершилось ли развёртывание, и расскажи, что произошло
```

Qwen Code разбирает интервал, преобразует его в cron-выражение, планирует задачу и подтверждает периодичность и ID задачи. Затем он сразу же выполняет запрос один раз — вам не нужно ждать первого срабатывания cron.

### Синтаксис интервала

Интервал необязателен. Его можно указать в начале, в конце или опустить вовсе.

| Форма                     | Пример                                    | Разобранный интервал     |
| :------------------------ | :---------------------------------------- | :----------------------- |
| Ведущий токен             | `/loop 30m проверь сборку`                | каждые 30 минут          |
| Завершающая конструкция every | `/loop проверь сборку every 2 hours`   | каждые 2 часа            |
| Без интервала             | `/loop проверь сборку`                    | по умолчанию каждые 10 минут |

Поддерживаемые единицы: `s` (секунды), `m` (минуты), `h` (часы), `d` (дни). Секунды округляются до ближайшей минуты, так как cron имеет минутную точность. Интервалы, которые делятся на свою единицу неравномерно, например `7m` или `90m`, округляются до ближайшего чистого интервала, и Qwen Code сообщает вам выбранное значение.

### Зациклить другую команду

Запланированный запрос сам может быть командой или вызовом скилла. Это удобно для повторного выполнения уже оформленного рабочего процесса.

```text
/loop 20m /review-pr 1234
```

Каждый раз при срабатывании задачи Qwen Code выполняет `/review-pr 1234` так, как если бы вы ввели эту команду сами.

### Управление циклами

`/loop` также поддерживает две подкоманды для управления существующими задачами:

```text
/loop list
```

Выводит список всех запланированных задач с их ID и cron-выражениями.

```text
/loop clear
```

Отменяет все запланированные задачи за один раз.

## Установить одноразовое напоминание

Для одноразовых напоминаний опишите желаемое на естественном языке вместо использования `/loop`. Qwen Code запланирует задачу с однократным срабатыванием, которая удалится после выполнения.

```text
напомни мне в 15:00 отправить ветку релиза
```

```text
через 45 минут проверь, прошли ли интеграционные тесты
```

Qwen Code привязывает время срабатывания к определённой минуте и часу с помощью cron-выражения и подтверждает, когда оно сработает.

## Управление запланированными задачами

Попросите Qwen Code на естественном языке вывести список задач или отменить их, либо напрямую обращайтесь к соответствующим инструментам.

```text
какие у меня есть запланированные задачи?
```

```text
отмени задачу проверки развёртывания
```

Под капотом Qwen Code использует следующие инструменты:

| Инструмент   | Назначение                                                                                                                 |
| :----------- | :------------------------------------------------------------------------------------------------------------------------- |
| `CronCreate` | Запланировать новую задачу. Принимает cron-выражение из 5 полей, запрос для выполнения и признак повторяемости (один раз или по расписанию). |
| `CronList`   | Вывести список всех запланированных задач с их ID, расписанием и запросами.                                                |
| `CronDelete` | Отменить задачу по ID.                                                                                                     |

Каждая запланированная задача имеет 8-символьный ID, который можно передать в `CronDelete`. В сессии может одновременно храниться до 50 запланированных задач.

## Как выполняются запланированные задачи

Планировщик проверяет наличие задач, готовых к выполнению, каждую секунду и ставит их в очередь, когда сессия простаивает. Запланированный запрос выполняется между вашими ходами, а не в тот момент, когда Qwen Code находится в середине ответа. Если Qwen Code занят в момент наступления задачи, запрос ожидает завершения текущего хода.

Всё время интерпретируется в вашем локальном часовом поясе. Cron-выражение вида `0 9 * * *` означает 9 утра там, где вы запустили Qwen Code, а не по UTC.

### Джиттер

Чтобы избежать одновременного обращения всех сессий к API в один и тот же момент времени, планировщик добавляет небольшой детерминированный сдвиг к моментам срабатывания:

- **Повторяющиеся задачи** срабатывают с задержкой до 10% от своего периода, но не более 15 минут. Ежечасовая задача может сработать в любое время от `:00` до `:06`.
- **Одноразовые задачи**, запланированные на начало или середину часа (минута `:00` или `:30`), срабатывают с опережением до 90 секунд.
Смещение вычисляется на основе ID задачи, поэтому одна и та же задача всегда получает одно и то же смещение. Если точное время важно, выберите минуту, отличную от `:00` или `:30`, например `3 9 * * *` вместо `0 9 * * *` — в этом случае однократный разброс применяться не будет.

### Истечение через три дня

Повторяющиеся задачи автоматически истекают через 3 дня после создания. Задача срабатывает последний раз, а затем удаляет себя. Это ограничивает время работы забытого цикла. Если вам нужно, чтобы повторяющаяся задача работала дольше, отмените её и создайте заново до истечения срока.

Одноразовые задачи не истекают по таймеру — они просто удаляют себя после однократного срабатывания.

## Справочник по cron-выражениям

`CronCreate` принимает стандартные 5-польные cron-выражения: `минута час день-месяца месяц день-недели`. Все поля поддерживают подстановочные знаки (`*`), одиночные значения (`5`), шаги (`*/15`), диапазоны (`1-5`) и списки, разделённые запятыми (`1,15,30`).

| Пример          | Значение                          |
| :-------------- | :-------------------------------- |
| `*/5 * * * *`   | Каждые 5 минут                    |
| `0 * * * *`     | Каждый час ровно                 |
| `7 * * * *`     | Каждый час в 7 минут              |
| `0 9 * * *`     | Каждый день в 9:00 по местному    |
| `0 9 * * 1-5`   | По будням в 9:00 по местному      |
| `30 14 15 3 *`  | 15 марта в 14:30 по местному      |

День недели использует `0` или `7` для воскресенья, от `1` до `6` для остальных дней (понедельник–суббота). Когда ограничены и день месяца, и день недели (ни один из них не равен `*`), дата считается подходящей, если совпадает хотя бы одно из полей — это соответствует стандартной семантике vixie-cron.

Расширенный синтаксис, такой как `L`, `W`, `?` и текстовые псевдонимы наподобие `MON` или `JAN`, не поддерживается.

## Ограничения

Планирование в рамках сеанса имеет свои ограничения:

- Задачи срабатывают только пока Qwen Code запущен и находится в режиме ожидания. Закрытие терминала или завершение сеанса отменяет все задачи.
- Пропущенные срабатывания не навёрстываются. Если запланированное время задачи наступило, пока Qwen Code занят длительным запросом, задача сработает один раз, когда Qwen Code снова станет свободным, но не по одному разу за каждый пропущенный интервал.
- Нет сохранения между перезапусками. Перезапуск Qwen Code удаляет все задачи, привязанные к сеансу.
