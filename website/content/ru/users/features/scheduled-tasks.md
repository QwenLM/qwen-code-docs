# Запуск промптов по расписанию

> Используйте `/loop` и инструменты планирования cron для многократного запуска промптов, опроса статуса или установки разовых напоминаний в рамках сессии Qwen Code.

Запланированные задачи позволяют Qwen Code автоматически перезапускать промпт через заданные интервалы. Используйте их для опроса статуса деплоя, отслеживания PR, проверки длительных сборок или напоминания себе о необходимости сделать что-то позже в рамках текущей сессии.

Задачи привязаны к сессии: они существуют только в текущем процессе Qwen Code и удаляются при выходе. На диск ничего не записывается.

> **Совет:** Запланированные задачи включены по умолчанию. Чтобы отключить их, установите `experimental.cron: false` в [настройках](../configuration/settings.md) или задайте переменную окружения `QWEN_CODE_DISABLE_CRON=1`.

## Планирование повторяющегося промпта с помощью /loop

Встроенный [skill](skills.md) `/loop` — это самый быстрый способ запланировать повторяющийся промпт. Передайте необязательный интервал и сам промпт, и Qwen Code настроит cron-задачу, которая будет запускаться в фоновом режиме, пока сессия открыта.

```text
/loop 5m проверь, завершился ли деплой, и расскажи, что произошло
```

Qwen Code разбирает интервал, преобразует его в cron-выражение, планирует задачу и подтверждает частоту и ID задачи. Затем он сразу же выполняет промпт один раз — вам не нужно ждать первого срабатывания cron.

### Синтаксис интервалов

Интервалы необязательны. Вы можете указать их в начале, в конце или не указывать вовсе.

| Форма | Пример | Распознанный интервал |
| :--- | :--- | :--- |
| Интервал в начале | `/loop 30m check the build` | каждые 30 минут |
| Конструкция `every` в конце | `/loop check the build every 2 hours` | каждые 2 часа |
| Без интервала | `/loop check the build` | по умолчанию каждые 10 минут |

Поддерживаемые единицы: `s` — секунды, `m` — минуты, `h` — часы и `d` — дни. Секунды округляются до ближайшей минуты, так как минимальный шаг cron составляет одну минуту. Интервалы, которые не делятся равномерно на свою единицу, например `7m` или `90m`, округляются до ближайшего чистого интервала, и Qwen Code сообщает, какой вариант был выбран.

### Запуск другой команды в цикле

Запланированный промпт может сам быть командой или вызовом skill. Это полезно для повторного запуска уже упакованного рабочего процесса.

```text
/loop 20m /review-pr 1234
```

При каждом срабатывании задачи Qwen Code выполняет `/review-pr 1234` так, как если бы вы ввели это сами.

### Автономный режим

Запуск `/loop` **без промпта** запускает автономный цикл вместо повторения фиксированного промпта. Qwen Code выступает в роли управляющего работой, уже заданной в диалоге, — он продолжает двигать задачу вперед, пока вас нет:

```text
/loop
```

Простой `/loop` (без промпта и интервала) запускает автономный цикл в собственном темпе; `/loop <интервал>` без промпта запускает тот же автономный цикл с фиксированной частотой (например, `/loop 10m`). При каждом срабатывании он продвигает то, что уже было задано в диалоге: завершает начатые вами дела, поддерживает PR в работе (отвечает на комментарии в ревью, чинит упавший CI, разрешает конфликты) и выполняет последующие обязательства. Он работает только с задачами, которые уже определены в истории диалога: он никогда не придумывает новую работу и не вносит необратимые изменения (push, delete, send) без явного разрешения и останавливается, когда все задачи выполнены.

### Управление циклами

`/loop` также поддерживает две подкоманды для управления существующими задачами:

```text
/loop list
```

Выводит список всех запланированных задач с их ID и cron-выражениями.

```text
/loop clear
```

Отменяет все запланированные задачи сразу.

## Установка разового напоминания

Для разовых напоминаний опишите то, что вам нужно, на естественном языке вместо использования `/loop`. Qwen Code запланирует задачу с однократным срабатыванием, которая удалит сама себя после выполнения.

```text
напомни мне в 15:00 запушить ветку релиза
```

```text
через 45 минут проверь, прошли ли интеграционные тесты
```

Qwen Code фиксирует время срабатывания на конкретной минуте и часе с помощью cron-выражения и подтверждает, когда оно произойдет.

## Управление запланированными задачами

Попросите Qwen Code на естественном языке вывести список или отменить задачи, либо обратитесь к базовым инструментам напрямую.

```text
какие у меня запланированные задачи?
```

```text
отмени задачу проверки деплоя
```

Под капотом Qwen Code использует следующие инструменты:

| Инструмент | Назначение |
| :--- | :--- |
| `CronCreate` | Планирует новую задачу. Принимает cron-выражение из 5 полей, промпт для запуска и указание, должна ли она повторяться или сработать один раз. |
| `CronList` | Выводит список всех запланированных задач с их ID, расписаниями и промптами. |
| `CronDelete` | Отменяет задачу по ID. |

Каждая запланированная задача имеет 8-символьный ID, который можно передать в `CronDelete`. В рамках одной сессии может быть до 50 запланированных задач одновременно.

## Как выполняются запланированные задачи

Планировщик каждую секунду проверяет наличие задач, готовых к выполнению, и ставит их в очередь, когда сессия простаивает. Запланированный промпт срабатывает между вашими запросами, а не во время формирования ответа Qwen Code. Если Qwen Code занят, когда наступает время выполнения задачи, промпт ждет до окончания текущего цикла.

Все время интерпретируется в вашем локальном часовом поясе. Cron-выражение вроде `0 9 * * *` означает 9 утра там, где вы запускаете Qwen Code, а не по UTC.

### Джиттер

Чтобы избежать одновременных обращений к API от всех сессий в одно и то же абсолютное время, планировщик добавляет небольшое детерминированное смещение ко времени срабатывания:

- **Повторяющиеся задачи** срабатывают с задержкой до 10% от их периода, но не более 15 минут. Часовая задача может сработать в любое время от `:00` до `:06`.
- **Разовые задачи**, запланированные на начало или середину часа (минута `:00` или `:30`), срабатывают максимум на 90 секунд раньше.

Смещение вычисляется на основе ID задачи, поэтому одна и та же задача всегда получает одно и то же смещение. Если точное время важно, выберите минуту, отличную от `:00` или `:30`, например `3 9 * * *` вместо `0 9 * * *`, и джиттер для разовых задач не будет применяться.

### Трехдневный срок действия

Повторяющиеся задачи автоматически истекают через 3 дня после создания. Задача срабатывает в последний раз, а затем удаляет сама себя. Это ограничивает время работы забытого цикла. Если вам нужно, чтобы повторяющаяся задача работала дольше, отмените и пересоздайте ее до истечения срока действия.

Разовые задачи не истекают по таймеру — они просто удаляют себя после однократного срабатывания.

## Справочник по cron-выражениям

`CronCreate` принимает стандартные cron-выражения из 5 полей: `минута час день-месяца месяц день-недели`. Все поля поддерживают wildcards (`*`), одиночные значения (`5`), шаги (`*/15`), диапазоны (`1-5`) и списки через запятую (`1,15,30`).

| Пример | Значение |
| :--- | :--- |
| `*/5 * * * *` | Каждые 5 минут |
| `0 * * * *` | Каждый час в начале часа |
| `7 * * * *` | Каждый час на 7-й минуте |
| `0 9 * * *` | Каждый день в 9:00 по местному времени |
| `0 9 * * 1-5` | По будням в 9:00 по местному времени |
| `30 14 15 3 *` | 15 марта в 14:30 по местному времени |

В поле «день недели» используется `0` или `7` для воскресенья и до `6` для субботы. Если ограничены оба поля: день месяца и день недели (ни одно не равно `*`), дата совпадает, если совпадает хотя бы одно из полей — это соответствует стандартной семантике vixie-cron.

Расширенный синтаксис, такой как `L`, `W`, `?`, и псевдонимы названий, например `MON` или `JAN`, не поддерживаются.

## Ограничения

Планирование, привязанное к сессии, имеет неизбежные ограничения:

- Задачи срабатывают только тогда, когда Qwen Code запущен и простаивает. Закрытие терминала или завершение сессии отменяет все задачи.
- Нет компенсации пропущенных срабатываний. Если время срабатывания задачи прошло, пока Qwen Code был занят длительным запросом, она сработает один раз, когда Qwen Code освободится, а не по одному разу за каждый пропущенный интервал.
- Нет сохранения между перезапусками. Перезапуск Qwen Code очищает все задачи, привязанные к сессии.
