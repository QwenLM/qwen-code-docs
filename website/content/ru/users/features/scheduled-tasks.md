# Выполнение запросов по расписанию

> Используйте `/loop` и инструменты планирования cron для повторяющегося выполнения запросов, опроса статуса или установки одноразовых напоминаний внутри сессии Qwen Code.

Запланированные задачи позволяют Qwen Code автоматически повторять запрос через заданные интервалы. Используйте их для опроса развертывания, наблюдения за PR, проверки статуса долгого билда или чтобы напомнить себе о чем-то позже в сессии.

Задачи привязаны к сессии: они живут в текущем процессе Qwen Code и исчезают при его завершении. Ничего не записывается на диск.

> [!tip]
> Запланированные задачи включены по умолчанию. Чтобы отключить их, установите `experimental.cron: false` в вашем [файле настроек](../configuration/settings.md) или задайте переменную окружения `QWEN_CODE_DISABLE_CRON=1`.

## Запланировать повторяющийся запрос с /loop

`/loop` — это [встроенный навык](skills.md), который позволяет быстрее всего запланировать повторяющийся запрос. Передайте опциональный интервал и сам запрос, и Qwen Code создаст задачу cron, которая будет выполняться в фоне, пока сессия открыта.

```text
/loop 5m check if the deployment finished and tell me what happened
```

Qwen Code разбирает интервал, преобразует его в cron-выражение, планирует задачу и подтверждает периодичность и ID задачи. Затем он сразу же выполняет запрос один раз — вам не нужно ждать первого срабатывания cron.

### Синтаксис интервалов

Интервалы опциональны. Вы можете указать их в начале, в конце или не указывать вовсе.

| Форма               | Пример                                | Разобранный интервал          |
| :------------------ | :------------------------------------ | :---------------------------- |
| В начале            | `/loop 30m check the build`           | каждые 30 минут               |
| В конце с `every`   | `/loop check the build every 2 hours` | каждые 2 часа                 |
| Без интервала       | `/loop check the build`               | по умолчанию каждые 10 минут  |

Поддерживаемые единицы: `s` для секунд, `m` для минут, `h` для часов, `d` для дней. Секунды округляются до ближайшей минуты, так как cron имеет точность до минуты. Интервалы, которые не делятся ровно на свою единицу, например `7m` или `90m`, округляются до ближайшего четкого интервала, и Qwen Code сообщает, что он выбрал.

### Запуск другой команды по расписанию

В качестве запроса можно указать команду или вызов навыка. Это удобно для повторного выполнения уже упакованного рабочего процесса.

```text
/loop 20m /review-pr 1234
```

Каждый раз при срабатывании задачи Qwen Code выполняет `/review-pr 1234` так, как если бы вы ввели это сами.

### Управление циклами

`/loop` также поддерживает две подкоманды для управления существующими задачами:

```text
/loop list
```

Выводит список всех запланированных задач с их ID и cron-выражениями.

```text
/loop clear
```

Отменяет все запланированные задачи сразу.

## Установка одноразового напоминания

Для одноразовых напоминаний опишите, что нужно, на естественном языке, не используя `/loop`. Qwen Code запланирует задачу, которая выполнится один раз и удалится после выполнения.

```text
remind me at 3pm to push the release branch
```

```text
in 45 minutes, check whether the integration tests passed
```

Qwen Code привязывает время срабатывания к конкретной минуте и часу с помощью cron-выражения и подтверждает, когда оно сработает.

## Управление запланированными задачами

Спросите Qwen Code на естественном языке, чтобы вывести список или отменить задачи, или используйте соответствующие инструменты напрямую.

```text
what scheduled tasks do I have?
```

```text
cancel the deploy check job
```

Под капотом Qwen Code использует эти инструменты:

| Инструмент   | Назначение                                                                                             |
| :----------- | :----------------------------------------------------------------------------------------------------- |
| `CronCreate` | Запланировать новую задачу. Принимает cron-выражение из 5 полей, запрос для выполнения и признак повторяемости. |
| `CronList`   | Вывести список всех запланированных задач с их ID, расписанием и запросами.                            |
| `CronDelete` | Отменить задачу по ID.                                                                                 |

Каждая запланированная задача имеет 8-символьный ID, который можно передать в `CronDelete`. В сессии может быть одновременно до 50 запланированных задач.

## Как выполняются запланированные задачи

Планировщик каждую секунду проверяет, не подошло ли время для задач, и добавляет их в очередь, когда сессия простаивает. Запланированный запрос выполняется между вашими сообщениями, а не в тот момент, когда Qwen Code уже отвечает. Если Qwen Code занят на момент срабатывания задачи, запрос ждет окончания текущего ответа.

Все времена интерпретируются в вашем локальном часовом поясе. Выражение cron вида `0 9 * * *` означает 9 утра там, где работает Qwen Code, а не по UTC.

### Джиттер

Чтобы разные сессии не обращались к API в один и тот же момент времени, планировщик добавляет небольшое детерминированное смещение к моментам срабатывания:

- **Повторяющиеся задачи** срабатывают с задержкой до 10% от их периода, но не более 15 минут. Например, задача с интервалом в час может сработать в любое время от `:00` до `:06`.
- **Одноразовые задачи**, запланированные на ровный час или половину часа (минуты `:00` или `:30`), срабатывают с опережением до 90 секунд.

Смещение вычисляется на основе ID задачи, поэтому одна и та же задача всегда получает одно и то же смещение. Если точное время важно, используйте минуту, отличную от `:00` или `:30`, например `3 9 * * *` вместо `0 9 * * *`, — тогда джиттер для одноразовых задач не применяется.

### Истечение через три дня

Повторяющиеся задачи автоматически истекают через 3 дня после создания. Задача выполняется последний раз, а затем удаляется. Это ограничивает время работы забытого цикла. Если нужно, чтобы повторяющаяся задача работала дольше, отмените и создайте ее заново до истечения.

Одноразовые задачи не истекают по таймеру — они просто удаляются после однократного выполнения.

## Справочник по cron-выражениям

`CronCreate` принимает стандартные cron-выражения из 5 полей: `минута час день-месяца месяц день-недели`. Все поля поддерживают подстановочные символы (`*`), одиночные значения (`5`), шаги (`*/15`), диапазоны (`1-5`) и списки через запятую (`1,15,30`).

| Пример           | Значение                              |
| :--------------- | :------------------------------------ |
| `*/5 * * * *`    | Каждые 5 минут                        |
| `0 * * * *`      | Каждый час в 00 минут                 |
| `7 * * * *`      | Каждый час в 7 минут                  |
| `0 9 * * *`      | Каждый день в 9:00 по местному времени |
| `0 9 * * 1-5`    | По будням в 9:00 по местному времени   |
| `30 14 15 3 *`   | 15 марта в 14:30 по местному времени   |

День недели: `0` или `7` — воскресенье, `6` — суббота. Когда одновременно заданы и день месяца, и день недели (ни одно не равно `*`), дата соответствует, если совпадает хотя бы одно из полей — это стандартная семантика vixie-cron.

Расширенный синтаксис, такой как `L`, `W`, `?` и именные синонимы (`MON`, `JAN`), не поддерживается.

## Ограничения

Планирование в рамках сессии имеет свои ограничения:

- Задачи выполняются только пока Qwen Code работает и в режиме ожидания. Закрытие терминала или завершение сессии отменяет все задачи.
- Нет наверстывания пропущенных срабатываний. Если время задачи прошло, пока Qwen Code был занят долгим запросом, задача выполнится один раз, когда Qwen Code освободится, а не один раз за каждый пропущенный интервал.
- Нет сохранения между перезапусками. Перезапуск Qwen Code удаляет все задачи сессии.
