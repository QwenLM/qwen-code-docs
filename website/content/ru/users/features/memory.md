# Память

Каждый сеанс Qwen Code начинается с чистого контекста. Два механизма переносят знания между сеансами, чтобы вам не приходилось каждый раз повторять одно и то же:

- **QWEN.md** — инструкции, которые вы пишете один раз, а Qwen читает их в каждом сеансе
- **Автоматическая память** — заметки, которые Qwen пишет сам на основе того, что узнаёт от вас

---

## QWEN.md: ваши инструкции для Qwen

QWEN.md — это обычный текстовый файл, в который вы записываете то, что Qwen должен всегда знать о вашем проекте или ваших предпочтениях. Воспринимайте его как постоянный брифинг, который загружается в начале каждого разговора.

### Что помещать в QWEN.md

Добавляйте то, что иначе вам пришлось бы повторять в каждом сеансе:

- Команды сборки и тестирования (`npm run test`, `make build`)
- Соглашения о коде, которым следует ваша команда («все новые файлы должны содержать JSDoc-комментарии»)
- Архитектурные решения («мы используем репозиторий, никогда не обращаемся к базе данных напрямую из контроллеров»)
- Личные предпочтения («всегда используй pnpm, а не npm»)

Не включайте то, что Qwen может выяснить, прочитав ваш код. QWEN.md лучше всего работает, когда он короткий и конкретный — чем он длиннее, тем менее надёжно Qwen ему следует.

### Где создать QWEN.md

| Файл                          | К кому применяется                                    |
| ----------------------------- | ----------------------------------------------------- |
| `~/.qwen/QWEN.md`             | К вам, во всех ваших проектах                         |
| `QWEN.md` в корне проекта     | Ко всей вашей команде (закоммитьте его в систему контроля версий) |
| `.qwen/QWEN.local.md`         | Только к вам, только в этом проекте (не добавляйте в git) |

Вы можете использовать любую комбинацию этих файлов. Qwen загружает их все при запуске сеанса.

Если в вашем репозитории уже есть файл `AGENTS.md` для других AI-инструментов, Qwen читает и его. Не нужно дублировать инструкции.

#### Когда использовать `.qwen/QWEN.local.md`

Используйте его для **проектно-специфичных, но личных** инструкций — того, что относится к данному проекту, но не должно быть общим для команды:

- Ваш собственный ID кластера, пространство имён реестра контейнеров или облачный аккаунт
- Личная отладочная команда, которая жёстко привязана к вашему локальному окружению
- Заметки, которые вы хотите, чтобы Qwen знал о вашей текущей работе, но не хотелось бы коммитить

Он загружается **после** общего проектного `QWEN.md`, поэтому ваши локальные инструкции могут дополнять или переопределять командные.

**Вы должны самостоятельно добавить его в .gitignore.** Хотя `.qwen/` часто воспринимается как локальная директория, qwen-code не создаёт для вас `.gitignore`, и некоторые проекты коммитят `.qwen/settings.json`. Добавьте эту строку в ваш `.gitignore` (или в глобальный gitignore):

```
.qwen/QWEN.local.md
```

### Автоматическая генерация с помощью `/init`

Запустите `/init`, и Qwen проанализирует вашу кодовую базу, чтобы создать стартовый QWEN.md с командами сборки, инструкциями по тестированию и найденными соглашениями. Если такой файл уже существует, он предложит дополнения вместо перезаписи.

### Ссылки на другие файлы

Вы можете указать в QWEN.md на другие файлы, чтобы Qwen тоже их читал:

```markdown
См. @README.md для обзора проекта.

# Соглашения

- Git-процесс: @docs/git-workflow.md
```

Используйте `@путь/к/файлу` в любом месте QWEN.md. Относительные пути разрешаются относительно самого файла QWEN.md.

## Автоматическая память: что Qwen узнаёт о вас

Автоматическая память работает в фоновом режиме. После каждого вашего разговора Qwen незаметно сохраняет полезные вещи, которые узнал — ваши предпочтения, оставленные вами замечания, контекст проекта — чтобы использовать их в будущих сеансах без необходимости повторяться.

Это отличается от QWEN.md: вы его не пишете, это делает Qwen.

### Что сохраняет Qwen

Qwen ищет четыре типа информации, которую стоит запомнить:

| Что                    | Примеры                                                 |
| ----------------------- | ------------------------------------------------------- |
| **О вас**               | Ваша роль, опыт, как вы любите работать                 |
| **Ваши замечания**      | Исправления, которые вы внесли, подтверждённые подходы  |
| **Контекст проекта**    | Текущая работа, решения, цели, не очевидные из кода     |
| **Внешние ссылки**      | Дашборды, трекеры задач, ссылки на документацию, которые вы упомянули |

Qwen сохраняет не всё — только то, что действительно может пригодиться в следующий раз.

### Где это хранится

Файлы автоматической памяти находятся в `~/.qwen/projects/<project>/memory/`. Все ветки и рабочие деревья одного репозитория используют одну и ту же папку памяти, поэтому то, что Qwen узнаёт в одной ветке, доступно и в других.

Всё сохраняемое — это обычный Markdown: вы можете в любой момент открыть, отредактировать или удалить любой файл.

### Периодическая очистка

Qwen периодически просматривает свои сохранённые воспоминания, чтобы удалить дубликаты и очистить устаревшие записи. Это происходит автоматически в фоновом режиме раз в день после накопления достаточного количества сеансов. Вы можете запустить очистку вручную с помощью `/dream`, если хотите сделать это сейчас.

Пока идёт очистка, в углу экрана отображается **✦ мечтает**. Ваш сеанс продолжается как обычно.

### Включение и отключение

Автоматическая память включена по умолчанию. Чтобы переключить её, откройте `/memory` и используйте переключатели вверху. Вы можете отключить только автоматическое сохранение, только периодическую очистку или оба параметра.

Также вы можете задать их в `~/.qwen/settings.json` (применяется ко всем проектам) или `.qwen/settings.json` (только для этого проекта):

```json
{
  "memory": {
    "enableManagedAutoMemory": true,
    "enableManagedAutoDream": true
  }
}
```

## Команды

### `/memory`

Открывает панель «Память». Отсюда вы можете:

- Включить или отключить автоматическое сохранение памяти
- Включить или отключить периодическую очистку (мечтание)
- Открыть ваш личный QWEN.md (`~/.qwen/QWEN.md`)
- Открыть проектный QWEN.md
- Просмотреть папку автоматической памяти

### `/init`

Генерирует стартовый QWEN.md для вашего проекта. Qwen читает вашу кодовую базу и заполняет команды сборки, инструкции по тестированию и обнаруженные соглашения.

### `/remember <текст>`

Немедленно сохраняет что-то в автоматическую память, не дожидаясь, пока Qwen подхватит это автоматически:

```
/remember всегда используй snake_case для имён переменных в Python
/remember тестовое окружение находится по адресу staging.example.com
```

### `/forget <текст>`

Удаляет записи автоматической памяти, соответствующие вашему описанию:

```
/forget старый обходной путь для бага с входом
```

### `/dream`

Запускает очистку памяти сейчас, вместо ожидания автоматического расписания:

```
/dream
```

## Устранение неполадок

### Qwen не следует моему QWEN.md

Откройте `/memory`, чтобы увидеть, какие файлы загружены. Если ваш файл не указан, Qwen его не видит — убедитесь, что он находится в корне проекта или в `~/.qwen/`.

Инструкции работают лучше, когда они конкретны:

- ✓ `Используйте отступы в 2 пробела для TypeScript-файлов`
- ✗ `Форматируйте код красиво`

Если у вас есть несколько файлов QWEN.md с противоречивыми инструкциями, Qwen может вести себя непоследовательно. Просмотрите их и удалите любые противоречия.

### Я хочу посмотреть, что сохранил Qwen

Выполните `/memory` и выберите **Открыть папку автоматической памяти**. Все сохранённые воспоминания — это читаемые Markdown-файлы, которые можно просматривать, редактировать или удалять.

### Qwen постоянно что-то забывает

Если автоматическая память включена, но Qwen, похоже, не помнит информацию между сеансами, попробуйте выполнить `/dream`, чтобы принудительно запустить очистку. Также проверьте `/memory`, чтобы убедиться, что оба переключателя включены.

Для того, что вы обязательно хотите, чтобы Qwen запомнил, лучше добавить это в QWEN.md — автоматическая память работает по принципу «лучшее из возможного», а QWEN.md гарантированно загружается.
