# Память

Каждый сеанс Qwen Code начинается с чистого окна контекста. Два механизма переносят знания между сеансами, чтобы вам не приходилось каждый раз объяснять всё заново:

- **QWEN.md** — инструкции, которые _вы_ пишете один раз, а Qwen читает каждый сеанс
- **Авто-память** — заметки, которые Qwen пишет сам на основе того, что узнаёт от вас

---

## QWEN.md: ваши инструкции для Qwen

QWEN.md — это обычный текстовый файл, в который вы записываете то, что Qwen всегда должен знать о вашем проекте или ваших предпочтениях. Представьте его как постоянный брифинг, который загружается в начале каждого разговора.

### Что помещать в QWEN.md

Добавляйте то, что иначе вам приходилось бы повторять каждый сеанс:

- Команды сборки и тестирования (`npm run test`, `make build`)
- Соглашения по стилю кода, которым следует ваша команда («все новые файлы должны содержать JSDoc-комментарии»)
- Архитектурные решения («мы используем шаблон репозитория, никогда не обращаемся к базе данных напрямую из контроллеров»)
- Личные предпочтения («всегда используй pnpm, не npm»)

Не включайте то, что Qwen может понять, читая ваш код. QWEN.md работает лучше всего, когда он короткий и конкретный — чем он длиннее, тем менее надёжно Qwen ему следует.

### Где создавать QWEN.md

| Файл                              | К кому применяется                                    |
| --------------------------------- | ----------------------------------------------------- |
| `~/.qwen/QWEN.md`                 | Только к вам, во всех ваших проектах                  |
| `QWEN.md` в корне проекта         | Ко всей вашей команде (закоммитьте в систему контроля версий) |
| `.qwen/QWEN.local.md`              | Только к вам и только в этом проекте (не добавляйте в git) |

Вы можете использовать любую комбинацию этих файлов. Qwen загружает их все при запуске сеанса.

Если в вашем репозитории уже есть файл `AGENTS.md`, предназначенный для других ИИ-инструментов, Qwen также читает его. Дублировать инструкции не нужно.

#### Когда использовать `.qwen/QWEN.local.md`

Используйте его для **проектно-специфичных, но личных** инструкций — того, что относится к этому проекту, но не должно быть доступно команде:

- Ваш собственный идентификатор кластера, пространство имён реестра контейнеров или облачный аккаунт
- Личная команда для отладки, которая жёстко завязана на ваше локальное окружение
- Заметки, которые Qwen должен знать о вашей текущей работе, но не нужно коммитить

Он загружается **после** общего проектного `QWEN.md`, так что ваши локальные инструкции могут дополнять или переопределять команды.

**Вы должны добавить его в .gitignore самостоятельно.** Хотя `.qwen/` часто рассматривается как локальная директория, qwen-code не создаёт для вас `.gitignore`, и некоторые проекты коммитят `.qwen/settings.json`. Добавьте эту строку в ваш `.gitignore` (или в глобальный gitignore):

```
.qwen/QWEN.local.md
```

### Автоматическая генерация с помощью `/init`

Запустите `/init`, и Qwen проанализирует вашу кодовую базу, чтобы создать начальный QWEN.md с командами сборки, инструкциями по тестированию и соглашениями, которые он обнаружит. Если такой файл уже существует, он предложит дополнения, а не перезапишет его.

### Ссылки на другие файлы

Вы можете указать QWEN.md на другие файлы, чтобы Qwen также читал их:

```markdown
См. @README.md для обзора проекта.

# Соглашения

- Git-процесс: @docs/git-workflow.md
```

Используйте `@путь/к/файлу` в любом месте QWEN.md. Относительные пути разрешаются относительно самого файла QWEN.md.

## Авто-память: что Qwen узнаёт о вас

Авто-память работает в фоновом режиме. После каждого вашего разговора Qwen незаметно сохраняет полезные сведения, которые он узнал — ваши предпочтения, полученные от вас замечания, контекст проекта — чтобы использовать их в будущих сеансах, избавляя вас от необходимости повторяться.

Это отличается от QWEN.md: вы не пишете её, это делает Qwen.

### Что Qwen сохраняет

Qwen ищет четыре типа вещей, которые стоит запомнить:

| Что                    | Примеры                                                              |
| ---------------------- | -------------------------------------------------------------------- |
| **О вас**              | Ваша роль, опыт, как вы любите работать                             |
| **Ваши замечания**     | Исправления, которые вы внесли, подходы, которые вы подтвердили     |
| **Контекст проекта**   | Текущая работа, решения, цели, неочевидные из кода                  |
| **Внешние ссылки**     | Дашборды, трекеры задач, ссылки на документацию, которые вы упомянули |

Qwen сохраняет не всё — только то, что действительно пригодится в следующий раз.

### Где это хранится

Файлы авто-памяти находятся в `~/.qwen/projects/<project>/memory/`. Все ветки и рабочие каталоги одного репозитория используют общую папку памяти, так что то, что Qwen узнал в одной ветке, доступно в других.

Всё сохранённое — обычный markdown: вы можете открыть, отредактировать или удалить любой файл в любое время.

### Периодическая чистка

Qwen периодически просматривает сохранённые воспоминания, чтобы удалить дубликаты и устаревшие записи. Это происходит автоматически в фоновом режиме раз в день после накопления достаточного количества сеансов. Вы можете запустить чистку вручную с помощью `/dream`, если хотите выполнить её сейчас.

Пока выполняется чистка, в углу экрана отображается **✦ dreaming**. Ваш сеанс продолжается как обычно.

### Включение и выключение

Авто-память включена по умолчанию. Чтобы переключить её, откройте `/memory` и используйте переключатели вверху. Вы можете отключить только автоматическое сохранение, только периодическую чистку или и то, и другое.
Вы также можете задать их в `~/.qwen/settings.json` (применяется ко всем проектам) или `.qwen/settings.json` (только для этого проекта):

```json
{
  "memory": {
    "enableManagedAutoMemory": true,
    "enableManagedAutoDream": true
  }
}
```

## Команды

### `/memory`

Открывает панель памяти. Здесь вы можете:

- Включить или выключить автоматическое сохранение памяти
- Включить или выключить периодическую очистку (dream)
- Открыть ваш личный QWEN.md (`~/.qwen/QWEN.md`)
- Открыть QWEN.md проекта
- Просмотреть папку автоматической памяти

### `/init`

Создаёт начальный QWEN.md для вашего проекта. Qwen читает вашу кодовую базу и заполняет команды сборки, инструкции по тестированию и обнаруженные соглашения.

### `/remember <текст>`

Немедленно сохраняет что-то в автоматическую память, не дожидаясь, пока Qwen подхватит это самостоятельно:

```
/remember всегда используй snake_case для имён переменных Python
/remember стенд staging находится на staging.example.com
```

### `/forget <текст>`

Удаляет записи автоматической памяти, соответствующие вашему описанию:

```
/forget старый обходной баг для ошибки входа
```

### `/dream`

Запускает очистку памяти немедленно, не дожидаясь автоматического расписания:

```
/dream
```

## Устранение неполадок

### Qwen не следует моему QWEN.md

Откройте `/memory`, чтобы увидеть, какие файлы загружены. Если ваш файл не указан, Qwen его не видит — убедитесь, что он находится в корне проекта или в `~/.qwen/`.

Инструкции работают лучше, когда они конкретны:

- ✓ Используйте отступы в 2 пробела для файлов TypeScript
- ✗ Форматируйте код красиво

Если у вас несколько файлов QWEN.md с противоречивыми инструкциями, Qwen может вести себя непоследовательно. Проверьте их и удалите любые противоречия.

### Я хочу посмотреть, что сохранил Qwen

Выполните `/memory` и выберите **Открыть папку автоматической памяти**. Все сохранённые воспоминания — это читаемые markdown-файлы, которые можно просматривать, редактировать или удалять.

### Qwen постоянно что-то забывает

Если автоматическая память включена, но Qwen, похоже, не запоминает информацию между сессиями, попробуйте выполнить `/dream`, чтобы принудительно запустить очистку. Также проверьте `/memory`, чтобы убедиться, что оба переключателя включены.

Если есть вещи, которые Qwen должен всегда помнить, лучше добавляйте их в QWEN.md — автоматическая память работает по принципу «best-effort», а QWEN.md гарантированно запоминается.
