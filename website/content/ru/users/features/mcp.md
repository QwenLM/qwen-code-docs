# Подключение Qwen Code к инструментам через MCP

Qwen Code может подключаться к внешним инструментам и источникам данных через [Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction). MCP-серверы дают Qwen Code доступ к вашим инструментам, базам данных и API.

## Что можно делать с помощью MCP

Подключив MCP-серверы, вы можете попросить Qwen Code:

- Работать с файлами и репозиториями (чтение/поиск/запись — в зависимости от разрешённых инструментов)
- Запрашивать базы данных (просмотр схемы, запросы, отчёты)
- Интегрировать внутренние сервисы (оборачивать свои API в MCP-инструменты)
- Автоматизировать рабочие процессы (повторяющиеся задачи, представленные в виде инструментов/подсказок)

> [!tip]
>
> Если вы ищете «одну команду для начала», переходите к разделу [Быстрый старт](#быстрый-старт).

## Быстрый старт

Qwen Code загружает MCP-серверы из раздела `mcpServers` в вашем `settings.json`. Настроить серверы можно двумя способами:

- редактируя `settings.json` напрямую,
- используя команды `qwen mcp` (см. [Справочник по CLI](#управление-mcp-серверами-с-помощью-qwen-mcp)).

### Добавление первого сервера

1. Добавьте сервер (пример: удалённый HTTP MCP-сервер):

```bash
qwen mcp add --transport http my-server http://localhost:3000/mcp
```

2. Запустите Qwen Code и откройте диалог управления MCP, чтобы просматривать серверы и управлять ими:

```bash
qwen
```

Затем введите:

```text
/mcp
```

3. Если Qwen Code уже был запущен до добавления сервера, перезапустите его в том же проекте. Затем попросите модель использовать инструменты с этого сервера.

## Где хранятся настройки (области видимости)

Большинству пользователей нужны только две области:

- **Область пользователя (по умолчанию)**: `~/.qwen/settings.json` — применяется ко всем проектам на вашем компьютере.
- **Область проекта**: `.qwen/settings.json` — в корне вашего проекта.

Запись в область пользователя:

```bash
qwen mcp add --scope user --transport http my-server http://localhost:3000/mcp
```

> [!tip]
>
> О продвинутых уровнях конфигурации (системные умолчания/системные настройки и правила приоритета) см. [Настройки](../configuration/settings).

## Настройка серверов

### Выбор транспорта

| Транспорт | Когда использовать                                                  | Поле(я) JSON                                  |
| --------- | ------------------------------------------------------------------- | --------------------------------------------- |
| `http`    | Рекомендуется для удалённых сервисов; хорошо работает с облачными MCP-серверами | `httpUrl` (+ опционально `headers`)            |
| `sse`     | Устаревшие/старые серверы, поддерживающие только Server-Sent Events | `url` (+ опционально `headers`)                |
| `stdio`   | Локальный процесс (скрипты, CLI, Docker) на вашем компьютере        | `command`, `args` (+ опционально `cwd`, `env`) |

> [!note]
>
> Если сервер поддерживает оба варианта, предпочитайте **HTTP** перед **SSE**.

### Настройка через `settings.json` vs `qwen mcp add`

Оба подхода создают одинаковые записи в разделе `mcpServers` в вашем `settings.json` — используйте тот, который вам удобнее.

#### Stdio-сервер (локальный процесс)

JSON (`.qwen/settings.json`):

```json
{
  "mcpServers": {
    "pythonTools": {
      "command": "python",
      "args": ["-m", "my_mcp_server", "--port", "8080"],
      "cwd": "./mcp-servers/python",
      "env": {
        "DATABASE_URL": "$DB_CONNECTION_STRING",
        "API_KEY": "${EXTERNAL_API_KEY}"
      },
      "timeout": 15000
    }
  }
}
```

CLI (по умолчанию записывает в область пользователя):

```bash
qwen mcp add pythonTools -e DATABASE_URL=$DB_CONNECTION_STRING -e API_KEY=$EXTERNAL_API_KEY \
  --timeout 15000 python -m my_mcp_server --port 8080
```

#### HTTP-сервер (удалённый потоковый HTTP)

JSON:

```json
{
  "mcpServers": {
    "httpServerWithAuth": {
      "httpUrl": "http://localhost:3000/mcp",
      "headers": {
        "Authorization": "Bearer your-api-token"
      },
      "timeout": 5000
    }
  }
}
```

CLI:

```bash
qwen mcp add --transport http httpServerWithAuth http://localhost:3000/mcp \
  --header "Authorization: Bearer your-api-token" --timeout 5000
```

#### SSE-сервер (удалённый Server-Sent Events)

JSON:

```json
{
  "mcpServers": {
    "sseServer": {
      "url": "http://localhost:8080/sse",
      "timeout": 30000
    }
  }
}
```

CLI:

```bash
qwen mcp add --transport sse sseServer http://localhost:8080/sse --timeout 30000
```

## Использование MCP-подсказок и ресурсов

Помимо инструментов, Qwen Code обнаруживает и предоставляет два других типа MCP-примитивов.

### Подсказки (слеш-команды)

Любая подсказка, которую сервер объявляет через `prompts/list`, становится исполняемой **слеш-командой**. После обнаружения введите `/`, и вы увидите подсказку в списке (помеченную как `MCP: <server>`); используйте её как любую другую команду:

```text
/my_prompt --arg1="значение" --arg2="значение"
# позиционная форма тоже работает:
/my_prompt "значение" "значение"
# показать аргументы подсказки:
/my_prompt help
```

Сообщения подсказки отправляются модели, которая затем выполняет соответствующие действия.

> Обнаружение работает, даже если сервер не заявляет о поддержке `prompts` в своих возможностях инициализации: Qwen Code всё равно пытается вызвать `prompts/list`. Те серверы, у которых действительно нет подсказок, просто отвечают `Method not found`, и этот ответ игнорируется.
### Ресурсы

Ресурсы, которые сервер рекламирует через `resources/list`, обнаруживаются для каждого сервера по отдельности. Откройте диалог управления с помощью `/mcp` и выберите сервер, чтобы увидеть количество его **Ресурсов** вместе с инструментами и подсказками. Выберите **Просмотреть ресурсы**, чтобы просмотреть URI ресурсов сервера; выбор одного из них покажет его описание и MIME-тип, а также точную ссылку `@server:uri` для вставки в сообщение. Как и для подсказок, возможность `resources` не обязательна для объявления.

Вставьте содержимое ресурса в сообщение с помощью синтаксиса `@server:uri` — введите `@`, затем имя сервера, двоеточие и URI ресурса:

```text
summarize @myserver:file:///docs/spec.md and list the open questions
```

При вводе `@myserver:` отобразится автодополняемый список ресурсов этого сервера; продолжайте вводить текст для фильтрации, сопоставляя (без учета регистра) либо URI ресурса, либо его понятное имя/заголовок. Вам не нужно знать URI наизусть — до того, как вы поставите двоеточие, ввод части имени сервера также предложит подходящие серверы, предоставляющие ресурсы, так что вы можете выбрать один из них и сразу перейти к его списку ресурсов. При отправке указанный ресурс считывается, и его содержимое добавляется к вашему сообщению (текст встроен, двоичные данные — как вложения); ссылка `@server:uri` сохраняется в подсказке, чтобы модель видела, с чем работает. Префикс `server` должен соответствовать настроенному MCP-серверу — иначе токен обрабатывается как обычный путь к файлу, поэтому существующие ссылки `@path/to/file` не затрагиваются. Чтение ресурсов отключено в ненадежных папках.

## Постепенная доступность и таймауты обнаружения

Qwen Code обнаруживает MCP-серверы в фоновом режиме после того, как интерфейс уже стал интерактивным. Вы видите первое приглашение cli в течение нескольких сотен миллисекунд, даже если один из ваших MCP-серверов отвечает несколько секунд (или не отвечает вовсе), а список инструментов модели обновляется примерно за один кадр (~16 мс) после завершения рукопожатия обнаружения каждым сервером.

- **Интерактивный режим**: интерфейс появляется немедленно; индикатор состояния MCP в правом нижнем углу показывает `N/M MCP servers ready`, пока идет обнаружение. Отправка подсказки до завершения MCP означает, что модель видит только те инструменты, которые готовы _на данный момент_; последующие подсказки будут видеть больше инструментов по мере подключения серверов.
- **Неинтерактивный режим** (`--prompt`, stream-json, ACP): cli по-прежнему ожидает завершения обнаружения MCP перед отправкой первого приглашения, поэтому сценарии/конвейерные вызовы видят тот же полный набор инструментов, который давало прежнее синхронное поведение.

### `discoveryTimeoutMs` для каждого сервера

Для каждого MCP-сервера устанавливается таймаут только на обнаружение, который ограничивает время, отведенное на начальное рукопожатие (`connect` + `tools/list` + `prompts/list` + `resources/list`). Значения по умолчанию:

- **stdio-серверы**: 30 с
- **удаленные HTTP/SSE-серверы**: 5 с (сетевой риск выше)

При необходимости переопределите для каждого сервера:

```jsonc
{
  "mcpServers": {
    "slow-stdio": {
      "command": "node",
      "args": ["./slow-server.js"],
      "discoveryTimeoutMs": 60000,
    },
    "flaky-remote": {
      "httpUrl": "https://example.com/mcp",
      "discoveryTimeoutMs": 10000,
    },
  },
}
}

Существующее поле `timeout` является таймаутом **вызова инструмента** (используется для каждого запроса `tools/call`, по умолчанию 10 минут) и не зависит от `discoveryTimeoutMs` — длительный вызов инструмента не является проблемой запуска.

### Откат постепенного MCP

Если вам нужно старое синхронное поведение (cli ожидает каждый MCP-сервер перед отображением какого-либо интерфейса), установите переменную окружения `QWEN_CODE_LEGACY_MCP_BLOCKING=1`. Это запасной вариант, который сохраняется как минимум на один релиз.

## Безопасность и контроль

### Доверие (пропуск подтверждений)

- **Доверие к серверу** (`trust: true`): отключает запросы подтверждения для этого сервера (используйте с осторожностью).

### OAuth-аутентификация

Qwen Code поддерживает аутентификацию OAuth 2.0 для MCP-серверов. Это полезно при доступе к удаленным серверам, требующим аутентификацию.

#### Базовое использование

При добавлении MCP-сервера с OAuth-учетными данными Qwen Code автоматически обработает процесс аутентификации:

```bash
qwen mcp add --transport sse oauth-server https://api.example.com/sse/ \
  --oauth-client-id your-client-id \
  --oauth-redirect-uri https://your-server.com/oauth/callback \
  --oauth-authorization-url https://provider.example.com/authorize \
  --oauth-token-url https://provider.example.com/token
```

#### Важно: настройка redirect URI

Процесс OAuth требует redirect URI, на который поставщик авторизации отправляет код аутентификации.

- **Локальная разработка**: По умолчанию Qwen Code использует `http://localhost:7777/oauth/callback`. Это работает при запуске Qwen Code на локальной машине с локальным браузером.

- **Удаленные развертывания/облачные**: При запуске Qwen Code на удаленных серверах, облачных IDE или веб-терминалах перенаправление на `localhost` по умолчанию НЕ будет работать. Вы ДОЛЖНЫ настроить `--oauth-redirect-uri` так, чтобы он указывал на общедоступный URL, который может получать обратный вызов OAuth.

Пример для удаленных серверов:

```bash
qwen mcp add --transport sse remote-server https://api.example.com/sse/ \
  --oauth-redirect-uri https://your-remote-server.example.com/oauth/callback
```
#### Ручная настройка через settings.json

Вы также можете настроить OAuth, напрямую отредактировав `settings.json`:

```json
{
  "mcpServers": {
    "oauthServer": {
      "url": "https://api.example.com/sse/",
      "oauth": {
        "enabled": true,
        "clientId": "your-client-id",
        "clientSecret": "your-client-secret",
        "authorizationUrl": "https://provider.example.com/authorize",
        "tokenUrl": "https://provider.example.com/token",
        "redirectUri": "https://your-server.com/oauth/callback",
        "scopes": ["read", "write"]
      }
    }
  }
}
```

Свойства настройки OAuth:

| Свойство           | Описание                                                                                                                     |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| `enabled`          | Включить OAuth для этого сервера (boolean)                                                                                   |
| `clientId`         | Идентификатор клиента OAuth (строка, опционально при динамической регистрации)                                               |
| `clientSecret`     | Секрет клиента OAuth (строка, опционально для публичных клиентов)                                                            |
| `authorizationUrl` | Конечная точка авторизации OAuth (строка, автоматически определяется, если опущена)                                         |
| `tokenUrl`         | Конечная точка токена OAuth (строка, автоматически определяется, если опущена)                                               |
| `scopes`           | Требуемые области OAuth (массив строк)                                                                                       |
| `redirectUri`      | Пользовательский URI перенаправления (строка). **Критически важно для удаленных развертываний**. По умолчанию: `http://localhost:7777/oauth/callback`                                                                          |
| `tokenParamName`   | Имя параметра запроса для токенов в URL SSE (строка)                                                                         |
| `audiences`        | Аудитории, для которых действителен токен (массив строк)                                                                     |

#### Управление токенами

Токены OAuth автоматически:

- **Хранятся** в файле `~/.qwen/mcp-oauth-tokens.json` (открытый текст, права доступа 0600) по умолчанию. Если установлена переменная `QWEN_CODE_FORCE_ENCRYPTED_FILE_STORAGE=true`, Qwen Code использует хранилище, привязанное к системной связке ключей, где это возможно, или `~/.qwen/mcp-oauth-tokens-v2.json` с шифрованием AES-256-GCM.
- **Обновляются** при истечении срока действия (если доступны токены обновления)
- **Проверяются** перед каждой попыткой подключения

> [!WARNING]
> По умолчанию токены OAuth хранятся на диске в незашифрованном виде. На общих или многопользовательских машинах установите `QWEN_CODE_FORCE_ENCRYPTED_FILE_STORAGE=true`, чтобы защитить учетные данные.

Используйте диалог `/mcp` в Qwen Code для просмотра MCP-серверов и интерактивного управления аутентификацией.

### Фильтрация инструментов (разрешить/запретить инструменты для сервера)

Используйте `includeTools` / `excludeTools` для ограничения инструментов, предоставляемых сервером (с точки зрения Qwen Code).

Пример: включить только несколько инструментов:

```json
{
  "mcpServers": {
    "filteredServer": {
      "command": "python",
      "args": ["-m", "my_mcp_server"],
      "includeTools": ["safe_tool", "file_reader", "data_processor"],
      "timeout": 30000
    }
  }
}
```

### Глобальные списки разрешений/запретов

Объект `mcp` в вашем `settings.json` задает глобальные правила для всех MCP-серверов:

- `mcp.allowed`: список разрешенных имен MCP-серверов (ключей в `mcpServers`)
- `mcp.excluded`: список запрещенных имен MCP-серверов

Пример:

```json
{
  "mcp": {
    "allowed": ["my-trusted-server"],
    "excluded": ["experimental-server"]
  }
}
```

## Устранение неполадок

- **Сервер показывает «Disconnected» в `qwen mcp list`**: проверьте правильность URL/команды, затем увеличьте `timeout`.
- **Stdio-сервер не запускается**: используйте абсолютный путь в `command` и перепроверьте `cwd`/`env`.
- **Переменные окружения в JSON не разрешаются**: убедитесь, что они существуют в окружении, где запущен Qwen Code (среда оболочки и GUI-приложения могут различаться).

## Справочная информация

### Структура `settings.json`

#### Конфигурация конкретного сервера (`mcpServers`)

Добавьте объект `mcpServers` в ваш файл `settings.json`:

```json
// ... файл содержит другие объекты конфигурации
{
  "mcpServers": {
    "serverName": {
      "command": "path/to/server",
      "args": ["--arg1", "value1"],
      "env": {
        "API_KEY": "$MY_API_TOKEN"
      },
      "cwd": "./server-directory",
      "timeout": 30000,
      "trust": false
    }
  }
}
```

Свойства конфигурации:

Обязательные (одно из следующих):

| Свойство  | Описание                                                     |
| --------- | ------------------------------------------------------------ |
| `command` | Путь к исполняемому файлу для транспорта Stdio               |
| `url`     | URL SSE-конечной точки (например, `"http://localhost:8080/sse"`) |
| `httpUrl` | URL HTTP-стриминговой конечной точки                         |

Необязательные:

| Свойство               | Тип/По умолчанию            | Описание                                                                                                                                                                                                                                                                    |
| ---------------------- | --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `args`                 | array                       | Аргументы командной строки для транспорта Stdio                                                                                                                                                                                                                             |
| `headers`              | object                      | Пользовательские HTTP-заголовки при использовании `url` или `httpUrl`                                                                                                                                                                                                       |
| `env`                  | object                      | Переменные окружения для процесса сервера. Значения могут ссылаться на переменные окружения с помощью синтаксиса `$VAR_NAME` или `${VAR_NAME}`                                                                                                                               |
| `cwd`                  | string                      | Рабочая директория для транспорта Stdio                                                                                                                                                                                                                                     |
| `timeout`              | number<br>(default: 600,000)| Тайм-аут запроса в миллисекундах (по умолчанию: 600,000 мс = 10 минут)                                                                                                                                                                                                      |
| `trust`                | boolean<br>(default: false) | Если `true`, отключает все подтверждения вызовов инструментов для этого сервера (по умолчанию: `false`)                                                                                                                                                                     |
| `includeTools`         | array                       | Список имен инструментов, которые следует включить с этого MCP-сервера. Если указан, с этого сервера будут доступны только перечисленные здесь инструменты (поведение белого списка). Если не указан, по умолчанию включены все инструменты сервера.                          |
| `excludeTools`         | array                       | Список имен инструментов, которые следует исключить с этого MCP-сервера. Инструменты, перечисленные здесь, не будут доступны модели, даже если они предоставляются сервером.<br>Примечание: `excludeTools` имеет приоритет над `includeTools` — если инструмент есть в обоих списках, он будет исключен. |
| `targetAudience`       | string                      | Идентификатор клиента OAuth, внесенный в белый список приложения, защищенного IAP, к которому вы пытаетесь получить доступ. Используется с `authProviderType: 'service_account_impersonation'`.                                                                              |
| `targetServiceAccount` | string                      | Адрес электронной почты сервисного аккаунта Google Cloud для олицетворения. Используется с `authProviderType: 'service_account_impersonation'`.                                                                                                                              |

### Управление MCP-серверами с помощью `qwen mcp`

Вы всегда можете настроить MCP-серверы, отредактировав `settings.json` вручную, но CLI обычно быстрее.

#### Добавление сервера (`qwen mcp add`)

```bash
qwen mcp add [options] <name> <commandOrUrl> [args...]
```

| Аргумент/Опция              | Описание                                                             | По умолчанию                          | Пример                                                             |
| --------------------------- | -------------------------------------------------------------------- | ------------------------------------- | ------------------------------------------------------------------ |
| `<name>`                    | Уникальное имя сервера.                                              | —                                     | `example-server`                                                   |
| `<commandOrUrl>`            | Команда для выполнения (`stdio`) или URL (`http`/`sse`).             | —                                     | `/usr/bin/python` или `http://localhost:8`                         |
| `[args...]`                 | Необязательные аргументы для команды `stdio`.                        | —                                     | `--port 5000`                                                      |
| `-s`, `--scope`             | Область настройки (user или project).                                | `user`                                | `-s user`                                                          |
| `-t`, `--transport`         | Тип транспорта (`stdio`, `sse`, `http`).                             | `stdio`                               | `-t sse`                                                           |
| `-e`, `--env`               | Задание переменных окружения.                                        | —                                     | `-e KEY=value`                                                     |
| `-H`, `--header`            | Задание HTTP-заголовков для транспортов SSE и HTTP.                  | —                                     | `-H "X-Api-Key: abc123"`                                           |
| `--timeout`                 | Задание времени ожидания подключения в миллисекундах.                | —                                     | `--timeout 30000`                                                  |
| `--trust`                   | Доверять серверу (пропускать все запросы подтверждения вызова инструмента). | — (`false`)                           | `--trust`                                                          |
| `--description`             | Задание описания сервера.                                            | —                                     | `--description "Local tools"`                                      |
| `--include-tools`           | Список инструментов для включения (через запятую).                   | все инструменты включены              | `--include-tools mytool,othertool`                                 |
| `--exclude-tools`           | Список инструментов для исключения (через запятую).                  | нет                                   | `--exclude-tools mytool`                                           |
| `--oauth-client-id`         | OAuth-идентификатор клиента для аутентификации MCP-сервера.          | —                                     | `--oauth-client-id your-client-id`                                 |
| `--oauth-client-secret`     | OAuth-секрет клиента для аутентификации MCP-сервера.                 | —                                     | `--oauth-client-secret your-client-secret`                         |
| `--oauth-redirect-uri`      | OAuth URI перенаправления для обратного вызова аутентификации.       | `http://localhost:7777/oauth/callback` | `--oauth-redirect-uri https://your-server.com/oauth/callback`      |
| `--oauth-authorization-url` | URL авторизации OAuth.                                               | —                                     | `--oauth-authorization-url https://provider.example.com/authorize` |
| `--oauth-token-url`         | URL получения токена OAuth.                                          | —                                     | `--oauth-token-url https://provider.example.com/token`             |
| `--oauth-scopes`            | OAuth-области (через запятую).                                       | —                                     | `--oauth-scopes scope1,scope2`                                     |

> Флаги `--oauth-*` применяются только к `--transport sse` и `--transport http`. Их использование с `--transport stdio` запрещено.

#### Удаление сервера (`qwen mcp remove`)

```bash
qwen mcp remove <name>
```
