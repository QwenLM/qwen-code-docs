# Строка состояния

> Отображение пользовательской информации в нижнем колонтитуле.

Строка состояния показывает информацию, зависящую от сессии — имя модели, использование токенов, ветку Git и другое — в левой части нижнего колонтитула. Есть два режима настройки:

- **Режим предустановок** — выбор из встроенных элементов данных через интерактивный диалог или JSON-конфигурацию. Не требует написания скриптов.
- **Командный режим** — выполнение shell-команды, которая получает структурированный JSON-контекст через stdin. Полная гибкость для собственного форматирования.

```
Однорядная строка состояния (режим одобрения по умолчанию — 1 строка):
┌─────────────────────────────────────────────────────────────────┐
│  user@host ~/project (main) ctx:34%   🔒 docker | Debug | 67%  │  ← строка состояния
└─────────────────────────────────────────────────────────────────┘

Многострочная строка состояния (до 2 строк — 2 строки):
┌─────────────────────────────────────────────────────────────────┐
│  user@host ~/project (main) ctx:34%   🔒 docker | Debug | 67%  │  ← строка состояния 1
│  ████████░░░░░░░░░░ 34% context                                │  ← строка состояния 2
└─────────────────────────────────────────────────────────────────┘

Многострочная строка состояния + нестандартный режим (максимум 3 строки):
┌─────────────────────────────────────────────────────────────────┐
│  user@host ~/project (main) ctx:34%   🔒 docker | Debug | 67%  │  ← строка состояния 1
│  ████████░░░░░░░░░░ 34% context                                │  ← строка состояния 2
│  auto-accept edits (shift + tab to cycle)                       │  ← индикатор режима
└─────────────────────────────────────────────────────────────────┘
```

При настройке строка состояния заменяет подсказку по умолчанию «? for shortcuts». Сообщения с высоким приоритетом (подсказки о выходе Ctrl+C/D, Esc, режим INSERT vim) временно переопределяют строку состояния. Текст строки состояния усекается, чтобы поместиться в доступную ширину.

## Быстрая настройка

Самый простой способ настроить строку состояния — команда `/statusline`. Она открывает интерактивный диалог, где можно выбрать элементы предустановок, переключить цвета темы и увидеть предварительный просмотр в реальном времени:

```
/statusline
```

Откроется конфигуратор режима предустановок. Используйте клавиши со стрелками для навигации, пробел для переключения элементов и Enter для подтверждения. Выбор автоматически сохраняется в настройках.

Вы также можете передать `/statusline` конкретные инструкции, чтобы сгенерировать конфигурацию командного режима:

```
/statusline show model name and context usage percentage
```

## Режим предустановок

Режим предустановок предоставляет набор встроенных элементов данных, которые можно выбирать и комбинировать — никаких shell-команд, никакого `jq`, никакого написания скриптов. Элементы отображаются как `item1 | item2 | item3` в одной строке.

### Конфигурация

Добавьте объект `statusLine` под ключом `ui` в `~/.qwen/settings.json`:

```json
{
  "ui": {
    "statusLine": {
      "type": "preset",
      "items": [
        "model-with-reasoning",
        "git-branch",
        "context-remaining",
        "current-dir",
        "context-used"
      ],
      "useThemeColors": true
    }
  }
}
```

| Поле                  | Тип        | Обязательное | Описание                                                                                                                              |
| --------------------- | ---------- | ------------ | ------------------------------------------------------------------------------------------------------------------------------------- |
| `type`                | `"preset"` | Да           | Должно быть `"preset"`                                                                                                                |
| `items`               | string[]   | Да           | Упорядоченный список идентификаторов элементов предустановок для отображения (см. таблицу ниже). Элементы объединяются с разделителем `\|`. |
| `useThemeColors`      | boolean    | Нет          | Применить цвет активной темы `/theme` к тексту строки состояния. По умолчанию `true`.                                               |
| `hideContextIndicator`| boolean    | Нет          | Скрыть встроенный индикатор использования контекста в правой части нижнего колонтитула. По умолчанию `false`.                         |

### Доступные элементы предустановок

| ID элемента            | По умолчанию | Описание                                                                      |
| ---------------------- | ------------ | ----------------------------------------------------------------------------- |
| `model-with-reasoning` | Да           | Имя текущей модели с уровнем рассуждения (например, `qwen-3-235b high`)       |
| `model`                |              | Имя текущей модели без уровня рассуждения                                     |
| `git-branch`           | Да           | Имя текущей ветки Git (скрыто, если не в git-репозитории)                     |
| `context-remaining`    | Да           | Процент оставшегося окна контекста (например, `Context 65.7% left`)           |
| `total-input-tokens`   |              | Суммарное количество входных токенов за сессию (например, `30.0k total in`)   |
| `total-output-tokens`  |              | Суммарное количество выходных токенов за сессию (например, `5.0k total out`)  |
| `current-dir`          | Да           | Текущая рабочая директория                                                    |
| `project-name`         |              | Имя проекта (базовое имя рабочей директории)                                   |
| `pull-request-number`  |              | Номер открытого PR для текущей ветки (требуется `gh` CLI)                     |
| `branch-changes`       |              | Статистика изменений файлов за сессию (например, `+120 -30`)                   |
| `context-used`         | Да           | Процент использованного окна контекста (например, `Context 34.3% used`)       |
| `run-state`            |              | Компактное состояние сессии (`Ready`, `Working` или `Confirm`)                |
| `qwen-version`         |              | Версия Qwen Code (например, `v0.14.1`)                                       |
| `context-window-size`  |              | Общий размер окна контекста (например, `131.1k window`)                       |
| `used-tokens`          |              | Количество токенов в текущем запросе (например, `45.0k used`)                 |
| `session-id`           |              | Идентификатор текущей сессии                                                  |

Элементы, отмеченные как **По умолчанию**, предварительно выбраны при первом открытии диалога `/statusline`.

`total-input-tokens` и `total-output-tokens` — это суммы за сессию. Они накапливают использование токенов по всем оборотам, поэтому входные токены могут быстро расти, так как каждый новый запрос к модели включает текущий контекст разговора снова. Используйте `used-tokens`, если вам нужен размер текущего запроса, а не кумулятивные затраты за сессию.

### Пример вывода

С элементами по умолчанию строка состояния выглядит так:

```
qwen-3-235b high | main | Context 65.7% left | /home/user/project | Context 34.3% used
```

## Командный режим

Командный режим выполняет shell-команду, вывод которой отображается в строке состояния. Команда получает структурированный JSON-контекст через stdin для формирования вывода с учётом сессии.

### Предварительные требования

- [`jq`](https://jqlang.github.io/jq/) рекомендуется для разбора JSON-ввода (установка: `brew install jq`, `apt install jq` и т.д.)
- Простые команды, не требующие данных JSON (например, `git branch --show-current`), работают без `jq`

### Конфигурация

Добавьте объект `statusLine` под ключом `ui` в `~/.qwen/settings.json`:

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); model=$(echo \"$input\" | jq -r '.model.display_name'); pct=$(echo \"$input\" | jq -r '.context_window.used_percentage'); echo \"$model  ctx:${pct}%\""
    }
  }
}
```

| Поле                  | Тип         | Обязательное | Описание                                                                                                                                |
| --------------------- | ----------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| `type`                | `"command"` | Да           | Должно быть `"command"`                                                                                                                 |
| `command`             | string      | Да           | Shell-команда для выполнения. Получает JSON через stdin, stdout отображается (до 2 строк).                                              |
| `refreshInterval`     | number      | Нет          | Повторно выполнять команду каждые N секунд (минимум 1). Полезно для данных, которые меняются без событий состояния агента (часы, квоты, время работы). |
| `respectUserColors`   | boolean     | Нет          | Сохранять ANSI-коды цветов в выводе команды вместо применения приглушённого стиля нижнего колонтитула. По умолчанию `false`.           |
| `hideContextIndicator`| boolean     | Нет          | Скрыть встроенный индикатор использования контекста в правой части нижнего колонтитула. По умолчанию `false`.                           |

### JSON-ввод

Команда получает JSON-объект через stdin со следующими полями:

```json
{
  "session_id": "abc-123",
  "version": "0.14.1",
  "model": {
    "display_name": "qwen-3-235b"
  },
  "context_window": {
    "context_window_size": 131072,
    "used_percentage": 34.3,
    "remaining_percentage": 65.7,
    "current_usage": 45000,
    "total_input_tokens": 30000,
    "total_output_tokens": 5000
  },
  "workspace": {
    "current_dir": "/home/user/project"
  },
  "git": {
    "branch": "main"
  },
  "worktree": {
    "name": "fix-auth",
    "path": "/home/user/project/.qwen/worktrees/fix-auth",
    "branch": "fix-auth",
    "original_cwd": "/home/user/project",
    "original_branch": "main"
  },
  "metrics": {
    "models": {
      "qwen-3-235b": {
        "api": {
          "total_requests": 10,
          "total_errors": 0,
          "total_latency_ms": 5000
        },
        "tokens": {
          "prompt": 30000,
          "completion": 5000,
          "total": 35000,
          "cached": 10000,
          "thoughts": 2000
        }
      }
    },
    "files": {
      "total_lines_added": 120,
      "total_lines_removed": 30
    }
  },
  "vim": {
    "mode": "INSERT"
  }
}
```

| Поле                                 | Тип              | Описание                                                                          |
| ------------------------------------ | ---------------- | --------------------------------------------------------------------------------- |
| `session_id`                         | string           | Уникальный идентификатор сессии                                                   |
| `version`                            | string           | Версия Qwen Code                                                                 |
| `model.display_name`                 | string           | Имя текущей модели                                                               |
| `context_window.context_window_size` | number           | Общий размер окна контекста в токенах                                            |
| `context_window.used_percentage`     | number           | Процент использования окна контекста (0–100)                                     |
| `context_window.remaining_percentage`| number           | Процент оставшегося окна контекста (0–100)                                       |
| `context_window.current_usage`       | number           | Количество токенов из последнего вызова API (текущий размер контекста)             |
| `context_window.total_input_tokens`  | number           | Всего входных токенов за сессию                                                  |
| `context_window.total_output_tokens` | number           | Всего выходных токенов за сессию                                                 |
| `workspace.current_dir`              | string           | Текущая рабочая директория                                                       |
| `git`                                | object \| absent | Присутствует только внутри git-репозитория.                                      |
| `git.branch`                         | string           | Имя текущей ветки                                                                |
| `worktree`                           | object \| absent | Присутствует только внутри активного worktree (созданного с помощью `enter_worktree`). |
| `worktree.name`                      | string           | Имя worktree (слаг)                                                              |
| `worktree.path`                      | string           | Абсолютный путь к директории worktree                                            |
| `worktree.branch`                    | string           | Ветка, извлечённая в worktree                                                    |
| `worktree.original_cwd`              | string           | Рабочая директория до входа в worktree                                           |
| `worktree.original_branch`           | string           | Ветка, активная до входа в worktree                                              |
| `metrics.models.<id>.api`            | object           | Статистика API для модели: `total_requests`, `total_errors`, `total_latency_ms`  |
| `metrics.models.<id>.tokens`         | object           | Использование токенов для модели: `prompt`, `completion`, `total`, `cached`, `thoughts` |
| `metrics.files`                      | object           | Статистика изменений файлов: `total_lines_added`, `total_lines_removed`          |
| `vim`                                | object \| absent | Присутствует только когда включён vim-режим. Содержит `mode` (`"INSERT"` или `"NORMAL"`). |

> **Важно:** stdin можно прочитать только один раз. Всегда сохраняйте его в переменную в первую очередь: `input=$(cat)`.

### Примеры

#### Модель и использование токенов

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); model=$(echo \"$input\" | jq -r '.model.display_name'); pct=$(echo \"$input\" | jq -r '.context_window.used_percentage'); echo \"$model  ctx:${pct}%\""
    }
  }
}
```

Вывод: `qwen-3-235b  ctx:34%`

#### Git-ветка + директория

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); branch=$(echo \"$input\" | jq -r '.git.branch // empty'); dir=$(basename \"$(echo \"$input\" | jq -r '.workspace.current_dir')\"); echo \"$dir${branch:+ ($branch)}\""
    }
  }
}
```

Вывод: `my-project (main)`

> Примечание: Поле `git.branch` предоставляется непосредственно в JSON-вводе — нет необходимости вызывать `git` отдельно.

#### Статистика изменений файлов

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); added=$(echo \"$input\" | jq -r '.metrics.files.total_lines_added'); removed=$(echo \"$input\" | jq -r '.metrics.files.total_lines_removed'); echo \"+$added/-$removed lines\""
    }
  }
}
```

Вывод: `+120/-30 lines`

#### Живые часы и git-ветка

Используйте `refreshInterval`, когда строка состояния отображает данные, которые меняются без события агента (например, часы, время работы или счётчики ограничений):

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); branch=$(echo \"$input\" | jq -r '.git.branch // \"no-git\"'); echo \"$(date +%H:%M:%S)  ($branch)\"",
      "refreshInterval": 1
    }
  }
}
```

Вывод (обновляется каждую секунду): `14:32:07  (main)`

#### Файл скрипта для сложных команд

Для длинных команд сохраните файл скрипта в `~/.qwen/statusline-command.sh`:

```bash
#!/bin/bash
input=$(cat)
model=$(echo "$input" | jq -r '.model.display_name')
pct=$(echo "$input" | jq -r '.context_window.used_percentage')
branch=$(echo "$input" | jq -r '.git.branch // empty')
added=$(echo "$input" | jq -r '.metrics.files.total_lines_added')
removed=$(echo "$input" | jq -r '.metrics.files.total_lines_removed')

parts=()
[ -n "$model" ] && parts+=("$model")
[ -n "$branch" ] && parts+=("($branch)")
[ "$pct" != "0" ] 2>/dev/null && parts+=("ctx:${pct}%")
([ "$added" -gt 0 ] || [ "$removed" -gt 0 ]) 2>/dev/null && parts+=("+${added}/-${removed}")

echo "${parts[*]}"
```

Затем укажите его в настройках:

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "bash ~/.qwen/statusline-command.sh"
    }
  }
}
```

## Поведение

**Оба режима:**

- **Триггеры обновления**: Строка состояния обновляется при изменении модели, отправке нового сообщения (меняется количество токенов), переключении vim-режима, изменении ветки git, завершении вызова инструментов или изменении файлов. Обновления дебаунсятся (300 мс).
- **Вывод**: До 2 строк. Каждая строка отображается как отдельный ряд в левой части нижнего колонтитула. Строки, превышающие доступную ширину, усекаются.
- **Горячая перезагрузка**: Изменения в `ui.statusLine` в настройках вступают в силу немедленно — перезапуск не требуется.
- **Удаление**: Удалите ключ `ui.statusLine` из настроек, чтобы отключить. Подсказка «? for shortcuts» возвращается.

**Только командный режим:**

- **Тайм-аут**: Команды, выполняющиеся дольше 5 секунд, завершаются. Строка состояния очищается при ошибке.
- **Обновление**: Установите `refreshInterval` (секунды), чтобы дополнительно перезапускать команду по таймеру — полезно для данных, которые меняются без события агента (часы, ограничения, статус сборки).
- **Shell**: Команды выполняются через `/bin/sh` на macOS/Linux. В Windows по умолчанию используется `cmd.exe` — оборачивайте POSIX-команды с помощью `bash -c "..."` или указывайте bash-скрипт (например, `bash ~/.qwen/statusline-command.sh`).

**Только режим предустановок:**

- **Без внешних зависимостей**: Элементы предустановок вычисляются внутри — никаких shell-команд, никакого `jq`, никаких тайм-аутов.
- **Интеграция с темой**: Если `useThemeColors` равно `true` (по умолчанию), текст строки состояния использует цвет активной темы `/theme`. Если `false`, применяется приглушённый стиль нижнего колонтитула.
- **Поиск PR**: Элемент `pull-request-number` выполняет `gh pr view` в фоне (тайм-аут 2 с). Он срабатывает только при изменении ветки, а не при каждом обновлении.

## Устранение неполадок

| Проблема                          | Причина                            | Исправление                                                                                                                                                                                                                                                                                                                                                                                    |
| --------------------------------- | ---------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Строка состояния не отображается  | Конфигурация не по тому пути       | Должен быть под `ui.statusLine`, а не на корневом уровне `statusLine`                                                                                                                                                                                                                                                                                                                           |
| Пустой вывод (командный режим)    | Команда молча завершается ошибкой  | Проверьте вручную: `echo '{"session_id":"test","version":"0.14.1","model":{"display_name":"test"},"context_window":{"context_window_size":0,"used_percentage":0,"remaining_percentage":100,"current_usage":0,"total_input_tokens":0,"total_output_tokens":0},"workspace":{"current_dir":"/tmp"},"metrics":{"models":{},"files":{"total_lines_added":0,"total_lines_removed":0}}}' \| sh -c 'ваша_команда'` |
| Устаревшие данные (командный режим)| Не сработал триггер                | Отправьте сообщение или переключите модели, чтобы вызвать обновление — или установите `refreshInterval` для повторного выполнения команды по таймеру                                                                                                                                                                                                                                             |
