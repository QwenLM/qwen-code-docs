# Строка состояния

> Отображение пользовательской информации в нижнем колонтитуле.

Строка состояния показывает информацию, учитывающую текущий сеанс — имя модели, использование токенов, ветку Git и другое — в левой части нижнего колонтитула. Есть два режима настройки:

- **Режим предустановок** — выбор из встроенных элементов данных через интерактивный диалог или JSON-конфиг. Не требует написания сценариев.
- **Режим команд** — запуск shell-команды, которая получает структурированный JSON-контекст через stdin. Полная гибкость для произвольного форматирования.

```
Однострочная строка состояния (режим одобрения по умолчанию — 1 строка):
┌─────────────────────────────────────────────────────────────────┐
│  user@host ~/project (main) ctx:34%   🔒 docker | Debug | 67%  │  ← строка состояния
└─────────────────────────────────────────────────────────────────┘

Многострочная строка состояния (до 2 строк — 2 строки):
┌─────────────────────────────────────────────────────────────────┐
│  user@host ~/project (main) ctx:34%   🔒 docker | Debug | 67%  │  ← строка состояния 1
│  ████████░░░░░░░░░░ 34% context                                │  ← строка состояния 2
└─────────────────────────────────────────────────────────────────┘

Многострочная строка состояния + нестандартный режим (максимум 3 строки):
┌─────────────────────────────────────────────────────────────────┐
│  user@host ~/project (main) ctx:34%   🔒 docker | Debug | 67%  │  ← строка состояния 1
│  ████████░░░░░░░░░░ 34% context                                │  ← строка состояния 2
│  auto-accept edits (shift + tab to cycle)                       │  ← индикатор режима
└─────────────────────────────────────────────────────────────────┘
```

При настройке строка состояния заменяет подсказку по умолчанию «? for shortcuts». Сообщения высокого приоритета (подсказки Ctrl+C/D о выходе, Esc, режим INSERT в vim) временно переопределяют строку состояния. Текст строки состояния обрезается, чтобы поместиться в доступную ширину.

## Быстрая настройка

Самый простой способ настроить строку состояния — команда `/statusline`. Она открывает интерактивный диалог, где можно выбрать элементы предустановок, переключить цвета темы и увидеть предварительный просмотр в реальном времени:

```
/statusline
```

Эта команда открывает конфигуратор режима предустановок. Используйте клавиши со стрелками для навигации, пробел для переключения элементов и Enter для подтверждения. Ваш выбор автоматически сохраняется в настройках.

Вы также можете передать `/statusline` конкретные инструкции, чтобы она сгенерировала конфигурацию в режиме команд:

```
/statusline показать имя модели и процент использования контекста
```

## Режим предустановок

Режим предустановок предоставляет набор встроенных элементов данных, которые можно выбирать и комбинировать — без shell-команд, без `jq`, без сценариев. Элементы отображаются в одной строке как `item1 | item2 | item3`.

### Конфигурация

Добавьте объект `statusLine` под ключом `ui` в файл `~/.qwen/settings.json`:

```json
{
  "ui": {
    "statusLine": {
      "type": "preset",
      "items": [
        "model-with-reasoning",
        "git-branch",
        "context-remaining",
        "current-dir",
        "context-used"
      ],
      "useThemeColors": true
    }
  }
}
```

| Поле                    | Тип        | Обязательно | Описание                                                                                                 |
| ----------------------- | ---------- | ----------- | -------------------------------------------------------------------------------------------------------- |
| `type`                  | `"preset"` | Да          | Должно быть `"preset"`                                                                                   |
| `items`                 | string[]   | Да          | Упорядоченный список идентификаторов элементов предустановок для отображения (см. таблицу ниже). Элементы объединяются через `\|` в качестве разделителя. |
| `useThemeColors`        | boolean    | Нет         | Применить цвет активной темы `/theme` к тексту строки состояния. По умолчанию `true`.                    |
| `hideContextIndicator`  | boolean    | Нет         | Скрыть встроенный индикатор использования контекста в правой части нижнего колонтитула. По умолчанию `false`. |

### Доступные элементы предустановок

| Идентификатор элемента     | По умолчанию | Описание                                                               |
| -------------------------- | ------------ | ---------------------------------------------------------------------- |
| `model-with-reasoning`     | Да           | Имя текущей модели с уровнем рассуждений (например, `qwen-3-235b high`)|
| `model`                    |              | Имя текущей модели без уровня рассуждений                              |
| `git-branch`               | Да           | Имя текущей Git-ветки (скрыто, если не в репозитории git)              |
| `context-remaining`        | Да           | Процент оставшегося контекстного окна (например, `Context 65.7% left`) |
| `total-input-tokens`       |              | Накопленные входные токены, использованные в сеансе (например, `30.0k total in`) |
| `total-output-tokens`      |              | Накопленные выходные токены, использованные в сеансе (например, `5.0k total out`) |
| `current-dir`              | Да           | Текущая рабочая директория                                              |
| `project-name`             |              | Имя проекта (базовое имя рабочей директории)                            |
| `pull-request-number`      |              | Номер открытого PR для текущей ветки (требуется CLI `gh`)               |
| `branch-changes`           |              | Статистика изменений файлов за сеанс (например, `+120 -30`)             |
| `context-used`             | Да           | Процент использованного контекстного окна (например, `Context 34.3% used`) |
| `run-state`                |              | Компактное состояние сеанса (`Ready`, `Working` или `Confirm`)          |
| `qwen-version`             |              | Версия Qwen Code (например, `v0.14.1`)                                 |
| `context-window-size`      |              | Общий размер контекстного окна (например, `131.1k window`)              |
| `used-tokens`              |              | Количество токенов текущего запроса (например, `45.0k used`)            |
| `session-id`               |              | Идентификатор текущего сеанса                                           |
Элементы, отмеченные **Default**, предварительно выбраны при первом открытии диалога `/statusline`.

`total-input-tokens` и `total-output-tokens` являются суммарными за сессию. Они накапливают использование токенов между обращениями, поэтому количество входных токенов может быстро расти, поскольку каждый новый запрос к модели снова включает текущий контекст беседы. Используйте `used-tokens`, если вам нужен текущий размер подсказки, а не совокупные расходы за сессию.

### Пример вывода

С элементами по умолчанию строка состояния выглядит так:

```
qwen-3-235b high | main | Context 65.7% left | /home/user/project | Context 34.3% used
```

## Режим команды

Режим команды выполняет shell-команду, вывод которой отображается в строке состояния. Команда получает структурированный JSON-контекст через stdin для создания вывода с учетом сессии.

### Предварительные требования

- [`jq`](https://jqlang.github.io/jq/) рекомендуется для разбора JSON-ввода (установите через `brew install jq`, `apt install jq` и т.д.)
- Простые команды, не требующие JSON-данных (например, `git branch --show-current`), работают без `jq`

### Конфигурация

Добавьте объект `statusLine` в раздел `ui` файла `~/.qwen/settings.json`:

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); model=$(echo \"$input\" | jq -r '.model.display_name'); pct=$(echo \"$input\" | jq -r '.context_window.used_percentage'); echo \"$model  ctx:${pct}%\""
    }
  }
}
```

| Поле                  | Тип        | Обязательно | Описание                                                                                                                       |
| ---------------------- | ----------- | -------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `type`                 | `"command"` | Да       | Должно быть `"command"`                                                                                                               |
| `command`              | строка      | Да       | Shell-команда для выполнения. Получает JSON через stdin, stdout отображается (до 2 строк).                                           |
| `refreshInterval`      | число       | Нет      | Повторно запускать команду каждые N секунд (минимум 1). Полезно для данных, которые изменяются без события состояния агента (часы, квота, время работы). |
| `respectUserColors`    | булево      | Нет       | Сохранять ANSI-цветовые коды в выводе команды вместо применения затемненного стиля нижнего колонтитула. По умолчанию `false`.                       |
| `hideContextIndicator` | булево      | Нет       | Скрыть встроенный индикатор использования контекста в правой части нижнего колонтитула. По умолчанию `false`.                                       |

### JSON-ввод

Команда получает JSON-объект через stdin со следующими полями:

```json
{
  "session_id": "abc-123",
  "version": "0.14.1",
  "model": {
    "display_name": "qwen-3-235b"
  },
  "context_window": {
    "context_window_size": 131072,
    "used_percentage": 34.3,
    "remaining_percentage": 65.7,
    "current_usage": 45000,
    "total_input_tokens": 30000,
    "total_output_tokens": 5000
  },
  "workspace": {
    "current_dir": "/home/user/project"
  },
  "git": {
    "branch": "main"
  },
  "worktree": {
    "name": "fix-auth",
    "path": "/home/user/project/.qwen/worktrees/fix-auth",
    "branch": "fix-auth",
    "original_cwd": "/home/user/project",
    "original_branch": "main"
  },
  "metrics": {
    "models": {
      "qwen-3-235b": {
        "api": {
          "total_requests": 10,
          "total_errors": 0,
          "total_latency_ms": 5000
        },
        "tokens": {
          "prompt": 30000,
          "completion": 5000,
          "total": 35000,
          "cached": 10000,
          "thoughts": 2000
        }
      }
    },
    "files": {
      "total_lines_added": 120,
      "total_lines_removed": 30
    }
  },
  "vim": {
    "mode": "INSERT"
  }
}
```
| Поле                                | Тип               | Описание                                                                                                    |
| ----------------------------------- | ----------------- | ----------------------------------------------------------------------------------------------------------- |
| `session_id`                        | string            | Уникальный идентификатор сессии                                                                              |
| `version`                           | string            | Версия Qwen Code                                                                                            |
| `model.display_name`                | string            | Текущее название модели                                                                                     |
| `context_window.context_window_size` | number            | Общий размер окна контекста в токенах                                                                       |
| `context_window.used_percentage`    | number            | Использование окна контекста в процентах (0–100)                                                            |
| `context_window.remaining_percentage`| number            | Оставшееся место в окне контекста в процентах (0–100)                                                       |
| `context_window.current_usage`      | number            | Количество токенов от последнего вызова API (текущий размер контекста)                                      |
| `context_window.total_input_tokens` | number            | Всего входных токенов, потреблённых за сессию                                                               |
| `context_window.total_output_tokens`| number            | Всего выходных токенов, потреблённых за сессию                                                              |
| `workspace.current_dir`             | string            | Текущая рабочая директория                                                                                  |
| `git`                               | object \| absent  | Присутствует только внутри git-репозитория.                                                                 |
| `git.branch`                        | string            | Имя текущей ветки                                                                                           |
| `worktree`                          | object \| absent  | Присутствует только при нахождении в активном worktree (созданном через `enter_worktree`).                  |
| `worktree.name`                     | string            | Имя-слаг worktree                                                                                           |
| `worktree.path`                     | string            | Абсолютный путь к директории worktree                                                                       |
| `worktree.branch`                   | string            | Ветка, которая развёрнута в worktree                                                                        |
| `worktree.original_cwd`             | string            | Рабочая директория до входа в worktree                                                                      |
| `worktree.original_branch`          | string            | Ветка, которая была активна до входа в worktree                                                             |
| `metrics.models.<id>.api`           | object            | Статистика по API для модели: `total_requests`, `total_errors`, `total_latency_ms`                          |
| `metrics.models.<id>.tokens`        | object            | Использование токенов для модели: `prompt`, `completion`, `total`, `cached`, `thoughts`                     |
| `metrics.files`                     | object            | Статистика изменений в файлах: `total_lines_added`, `total_lines_removed`                                   |
| `vim`                               | object \| absent  | Присутствует только когда включён vim-режим. Содержит `mode` (`"INSERT"` или `"NORMAL"`).                  |

> **Важно:** stdin можно прочитать только один раз. Всегда сначала сохраняйте его в переменную: `input=$(cat)`.

### Примеры

#### Использование модели и токены

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); model=$(echo \"$input\" | jq -r '.model.display_name'); pct=$(echo \"$input\" | jq -r '.context_window.used_percentage'); echo \"$model  ctx:${pct}%\""
    }
  }
}
```

Вывод: `qwen-3-235b  ctx:34%`

#### Ветка git + директория

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); branch=$(echo \"$input\" | jq -r '.git.branch // empty'); dir=$(basename \"$(echo \"$input\" | jq -r '.workspace.current_dir')\"); echo \"$dir${branch:+ ($branch)}\""
    }
  }
}
```

Вывод: `my-project (main)`

> Примечание: Поле `git.branch` передаётся непосредственно в JSON-входе — нет необходимости вызывать `git` из оболочки.

#### Статистика изменений в файлах

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); added=$(echo \"$input\" | jq -r '.metrics.files.total_lines_added'); removed=$(echo \"$input\" | jq -r '.metrics.files.total_lines_removed'); echo \"+$added/-$removed lines\""
    }
  }
}
```

Вывод: `+120/-30 lines`

#### Часы реального времени и ветка git

Используйте `refreshInterval`, когда строка состояния показывает данные, которые меняются без события Agent (например, часы, время работы или счётчики лимитов скорости):

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "input=$(cat); branch=$(echo \"$input\" | jq -r '.git.branch // \"no-git\"'); echo \"$(date +%H:%M:%S)  ($branch)\"",
      "refreshInterval": 1
    }
  }
}
```
Вывод (обновляется каждую секунду): `14:32:07  (main)`

#### Файл сценария для сложных команд

Для длинных команд сохраните файл сценария по пути `~/.qwen/statusline-command.sh`:

```bash
#!/bin/bash
input=$(cat)
model=$(echo "$input" | jq -r '.model.display_name')
pct=$(echo "$input" | jq -r '.context_window.used_percentage')
branch=$(echo "$input" | jq -r '.git.branch // empty')
added=$(echo "$input" | jq -r '.metrics.files.total_lines_added')
removed=$(echo "$input" | jq -r '.metrics.files.total_lines_removed')

parts=()
[ -n "$model" ] && parts+=("$model")
[ -n "$branch" ] && parts+=("($branch)")
[ "$pct" != "0" ] 2>/dev/null && parts+=("ctx:${pct}%")
([ "$added" -gt 0 ] || [ "$removed" -gt 0 ]) 2>/dev/null && parts+=("+${added}/-${removed}")

echo "${parts[*]}"
```

Затем укажите его в настройках:

```json
{
  "ui": {
    "statusLine": {
      "type": "command",
      "command": "bash ~/.qwen/statusline-command.sh"
    }
  }
}
```

## Поведение

**Оба режима:**

- **Триггеры обновления**: Строка состояния обновляется при смене модели, отправке нового сообщения (изменение количества токенов), переключении режима vim, изменении ветки git, завершении вызовов инструментов или изменении файлов. Обновления сглаживаются (300 мс).
- **Вывод**: До 2 строк. Каждая строка отображается как отдельная строка в левой части нижнего колонтитула. Строки, превышающие доступную ширину, обрезаются.
- **Горячая перезагрузка**: Изменения `ui.statusLine` в настройках вступают в силу немедленно — перезапуск не требуется.
- **Удаление**: Удалите ключ `ui.statusLine` из настроек, чтобы отключить. Подсказка '? для сочетаний клавиш' возвращается.

**Только режим команды:**

- **Тайм-аут**: Команды, выполняющиеся дольше 5 секунд, завершаются. Строка состояния очищается при ошибке.
- **Обновление**: Установите `refreshInterval` (секунды), чтобы дополнительно повторно запускать команду по таймеру — полезно для данных, изменяющихся без события Agent (часы, лимиты запросов, статус сборки).
- **Оболочка**: Команды выполняются через `/bin/sh` на macOS/Linux. В Windows по умолчанию используется `cmd.exe` — оберните POSIX-команды в `bash -c "..."` или укажите bash-скрипт (например, `bash ~/.qwen/statusline-command.sh`).

**Только режим предустановок:**

- **Нет внешних зависимостей**: Элементы предустановок вычисляются внутренне — никаких команд оболочки, никакого `jq`, никаких тайм-аутов.
- **Интеграция с темой**: Если `useThemeColors` имеет значение `true` (по умолчанию), текст строки состояния использует активный цвет `/theme`. Если `false`, применяется стиль тусклого нижнего колонтитула.
- **Поиск PR**: Элемент `pull-request-number` запускает `gh pr view` в фоне (тайм-аут 2 с). Он срабатывает только при изменении ветки, а не при каждом обновлении.

## Устранение неполадок

| Проблема                     | Причина                          | Решение                                                                                                                                                                                                                                                                                                                                                                                               |
| --------------------------- | ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Строка состояния не отображается     | Неправильный путь в конфиге           | Должно быть внутри `ui.statusLine`, а не на корневом уровне `statusLine`                                                                                                                                                                                                                                                                                                                             |
| Пустой вывод (режим команды) | Команда молча завершается с ошибкой         | Протестируйте вручную: `echo '{"session_id":"test","version":"0.14.1","model":{"display_name":"test"},"context_window":{"context_window_size":0,"used_percentage":0,"remaining_percentage":100,"current_usage":0,"total_input_tokens":0,"total_output_tokens":0},"workspace":{"current_dir":"/tmp"},"metrics":{"models":{},"files":{"total_lines_added":0,"total_lines_removed":0}}}' \| sh -c 'your_command'` |
