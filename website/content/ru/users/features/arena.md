# Agent Arena

> Одновременный запуск нескольких ИИ-моделей для выполнения одной задачи, сравнение их решений бок о бок и выбор лучшего результата для применения в вашем рабочем пространстве.

> [!warning]
> Agent Arena является экспериментальной. У неё есть [известные ограничения](#limitations) касающиеся режимов отображения и управления сессиями.

Agent Arena позволяет столкнуть несколько ИИ-моделей друг с другом при выполнении одной задачи. Каждая модель работает как полностью независимый агент в своей изолированной рабочей копии Git, так что файловые операции никогда не пересекаются. Когда все агенты завершают работу, вы сравниваете результаты и выбираете победителя, чтобы объединить его с основным рабочим пространством.

В отличие от [подагентов](./sub-agents.md), которые выполняют узкие подзадачи в рамках одной сессии, агенты Arena — это полноценные, самостоятельные экземпляры агентов, каждый со своей моделью, контекстом и полным доступом к инструментам.

## Когда использовать Agent Arena

Agent Arena наиболее эффективна, когда вы хотите **оценить или сравнить**, как разные модели справляются с одной и той же задачей. Самые сильные сценарии использования:

- **Бенчмаркинг моделей**: оценка возможностей разных моделей на реальных задачах в вашей фактической кодовой базе, а не на синтетических тестах.
- **Выбор лучшего из N**: получение нескольких независимых решений и выбор наилучшей реализации.
- **Исследование подходов**: наблюдение за тем, как разные модели размышляют и решают одну и ту же задачу — полезно для обучения и понимания.
- **Снижение рисков**: для критических изменений — проверка, что несколько моделей сходятся к схожему подходу перед фиксацией.

Agent Arena использует существенно больше токенов, чем одиночная сессия (у каждого агента свой контекст и вызовы модели). Она лучше всего подходит, когда ценность сравнения оправдывает затраты. Для рутинных задач, когда вы доверяете своей модели по умолчанию, одиночная сессия эффективнее.

## Запуск сессии Arena

Используйте слэш-команду `/arena`, чтобы запустить сессию. Укажите модели, которые должны соревноваться, и задачу:

```
/arena --models qwen3.5-plus,glm-5,kimi-k2.5 "Рефакторинг модуля аутентификации для использования JWT-токенов"
```

Если опустить `--models`, появится интерактивный диалог выбора моделей, позволяющий выбрать из настроенных провайдеров.

### Что происходит при запуске

1. **Настройка рабочей копии**: Qwen Code создаёт изолированные рабочие копии Git для каждого агента в `~/.qwen/arena/<session-id>/worktrees/<model-name>/`. Каждая рабочая копия в точности отражает текущее состояние вашего рабочего каталога — включая проиндексированные изменения, неиндексированные изменения и неотслеживаемые файлы.
2. **Запуск агентов**: Каждый агент запускается в своей собственной рабочей копии с полным доступом к инструментам и настроенной моделью. Агенты запускаются последовательно, но выполняются параллельно.
3. **Выполнение**: Все агенты работают над задачей независимо, без общего состояния или связи. Вы можете отслеживать их прогресс и взаимодействовать с любым из них.
4. **Завершение**: Когда все агенты завершат работу (или потерпят неудачу), вы переходите к фазе сравнения результатов.

## Взаимодействие с агентами

### Режимы отображения

В настоящее время Agent Arena поддерживает **режим в процессе**, когда все агенты выполняются асинхронно в одном и том же процессе терминала. Панель вкладок внизу терминала позволяет переключаться между агентами.

> [!note]
> **Режимы отображения с разделёнными панелями запланированы на будущее.** Мы планируем поддерживать макеты с разделением на панели на основе tmux и iTerm2, где каждый агент получит собственную панель терминала для настоящего просмотра бок о бок. В настоящее время доступно только переключение вкладок внутри процесса.

### Навигация между агентами

В режиме в процессе используйте сочетания клавиш для переключения между представлениями агентов:

| Сочетание  | Действие                                   |
| :--------- | :----------------------------------------- |
| `Вправо`   | Переключиться на следующую вкладку агента  |
| `Влево`    | Переключиться на предыдущую вкладку агента |
| `Вверх`    | Перевести фокус на поле ввода              |
| `Вниз`     | Перевести фокус на панель вкладок агентов  |

Панель вкладок показывает текущий статус каждого агента:

| Индикатор | Значение                  |
| :-------- | :------------------------ |
| `●`       | Выполняется или ожидает   |
| `✓`       | Успешно завершён          |
| `✗`       | Завершился с ошибкой      |
| `○`       | Отменён                   |

### Взаимодействие с отдельными агентами

Просматривая вкладку агента, вы можете:

- **Отправлять сообщения** — вводить текст в область ввода, чтобы дать агенту дополнительные инструкции
- **Подтверждать вызовы инструментов** — если агент запрашивает подтверждение использования инструмента, диалог подтверждения появляется в его вкладке
- **Просматривать полную историю** — пролистывать весь диалог агента: вывод модели, вызовы инструментов и результаты

Каждый агент — это полноценная независимая сессия. Всё, что можно сделать с основным агентом, можно сделать и с агентом в Arena.

## Сравнение результатов и выбор победителя

Когда все агенты завершат работу, Arena переходит в фазу сравнения результатов. Вы увидите:
- **Сводка статуса**: Какие агенты успешно завершили работу, какие завершились с ошибкой или были отменены  
- **Метрики выполнения**: Продолжительность, количество раундов рассуждений, использование токенов и количество вызовов инструментов для каждого агента  
- **Сводка сравнения арены**: Файлы, изменённые совместно или только одним агентом, количество изменённых строк, эффективность токенов и краткое описание подхода, сформированное на основе diff, метрик и истории беседы каждого агента  

Диалог выбора отображает успешных агентов. Выберите одного, чтобы применить его изменения к вашему основному рабочему пространству, или отклоните все результаты. Нажмите `p`, чтобы включить быстрый предварительный просмотр для выделенного агента, или `d`, чтобы включить подробный diff этого агента перед выбором победителя.

### Что происходит при выборе победителя

1. Изменения агента-победителя извлекаются в виде diff относительно базовой версии  
2. Diff применяется к вашей основной рабочей директории  
3. Все рабочие деревья (worktrees) и временные ветки автоматически очищаются  

Если вы хотите проверить полный путь рассуждений перед принятием решения, полная история беседы каждого агента по-прежнему доступна через панель вкладок, пока активен диалог выбора.

## Конфигурация

Поведение арены можно настроить в [settings.json](../configuration/settings.md):

```json
{
  "arena": {
    "worktreeBaseDir": "~/.qwen/arena",
    "maxRoundsPerAgent": 50,
    "timeoutSeconds": 600
  }
}
```

| Параметр                    | Описание                                  | Значение по умолчанию |
| :-------------------------- | :---------------------------------------- | :-------------------- |
| `arena.worktreeBaseDir`     | Базовая директория для рабочих деревьев   | `~/.qwen/arena`       |
| `arena.maxRoundsPerAgent`   | Максимальное количество раундов рассуждений | `50`                  |
| `arena.timeoutSeconds`      | Тайм-аут для каждого агента (в секундах)  | `600`                 |

## Рекомендации

### Выбирайте модели, дополняющие друг друга

Арена наиболее ценна, когда вы сравниваете модели со значимо разными сильными сторонами. Например:

```
/arena --models qwen3.5-plus,glm-5,kimi-k2.5 "Оптимизировать запросы к базе данных"
```

Сравнение трёх версий одного семейства моделей даёт меньше пользы, чем сравнение моделей от разных провайдеров.

### Формулируйте задачи как самодостаточные

Агенты арены работают независимо, без общения друг с другом. Задачи должны быть полностью описаны в промпте, не требуя обмена уточнениями:

**Хорошо**: «Рефакторинг платёжного модуля с использованием паттерна «Стратегия». Обновить все тесты.»

**Менее эффективно**: «Давайте обсудим, как улучшить платёжный модуль» — это выигрывает от обсуждения, которое лучше подходит для одного сеанса.

### Ограничьте количество агентов

До 5 агентов могут выполняться одновременно. На практике 2-3 агента дают наилучший баланс между ценностью сравнения и затратами ресурсов. Больше агентов означает:

- Более высокие затраты на токены (каждый агент имеет свой контекст)
- Увеличение общего времени выполнения
- Больше результатов для сравнения

Начните с 2-3 и увеличивайте количество только когда ценность сравнения того стоит.

### Используйте агену для решений с высокой ценностью

Арена особенно полезна, когда ставки оправдывают запуск нескольких моделей:

- Выбор архитектуры для нового модуля
- Выбор подхода для сложного рефакторинга
- Проверка критического исправления ошибки с разных сторон

Для рутинных изменений, таких как переименование переменной или обновление конфигурационного файла, одного сеанса быстрее и дешевле.

## Устранение неполадок

### Агенты не запускаются

- Убедитесь, что каждая модель в `--models` правильно настроена и имеет действительные учётные данные API
- Проверьте, что ваша рабочая директория является репозиторием Git (для рабочих деревьев требуется Git)
- Убедитесь, что у вас есть права на запись в базовую директорию рабочих деревьев (`~/.qwen/arena/` по умолчанию)

### Не удаётся создать рабочее дерево

- Выполните `git worktree list`, чтобы проверить наличие устаревших рабочих деревьев от предыдущих сеансов
- Очистите устаревшие рабочие деревья с помощью `git worktree prune`
- Убедитесь, что ваша версия Git поддерживает рабочие деревья (`git --version`, требуется Git 2.5+)

### Агент работает слишком долго

- Увеличьте тайм-аут: установите `arena.timeoutSeconds` в настройках
- Уменьшите сложность задачи — задачи арены должны быть сфокусированными и чётко определёнными
- Снизьте `arena.maxRoundsPerAgent`, если агенты тратят слишком много раундов

### Не удаётся применить изменения победителя

- Проверьте наличие незакоммиченных изменений в вашей основной рабочей директории, которые могут вызвать конфликты
- Diff применяется в виде патча — возможны конфликты слияния, если ваша рабочая директория изменилась во время сеанса

## Ограничения

Агентная Арена является экспериментальной. Текущие ограничения:

- **Только режим в одном процессе**: Отображение разделённого экрана через tmux или iTerm2 пока недоступно. Все агенты работают в одном окне терминала с переключением вкладок.
- **Нет предпросмотра diff перед выбором**: Вы можете просмотреть историю беседы каждого агента, но нет единого diff-просмотрщика для сравнения решений бок о бок перед выбором победителя.
- **Рабочие деревья не сохраняются**: Рабочие деревья всегда очищаются после выбора. Нет опции для их сохранения для дальнейшего анализа.
- **Сеанс не возобновляется**: Сеансы арены нельзя возобновить после выхода. Если вы закроете терминал в середине сеанса, рабочие деревья останутся на диске и должны быть очищены вручную с помощью `git worktree prune`.
- **Максимум 5 агентов**: Жёсткое ограничение в 5 одновременных агентов не может быть изменено.
- **Требуется репозиторий Git**: Для изоляции через рабочие деревья требуется репозиторий Git. Арену нельзя использовать в не-Git-директориях.
## Сравнение с другими многопользовательскими режимами

Agent Arena — один из нескольких запланированных многопользовательских режимов в Qwen Code. **Agent Team** и **Agent Swarm** пока не реализованы — в таблице ниже описана их предполагаемая схема для ознакомления.

|                   | **Agent Arena**                                        | **Agent Team** (планируется)                        | **Agent Swarm** (планируется)                                |
| :---------------- | :----------------------------------------------------- | :-------------------------------------------------- | :----------------------------------------------------------- |
| **Цель**          | Конкурентный: найти лучшее решение для _одной_ задачи  | Совместный: работать над _разными_ аспектами вместе | Пакетно-параллельный: динамически создавать воркеров для массовых задач |
| **Агенты**        | Предварительно настроенные модели соревнуются независимо | Участники команды сотрудничают с назначенными ролями | Воркеры создаются на лету, уничтожаются после завершения     |
| **Коммуникация**  | Нет общения между агентами                              | Прямой обмен сообщениями между участниками          | Однонаправленная: результаты собираются родительским агентом |
| **Изоляция**      | Полная: отдельные Git worktree                         | Независимые сессии с общим списком задач            | Легковесный эфемерный контекст для каждого воркера          |
| **Результат**     | Одно выбранное решение применяется к рабочей области   | Синтезированные результаты с разных точек зрения    | Агрегированные результаты параллельной обработки             |
| **Лучше всего**   | Бенчмаркинг, выбор между подходами моделей             | Исследования, сложное взаимодействие, межслоевая работа | Пакетные операции, обработка данных, задачи map-reduce      |

## Следующие шаги

Изучите смежные подходы для параллельной и делегированной работы:

- **Лёгкое делегирование**: [Subagents](./sub-agents.md) обрабатывают отдельные подзадачи в вашем сеансе — лучше, когда не нужно сравнение моделей
- **Ручные параллельные сеансы**: запустите несколько сеансов Qwen Code самостоятельно в отдельных терминалах с помощью [Git worktree](https://git-scm.com/docs/git-worktree) для полного ручного контроля
