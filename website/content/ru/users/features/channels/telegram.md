# Telegram

Это руководство описывает настройку канала Qwen Code в Telegram.

## Предварительные требования

- Учётная запись Telegram
- Токен бота Telegram (см. ниже)

## Создание бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot` и следуйте инструкциям, чтобы выбрать имя и username
3. BotFather выдаст токен бота — сохраните его в безопасности

## Поиск вашего User ID

Чтобы использовать `senderPolicy: "allowlist"` или `"pairing"`, вам понадобится ваш Telegram user ID (числовой идентификатор, не username).

Самый простой способ его найти:

1. Найдите [@userinfobot](https://t.me/userinfobot) в Telegram
2. Отправьте ему любое сообщение — он ответит вашим user ID

## Конфигурация

Добавьте канал в `~/.qwen/settings.json`:

```json
{
  "channels": {
    "my-telegram": {
      "type": "telegram",
      "token": "$TELEGRAM_BOT_TOKEN",
      "senderPolicy": "allowlist",
      "allowedUsers": ["YOUR_USER_ID"],
      "sessionScope": "user",
      "cwd": "/path/to/your/project",
      "instructions": "You are a concise coding assistant responding via Telegram. Keep responses short.",
      "groupPolicy": "disabled",
      "groups": {
        "*": { "requireMention": true }
      }
    }
  }
}
```

Установите токен бота как переменную окружения:

```bash
export TELEGRAM_BOT_TOKEN=<ваш-токен-от-botfather>
```

Или добавьте его в файл `.env`, который загружается перед запуском.

## Запуск

```bash
# Запустить только Telegram канал
qwen channel start my-telegram

# Или запустить все настроенные каналы вместе
qwen channel start
```

Затем откройте своего бота в Telegram и отправьте сообщение. Вы должны сразу увидеть "Working...", а затем ответ агента.

## Групповые чаты

Чтобы использовать бота в группах Telegram:

1. Установите `groupPolicy` в `"allowlist"` или `"open"` в конфигурации канала
2. **Отключите режим приватности** в BotFather: `/mybots` → выберите бота → Bot Settings → Group Privacy → Turn Off
3. Добавьте бота в группу. Если он уже был в группе, **удалите и добавьте его заново** (Telegram кэширует настройки приватности с момента добавления бота)
4. Если используете `groupPolicy: "allowlist"`, добавьте chat ID группы в `groups` в вашей конфигурации

По умолчанию бот требует @упоминание или ответ в группах. Установите `"requireMention": false` для конкретной группы, чтобы он отвечал на все сообщения (полезно для выделенных групп задач). Подробнее см. [Групповые чаты](./overview#group-chats).

## Изображения и файлы

Вы можете отправлять боту фотографии и документы, не только текст.

**Фотографии:** Отправьте фото, и агент проанализирует его с помощью возможностей зрения. Для этого требуется мультимодальная модель — добавьте `"model": "qwen3.5-plus"` (или другую модель с поддержкой зрения) в конфигурацию канала. Подписи к фото передаются как текст сообщения.

**Документы:** Отправьте PDF, файл с кодом или любой документ. Бот скачивает его и сохраняет локально, чтобы агент мог прочитать его с помощью своих файловых инструментов. Это работает с любой моделью. Ограничение Telegram на размер файла — 20MB.

## Советы

- **Делайте инструкции краткими** — в Telegram есть ограничение в 4096 символов на сообщение. Добавление инструкций вроде "keep responses short" помогает агенту оставаться в рамках.
- **Используйте `sessionScope: "user"`** — это даёт каждому пользователю свой собственный диалог. Используйте `/clear`, чтобы начать заново.
- **Ограничьте доступ** — используйте `senderPolicy: "allowlist"` для фиксированного набора пользователей или `"pairing"`, чтобы новые пользователи могли запросить доступ с кодом, который вы одобряете через CLI. Подробнее см. [DM Pairing](./overview#dm-pairing).

## Форматирование сообщений

Ответы агента в формате Markdown автоматически преобразуются в совместимый с Telegram HTML. Поддерживаются блоки кода, полужирный, курсив, ссылки и списки.

## Устранение неполадок

### Бот не отвечает

- Проверьте, что токен бота верен и переменная окружения установлена
- Убедитесь, что ваш user ID указан в `allowedUsers` при использовании `senderPolicy: "allowlist"`, или что вы одобрены при использовании `"pairing"`
- Проверьте вывод терминала на наличие ошибок

### Бот не отвечает в группах

- Убедитесь, что `groupPolicy` установлен в `"allowlist"` или `"open"` (по умолчанию `"disabled"`)
- При использовании `"allowlist"` проверьте, что chat ID группы указан в конфигурации `groups`
- Убедитесь, что **Group Privacy отключён** в BotFather — без этого бот не видит некомандные сообщения в группах
- Если вы изменили режим приватности после добавления бота в группу, **удалите и добавьте бота заново**
- По умолчанию бот требует @упоминание или ответ. Отправьте `@имя_вашего_бота привет` для проверки

### "Sorry, something went wrong processing your message"

Обычно это означает, что агент столкнулся с ошибкой. Проверьте вывод терминала для получения подробностей.

### Бот долго отвечает

Агент может выполнять несколько вызовов инструментов (чтение файлов, поиск и т.д.). Индикатор "Working..." показывает, что агент обрабатывает запрос. Сложные задачи могут занимать минуту или больше.
