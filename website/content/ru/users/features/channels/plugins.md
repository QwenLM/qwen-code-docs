# Пользовательские плагины каналов

Вы можете расширить систему каналов с помощью пользовательских адаптеров платформ, упакованных как [расширения](../../extension/introduction). Это позволяет подключать Qwen Code к любой платформе обмена сообщениями, веб-хуку или пользовательскому транспорту.

## Как это работает

Плагины каналов загружаются при запуске из активных расширений. Когда выполняется `qwen channel start`, он:

1. Сканирует все включенные расширения на наличие записей `channels` в их `qwen-extension.json`
2. Динамически импортирует точку входа каждого канала
3. Регистрирует тип канала, чтобы на него можно было ссылаться в `settings.json`
4. Создает экземпляры каналов с помощью фабричной функции плагина

Ваш пользовательский канал получает полный общий конвейер бесплатно: фильтрация отправителей, групповые политики, маршрутизация сессий, слеш-команды, восстановление после сбоев и мост ACP к агенту.

## Установка пользовательского канала

Установите расширение, которое предоставляет плагин канала:

```bash
# Из локальной папки (для разработки или закрытых плагинов)
qwen extensions install /path/to/my-channel-extension

# Или привяжите его для разработки (изменения применяются немедленно)
qwen extensions link /path/to/my-channel-extension
```

## Настройка пользовательского канала

Добавьте запись канала в `~/.qwen/settings.json`, используя пользовательский тип, предоставленный расширением:

```json
{
  "channels": {
    "my-bot": {
      "type": "my-platform",
      "apiKey": "$MY_PLATFORM_API_KEY",
      "senderPolicy": "open",
      "cwd": "/path/to/project"
    }
  }
}
```

`type` должен совпадать с типом канала, зарегистрированным установленным расширением. Обратитесь к документации расширения, чтобы узнать, какие поля, специфичные для плагина, являются обязательными (например, `apiKey`, `webhookUrl`).

Все стандартные опции канала работают с пользовательскими каналами:

| Опция             | Описание                                                              |
| ----------------- | --------------------------------------------------------------------- |
| `senderPolicy`    | `allowlist`, `pairing` или `open`                                     |
| `allowedUsers`    | Статический белый список идентификаторов отправителей                  |
| `sessionScope`    | `user`, `thread` или `single`                                         |
| `cwd`             | Рабочая директория для агента                                          |
| `instructions`    | Добавляется к первому сообщению каждой сессии                         |
| `model`           | Переопределение модели для канала                                     |
| `groupPolicy`     | `disabled`, `allowlist` или `open`                                    |
| `groups`          | Настройки для каждой группы                                            |

См. [Обзор](./overview) для получения подробной информации о каждой опции.

## Запуск канала

```bash
# Запуск всех каналов, включая пользовательские
qwen channel start

# Запуск только вашего пользовательского канала
qwen channel start my-bot
```

## Что вы получаете бесплатно

Пользовательские каналы автоматически поддерживают всё, что делают встроенные каналы:

- **Политики отправителей** — контроль доступа `allowlist`, `pairing` и `open`
- **Групповые политики** — Настройки для каждой группы с опциональной фильтрацией по @упоминаниям
- **Маршрутизация сессий** — Сессии для каждого пользователя, каждого треда или одна общая
- **Сопряжение ЛС** — Полный процесс кода сопряжения для неизвестных пользователей
- **Слеш-команды** — `/help`, `/clear`, `/status` работают из коробки
- **Пользовательские инструкции** — Добавляются к первому сообщению в каждой сессии
- **Восстановление после сбоев** — Автоматический перезапуск с сохранением сессии
- **Сериализация сессий** — Сообщения ставятся в очередь для предотвращения состояний гонки
