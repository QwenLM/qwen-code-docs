# Пользовательские каналы-плагины

Вы можете расширить систему каналов с помощью собственных адаптеров платформ, упакованных как [расширения](../../extension/introduction). Это позволяет подключить Qwen Code к любой платформе обмена сообщениями, вебхуку или собственному транспорту.

## Как это работает

Каналы-плагины загружаются при запуске из активных расширений. Когда выполняется `qwen channel start`:

1. Сканируются все включённые расширения на наличие записей `channels` в их `qwen-extension.json`
2. Динамически импортируется точка входа каждого канала
3. Регистрируется тип канала, чтобы на него можно было ссылаться в `settings.json`
4. Создаются экземпляры каналов с помощью фабричной функции плагина

Ваш собственный канал получает полный общий конвейер бесплатно: ограничение отправителей, групповые политики, маршрутизация сессий, слэш-команды, восстановление после сбоев и мост ACP к агенту.

## Установка собственного канала

Установите расширение, предоставляющее канал-плагин:

```bash
# Из локального пути (для разработки или частных плагинов)
qwen extensions install /path/to/my-channel-extension

# Или свяжите его для разработки (изменения применяются мгновенно)
qwen extensions link /path/to/my-channel-extension
```

## Настройка собственного канала

Добавьте запись канала в `~/.qwen/settings.json`, используя собственный тип, предоставленный расширением:

```json
{
  "channels": {
    "my-bot": {
      "type": "my-platform",
      "apiKey": "$MY_PLATFORM_API_KEY",
      "senderPolicy": "open",
      "cwd": "/path/to/project"
    }
  }
}
```

Поле `type` должно соответствовать типу канала, зарегистрированному установленным расширением. Проверьте документацию расширения, чтобы узнать, какие поля, специфичные для плагина, требуются (например, `apiKey`, `webhookUrl`).

Все стандартные параметры канала работают с пользовательскими каналами:

| Параметр         | Описание                                           |
| ---------------- | -------------------------------------------------- |
| `senderPolicy`   | `allowlist`, `pairing` или `open`                  |
| `allowedUsers`   | Статический белый список идентификаторов отправителей |
| `sessionScope`   | `user`, `thread` или `single`                      |
| `cwd`            | Рабочая директория для агента                      |
| `instructions`   | Добавляется к первому сообщению каждой сессии        |
| `model`          | Переопределение модели для канала                    |
| `groupPolicy`    | `disabled`, `allowlist` или `open`                 |
| `groups`         | Настройки для каждой группы                          |

Подробнее о каждом параметре см. в разделе [Обзор](./overview).

## Запуск канала

```bash
# Запустить все каналы, включая собственные
qwen channel start

# Запустить только ваш собственный канал
qwen channel start my-bot
```

## Что вы получаете бесплатно

Пользовательские каналы автоматически поддерживают всё, что и встроенные каналы:

- **Политики отправителей** — контроль доступа `allowlist`, `pairing` и `open`
- **Групповые политики** — настройки для каждой группы с опциональным ограничением по @упоминаниям
- **Маршрутизация сессий** — отдельные сессии для пользователя, треда или общие
- **Парная связь через ЛС** — полный поток с кодом пары для неизвестных пользователей
- **Слэш-команды** — `/help`, `/clear`, `/status` работают из коробки
- **Пользовательские инструкции** — добавляются к первому сообщению в каждой сессии
- **Восстановление после сбоев** — автоматический перезапуск с сохранением сессии
- **Последовательная сериализация сессий** — сообщения ставятся в очередь для предотвращения состояний гонки
