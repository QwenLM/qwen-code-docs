# Рекомендации продолжения

Qwen Code может предсказывать, что вы хотите ввести дальше, и показывать это в виде подсказки (placeholder) в области ввода. Эта функция использует вызов LLM для анализа контекста беседы и генерации естественного предложения следующего шага.

Эта функция полностью работает в CLI. В WebUI доступны сам хук и связка с интерфейсом, но приложения-хосты должны самостоятельно запускать генерацию предложений и подключать состояние рекомендаций, чтобы они отображались.

## Как это работает

После того как Qwen Code завершает ответ, через небольшую задержку (~300 мс) в области ввода появляется затемнённая подсказка. Например, после исправления ошибки вы можете увидеть:

```
> запустить тесты
```

Предложение генерируется путём отправки истории беседы модели, которая предсказывает, что вы естественным образом ввели бы дальше. Если ответ содержит явную подсказку (например, `Совет: введите "post comments" для публикации результатов`), предлагаемое действие извлекается автоматически.

## Принятие предложений

| Клавиша      | Действие                                              |
| ------------ | ----------------------------------------------------- |
| `Tab`        | Принять предложение и вставить его в поле ввода       |
| `Enter`      | Принять предложение и вставить его в поле ввода       |
| `Правая стрелка` | Принять предложение и вставить его в поле ввода   |
| Любой ввод   | Отклонить предложение и вводить как обычно            |

`Enter` заполняет поле ввода, но не отправляет его, поэтому принятие предложенной слэш-команды (например, `/clear`) никогда не выполняется автоматически — вы отправляете её сами вторым нажатием `Enter`.

## Когда появляются предложения

Предложения генерируются, когда выполнены все следующие условия:

- Модель завершила свой ответ (не во время потоковой передачи)
- В диалоге было не менее 2 обращений модели
- В последнем ответе нет ошибок
- Нет ожидающих диалогов подтверждения (например, подтверждение оболочки, разрешения)
- Режим подтверждения не установлен в `plan`
- Функция включена (по умолчанию включена — установите `ui.enableFollowupSuggestions` в `false`, чтобы отключить)

Предложения не будут появляться в неинтерактивном режиме (например, headless/SDK).

Предложения автоматически отклоняются, когда:

- Вы начинаете печатать
- Начинается новый ответ модели
- Предложение принято

## Быстрая модель

По умолчанию предложения используют ту же модель, что и ваш основной диалог. Для предложений с меньшей задержкой настройте отдельную быструю модель:

### Через команду

```
/model --fast qwen3-coder-flash
```

Или используйте `/model --fast` (без имени модели), чтобы открыть диалог выбора.

### Через settings.json

```json
{
  "fastModel": "qwen3-coder-flash"
}
```

Быстрая модель используется для предложений ввода и спекулятивного выполнения. Если она не настроена, используется основная модель диалога как запасной вариант.

> **Замечание о стоимости:** Быстрая модель снижает задержку, но не всегда снижает стоимость. Генерация предложений использует кэш префиксов вашего диалога (через `ui.enableCacheSharing`, по умолчанию включено) — но кэш префиксов привязан к модели. При указании `fastModel` на другую модель она переключается на отдельный кэш, и вся история диалога пересчитывается как некэшированный ввод на быстрой модели. На длинных диалогах настройка по умолчанию (основная модель + общий кэш) может быть **дешевле**, чем быстрая модель, поскольку большая часть истории оплачивается по сниженной ставке кэширования. Устанавливайте `fastModel`, когда задержка важнее стоимости за обращение.

Режим размышлений/рассуждений автоматически отключается для всех фоновых задач (генерация предложений и спекуляция), независимо от настройки размышлений вашей основной модели. Это позволяет не тратить токены на внутренние рассуждения, которые не нужны для этих задач.

## Конфигурация

Эти настройки можно задать в `settings.json`:

| Параметр                      | Тип     | По умолчанию | Описание                                                                 |
| ----------------------------- | ------- | ------------ | ------------------------------------------------------------------------ |
| `ui.enableFollowupSuggestions` | boolean | `true`       | Включить или отключить предложения продолжения                           |
| `ui.enableCacheSharing`        | boolean | `true`       | Использовать кэшированные ответвлённые запросы для снижения стоимости (экспериментально) |
| `ui.enableSpeculation`         | boolean | `false`      | Спекулятивно выполнять предложения до отправки (экспериментально)        |
| `fastModel`                    | string  | `""`         | Модель для предложений ввода и спекулятивного выполнения                 |

### Пример

```json
{
  "fastModel": "qwen3-coder-flash",
  "ui": {
    "enableFollowupSuggestions": true,
    "enableCacheSharing": true
  }
}
```

## Мониторинг

Использование модели для предложений отображается в выводе `/stats`, показывая токены, потреблённые быстрой моделью для генерации предложений.

Быстрая модель также отображается в выводе `/about` в строке "Fast Model".

## Качество предложений

Предложения проходят фильтры качества, чтобы гарантировать их полезность:

- Длина: 2-12 слов (для CJK: 2-30 символов), не более 100 символов всего
- Не может быть оценочным ("выглядит хорошо", "спасибо")
- Не может использовать голос ИИ ("Позвольте мне...", "Я...")
- Не может состоять из нескольких предложений или содержать форматирование (markdown, переводы строк)
- Не может быть мета-комментарием ("нечего предложить", "тишина")
- Не может быть сообщениями об ошибках или помеченными префиксами ("Предложение: ...")
- Однословные предложения допускаются только для распространённых команд (yes, commit, push и т.д.)
- Слэш-команды (например, `/commit`) всегда допускаются как однословные предложения
