# Интернационализация (i18n) и язык

Qwen Code создан для многоязычных рабочих процессов: он поддерживает локализацию интерфейса (i18n/l10n) в CLI, позволяет выбирать язык вывода ассистента и даёт возможность создавать собственные языковые пакеты для интерфейса.

## Обзор

С точки зрения пользователя, «интернационализация» Qwen Code охватывает несколько уровней:

| Возможность / Настройка | Что управляет                                                        | Где хранится                     |
|------------------------|----------------------------------------------------------------------|----------------------------------|
| `/language ui`         | Текст интерфейса терминала (меню, системные сообщения, приглашения) | `~/.qwen/settings.json`          |
| `/language output`     | Язык, на котором отвечает ИИ (предпочтение вывода, не перевод UI)   | `~/.qwen/output-language.md`     |
| Пользовательские языковые пакеты UI | Переопределение/расширение встроенных переводов интерфейса       | `~/.qwen/locales/*.js`           |

## Язык интерфейса

Это слой локализации интерфейса CLI (i18n/l10n): он определяет язык меню, подсказок и системных сообщений.

### Установка языка интерфейса

Используйте команду `/language ui`:

```bash
/language ui zh-CN    # Китайский
/language ui en-US    # Английский
/language ui ru-RU    # Русский
/language ui de-DE    # Немецкий
/language ui ja-JP    # Японский
/language ui pt-BR    # Португальский (Бразилия)
/language ui fr-FR    # Французский
/language ui ca-ES    # Каталанский
```

Также поддерживаются псевдонимы:

```bash
/language ui zh       # Китайский
/language ui en       # Английский
/language ui ru       # Русский
/language ui de       # Немецкий
/language ui ja       # Японский
/language ui pt       # Португальский
/language ui fr       # Французский
/language ui ca       # Каталанский
```

### Автоопределение

При первом запуске Qwen Code определяет системную локаль и автоматически устанавливает язык интерфейса.

Приоритет определения:

1. Переменная окружения `QWEN_CODE_LANG`
2. Переменная окружения `LANG`
3. Системная локаль через JavaScript Intl API
4. По умолчанию: английский

## Язык вывода LLM

Язык вывода LLM определяет, на каком языке ИИ-ассистент отвечает, независимо от языка, на котором вы задаёте вопросы.

### Как это работает

Язык вывода LLM задаётся файлом правил `~/.qwen/output-language.md`. Этот файл автоматически включается в контекст LLM при запуске, предписывая ассистенту отвечать на указанном языке.

### Автоопределение

При первом запуске, если файл `output-language.md` не существует, Qwen Code автоматически создаёт его на основе системной локали. Например:

- Системная локаль `zh` — создаётся правило для ответов на китайском
- Системная локаль `en` — создаётся правило для ответов на английском
- Системная локаль `ru` — создаётся правило для ответов на русском
- Системная локаль `de` — создаётся правило для ответов на немецком
- Системная локаль `ja` — создаётся правило для ответов на японском
- Системная локаль `pt` — создаётся правило для ответов на португальском
- Системная локаль `fr` — создаётся правило для ответов на французском
- Системная локаль `ca` — создаётся правило для ответов на каталанском

### Ручная установка

Используйте `/language output <язык>` для изменения:

```bash
/language output Chinese
/language output English
/language output Japanese
/language output German
```

Подходит любое название языка. LLM получит инструкцию отвечать на этом языке.

> [!note]
>
> После изменения языка вывода перезапустите Qwen Code, чтобы изменение вступило в силу.

### Расположение файла

```
~/.qwen/output-language.md
```

## Конфигурация

### Через диалог настроек

1. Выполните `/settings`
2. Найдите «Language» в разделе «General»
3. Выберите предпочитаемый язык интерфейса

### Через переменную окружения

```bash
export QWEN_CODE_LANG=zh
```

Это влияет на автоопределение при первом запуске (если вы ещё не установили язык интерфейса и файл `output-language.md` ещё не существует).

## Пользовательские языковые пакеты

Для переводов интерфейса вы можете создавать собственные языковые пакеты в `~/.qwen/locales/`:

- Пример: `~/.qwen/locales/es.js` для испанского
- Пример: `~/.qwen/locales/fr.js` для французского

Пользовательская директория имеет приоритет над встроенными переводами.

> [!tip]
>
> Приветствуются любые доработки! Если вы хотите улучшить встроенные переводы или добавить новые языки.
> Пример — [PR #1238: feat(i18n): add Russian language support](https://github.com/QwenLM/qwen-code/pull/1238).

### Поддержка `zh-TW` (традиционный китайский для Тайваня)

`zh-TW` **не является** автоматическим преобразованием s2t OpenCC из `zh.js` — это вручную поддерживаемый перевод с тайваньской лексикой. При добавлении или изменении ключей следуйте приведённым ниже соглашениям.

Столбец «Проверяется CI?» указывает, приведёт ли нарушение к ошибке сборки при выполнении `npm run check-i18n`. Строки с пометкой **Нет** соблюдаются только при ревью — обычно потому, что проблемная форма имеет допустимое неинтерфейсное значение (например, `文件` может означать «документ», `打開` разговорно допустимо на Тайване).

| Избегайте | Используйте вместо | Проверяется CI? | Причина |
|-----------|-------------------|-----------------|---------|
| 文件 (file) | 檔案 | Нет | Тайваньский термин для файлов файловой системы (но `文件` может означать «документ») |
| 服務器 / 服务器 | 伺服器 | Да | Тайваньский термин для «сервер» |
| 菜單 / 菜单 | 選單 | Да | Тайваньский термин для «меню» |
| 鏈接 / 链接 | 連結 | Да | Тайваньский термин для «ссылка» (голый `鏈` допустим, например, 區塊鏈) |
| 打開 | 開啟 | Нет | Предпочтительный на Тайване глагол для «открыть» (интерфейс); `打開` разговорно распространён |
| 爲 / 啓 / 曆史 / 鏈接 | 為 / 啟 / 歷史 / 連結 | Да | Варианты традиционных форм из прямого s2t OpenCC. Примечание: `曆` контекстно-зависим и верен в календарных терминах (日曆, 農曆, 西曆); CI проверяет только биграмму `曆史`, не единичное `曆`. |
Если вы не являетесь носителем традиционного китайского языка и вам нужно создать перевод, **не вставляйте сырой вывод OpenCC `s2t`**: профиль `s2t` по умолчанию выдаёт варианты традиционных иероглифов (например, 爲, 啓), которые не используются на Тайване, и никогда не заменяет лексику материкового Китая (服務器, 菜單). Предпочтительнее использовать `s2twp.json` (упрощённый → тайваньский с заменой фраз) в качестве отправной точки, а затем попросить носителя тайваньского китайского проверить.

Скрипт `check-i18n` (запускается в CI через `npm run check-i18n`) завершит сборку ошибкой, если любой из перечисленных ниже подстрок, проверяемых в CI, окажется в значении для `zh-TW`. Полный список см. в `scripts/check-i18n.ts → ZH_TW_FORBIDDEN_PATTERNS`. Если перевод легитимно должен содержать запрещённую CI подстроку, добавьте его ключ в `ZH_TW_ALLOWED_EXCEPTIONS` в том же файле с кратким обоснованием.

> [!note]
>
> Проверка использует простое сопоставление подстрок, которое не понимает границы китайских слов. Поэтому шаблон биграммы может дать ложное срабатывание на границе составных слов — например, `區塊鏈接口` (= `區塊鏈` + `接口`) содержит подстроку `鏈接`, хотя ни одно из слов не является некорректным. Если вы столкнулись с неожиданным отказом CI такого рода, добавьте ключ перевода в `ZH_TW_ALLOWED_EXCEPTIONS`, а не удаляйте шаблон.

### Формат языкового пакета

```javascript
// ~/.qwen/locales/es.js
export default {
  Hello: 'Hola',
  Settings: 'Configuracion',
  // ... more translations
};
```

## Связанные команды

- `/language` — Показать текущие языковые настройки
- `/language ui [lang]` — Установить язык интерфейса
- `/language output <language>` — Установить язык вывода LLM
- `/settings` — Открыть диалог настроек
