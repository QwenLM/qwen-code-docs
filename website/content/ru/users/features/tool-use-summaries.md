# Сводки об использовании инструментов

Qwen Code может генерировать короткую метку в стиле темы коммита git после завершения каждого пакета вызовов инструментов, обобщая, что было сделано в этом пакете. Метка отображается непосредственно в транскрипте и заменяет стандартный заголовок `Tool × N` в компактном режиме.

Это вспомогательный элемент для параллельных вызовов инструментов: когда модель выполняет несколько вызовов `Read` + `Grep` + `Bash` одновременно, сводка сразу показывает намерение, не заставляя просматривать весь список инструментов.

Функция включена по умолчанию и работает в фоновом режиме. Для её работы требуется настроенная [быстрая модель](./followup-suggestions#fast-model).

## Что вы видите

### Полный режим (по умолчанию)

Сводка отображается в виде приглушённого бейджа сразу под группой инструментов:

```
╭──────────────────────────────────────────────╮
│ ✓  ReadFile a.txt                            │
│ ✓  ReadFile b.txt                            │
│ ✓  ReadFile c.txt                            │
│ ✓  ReadFile d.txt                            │
╰──────────────────────────────────────────────╯

 ● Прочитано 4 текстовых файла
```

### Компактный режим (`Ctrl+O` или `ui.compactMode: true`)

Метка заменяет стандартный заголовок `Tool × N` в компактной однострочной форме:

```
╭──────────────────────────────────────────────╮
│✓  Чтение txt-файлов  · 4 инструмента         │
│Нажмите Ctrl+O для полного вывода инструментов │
╰──────────────────────────────────────────────╯
```

Отдельные вызовы инструментов по-прежнему доступны одним нажатием (`Ctrl+O` для переключения в полный режим).

## Как это работает

После завершения пакета вызовов инструментов Qwen Code отправляет асинхронный запрос к настроенной быстрой модели, содержащий:

- Имена инструментов, усечённые аргументы и усечённые результаты (каждый не более 300 символов).
- Последний текстовый вывод ассистента (первые 200 символов) как указание на намерение.
- Системный промпт, предписывающий модели вернуть метку в прошедшем времени, длиной до 30 символов, в стиле темы коммита git.

Запрос выполняется параллельно с потоковой передачей следующего шага диалога, так что задержка около 1 с скрыта за ответом основной модели. Когда метка получена, она добавляется в транскрипт как запись `tool_use_summary`.

Примеры меток: `Поиск в auth/`, `Исправлен NPE в UserService`, `Создан эндпоинт signup`, `Прочитан config.json`, `Запущены падающие тесты`.

## Когда она появляется

Сводка генерируется, если **все** следующие условия истинны:

- `experimental.emitToolUseSummaries` равно `true` (по умолчанию).
- Настроена `fastModel` (через настройки или `/model --fast`).
- В пакете завершён хотя бы один вызов инструмента.
- Шаг не был прерван до завершения вызовов инструментов.
- Быстрая модель вернула непустой ответ без ошибок.

Вызовы инструментов саб-агента не вызывают генерацию сводки — только пакеты инструментов основной сессии.

## Когда она не появляется

Сводка молча пропускается (без ошибок и изменений в интерфейсе), если:

- Не настроена быстрая модель.
- Запрос к быстрой модели завершился ошибкой, таймаутом или вернул пустой ответ.
- Модель вернула очевидно ошибочную строку (например, `Error: ...`, `I cannot ...`) — такие строки отфильтровываются клиентом, чтобы интерфейс не показывал вводящие в заблуждение метки.
- Шаг был прерван (`Ctrl+C`) до завершения модели.

Во всех этих случаях группа инструментов отображается как обычно.

## Быстрая модель

Метка генерируется с помощью [быстрой модели](./followup-suggestions#fast-model) — той же модели, которую вы настраиваете для подсказок и спекулятивного выполнения. Настройка:

### Через команду

```
/model --fast qwen3-coder-flash
```

### Через `settings.json`

```json
{
  "fastModel": "qwen3-coder-flash"
}
```

Если быстрая модель не настроена, генерация сводки полностью пропускается — функция не действует, пока вы её не настроите.

## Конфигурация

Эти параметры можно задать в `settings.json`:

| Параметр                            | Тип    | По умолчанию | Описание                                                                                        |
| ----------------------------------- | ------ | ------------ | ------------------------------------------------------------------------------------------------ |
| `experimental.emitToolUseSummaries` | boolean| `true`       | Главный включатель генерации сводок. Выключите, чтобы отключить дополнительный запрос к быстрой модели.               |
| `fastModel`                         | string | `""`         | Быстрая модель, используемая для генерации сводок (общая с подсказками). Обязательна; при пустой строке не действует. |

### Переопределение через переменную окружения

`QWEN_CODE_EMIT_TOOL_USE_SUMMARIES` переопределяет настройку `experimental.emitToolUseSummaries` для текущей сессии:

- `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=0` или `=false` — принудительно выключить.
- `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=1` или `=true` — принудительно включить.
- Не задано — используется настройка `experimental.emitToolUseSummaries`.

### Пример

```json
{
  "fastModel": "qwen3-coder-flash",
  "experimental": {
    "emitToolUseSummaries": true
  }
}
```

## Область действия и жизненный цикл

Три момента, которые часто вызывают вопросы при первом знакомстве с этой функцией:

1. **Одна генерация на пакет, общая для обоих режимов отображения.** Запрос к быстрой модели происходит ровно один раз в `handleCompletedTools`, когда завершается пакет инструментов. Последующее переключение `Ctrl+O` **не** вызывает новый запрос — оба режима читают одну и ту же запись `tool_use_summary` в истории, которая была сохранена при первой генерации. Вы можете свободно переключать компактный режим без дополнительных затрат.
2. **Нет обратного заполнения при переключении или возобновлении сессии.** Группа инструментов (`tool_group`), завершённая до включения функции (или до изменения настройки, или в возобновлённой сессии — `ChatRecordingService` не сохраняет записи сводок), никогда не получит метку. Нет прохода по существующей истории. Если вы включите эту настройку в середине сессии, метки появятся только у **будущих** пакетов; старые группы останутся в стандартном отображении без указания на отсутствие метки.
3. **Только пакеты основного агента.** Триггер находится в цикле шагов основной сессии (`useGeminiStream`), поэтому:
   - ✅ Сводятся вызовы Shell, MCP, файловых операций, а также сам вызов инструмента `Task` / саб-агента (как он фигурирует в основном пакете).
   - ❌ Внутренние пакеты инструментов саб-агента (выполняемые через `packages/core/src/agents/runtime/`) не сводятся.
Внешний пакет, который _содержит_ инструмент `Task`, по-прежнему будет помечен, но быстрая модель видит только вызов сабагентского инструмента и его агрегированный результат, а не отдельные вызовы инструментов внутри сабагента. Ожидайте такие метки, как `Ran research-agent` или `Delegated file search`, а не `Searched 14 files`. Это сделано намеренно — обобщение внутренностей сабагента привело бы к многократному увеличению стоимости быстрой модели и появлению шума, который никогда не отображается в основном интерфейсе.

## Рекомендуемое сочетание: включение компактного режима

Для пакетов из 3+ параллельных вызовов инструментов включение этой функции вместе с `ui.compactMode: true` даёт самую чистую запись диалога. Компактный вид сворачивает весь пакет в одну помеченную строку (`✓  Read txt files  · 4 tools`) вместо отображения всех строк инструментов и последующей сводки. Подробности остаются на расстоянии одного нажатия `Ctrl+O`.

```json
{
  "fastModel": "qwen3-coder-flash",
  "ui": {
    "compactMode": true
  },
  "experimental": {
    "emitToolUseSummaries": true
  }
}
```

В полном режиме (по умолчанию) сводка отображается как завершающая строка `● <label>` под группой инструментов — это удобно для больших или разнородных пакетов, но для небольших однотипных пакетов (например, `Read × 3`) метка может восприниматься как повторение уже видимых строк инструментов. Если это соответствует вашему обычному рабочему процессу, либо включите компактный режим как указано выше, либо полностью отключите сводку через `experimental.emitToolUseSummaries: false`.

## Мониторинг

Использование модели для сводки отображается в выводе `/stats` в общих токенах быстрой модели с `prompt_id` `tool_use_summary_generation`, чтобы её можно было отличить от подсказок и других фоновых задач.

## Поток данных и конфиденциальность

Вызов сводки отправляет имя каждого успешного инструмента, его усечённые `args` и усечённый результат (каждое поле ограничено 300 символами) в **быструю модель**, а также первые 200 символов последнего текста ассистента в качестве префикса намерения.

Если ваша быстрая модель настроена с тем же провайдером/авторизацией, что и основная модель сессии, данные передаются по тем же границам, которые уже использует основная сессия — доверие не изменяется. Если вы настроили быструю модель от **другого провайдера**, входные и выходные данные инструментов (включая содержимое файлов, прочитанное `read_file`, вывод команд из вызовов shell или значения, полученные через инструменты MCP) будут отправлены этому другому провайдеру в составе запроса на сводку. Это значительно более широкий объём обмена данными, чем в основной сессии.

Если это важно для вашего рабочего процесса, у вас есть два чистых варианта:

- Настройте `fastModel` на модель под тем же провайдером, что и основная сессия, чтобы вызов сводки не пересекал новые границы аутентификации/данных.
- Отключите функцию полностью с помощью `experimental.emitToolUseSummaries: false` (или `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=0`).

Ограничение в 300 символов на поле снижает объём передаваемых данных, но не устраняет его полностью — секреты, обнаруженные в выводе инструмента в пределах этого ограничения, всё ещё могут быть отправлены. Относитесь к границе данных быстрой модели так же, как к границе данных основной модели.

## Стоимость

Один вызов быстрой модели на каждый подходящий пакет инструментов. Входные данные — небольшой фиксированный системный промпт плюс усечённые входные/выходные данные инструментов (каждое поле ограничено 300 символами). Выходные данные — одна короткая строка (ограничена 100 символами, обычно 20 токенов или меньше). На типичной быстрой модели это примерно $0,001 за пакет.

Если вы не хотите дополнительных затрат, отключите функцию через `experimental.emitToolUseSummaries: false` или `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=0`.
