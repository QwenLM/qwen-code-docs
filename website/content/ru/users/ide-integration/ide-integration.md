# Интеграция с IDE

Qwen Code может интегрироваться с вашей IDE, чтобы обеспечить более удобную работу с учетом контекста. Эта интеграция позволяет CLI лучше понимать ваше рабочее пространство и предоставляет такие мощные функции, как нативный просмотр изменений (diff) прямо в редакторе.

В настоящее время единственной поддерживаемой IDE является [Visual Studio Code](https://code.visualstudio.com/) и другие редакторы, поддерживающие расширения VS Code. Информацию о создании поддержки для других редакторов см. в [IDE Companion Extension Spec](../ide-integration/ide-companion-spec).

## Возможности

- **Контекст рабочего пространства:** CLI автоматически получает информацию о вашем рабочем пространстве для предоставления более релевантных и точных ответов. Этот контекст включает:
  - **10 последних использованных файлов** в вашем рабочем пространстве.
  - Позицию активного курсора.
  - Любой выделенный вами текст (до 16 КБ; более длинные выделения будут обрезаны).

- **Нативный просмотр изменений:** Когда Qwen предлагает изменения в коде, вы можете просматривать их прямо в нативном просмотрщике различий вашей IDE. Это позволяет удобно просматривать, редактировать, принимать или отклонять предложенные изменения.

- **Команды VS Code:** Вы можете получить доступ к функциям Qwen Code напрямую из палитры команд VS Code (`Cmd+Shift+P` или `Ctrl+Shift+P`):
  - `Qwen Code: Run`: Запускает новый сеанс Qwen Code во встроенном терминале.
  - `Qwen Code: Accept Diff`: Принимает изменения в активном редакторе различий.
  - `Qwen Code: Close Diff Editor`: Отклоняет изменения и закрывает активный редактор различий.
  - `Qwen Code: View Third-Party Notices`: Отображает уведомления о сторонних компонентах для расширения.

## Установка и настройка

Существует три способа настройки интеграции с IDE:

### 1. Автоматическое предложение (Рекомендуется)

При запуске Qwen Code в поддерживаемом редакторе он автоматически определит ваше окружение и предложит подключиться. Ответ «Да» автоматически выполнит необходимую настройку, включая установку companion-расширения и активацию соединения.

### 2. Ручная установка из CLI

Если вы ранее отклонили предложение или хотите установить расширение вручную, вы можете выполнить следующую команду внутри Qwen Code:

```
/ide install
```

Команда найдет нужное расширение для вашей IDE и установит его.

### 3. Ручная установка из маркетплейса

Вы также можете установить расширение напрямую из маркетплейса.

- **Для Visual Studio Code:** Установите из [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=qwenlm.qwen-code-vscode-ide-companion).
- **Для форков VS Code:** Для поддержки форков VS Code расширение также опубликовано в [Open VSX Registry](https://open-vsx.org/extension/qwenlm/qwen-code-vscode-ide-companion). Следуйте инструкциям вашего редактора по установке расширений из этого реестра.

> NOTE:
> Расширение «Qwen Code Companion» может отображаться в конце результатов поиска. Если вы не видите его сразу, попробуйте прокрутить вниз или отсортировать по «Newly Published».
>
> После ручной установки расширения необходимо выполнить команду `/ide enable` в CLI для активации интеграции.

## Использование

### Включение и отключение

Вы можете управлять интеграцией с IDE из CLI:

- Чтобы включить соединение с IDE, выполните:
  ```
  /ide enable
  ```
- Чтобы отключить соединение, выполните:
  ```
  /ide disable
  ```

При включении Qwen Code будет автоматически пытаться подключиться к companion-расширению IDE.

### Проверка статуса

Чтобы проверить статус соединения и просмотреть контекст, полученный CLI от IDE, выполните:

```
/ide status
```

При успешном подключении эта команда покажет IDE, к которой установлено соединение, и список недавно открытых файлов, о которых ей известно.

(Примечание: список файлов ограничен 10 последними использованными файлами в вашем рабочем пространстве и включает только локальные файлы на диске.)

### Работа с изменениями (diff)

Когда вы просите модель Qwen изменить файл, она может открыть просмотр различий прямо в вашем редакторе.

**Чтобы принять изменения**, вы можете выполнить любое из следующих действий:

- Нажмите на **значок галочки** в заголовке редактора различий.
- Сохраните файл (например, с помощью `Cmd+S` или `Ctrl+S`).
- Откройте палитру команд и выполните **Qwen Code: Accept Diff**.
- Ответьте `yes` в CLI при появлении запроса.

**Чтобы отклонить изменения**, вы можете:

- Нажмите на **значок «x»** в заголовке редактора различий.
- Закройте вкладку редактора различий.
- Откройте палитру команд и выполните **Qwen Code: Close Diff Editor**.
- Ответьте `no` в CLI при появлении запроса.

Вы также можете **изменить предложенные правки** прямо в окне просмотра различий перед их принятием.

Если вы выберете «Yes, allow always» в CLI, изменения больше не будут отображаться в IDE, так как они будут приниматься автоматически.

## Использование в песочнице

Если вы используете Qwen Code в песочнице, обратите внимание на следующее:

- **В macOS:** Интеграция с IDE требует доступа к сети для связи с companion-расширением IDE. Вы должны использовать профиль Seatbelt, разрешающий доступ к сети.
- **В контейнере Docker:** Если вы запускаете Qwen Code внутри контейнера Docker (или Podman), интеграция с IDE все равно может подключиться к расширению VS Code, запущенному на вашей хост-машине. CLI настроен на автоматический поиск IDE-сервера по адресу `host.docker.internal`. Обычно специальная настройка не требуется, но вам может потребоваться убедиться, что ваша сетевая конфигурация Docker разрешает подключения из контейнера к хосту.

## Устранение неполадок

Если вы столкнулись с проблемами при интеграции с IDE, ниже приведены некоторые распространенные сообщения об ошибках и способы их устранения.

### Ошибки подключения

- **Сообщение:** `● Disconnected: Failed to connect to IDE companion extension for [IDE Name]. Please ensure the extension is running and try restarting your terminal. To install the extension, run /ide install.`
  - **Причина:** Qwen Code не смог найти необходимые переменные окружения (`QWEN_CODE_IDE_WORKSPACE_PATH` или `QWEN_CODE_IDE_SERVER_PORT`) для подключения к IDE. Обычно это означает, что companion-расширение IDE не запущено или не было корректно инициализировано.
  - **Решение:**
    1.  Убедитесь, что вы установили расширение **Qwen Code Companion** в вашу IDE и что оно включено.
    2.  Откройте новое окно терминала в вашей IDE, чтобы убедиться, что оно подхватывает правильное окружение.

- **Сообщение:** `● Disconnected: IDE connection error. The connection was lost unexpectedly. Please try reconnecting by running /ide enable`
  - **Причина:** Соединение с companion-расширением IDE было потеряно.
  - **Решение:** Выполните `/ide enable`, чтобы попытаться переподключиться. Если проблема сохраняется, откройте новое окно терминала или перезапустите IDE.

### Ошибки конфигурации

- **Сообщение:** `● Disconnected: Directory mismatch. Qwen Code is running in a different location than the open workspace in [IDE Name]. Please run the CLI from the same directory as your project's root folder.`
  - **Причина:** Текущий рабочий каталог CLI находится за пределами папки или рабочего пространства, открытого в вашей IDE.
  - **Решение:** Выполните `cd` в тот же каталог, который открыт в вашей IDE, и перезапустите CLI.

- **Сообщение:** `● Disconnected: To use this feature, please open a workspace folder in [IDE Name] and try again.`
  - **Причина:** В вашей IDE не открыто рабочее пространство.
  - **Решение:** Откройте рабочее пространство в вашей IDE и перезапустите CLI.

### Общие ошибки

- **Сообщение:** `IDE integration is not supported in your current environment. To use this feature, run Qwen Code in one of these supported IDEs: [List of IDEs]`
  - **Причина:** Вы запускаете Qwen Code в терминале или окружении, которое не является поддерживаемой IDE.
  - **Решение:** Запустите Qwen Code из встроенного терминала поддерживаемой IDE, например VS Code.

- **Сообщение:** `No installer is available for IDE. Please install the Qwen Code Companion extension manually from the marketplace.`
  - **Причина:** Вы выполнили `/ide install`, но в CLI нет автоматического установщика для вашей конкретной IDE.
  - **Решение:** Откройте маркетплейс расширений вашей IDE, найдите «Qwen Code Companion» и установите его вручную.
