# Интеграция с IDE

Qwen Code может интегрироваться с вашей IDE для обеспечения более бесшовного и учитывающего контекст опыта. Эта интеграция позволяет CLI лучше понимать ваше рабочее пространство и включает мощные функции, такие как встроенный в редактор режим сравнения различий (diff).

В настоящее время поддерживается только [Visual Studio Code](https://code.visualstudio.com/) и другие редакторы, совместимые с расширениями VS Code. Чтобы добавить поддержку других редакторов, ознакомьтесь со [спецификацией расширения-компаньона для IDE](../ide-integration/ide-companion-spec).

## Возможности

- **Контекст рабочей области:** CLI автоматически получает информацию о вашей рабочей области, чтобы предоставлять более релевантные и точные ответы. Этот контекст включает:
  - **10 последних файлов**, к которым вы обращались в рабочей области.
  - Текущую позицию курсора.
  - Любой выделенный вами текст (до лимита в 16 КБ; при превышении лимита выделение будет обрезано).

- **Встроенное сравнение различий (diff):** Когда Qwen предлагает изменения в коде, вы можете просматривать их непосредственно в родном средстве сравнения различий вашей IDE. Это позволяет вам удобно просматривать, редактировать и принимать или отклонять предложенные изменения.

- **Команды VS Code:** Вы можете получить доступ к возможностям Qwen Code напрямую через палитру команд VS Code (`Cmd+Shift+P` или `Ctrl+Shift+P`):
  - `Qwen Code: Запустить`: запускает новую сессию Qwen Code в интегрированном терминале;
  - `Qwen Code: Принять различия`: принимает изменения в активном редакторе различий;
  - `Qwen Code: Закрыть редактор различий`: отклоняет изменения и закрывает активный редактор различий;
  - `Qwen Code: Просмотреть уведомления о сторонних компонентах`: отображает уведомления о сторонних компонентах для этого расширения.

## Установка и настройка

Существует три способа настроить интеграцию с IDE:

### 1. Автоматическое напоминание (рекомендуется)

При запуске Qwen Code внутри поддерживаемого редактора он автоматически определит вашу среду и предложит установить соединение. Ответ «Да» приведёт к автоматическому выполнению необходимых действий по настройке, включая установку вспомогательного расширения и включение соединения.

### 2. Ручная установка из CLI

Если вы ранее отклонили запрос или хотите установить расширение вручную, выполните следующую команду внутри Qwen Code:

```
/ide install
```

Эта команда найдёт подходящее расширение для вашей IDE и установит его.

### 3. Ручная установка из маркетплейса

Расширение также можно установить напрямую из маркетплейса.

- **Для Visual Studio Code:** Установите его из [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=qwenlm.qwen-code-vscode-ide-companion).
- **Для форков VS Code:** Чтобы поддержать форки VS Code, расширение также опубликовано в [Open VSX Registry](https://open-vsx.org/extension/qwenlm/qwen-code-vscode-ide-companion). Следуйте инструкциям вашего редактора по установке расширений из этого реестра.

> ПРИМЕЧАНИЕ:
> Расширение «Qwen Code Companion» может отображаться ближе к концу результатов поиска. Если вы не видите его сразу, попробуйте прокрутить список вниз или отсортировать результаты по параметру «Недавно опубликовано».
>
> После ручной установки расширения необходимо выполнить команду `/ide enable` в CLI для активации интеграции.

## Использование

### Включение и отключение

Управление интеграцией с IDE можно осуществлять из командной строки:

- Чтобы включить подключение к IDE, выполните:
  ```
  /ide enable
  ```
- Чтобы отключить подключение, выполните:
  ```
  /ide disable
  ```

После включения Qwen Code автоматически попытается подключиться к расширению-компаньону IDE.

### Проверка состояния

Чтобы проверить состояние подключения и просмотреть контекст, полученный CLI от IDE, выполните:

```
/ide status
```

Если подключение установлено, команда покажет, к какой IDE выполнено подключение, а также список недавно открытых файлов, известных CLI.

(Примечание: список файлов ограничен 10 последними открытыми файлами в рабочей области и включает только локальные файлы на диске.)

### Работа с различиями (diff)

Когда вы просите модель Qwen изменить файл, она может открыть представление различий (diff) непосредственно в вашем редакторе.

**Чтобы принять различия**, выполните одно из следующих действий:

- Нажмите **значок галочки** на панели заголовка редактора diff.
- Сохраните файл (например, с помощью комбинации клавиш `Cmd+S` или `Ctrl+S`).
- Откройте палитру команд и выполните команду **Qwen Code: Accept Diff**.
- В ответ на запрос в CLI введите `yes`.

**Чтобы отклонить различия**, выполните одно из следующих действий:

- Нажмите **значок «x»** на панели заголовка редактора diff.
- Закройте вкладку редактора diff.
- Откройте палитру команд и выполните команду **Qwen Code: Close Diff Editor**.
- В ответ на запрос в CLI введите `no`.

Вы также можете **непосредственно изменять предложенные изменения** в представлении diff перед их принятием.

Если в CLI вы выберете вариант «Да, разрешить всегда», изменения больше не будут отображаться в IDE, поскольку они будут автоматически приниматься.

## Использование в песочнице

Если вы используете Qwen Code в песочнице, обратите внимание на следующее:

- **В macOS:** Интеграция с IDE требует сетевого доступа для взаимодействия с расширением-компаньоном IDE. Необходимо использовать профиль Seatbelt, разрешающий сетевой доступ.
- **В контейнере Docker:** Если вы запускаете Qwen Code внутри контейнера Docker (или Podman), интеграция с IDE по-прежнему может подключаться к расширению VS Code, работающему на вашем хост-компьютере. CLI настроен на автоматический поиск сервера IDE по адресу `host.docker.internal`. Как правило, специальная конфигурация не требуется, однако убедитесь, что ваша сетевая конфигурация Docker позволяет установить соединение из контейнера с хост-машиной.

## Устранение неполадок

Если возникают проблемы с интеграцией IDE, ниже приведены распространённые сообщения об ошибках и способы их устранения.

### Ошибки подключения

- **Сообщение:** `🔴 Отключено: не удалось подключиться к расширению-компаньону IDE для [названия IDE]. Убедитесь, что расширение запущено, и попробуйте перезапустить терминал. Чтобы установить расширение, выполните команду /ide install.`
  - **Причина:** Qwen Code не смог найти необходимые переменные окружения (`QWEN_CODE_IDE_WORKSPACE_PATH` или `QWEN_CODE_IDE_SERVER_PORT`) для подключения к IDE. Обычно это означает, что расширение-компаньон IDE не запущено или не инициализировалось корректно.
  - **Решение:**
    1.  Убедитесь, что вы установили расширение **Qwen Code Companion** в вашей IDE и оно включено.
    2.  Откройте новое окно терминала в вашей IDE, чтобы гарантировать, что оно получит правильные переменные окружения.

- **Сообщение:** `🔴 Отключено: ошибка подключения к IDE. Соединение было неожиданно потеряно. Попробуйте повторно подключиться, выполнив команду /ide enable`
  - **Причина:** Подключение к расширению-компаньону IDE было потеряно.
  - **Решение:** Выполните команду `/ide enable`, чтобы попытаться восстановить подключение. Если проблема сохраняется, откройте новое окно терминала или перезапустите IDE.

### Ошибки конфигурации

- **Сообщение:** `🔴 Отключено: несоответствие каталогов. Qwen Code запущен в другом месте, чем открытая рабочая область в [IDE Name]. Запустите CLI из того же каталога, что и корневой каталог вашего проекта.`
  - **Причина:** Текущий рабочий каталог CLI находится вне папки или рабочей области, открытой в вашей IDE.
  - **Решение:** Выполните `cd` в тот же каталог, который открыт в вашей IDE, и перезапустите CLI.

- **Сообщение:** `🔴 Отключено: чтобы использовать эту функцию, откройте папку рабочей области в [IDE Name] и повторите попытку.`
  - **Причина:** В вашей IDE не открыто ни одной рабочей области.
  - **Решение:** Откройте рабочую область в вашей IDE и перезапустите CLI.

### Общие ошибки

- **Сообщение:** `Интеграция с IDE не поддерживается в вашей текущей среде. Чтобы использовать эту функцию, запустите Qwen Code в одном из поддерживаемых IDE: [список IDE]`  
  - **Причина:** Вы запускаете Qwen Code в терминале или среде, которая не является поддерживаемой IDE.  
  - **Решение:** Запустите Qwen Code из интегрированного терминала поддерживаемой IDE, например, VS Code.

- **Сообщение:** `Для вашей IDE недоступен установщик. Установите расширение Qwen Code Companion вручную из магазина расширений.`  
  - **Причина:** Вы выполнили команду `/ide install`, однако CLI не содержит автоматического установщика для вашей конкретной IDE.  
  - **Решение:** Откройте магазин расширений вашей IDE, выполните поиск по запросу «Qwen Code Companion» и установите его вручную.
