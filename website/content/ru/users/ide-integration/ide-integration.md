# Интеграция с IDE

Qwen Code может интегрироваться с вашей IDE для обеспечения более плавной работы с учётом контекста. Эта интеграция позволяет CLI лучше понимать ваше рабочее пространство и открывает доступ к мощным функциям, таким как встроенная проверка изменений (diff) прямо в редакторе.

В настоящее время поддерживается только [Visual Studio Code](https://code.visualstudio.com/) и другие редакторы, поддерживающие расширения VS Code. Чтобы добавить поддержку других редакторов, см. [Спецификацию расширения-компаньона для IDE](../ide-integration/ide-companion-spec).

## Возможности

- **Контекст рабочей области:** CLI автоматически получает информацию о вашей рабочей области, чтобы давать более релевантные и точные ответы. Этот контекст включает:
  - **10 последних открытых файлов** в вашей рабочей области.
  - Позицию активного курсора.
  - Любой выделенный текст (до 16 КБ; более длинные выделения будут обрезаны).

- **Нативная проверка изменений (diff):** Когда Qwen предлагает изменения кода, вы можете просмотреть их непосредственно во встроенном средстве просмотра изменений (diff viewer) вашей IDE. Это позволяет удобно просматривать, редактировать, принимать или отклонять предложенные изменения.

- **Команды VS Code:** Вы можете вызывать функции Qwen Code прямо из палитры команд VS Code (`Cmd+Shift+P` или `Ctrl+Shift+P`):
  - `Qwen Code: Run`: Запускает новый сеанс Qwen Code во встроенном терминале.
  - `Qwen Code: Accept Diff`: Принимает изменения в активном редакторе diff.
  - `Qwen Code: Close Diff Editor`: Отклоняет изменения и закрывает активный редактор diff.
  - `Qwen Code: View Third-Party Notices`: Отображает уведомления о сторонних компонентах расширения.

## Установка и настройка

Существует три способа настройки интеграции с IDE:

### 1. Автоматическое предложение (рекомендуется)

Когда вы запускаете Qwen Code в поддерживаемом редакторе, он автоматически обнаружит ваше окружение и предложит подключиться. Ответ «Да» автоматически выполнит необходимую настройку, включая установку расширения-компаньона и активацию подключения.

### 2. Ручная установка из CLI

Если вы ранее отклонили предложение или хотите установить расширение вручную, выполните следующую команду внутри Qwen Code:

```
/ide install
```

Это найдет подходящее расширение для вашей IDE и установит его.

### 3. Ручная установка из маркетплейса

Вы также можете установить расширение напрямую из маркетплейса.

- **Для Visual Studio Code:** Установите из [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=qwenlm.qwen-code-vscode-ide-companion).
- **Для форков VS Code:** Чтобы поддерживать форки VS Code, расширение также опубликовано в [Open VSX Registry](https://open-vsx.org/extension/qwenlm/qwen-code-vscode-ide-companion). Следуйте инструкциям вашего редактора по установке расширений из этого реестра.

> ПРИМЕЧАНИЕ:
> Расширение «Qwen Code Companion» может отображаться в нижней части результатов поиска. Если вы не видите его сразу, попробуйте прокрутить вниз или отсортировать по «Новым публикациям».
>
> После ручной установки расширения необходимо выполнить `/ide enable` в CLI, чтобы активировать интеграцию.

## Использование

### Включение и отключение

Вы можете управлять интеграцией с IDE непосредственно из CLI:

- Чтобы включить подключение к IDE, выполните:
  ```
  /ide enable
  ```
- Чтобы отключить подключение, выполните:
  ```
  /ide disable
  ```

Когда интеграция включена, Qwen Code автоматически попытается подключиться к расширению-компаньону IDE.

### Проверка статуса

Чтобы проверить статус подключения и увидеть контекст, полученный CLI от IDE, выполните:

```
/ide status
```

Если подключение установлено, команда покажет IDE, к которой выполнено подключение, и список недавно открытых файлов, о которых знает CLI.

(Примечание: список файлов ограничен 10 последними открытыми файлами в вашей рабочей области и включает только локальные файлы на диске.)

### Работа с изменениями (diff)

Когда вы просите модель Qwen изменить файл, она может открыть представление diff прямо в вашем редакторе.

**Чтобы принять изменения (diff)**, выполните одно из следующих действий:

- Нажмите на **значок галочки** в заголовке редактора diff.
- Сохраните файл (например, с помощью `Cmd+S` или `Ctrl+S`).
- Откройте палитру команд и выполните **Qwen Code: Accept Diff**.
- Ответьте `yes` в CLI при запросе.

**Чтобы отклонить изменения (diff)**, вы можете:

- Нажмите на **значок 'x'** в заголовке редактора diff.
- Закройте вкладку редактора diff.
- Откройте палитру команд и выполните **Qwen Code: Close Diff Editor**.
- Ответьте `no` в CLI при запросе.

Вы также можете **изменить предложенные правки** непосредственно в представлении diff перед их принятием.

Если вы выберете «Yes, allow always» («Да, разрешить всегда») в CLI, изменения больше не будут отображаться в IDE, так как будут приниматься автоматически.

## Использование с изолированной средой (sandboxing)

Если вы используете Qwen Code в изолированной среде, учтите следующее:

- **На macOS:** Для работы интеграции с IDE требуется сетевой доступ для взаимодействия с расширением-компаньоном IDE. Необходимо использовать профиль Seatbelt, разрешающий сетевой доступ.
- **В Docker-контейнере:** Если вы запускаете Qwen Code внутри контейнера Docker (или Podman), интеграция с IDE всё равно может подключаться к расширению VS Code на хост-машине. CLI автоматически настраивается на поиск сервера IDE на `host.docker.internal`. Обычно никакой специальной настройки не требуется, но вам может потребоваться убедиться, что настройки сети Docker разрешают подключения из контейнера к хосту.

## Устранение неполадок

Если у вас возникли проблемы с интеграцией IDE, вот несколько распространённых сообщений об ошибках и способы их решения.

### Ошибки подключения

- **Сообщение:** `🔴 Disconnected: Failed to connect to IDE companion extension for [IDE Name]. Please ensure the extension is running and try restarting your terminal. To install the extension, run /ide install.`
  - **Причина:** Qwen Code не смог найти переменные окружения (`QWEN_CODE_IDE_WORKSPACE_PATH` или `QWEN_CODE_IDE_SERVER_PORT`), необходимые для подключения к IDE. Обычно это означает, что расширение-компаньон IDE не запущено или не инициализировалось должным образом.
  - **Решение:**
    1.  Убедитесь, что расширение **Qwen Code Companion** установлено в вашей IDE и включено.
    2.  Откройте новое окно терминала в IDE, чтобы оно получило правильные переменные окружения.

- **Сообщение:** `🔴 Disconnected: IDE connection error. The connection was lost unexpectedly. Please try reconnecting by running /ide enable`
  - **Причина:** Соединение с расширением-компаньоном IDE было потеряно.
  - **Решение:** Выполните `/ide enable`, чтобы попробовать переподключиться. Если проблема не исчезнет, откройте новое окно терминала или перезапустите IDE.

### Ошибки конфигурации

- **Сообщение:** `🔴 Disconnected: Directory mismatch. Qwen Code is running in a different location than the open workspace in [IDE Name]. Please run the CLI from the same directory as your project's root folder.`
  - **Причина:** Текущая рабочая директория CLI находится за пределами папки или рабочей области, открытой в вашей IDE.
  - **Решение:** Выполните `cd` в ту же директорию, которая открыта в IDE, и перезапустите CLI.

- **Сообщение:** `🔴 Disconnected: To use this feature, please open a workspace folder in [IDE Name] and try again.`
  - **Причина:** В вашей IDE не открыта ни одна рабочая область.
  - **Решение:** Откройте рабочую область в IDE и перезапустите CLI.

### Общие ошибки

- **Сообщение:** `IDE integration is not supported in your current environment. To use this feature, run Qwen Code in one of these supported IDEs: [List of IDEs]`
  - **Причина:** Вы запускаете Qwen Code в терминале или окружении, которое не является поддерживаемой IDE.
  - **Решение:** Запустите Qwen Code из встроенного терминала поддерживаемой IDE, например VS Code.

- **Сообщение:** `No installer is available for IDE. Please install the Qwen Code Companion extension manually from the marketplac`
  - **Причина:** Вы выполнили `/ide install`, но у CLI нет автоматического установщика для вашей конкретной IDE.
  - **Решение:** Откройте маркетплейс расширений вашей IDE, найдите «Qwen Code Companion» и установите его вручную.
