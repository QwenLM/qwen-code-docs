# Плагин-компаньон Qwen Code: спецификация интерфейса

> Последнее обновление: 15 сентября 2025 г.

В этом документе определён контракт для разработки плагина-компаньона, который включает режим IDE в Qwen Code. В VS Code эти функции (встроенное сравнение версий, осознание контекста) предоставляются официальным расширением ([рынок расширений](https://marketplace.visualstudio.com/items?itemName=qwenlm.qwen-code-vscode-ide-companion)). Данная спецификация предназначена для участников, желающих реализовать аналогичный функционал в других редакторах, например, в IDE JetBrains, Sublime Text и др.

## I. Интерфейс взаимодействия

Qwen Code и плагин IDE взаимодействуют через локальный канал связи.

### 1. Транспортный уровень: MCP поверх HTTP

Плагин **ОБЯЗАН** запускать локальный HTTP-сервер, реализующий **Protocol Model Context (MCP)**.

- **Протокол:** Сервер должен быть корректным сервером MCP. Мы рекомендуем использовать существующий SDK для MCP на выбранном вами языке программирования, если он доступен.
- **Конечная точка:** Сервер должен предоставлять одну конечную точку (например, `/mcp`) для всей коммуникации по протоколу MCP.
- **Порт:** Сервер **ОБЯЗАН** прослушивать динамически назначенный порт (то есть прослушивать порт `0`).

### 2. Механизм обнаружения: файл блокировки

Чтобы Qwen Code мог подключиться, он должен определить порт, на котором работает ваш сервер. Для этого плагин **ОБЯЗАН** создать «файл блокировки» и установить переменную окружения с портом.

- **Как CLI находит файл:** CLI считывает порт из переменной `QWEN_CODE_IDE_SERVER_PORT`, а затем читает файл `~/.qwen/ide/<PORT>.lock`. (Для устаревших расширений предусмотрены резервные механизмы; см. примечание ниже.)
- **Расположение файла:** Файл должен быть создан в строго определённой директории: `~/.qwen/ide/`. Ваш плагин обязан создать эту директорию, если она ещё не существует.
- **Соглашение об именовании файлов:** Имя файла критически важно и **ОБЯЗАНО** соответствовать шаблону:  
  `<PORT>.lock`  
  - `<PORT>` — порт, на котором ожидает подключений ваш MCP-сервер.
- **Содержимое файла и проверка рабочей области:** Файл **ОБЯЗАН** содержать JSON-объект со следующей структурой:

  ```json
  {
    "port": 12345,
    "workspacePath": "/path/to/project1:/path/to/project2",
    "authToken": "a-very-secret-token",
    "ppid": 1234,
    "ideName": "VS Code"
  }
  ```
  - `port` (число, обязательно): порт MCP-сервера.
  - `workspacePath` (строка, обязательно): список всех открытых корневых путей рабочих областей, разделённых системным разделителем путей (`:` для Linux/macOS, `;` для Windows). CLI использует этот путь, чтобы убедиться, что он запущен в той же директории проекта, которая открыта в IDE. Если текущая рабочая директория CLI не является поддиректорией `workspacePath`, подключение будет отклонено. Ваш плагин **ОБЯЗАН** предоставить корректные абсолютные пути к корню открытой(ых) рабочей(их) области(ей).
  - `authToken` (строка, обязательно): секретный токен для защиты соединения. CLI будет включать этот токен в заголовок `Authorization: Bearer <token>` во всех запросах.
  - `ppid` (число, обязательно): идентификатор родительского процесса IDE.
  - `ideName` (строка, обязательно): понятное пользователю имя IDE (например, `VS Code`, `JetBrains IDE`).

- **Аутентификация:** Для защиты соединения плагин **ОБЯЗАН** сгенерировать уникальный секретный токен и включить его в файл обнаружения. CLI затем будет передавать этот токен в заголовке `Authorization` при всех запросах к MCP-серверу (например, `Authorization: Bearer a-very-secret-token`). Ваш сервер **ОБЯЗАН** проверять этот токен при каждом запросе и отклонять все неавторизованные запросы.
- **Переменные окружения (обязательно):** Ваш плагин **ОБЯЗАН** установить переменную окружения `QWEN_CODE_IDE_SERVER_PORT` в интегрированном терминале, чтобы CLI смог найти соответствующий файл `<PORT>.lock`.

**Примечание об устаревших версиях:** Для расширений старше версии v0.5.1 Qwen Code может использовать резервный механизм чтения JSON-файлов из системной временной директории с именами `qwen-code-ide-server-<PID>.json` или `qwen-code-ide-server-<PORT>.json`. Новые интеграции не должны полагаться на эти устаревшие файлы.

## II. Интерфейс контекста

Чтобы обеспечить осведомлённость о контексте, плагин **МОЖЕТ** предоставлять CLI информацию в реальном времени о действиях пользователя в IDE.

### Уведомление `ide/contextUpdate`

Плагин **МОЖЕТ** отправлять уведомление `ide/contextUpdate` ([notification](https://modelcontextprotocol.io/specification/2025-06-18/basic/index#notifications)) в CLI каждый раз, когда контекст пользователя изменяется.

- **События, вызывающие отправку:** Это уведомление следует отправлять (с рекомендуемой задержкой 50 мс) при следующих событиях:
  - Открытии, закрытии или фокусировке файла.
  - Изменении позиции курсора или выделенного текста в активном файле.
- **Полезная нагрузка (`IdeContext`):** Параметры уведомления **ДОЛЖНЫ** представлять собой объект `IdeContext`:

  ```typescript
  interface IdeContext {
    workspaceState?: {
      openFiles?: File[];
      isTrusted?: boolean;
    };
  }

  interface File {
    // Абсолютный путь к файлу
    path: string;
    // Временная метка последней фокусировки в формате Unix timestamp (для сортировки)
    timestamp: number;
    // Значение `true`, если это текущий активный файл
    isActive?: boolean;
    cursor?: {
      // Номер строки (нумерация начинается с 1)
      line: number;
      // Номер символа в строке (нумерация начинается с 1)
      character: number;
    };
    // Текст, выделенный пользователем в данный момент
    selectedText?: string;
  }
  ```

  **Примечание:** Список `openFiles` должен включать только файлы, существующие на диске. Виртуальные файлы (например, несохранённые файлы без пути или страницы настроек редактора) **ДОЛЖНЫ** быть исключены.

### Как CLI использует этот контекст

Получив объект `IdeContext`, CLI выполняет несколько шагов нормализации и усечения перед отправкой информации модели.

- **Порядок файлов:** CLI использует поле `timestamp` для определения наиболее недавно использованных файлов. Список `openFiles` сортируется по этому значению. Поэтому ваш плагин **ОБЯЗАН** предоставлять корректную Unix-метку времени последнего фокуса на файле.
- **Активный файл:** CLI считает активным только самый недавний файл (после сортировки). Флаг `isActive` для всех остальных файлов игнорируется, а поля `cursor` и `selectedText` в них очищаются. Ваш плагин должен устанавливать `isActive: true` и указывать данные о курсоре и выделении **только** для текущего фокусируемого файла.
- **Усечение:** Чтобы уложиться в лимит токенов, CLI усекает как список файлов (до 10 файлов), так и `selectedText` (до 16 КБ).

Хотя окончательное усечение выполняет CLI, настоятельно рекомендуется, чтобы ваш плагин также ограничивал объём передаваемого контекста.

## III. Интерфейс сравнения различий

Чтобы обеспечить интерактивное изменение кода, плагин **МОЖЕТ** предоставлять интерфейс сравнения различий. Это позволяет CLI запросить у IDE открытие представления различий (diff), в котором отображаются предлагаемые изменения для файла. Пользователь может просмотреть эти изменения, отредактировать их и окончательно принять или отклонить непосредственно в IDE.

### Инструмент `openDiff`

Плагин **ОБЯЗАН** зарегистрировать инструмент `openDiff` на своём сервере MCP.

- **Описание:** Этот инструмент указывает IDE открыть редактируемое окно сравнения (diff) для указанного файла.
- **Запрос (`OpenDiffRequest`):** Инструмент вызывается через запрос `tools/call`. Поле `arguments` в объекте `params` запроса **ОБЯЗАНО** содержать объект `OpenDiffRequest`.

  ```typescript
  interface OpenDiffRequest {
    // Абсолютный путь к файлу, который необходимо сравнить.
    filePath: string;
    // Предлагаемое новое содержимое файла.
    newContent: string;
  }
  ```

- **Ответ (`CallToolResult`):** Инструмент **ОБЯЗАН** немедленно вернуть `CallToolResult`, чтобы подтвердить получение запроса и сообщить об успешном открытии окна сравнения или об ошибке.
  - При успехе: если окно сравнения было успешно открыто, ответ **ОБЯЗАН** содержать пустое тело (т.е. `content: []`).
  - При ошибке: если ошибка помешала открытию окна сравнения, ответ **ОБЯЗАН** иметь значение `isError: true` и включать блок `TextContent` в массиве `content` с описанием ошибки.

  Фактический результат операции сравнения (принятие или отклонение изменений) передаётся асинхронно через уведомления.

### Инструмент `closeDiff`

Плагин **ОБЯЗАН** зарегистрировать инструмент `closeDiff` на своём сервере MCP.

- **Описание:** Этот инструмент указывает IDE закрыть открытый просмотр различий (diff) для указанного файла.  
- **Запрос (`CloseDiffRequest`):** Инструмент вызывается через запрос `tools/call`. Поле `arguments` в объекте `params` запроса **ОБЯЗАНО** содержать объект `CloseDiffRequest`.

  ```typescript
  interface CloseDiffRequest {
    // Абсолютный путь к файлу, просмотр различий для которого следует закрыть.
    filePath: string;
  }
  ```

- **Ответ (`CallToolResult`):** Инструмент **ОБЯЗАН** вернуть объект `CallToolResult`.
  - При успешном выполнении: если просмотр различий был успешно закрыт, в ответе **ОБЯЗАНО** присутствовать ровно один блок **TextContent** в массиве `content`, содержащий финальное содержимое файла перед закрытием.
  - При ошибке: если ошибка помешала закрытию просмотра различий, в ответе **ОБЯЗАНО** установить `isError: true` и включить блок `TextContent` в массив `content`, описывающий ошибку.

### Уведомление `ide/diffAccepted`

Когда пользователь принимает изменения в представлении различий (например, нажимая кнопку «Применить» или «Сохранить»), плагин **ОБЯЗАН** отправить уведомление `ide/diffAccepted` в CLI.

- **Полезная нагрузка:** Параметры уведомления **ОБЯЗАНЫ** включать путь к файлу и его окончательное содержимое. Содержимое может отличаться от исходного `newContent`, если пользователь внес ручные правки в представлении различий.

  ```typescript
  {
    // Абсолютный путь к файлу, для которого было выполнено сравнение.
    filePath: string;
    // Полное содержимое файла после принятия изменений.
    content: string;
  }
  ```

### Уведомление `ide/diffRejected`

Когда пользователь отклоняет изменения (например, закрывая представление различий без их принятия), плагин **ОБЯЗАН** отправить уведомление `ide/diffRejected` в CLI.

- **Полезная нагрузка:** Параметры уведомления **ОБЯЗАНЫ** включать путь к файлу, для которого было выполнено сравнение и которое было отклонено.

  ```typescript
  {
    // Абсолютный путь к файлу, для которого было выполнено сравнение.
    filePath: string;
  }
  ```

## IV. Интерфейс жизненного цикла

Плагин **ОБЯЗАН** корректно управлять своими ресурсами и файлом обнаружения в соответствии с жизненным циклом IDE.

- **При активации (запуск IDE / включение плагина):**
  1.  Запустить сервер MCP.
  2.  Создать файл обнаружения.
- **При деактивации (завершение работы IDE / отключение плагина):**
  1.  Остановить сервер MCP.
  2.  Удалить файл обнаружения.
