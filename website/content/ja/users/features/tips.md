# コンテキストヒント

Qwen Code には、機能の発見やセッション状態の把握を支援するコンテキストヒントシステムが搭載されています。

## 起動時のヒント

Qwen Code を起動するたびに、ヘッダー領域にヒントが表示されます。ヒントはまず優先度に基づいて選択され、同じ優先度のヒント間では LRU（Least Recently Used）方式でセッションごとにローテーションされるため、毎回異なるヒントが表示されます。

新規ユーザーは、初期セッション中にオンボーディング向けのヒントが表示されます：

| セッション数 | ヒントの例                                         |
| -------- | ---------------------------------------------------- |
| < 5      | スラッシュコマンド（`/`）、`Tab` 補完               |
| < 10     | `QWEN.md` プロジェクトコンテキスト、`--continue` / `--resume` |
| < 15     | `!` プレフィックス付きシェルコマンド                       |

それ以降は、`/compress`、`/approval-mode`、`/insight`、`/btw` などの一般的な機能に関するヒントがローテーションで表示されます。

## 応答後のヒント

会話中、Qwen Code はコンテキストウィンドウの使用量を監視し、何らかの操作が必要になった時点でヒントを表示します：

| コンテキスト使用量 | 条件                      | ヒント                                               |
| ------------- | ------------------------------ | ------------------------------------------------- |
| 50-80%        | セッション内で数回プロンプトを送信した後 | コンテキストを解放するために `/compress` を提案           |
| 80-95%        | —                              | コンテキストが満杯に近づいていることを警告                     |
| >= 95%        | —                              | 緊急：すぐに `/compress` を実行するか、続行するには `/new` を実行 |

応答後のヒントは、繰り返し表示されないよう、ヒントごとにクールダウン期間が設定されています。

## ヒントの表示履歴

ヒントの表示履歴は `~/.qwen/tip_history.json` に保存されます。このファイルは以下の情報を追跡します：

- セッション数（新規ユーザー向けヒントの選択に使用）
- 表示されたヒントとその時刻（LRU ローテーションとクールダウンに使用）

このファイルを削除してヒントの履歴をリセットしても問題ありません。

## ヒントの無効化

すべてのヒント（起動時および応答後）を非表示にするには、`~/.qwen/settings.json` で `ui.hideTips` を `true` に設定します：

```json
{
  "ui": {
    "hideTips": true
  }
}
```

`/settings` コマンドで設定ダイアログを開き、ここで切り替えることもできます。

スクリーンリーダーモードが有効になっている場合、ヒントは自動的に非表示になります。
