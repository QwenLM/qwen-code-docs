# プロンプトをスケジュール実行する

> `/loop` と cron スケジューリングツールを使用して、Qwen Code セッション内でプロンプトを繰り返し実行したり、ステータスをポーリングしたり、ワンタイムリマインダーを設定したりします。

スケジュールされたタスクを使用すると、Qwen Code は指定された間隔でプロンプトを自動的に再実行できます。デプロイのポーリング、PR の監視、長時間実行されるビルドの状況確認、またはセッションの後半で何かを行うためのリマインダー設定などに使用します。

タスクはセッションスコープです。現在の Qwen Code プロセス内で保持され、終了すると消滅します。ディスクには何も書き込まれません。

> **Tip:** スケジュールタスクはデフォルトで有効になっています。無効にするには、[settings](../configuration/settings.md) で `experimental.cron: false` を設定するか、環境変数に `QWEN_CODE_DISABLE_CRON=1` を設定します。

## /loop で繰り返しプロンプトをスケジュールする

`/loop` [バンドル skill](skills.md) は、繰り返しプロンプトをスケジュールする最も迅速な方法です。オプションの間隔とプロンプトを渡すと、Qwen Code はセッションが開いている間バックグラウンドで実行される cron ジョブを設定します。

```text
/loop 5m デプロイが完了したか確認し、どうなったか教えて
```

Qwen Code は間隔を解析して cron 式に変換し、ジョブをスケジュールして、実行間隔とジョブ ID を確認します。その後、プロンプトをすぐに 1 回実行します。最初の cron 実行を待つ必要はありません。

### 間隔の構文

間隔はオプションです。先頭に指定することも、末尾に指定することも、完全に省略することもできます。

| 形式 | 例 | 解析される間隔 |
| :--- | :--- | :--- |
| 先頭のトークン | `/loop 30m check the build` | 30 分ごと |
| 末尾の `every` 句 | `/loop check the build every 2 hours` | 2 時間ごと |
| 間隔なし | `/loop check the build` | デフォルトの 10 分ごと |

サポートされている単位は、秒の `s`、分の `m`、時間の `h`、日の `d` です。cron の粒度は 1 分のため、秒は最も近い分に切り上げられます。`7m` や `90m` のように単位にきれいに割り切れない間隔は、最も近いきれいな間隔に丸められ、Qwen Code が選択した値を通知します。

### 別コマンドのループ

スケジュールされたプロンプト自体を、コマンドや skill の呼び出しにすることができます。これは、すでにパッケージ化したワークフローを再実行する場合に便利です。

```text
/loop 20m /review-pr 1234
```

ジョブが実行されるたびに、Qwen Code はあなたが直接入力したかのように `/review-pr 1234` を実行します。

### 自律モード

プロンプトを指定せずに `/loop` を実行すると、固定されたプロンプトを繰り返す代わりに自律ループが開始されます。Qwen Code は会話で既に確立された作業の管理者として機能し、あなたが離席している間も作業を前進させ続けます。

```text
/loop
```

プロンプトも間隔も指定しない単独の `/loop` は、自己ペースの自律ループを実行します。プロンプトなしの `/loop <interval>` は、同じ自律ループを固定間隔（例: `/loop 10m`）で実行します。実行のたびに、会話で既に設定された作業を前進させます。開始した作業の完了、進行中の PR の維持（レビューコメントへの対応、失敗した CI の修正、コンフリクトの解決）、およびフォローアップの約束を果たします。これはトランスクリプトで既に確立された作業にのみ作用します。明確な許可なく新しい作業を作り出したり、不可逆な変更（push、delete、send）を行ったりすることはなく、すべてが静まると停止します。

### ループの管理

`/loop` は、既存のジョブを管理するための 2 つのサブコマンドもサポートしています。

```text
/loop list
```

スケジュールされたすべてのジョブを、その ID と cron 式とともに一覧表示します。

```text
/loop clear
```

スケジュールされたすべてのジョブを一度にキャンセルします。

## ワンタイムリマインダーを設定する

ワンショットリマインダーの場合は、`/loop` を使用する代わりに、自然言語で必要な内容を記述します。Qwen Code は、実行後に自身を削除する単発実行タスクをスケジュールします。

```text
午後3時にリリースブランチを push するようリマインドして
```

```text
45分後に、統合テストがパスしたか確認して
```

Qwen Code は cron 式を使用して実行時刻を特定の分と時に固定し、いつ実行されるかを確認します。

## スケジュールタスクの管理

自然言語で Qwen Code にタスクの一覧表示やキャンセルを依頼するか、基盤となるツールを直接参照します。

```text
どのスケジュールタスクが登録されている？
```

```text
デプロイチェックジョブをキャンセルして
```

内部では、Qwen Code は以下のツールを使用しています。

| ツール | 目的 |
| :--- | :--- |
| `CronCreate` | 新しいタスクをスケジュールします。5 フィールドの cron 式、実行するプロンプト、および繰り返し実行するか単発実行するかを受け付けます。 |
| `CronList` | ID、スケジュール、およびプロンプトとともに、すべてのスケジュールタスクを一覧表示します。 |
| `CronDelete` | ID を指定してタスクをキャンセルします。 |

各スケジュールタスクには 8 文字の ID があり、これを `CronDelete` に渡すことができます。1 つのセッションで保持できるスケジュールタスクは最大 50 個です。

## スケジュールタスクの実行方法

スケジューラは毎秒、期限切れのタスクがないかチェックし、セッションがアイドル状態になったときにそれらをキューに入れます。スケジュールされたプロンプトは、Qwen Code が応答中の間ではなく、あなたのターンの合間に実行されます。タスクの実行時刻になったときに Qwen Code がビジー状態の場合、プロンプトは現在のターンが終了するまで待機します。

すべての時刻はローカルタイムゾーンで解釈されます。`0 9 * * *` のような cron 式は、UTC ではなく、Qwen Code を実行している場所の午前 9 時を意味します。

### ジッター

すべてのセッションが同時に API を叩くのを避けるため、スケジューラは実行時刻に小さな決定的なオフセットを追加します。

- **繰り返しタスク** は、期間の最大 10%（上限 15 分）遅れて実行されます。1 時間ごとのジョブは、`:00` から `:06` の間の任意の時刻に実行される可能性があります。
- **ワンショットタスク** で、正時または半時（`:00` または `:30`）にスケジュールされたものは、最大 90 秒早く実行されます。

オフセットはタスク ID から導出されるため、同じタスクは常に同じオフセットを受け取ります。正確なタイミングが重要な場合は、`0 9 * * *` ではなく `3 9 * * *` のように、`:00` や `:30` 以外の分を指定すると、ワンショットジッターは適用されません。

### 3 日での有効期限切れ

繰り返しタスクは、作成から 3 日後に自動的に期限切れになります。タスクは最後に 1 回実行された後、自身を削除します。これにより、忘れられたループが実行され続ける期間に上限が設けられます。繰り返しタスクをそれ以上継続させる必要がある場合は、期限切れになる前にキャンセルして再作成してください。

ワンショットタスクはタイマーによって期限切れになることはありません。1 回実行された後に単に自身を削除します。

## cron 式リファレンス

`CronCreate` は、標準的な 5 フィールドの cron 式を受け付けます。`分 時 日 月 曜日` です。すべてのフィールドは、ワイルドカード (`*`)、単一の値 (`5`)、ステップ (`*/15`)、範囲 (`1-5`)、およびカンマ区切りのリスト (`1,15,30`) をサポートしています。

| 例 | 意味 |
| :--- | :--- |
| `*/5 * * * *` | 5 分ごと |
| `0 * * * *` | 毎時 0 分（1 時間ごと） |
| `7 * * * *` | 毎時 7 分 |
| `0 9 * * *` | 毎日ローカルタイムの午前 9 時 |
| `0 9 * * 1-5` | 平日のローカルタイム午前 9 時 |
| `30 14 15 3 *` | 3 月 15 日のローカルタイム午後 2 時 30 分 |

曜日は、日曜日が `0` または `7`、土曜日が `6` です。日と曜日の両方が制約されている場合（どちらも `*` でない場合）、どちらかのフィールドが一致すれば日付が一致します。これは標準的な vixie-cron のセマンティクスに従います。

`L`、`W`、`?` などの拡張構文や、`MON` や `JAN` などの名前エイリアスはサポートされていません。

## 制限事項

セッションスコープのスケジューリングには、固有の制約があります。

- タスクは、Qwen Code が実行中でアイドル状態のときにのみ実行されます。ターミナルを閉じたり、セッションを終了させたりすると、すべてキャンセルされます。
- 実行逃しのキャッチアップはありません。長時間実行されるリクエストで Qwen Code がビジー状態の間にタスクのスケジュール時刻を過ぎた場合、Qwen Code がアイドル状態になったときに 1 回だけ実行され、逃した間隔ごとに実行されるわけではありません。
- 再起動をまたいだ永続性はありません。Qwen Code を再起動すると、すべてのセッションスコープタスクがクリアされます。
