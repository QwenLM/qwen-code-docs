# スケジュールでプロンプトを実行する

> `/loop` とcronスケジューリングツールを使用して、Qwen Codeセッション内でプロンプトを繰り返し実行したり、ステータスをポーリングしたり、一度限りのリマインダーを設定したりできます。

スケジュールタスクを使用すると、Qwen Codeが一定間隔でプロンプトを自動的に再実行します。デプロイのポーリング、PRの監視、長時間実行中のビルドの確認、またはセッション中に後で何かをするためのリマインダーとして活用できます。

タスクはセッションスコープです。現在のQwen Codeプロセスに存在し、終了すると消えます。ディスクには何も書き込まれません。

> **Tip:** スケジュールタスクはデフォルトで有効になっています。無効にするには、[設定](../configuration/settings.md)で `experimental.cron: false` を設定するか、環境変数 `QWEN_CODE_DISABLE_CRON=1` を設定してください。

## /loop で定期的なプロンプトをスケジュールする

`/loop` [バンドルスキル](skills.md)は、定期的なプロンプトをスケジュールする最も手軽な方法です。オプションの間隔とプロンプトを渡すと、Qwen Codeがセッションを開いたままバックグラウンドで実行されるcronジョブをセットアップします。

```text
/loop 5m check if the deployment finished and tell me what happened
```

Qwen Codeは間隔を解析してcron式に変換し、ジョブをスケジュールして、実行ペースとジョブIDを確認します。その後、最初のcron発火を待たずにプロンプトをすぐに一度実行します。

### 間隔の構文

間隔はオプションです。先頭に付けても、末尾に付けても、まったく省略しても構いません。

| 形式                    | 例                                    | 解析される間隔              |
| :---------------------- | :------------------------------------ | :--------------------------- |
| 先頭トークン           | `/loop 30m check the build`           | 30分ごと             |
| 末尾の `every` 句 | `/loop check the build every 2 hours` | 2時間ごと                |
| 間隔なし             | `/loop check the build`               | デフォルトで10分ごと |

サポートされる単位は、秒に `s`、分に `m`、時間に `h`、日に `d` です。cronは1分の粒度しかないため、秒は最も近い分に切り上げられます。`7m` や `90m` のように単位に均等に割り切れない間隔は、最も近いきりの良い間隔に丸められ、Qwen Codeが選択した値を通知します。

### 別のコマンドをループする

スケジュールされたプロンプト自体をコマンドやスキルの呼び出しにすることができます。これは、すでにパッケージ化したワークフローを再実行するときに便利です。

```text
/loop 20m /review-pr 1234
```

ジョブが発火するたびに、Qwen Codeは `/review-pr 1234` をタイプしたかのように実行します。

### ループの管理

`/loop` は既存のジョブを管理するためのサブコマンドも2つサポートしています。

```text
/loop list
```

すべてのスケジュールジョブをIDとcron式とともに一覧表示します。

```text
/loop clear
```

すべてのスケジュールジョブを一度にキャンセルします。

## 一度限りのリマインダーを設定する

一度限りのリマインダーには、`/loop` を使う代わりに自然言語で内容を説明してください。Qwen Codeは実行後に自動削除される単発タスクをスケジュールします。

```text
remind me at 3pm to push the release branch
```

```text
in 45 minutes, check whether the integration tests passed
```

Qwen Codeはcron式を使用して発火時刻を特定の分と時間に固定し、いつ発火するかを確認します。

## スケジュールタスクの管理

Qwen Codeに自然言語でタスクの一覧表示やキャンセルを依頼するか、基盤となるツールを直接参照してください。

```text
what scheduled tasks do I have?
```

```text
cancel the deploy check job
```

内部的に、Qwen Codeは以下のツールを使用します。

| ツール         | 目的                                                                                                         |
| :----------- | :-------------------------------------------------------------------------------------------------------------- |
| `CronCreate` | 新しいタスクをスケジュールします。5フィールドのcron式、実行するプロンプト、繰り返しか一度限りかを受け付けます。 |
| `CronList`   | すべてのスケジュールタスクをID、スケジュール、プロンプトとともに一覧表示します。                                                |
| `CronDelete` | IDでタスクをキャンセルします。                                                                            |

各スケジュールタスクには8文字のIDがあり、`CronDelete` に渡すことができます。1つのセッションで最大50のスケジュールタスクを保持できます。

## スケジュールタスクの実行方法

スケジューラは毎秒期限切れのタスクを確認し、セッションがアイドル状態のときにキューに入れます。スケジュールされたプロンプトは、Qwen Codeが応答中ではなく、ターンとターンの間に発火します。タスクの期限が来たときにQwen Codeがビジー状態の場合、現在のターンが終了するまでプロンプトは待機します。

すべての時刻はローカルタイムゾーンで解釈されます。`0 9 * * *` のようなcron式は、UTCではなく、Qwen Codeを実行している場所の午前9時を意味します。

### ジッター

すべてのセッションが同じ時刻にAPIにアクセスするのを避けるため、スケジューラは発火時刻に小さな決定論的オフセットを追加します。

- **定期タスク**は期間の最大10%遅れて発火し、15分を上限とします。1時間ごとのジョブは `:00` から `:06` の間のどこかで発火する可能性があります。
- **一度限りのタスク**で時や30分の区切り（分が `:00` または `:30`）にスケジュールされたものは、最大90秒早く発火します。

オフセットはタスクIDから導出されるため、同じタスクは常に同じオフセットを取得します。正確なタイミングが重要な場合は、`:00` や `:30` 以外の分を選択してください。たとえば `0 9 * * *` の代わりに `3 9 * * *` を使用すると、一度限りのジッターは適用されません。

### 3日間の有効期限

定期タスクは作成から3日後に自動的に期限切れになります。タスクは最後に一度発火してから自分自身を削除します。これにより、忘れられたループが実行され続ける期間を制限します。定期タスクをより長く継続させる必要がある場合は、期限切れになる前にキャンセルして再作成してください。

一度限りのタスクはタイマーで期限切れになりません。一度発火した後に自分自身を削除するだけです。

## cron式リファレンス

`CronCreate` は標準の5フィールドcron式を受け付けます: `minute hour day-of-month month day-of-week`。すべてのフィールドはワイルドカード（`*`）、単一値（`5`）、ステップ（`*/15`）、範囲（`1-5`）、カンマ区切りリスト（`1,15,30`）をサポートします。

| 例        | 意味                      |
| :------------- | :--------------------------- |
| `*/5 * * * *`  | 5分ごと              |
| `0 * * * *`    | 毎時0分       |
| `7 * * * *`    | 毎時7分 |
| `0 9 * * *`    | 毎日ローカル午前9時       |
| `0 9 * * 1-5`  | 平日のローカル午前9時        |
| `30 14 15 3 *` | 3月15日のローカル午後2時30分     |

曜日は日曜日に `0` または `7`、土曜日に `6` を使用します。日付と曜日の両方が制約されている（どちらも `*` でない）場合、いずれかのフィールドが一致すれば日付が一致します。これは標準のvixie-cronのセマンティクスに従います。

`L`、`W`、`?` などの拡張構文や `MON`、`JAN` などの名前エイリアスはサポートされていません。

## 制限事項

セッションスコープのスケジューリングには固有の制約があります。

- タスクはQwen Codeが実行中かつアイドル状態のときのみ発火します。ターミナルを閉じるかセッションが終了すると、すべてがキャンセルされます。
- 失われた発火のキャッチアップはありません。Qwen Codeが長時間実行中のリクエストでビジーな間にタスクのスケジュール時刻が過ぎた場合、Qwen Codeがアイドル状態になったときに一度だけ発火します。失われた間隔ごとに発火するわけではありません。
- 再起動をまたいで永続しません。Qwen Codeを再起動すると、すべてのセッションスコープタスクがクリアされます。
