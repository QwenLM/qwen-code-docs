# カスタムチャンネルプラグイン

チャンネルシステムは、[拡張機能](../../extension/introduction)としてパッケージ化されたカスタムプラットフォームアダプターで拡張できます。これにより、Qwen Code を任意のメッセージングプラットフォーム、Webhook、またはカスタムトランスポートに接続できます。

## 仕組み

チャンネルプラグインは、有効な拡張機能から起動時に読み込まれます。`qwen channel start` を実行すると、次の処理が行われます。

1. 有効なすべての拡張機能の `qwen-extension.json` にある `channels` エントリをスキャンする
2. 各チャンネルのエントリーポイントを動的にインポートする
3. `settings.json` で参照できるようにチャンネルタイプを登録する
4. プラグインのファクトリー関数を使用してチャンネルインスタンスを作成する

カスタムチャンネルは、送信者ゲーティング、グループポリシー、セッションルーティング、スラッシュコマンド、クラッシュリカバリー、エージェントへの ACP ブリッジなど、共有パイプラインの全機能をそのまま利用できます。

## カスタムチャンネルのインストール

チャンネルプラグインを提供する拡張機能をインストールします。

```bash
# ローカルパスからインストール（開発用またはプライベートプラグイン）
qwen extensions install /path/to/my-channel-extension

# 開発用にリンク（変更が即座に反映される）
qwen extensions link /path/to/my-channel-extension
```

## カスタムチャンネルの設定

拡張機能が提供するカスタムタイプを使用して、`~/.qwen/settings.json` にチャンネルエントリを追加します。

```json
{
  "channels": {
    "my-bot": {
      "type": "my-platform",
      "apiKey": "$MY_PLATFORM_API_KEY",
      "senderPolicy": "open",
      "cwd": "/path/to/project"
    }
  }
}
```

`type` には、インストール済みの拡張機能が登録したチャンネルタイプを指定する必要があります。プラグイン固有の必須フィールド（`apiKey`、`webhookUrl` など）については、拡張機能のドキュメントを確認してください。

カスタムチャンネルでも標準のチャンネルオプションがすべて使用できます。

| Option         | Description                                    |
| -------------- | ---------------------------------------------- |
| `senderPolicy` | `allowlist`、`pairing`、または `open`              |
| `allowedUsers` | 送信者 ID の静的許可リスト                 |
| `sessionScope` | `user`、`thread`、または `single`                  |
| `cwd`          | エージェントの作業ディレクトリ                |
| `instructions` | 各セッションの最初のメッセージの先頭に追加される |
| `model`        | チャンネルのモデルオーバーライド               |
| `groupPolicy`  | `disabled`、`allowlist`、または `open`             |
| `groups`       | グループごとの設定                             |

各オプションの詳細は [Overview](./overview) を参照してください。

## チャンネルの起動

```bash
# カスタムチャンネルを含むすべてのチャンネルを起動
qwen channel start

# カスタムチャンネルのみを起動
qwen channel start my-bot
```

## 無償で利用できる機能

カスタムチャンネルは、組み込みチャンネルと同じすべての機能を自動的にサポートします。

- **送信者ポリシー** — `allowlist`、`pairing`、`open` によるアクセス制御
- **グループポリシー** — オプションの @mention ゲーティングを備えたグループごとの設定
- **セッションルーティング** — ユーザーごと、スレッドごと、または単一の共有セッション
- **DM ペアリング** — 未知のユーザーに対する完全なペアリングコードフロー
- **スラッシュコマンド** — `/help`、`/clear`、`/status` がすぐに使用可能
- **カスタムインストラクション** — 各セッションの最初のメッセージの先頭に追加される
- **クラッシュリカバリー** — セッションを保持したまま自動再起動
- **セッションごとのシリアライズ** — 競合状態を防ぐためにメッセージがキューイングされる

## 独自のチャンネルプラグインの作成

新しいプラットフォーム向けにチャンネルプラグインを作成したい場合は、`ChannelPlugin` インターフェース、`Envelope` フォーマット、および拡張ポイントについて [チャンネルプラグイン開発者ガイド](../../../developers/channel-plugins.md) を参照してください。
