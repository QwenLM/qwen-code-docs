# Memory

Qwen Code のセッションは毎回新しいコンテキストウィンドウから始まります。セッションをまたいで知識を引き継ぐための仕組みが2つあり、毎回同じことを説明し直す必要がなくなります:

- **QWEN.md** — あなたが一度書いておくと、Qwen が毎回のセッションで読み込む指示
- **Auto-memory** — Qwen があなたから学んだことを自動的に記録するメモ

---

## QWEN.md: Qwen へのあなたの指示

QWEN.md は、プロジェクトや好みについて Qwen が常に把握すべきことを書いておくプレーンテキストファイルです。会話が始まるたびに読み込まれる「永続的なブリーフィング」として機能します。

### QWEN.md に書くべき内容

毎回説明し直す必要があるようなことを書いておきましょう:

- ビルドやテストのコマンド (`npm run test`、`make build`)
- チームのコーディング規約（「新しいファイルには必ず JSDoc コメントを付ける」など）
- アーキテクチャの意思決定（「リポジトリパターンを使用し、コントローラーから直接データベースを呼び出さない」など）
- 個人の好み（「npm ではなく pnpm を使う」など）

Qwen がコードを読めば分かることは書かないようにしましょう。QWEN.md は短く具体的なほど効果的です — 長くなるほど Qwen が確実に従う可能性が低くなります。

### QWEN.md の作成場所

| ファイル                          | 適用対象                                             |
| ----------------------------- | ------------------------------------------------ |
| `~/.qwen/QWEN.md`             | あなたのすべてのプロジェクト全体に適用                    |
| プロジェクトルートの `QWEN.md` | チーム全員（ソース管理にコミットする）                    |
| `.qwen/QWEN.local.md`         | このプロジェクトのあなた専用（git に含めない）             |

これらは任意の組み合わせで使用できます。セッション開始時に Qwen はすべてを読み込みます。

リポジトリに他の AI ツール向けの `AGENTS.md` ファイルがある場合、Qwen はそれも読み込みます。指示を重複して書く必要はありません。

#### `.qwen/QWEN.local.md` を使うべき場面

**プロジェクト固有かつ個人的な**指示に使いましょう — このプロジェクトに関するものだが、チームと共有すべきでない内容:

- あなた自身のクラスター ID、コンテナレジストリの名前空間、クラウドアカウント
- ローカル環境をハードコードした個人用デバッグコマンド
- 作業中の内容について Qwen に知らせておきたいが、コミットはしたくないメモ

共有プロジェクトの `QWEN.md` の**後に**読み込まれるため、ローカルの指示でチームの設定を補完・上書きできます。

**gitignore の設定はご自身で行ってください。** `.qwen/` はローカルディレクトリとして扱われることが多いですが、qwen-code は自動で `.gitignore` を生成しません。また、`.qwen/settings.json` をコミットするプロジェクトもあります。`.gitignore`（またはグローバルな git ignore）に次の行を追加してください:

```
.qwen/QWEN.local.md
```

### `/init` で自動生成する

`/init` を実行すると、Qwen がコードベースを解析して、ビルドコマンド・テスト手順・見つかった規約を含む QWEN.md のスターターを作成します。すでに存在する場合は、上書きせずに追記候補を提案します。

### 他のファイルを参照する

QWEN.md から他のファイルを参照させることで、Qwen にそのファイルも読み込ませることができます:

```markdown
See @README.md for project overview.

# Conventions

- Git workflow: @docs/git-workflow.md
```

QWEN.md の中で `@path/to/file` を使うと参照できます。相対パスは QWEN.md ファイル自体の場所を基準に解決されます。

## Auto-memory: Qwen があなたから学ぶこと

Auto-memory はバックグラウンドで動作します。会話が終わるたびに、Qwen は学んだ有用な情報（あなたの好み、もらったフィードバック、プロジェクトのコンテキスト）をひっそりと保存します。これにより、次のセッションでも同じことを繰り返し説明する必要がなくなります。

これは QWEN.md とは異なります: あなたが書くのではなく、Qwen が書きます。

### Qwen が保存する内容

Qwen は記憶しておく価値のある4種類の情報を探します:

| 種類                    | 例                                                       |
| ----------------------- | -------------------------------------------------------- |
| **あなたについて**      | あなたの役割、バックグラウンド、好きな作業スタイル         |
| **あなたのフィードバック** | あなたが修正した内容、確認したアプローチ                 |
| **プロジェクトのコンテキスト** | 進行中の作業、意思決定、コードから読み取れない目標    |
| **外部参照**            | 言及したダッシュボード、チケットトラッカー、ドキュメントのリンク |

Qwen はすべてを保存するわけではなく、次回に実際に役立つことだけを保存します。

### 保存場所

Auto-memory ファイルは `~/.qwen/projects/<project>/memory/` に保存されます。同じリポジトリのすべてのブランチとワークツリーは同じメモリフォルダを共有するため、あるブランチで学んだことは他のブランチでも利用できます。

保存されるのはすべてプレーンなマークダウンです — いつでも任意のファイルを開いて編集・削除できます。

### 定期的なクリーンアップ

Qwen は定期的に保存済みのメモリを確認し、重複を削除したり古くなったエントリを整理したりします。これは十分なセッションが蓄積された後、1日1回バックグラウンドで自動実行されます。すぐに実行したい場合は `/dream` で手動トリガーできます。

クリーンアップ実行中は、画面の隅に **✦ dreaming** と表示されます。セッションは通常どおり継続されます。

### 有効・無効の切り替え

Auto-memory はデフォルトで有効になっています。切り替えるには `/memory` を開き、上部のスイッチを使います。自動保存だけ、定期クリーンアップだけ、またはその両方をオフにすることができます。

`~/.qwen/settings.json`（すべてのプロジェクトに適用）または `.qwen/settings.json`（このプロジェクトのみ）で設定することもできます:

```json
{
  "memory": {
    "enableManagedAutoMemory": true,
    "enableManagedAutoDream": true
  }
}
```

## コマンド

### `/memory`

Memory パネルを開きます。ここから以下の操作ができます:

- Auto-memory の保存をオン・オフする
- 定期クリーンアップ（dream）をオン・オフする
- 個人の QWEN.md（`~/.qwen/QWEN.md`）を開く
- プロジェクトの QWEN.md を開く
- Auto-memory フォルダを閲覧する

### `/init`

プロジェクト用のスターター QWEN.md を生成します。Qwen がコードベースを読み込み、ビルドコマンド・テスト手順・見つかった規約を入力します。

### `/remember <text>`

Qwen が自動的に拾うのを待たずに、すぐに Auto-memory に何かを保存します:

```
/remember always use snake_case for Python variable names
/remember the staging environment is at staging.example.com
```

### `/forget <text>`

説明に一致する Auto-memory のエントリを削除します:

```
/forget old workaround for the login bug
```

### `/dream`

自動スケジュールを待たずに、今すぐメモリのクリーンアップを実行します:

```
/dream
```

## トラブルシューティング

### Qwen が QWEN.md の指示に従わない

`/memory` を開いて、どのファイルが読み込まれているか確認してください。ファイルが一覧にない場合、Qwen はそれを認識できていません — プロジェクトルートまたは `~/.qwen/` に配置されているか確認してください。

指示は具体的なほど効果的です:

- ✓ `Use 2-space indentation for TypeScript files`
- ✗ `Format code nicely`

複数の QWEN.md ファイルで指示が競合している場合、Qwen の動作が一貫しなくなることがあります。内容を見直して矛盾を解消してください。

### Qwen が保存した内容を確認したい

`/memory` を実行して **Open auto-memory folder** を選択してください。保存されたメモリはすべて読み取り可能なマークダウンファイルとして閲覧・編集・削除できます。

### Qwen がものを忘れてしまう

Auto-memory が有効なのにセッションをまたいで記憶が引き継がれない場合は、`/dream` を実行して強制的にクリーンアップを行ってみてください。また、`/memory` で両方のトグルが有効になっているか確認してください。

常に Qwen に覚えておいてほしいことは QWEN.md に追記しましょう — Auto-memory はベストエフォートですが、QWEN.md は確実に読み込まれます。
