# メモリ

Qwen Code のセッションは毎回新しいコンテキストウィンドウで開始されます。毎回同じ説明を繰り返さなくても済むよう、セッション間で知識を引き継ぐための 2 つの仕組みが用意されています：

- **QWEN.md** — あなたが一度だけ記述し、Qwen が毎セッション読み込む指示
- **Auto-memory** — Qwen があなたとの対話から学習し、自身で記述するメモ

---

## QWEN.md: Qwen への指示書

QWEN.md は、プロジェクトやあなたの好みについて Qwen が常に把握しておくべき事項を記述するプレーンテキストファイルです。会話の開始時に毎回読み込まれる、恒久的なブリーフィング資料のようなものだと考えてください。

### QWEN.md に記述する内容

毎セッション繰り返し説明する必要があった事項を記述します：

- ビルドおよびテストコマンド（`npm run test`、`make build`）
- チームのコーディング規約（「すべての新規ファイルに JSDoc コメントを必須とする」など）
- アーキテクチャ上の決定事項（「リポジトリパターンを採用し、コントローラーから直接データベースを呼び出さない」など）
- 個人の好み（「npm ではなく pnpm を常に使用する」など）

コードを読めば Qwen が自行で把握できる内容は含めないでください。QWEN.md は短く具体的であるほど効果を発揮します。長くなるほど、Qwen が指示を確実に遵守できなくなります。

### QWEN.md の作成場所

| ファイル                          | 適用対象                             |
| ----------------------------- | --------------------------------------------- |
| `~/.qwen/QWEN.md`             | あなた（すべてのプロジェクトに適用）                 |
| プロジェクトルートの `QWEN.md` | チーム全体（ソース管理にコミット） |

両方を併用できます。セッション開始時、Qwen は検出したすべての QWEN.md ファイル（個人用とプロジェクト用）を読み込みます。

リポジトリに他の AI ツール用の `AGENTS.md` ファイルが既に存在する場合、Qwen はそれも読み込みます。指示を重複して記述する必要はありません。

### `/init` による自動生成

`/init` を実行すると、Qwen がコードベースを分析し、検出したビルドコマンド、テスト手順、規約を含む初期版の QWEN.md を作成します。既にファイルが存在する場合は、上書きせずに追加事項を提案します。

### 他のファイルの参照

QWEN.md から他のファイルを指定し、Qwen に読み込ませることができます：

```markdown
See @README.md for project overview.

# Conventions

- Git workflow: @docs/git-workflow.md
```

QWEN.md 内の任意の場所で `@path/to/file` を使用できます。相対パスは QWEN.md ファイル自体の場所を基準に解決されます。

## Auto-memory: Qwen があなたについて学習する内容

Auto-memory はバックグラウンドで動作します。各会話の終了後、Qwen は学習した有用な情報（あなたの好み、提供したフィードバック、プロジェクトのコンテキストなど）を静かに保存し、今後のセッションであなたが繰り返し説明しなくても活用できるようにします。

これは QWEN.md とは異なります。あなたが記述するのではなく、Qwen が自動で記述します。

### Qwen が保存する内容

Qwen は、記憶する価値のある 4 種類の情報を検出します：

| 項目                    | 例                                                 |
| ----------------------- | -------------------------------------------------------- |
| **あなたについて**           | 役割、背景、作業の好み              |
| **あなたのフィードバック**       | 修正した内容、承認したアプローチ           |
| **プロジェクトのコンテキスト**     | 進行中の作業、決定事項、コードからは読み取れない目標 |
| **外部リファレンス** | 言及したダッシュボード、チケットトラッカー、ドキュメントリンク    |

Qwen はすべてを保存するわけではありません。次回実際に有用となる情報のみを選択して保存します。

### 保存場所

Auto-memory ファイルは `~/.qwen/projects/<project>/memory/` に保存されます。同じリポジトリのすべてのブランチと worktree で同じメモリフォルダを共有するため、あるブランチで Qwen が学習した内容は他のブランチでも利用可能です。

保存される内容はすべてプレーンな Markdown です。いつでもファイルを開いて編集または削除できます。

### 定期的なクリーンアップ

Qwen は定期的に保存されたメモリをスキャンし、重複の削除や期限切れのエントリの整理を行います。十分なセッションが蓄積されると、1 日 1 回バックグラウンドで自動的に実行されます。今すぐ実行したい場合は、`/dream` で手動でトリガーできます。

クリーンアップ実行中は、画面の隅に **✦ dreaming** が表示されます。セッションは通常通り継続されます。

### 有効化と無効化

Auto-memory はデフォルトで有効です。切り替えるには `/memory` を開き、上部のスイッチを使用します。自動保存のみ、定期的なクリーンアップのみ、またはその両方を無効にできます。

以下の設定ファイルでも構成可能です：`~/.qwen/settings.json`（すべてのプロジェクトに適用）または `.qwen/settings.json`（このプロジェクトのみ）：

```json
{
  "memory": {
    "enableManagedAutoMemory": true,
    "enableManagedAutoDream": true
  }
}
```

## コマンド

### `/memory`

メモリパネルを開きます。ここから以下の操作が可能です：

- Auto-memory の保存を有効/無効にする
- 定期的なクリーンアップ（dream）を有効/無効にする
- 個人用 QWEN.md（`~/.qwen/QWEN.md`）を開く
- プロジェクト用 QWEN.md を開く
- Auto-memory フォルダを参照する

### `/init`

プロジェクト用の初期 QWEN.md を生成します。Qwen がコードベースを読み込み、検出したビルドコマンド、テスト手順、規約を自動で記入します。

### `/remember <text>`

Qwen が自動で検出するのを待たず、即座に Auto-memory に保存します：

```
/remember always use snake_case for Python variable names
/remember the staging environment is at staging.example.com
```

### `/forget <text>`

説明に一致する Auto-memory エントリを削除します：

```
/forget old workaround for the login bug
```

### `/dream`

自動スケジュールを待たず、今すぐメモリクリーンアップを実行します：

```
/dream
```

## トラブルシューティング

### Qwen が QWEN.md の指示に従わない

`/memory` を開き、読み込まれているファイルを確認してください。ファイルが一覧にない場合、Qwen はそれを認識できません。プロジェクトルートまたは `~/.qwen/` に配置されているか確認してください。

指示は具体的であるほど効果的です：

- ✓ `Use 2-space indentation for TypeScript files`
- ✗ `Format code nicely`

指示が矛盾する複数の QWEN.md ファイルが存在する場合、Qwen の動作が不安定になる可能性があります。ファイルを確認し、矛盾する記述を削除してください。

### Qwen が保存した内容を確認したい

`/memory` を実行し、**Open auto-memory folder** を選択してください。保存されたメモリはすべて閲覧・編集・削除可能な Markdown ファイルです。

### Qwen が内容を忘れてしまう

Auto-memory が有効なのにセッション間で記憶が引き継がれない場合は、`/dream` を実行してクリーンアップを強制実行してください。また、`/memory` で両方のトグルが有効になっているか確認してください。

常に Qwen に記憶させておきたい事項は、代わりに QWEN.md に追加してください。Auto-memory はベストエフォート型ですが、QWEN.md は確実に適用されます。
