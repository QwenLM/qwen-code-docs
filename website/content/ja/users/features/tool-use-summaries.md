# ツール使用サマリー

Qwen Code は、各ツールバッチの完了後に、そのバッチが行った内容を要約するgitコミットサブジェクト形式の短いラベルを生成できます。ラベルはトランスクリプトにインラインで表示され、コンパクトモードでは汎用の `Tool × N` ヘッダーを置き換えます。

これは並列ツール呼び出しのUX補助機能です。モデルが複数の `Read` + `Grep` + `Bash` 呼び出しを一度に展開する場合、ツールリストをスキャンする手間なく、サマリーが意図を一目で伝えます。

この機能はデフォルトで有効になっており、バックグラウンドで静かに動作します。設定済みの [fast model](./followup-suggestions#fast-model) が必要です。

## 表示内容

### フルモード（デフォルト）

サマリーはツールグループの直下に薄いバッジ行として表示されます：

```
╭──────────────────────────────────────────────╮
│ ✓  ReadFile a.txt                            │
│ ✓  ReadFile b.txt                            │
│ ✓  ReadFile c.txt                            │
│ ✓  ReadFile d.txt                            │
╰──────────────────────────────────────────────╯

 ● Read 4 text files
```

### コンパクトモード（`Ctrl+O` または `ui.compactMode: true`）

ラベルがコンパクトな1行表示で汎用の `Tool × N` ヘッダーを置き換えます：

```
╭──────────────────────────────────────────────╮
│✓  Read txt files  · 4 tools                  │
│Press Ctrl+O to show full tool output         │
╰──────────────────────────────────────────────╯
```

個々のツール呼び出しは1キー操作で確認できます（`Ctrl+O` でフルモードに切り替え）。

## 仕組み

ツールバッチが完了すると、Qwen Code は設定済みのfast modelに対してfire-and-forgetの呼び出しを行います。その内容は以下のとおりです：

- ツール名、切り詰めた引数、切り詰めた結果（各フィールドは300文字でキャップ）。
- アシスタントの直近のテキスト出力（最初の200文字）を意図プレフィックスとして。
- 過去形で30文字のgitコミットサブジェクト形式のラベルを返すよう指示するシステムプロンプト。

この呼び出しは次のターンのAPIストリーミングと並列で実行されるため、約1秒のレイテンシはメインモデルのレスポンスの陰に隠れます。ラベルが解決されると、`tool_use_summary` エントリとしてトランスクリプトに追加されます。

ラベルの例：`Searched in auth/`、`Fixed NPE in UserService`、`Created signup endpoint`、`Read config.json`、`Ran failing tests`。

## 表示されるタイミング

以下の条件を**すべて**満たした場合にサマリーが生成されます：

- `experimental.emitToolUseSummaries` が `true`（デフォルト）。
- `fastModel` が設定済み（設定またはコマンド `/model --fast` で）。
- バッチ内で少なくとも1つのツールが完了している。
- ツール完了前にターンが中断されていない。
- fast modelが空でないエラーなしのレスポンスを返した。

サブエージェントのツール呼び出しはサマリー生成のトリガーにはなりません。メインセッションのツールバッチのみが対象です。

## 表示されないタイミング

以下の場合、サマリーは静かにスキップされます（エラーなし、UI変更なし）：

- fast modelが設定されていない。
- fast modelの呼び出しが失敗、タイムアウト、または空を返した。
- モデルが明らかにエラーメッセージのような文字列（例：`Error: ...`、`I cannot ...`）を返した場合 — クライアントがフィルタリングして誤解を招くラベルをUIに表示しません。
- モデルが完了する前にターンが中断された（`Ctrl+C`）。

これらの場合、ツールグループは通常どおり表示されます。

## Fast Model

ラベルはプロンプトサジェストや投機的実行と同じ [fast model](./followup-suggestions#fast-model) を使って生成されます。設定方法は以下のとおりです：

### コマンドで設定

```
/model --fast qwen3-coder-flash
```

### `settings.json` で設定

```json
{
  "fastModel": "qwen3-coder-flash"
}
```

fast modelが設定されていない場合、サマリー生成は完全にスキップされます。設定するまでこの機能は動作しません。

## 設定

以下の設定を `settings.json` で構成できます：

| Setting                             | Type    | Default | Description                                                                                        |
| ----------------------------------- | ------- | ------- | -------------------------------------------------------------------------------------------------- |
| `experimental.emitToolUseSummaries` | boolean | `true`  | サマリー生成のマスタースイッチ。オフにするとfast modelへの追加呼び出しが無効になります。           |
| `fastModel`                         | string  | `""`    | サマリー生成に使用するfast model（プロンプトサジェストと共有）。必須。空の場合は何も行いません。   |

### 環境変数によるオーバーライド

`QWEN_CODE_EMIT_TOOL_USE_SUMMARIES` は現在のセッションの `experimental.emitToolUseSummaries` 設定を上書きします：

- `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=0` または `=false` — 強制オフ。
- `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=1` または `=true` — 強制オン。
- 未設定 — `experimental.emitToolUseSummaries` の設定を使用。

### 例

```json
{
  "fastModel": "qwen3-coder-flash",
  "experimental": {
    "emitToolUseSummaries": true
  }
}
```

## スコープとライフサイクル

この機能を初めて読む際につまずきやすい3つのポイント：

1. **1バッチにつき1回の生成、両表示モードで共有。** fast modelの呼び出しは `handleCompletedTools` でツールバッチが完了した時点で厳密に1回だけ行われます。その後 `Ctrl+O` を切り替えても**新しい呼び出しは発生しません**。両モードは最初にキャプチャされた同じ `tool_use_summary` 履歴エントリから読み取ります。コンパクトモードのオン/オフを自由に切り替えても追加コストはかかりません。
2. **切り替え時やセッション再開時のバックフィルなし。** 機能が有効になる前（または設定をオンにする前、または再開したセッション — `ChatRecordingService` はサマリーエントリを永続化しません）に完了した `tool_group` にはラベルが付きません。「既存の履歴をスキャン」する処理はありません。セッション途中でこの設定をオンにした場合、_以降の_バッチにのみラベルが表示され、古いグループはラベルがないことを示す表示なしにデフォルト表示のままになります。
3. **メインエージェントのバッチのみ。** トリガーはメインセッションのターンループ（`useGeminiStream`）にあります：
   - ✅ シェル、MCP、ファイル操作、および `Task` / サブエージェントツールの_呼び出し自体_（メインバッチに表示されるもの）はサマリー化されます。
   - ❌ サブエージェントの**内部**ツールバッチ（`packages/core/src/agents/runtime/` 経由で実行）はサマリー化されません。

   `Task` ツールを_含む_外側のバッチはラベルが付きますが、fast modelはサブエージェントツール呼び出しとその集約出力のみを参照し、サブエージェント内部の個別ツール呼び出しは参照しません。`Searched 14 files` ではなく `Ran research-agent` や `Delegated file search` のようなラベルが期待されます。これは意図的な設計です。サブエージェントの内部をサマリー化するとfast modelのコストが倍増し、プライマリUIには表示されないノイズが表面化します。

## 推奨の組み合わせ：コンパクトモードを有効にする

3件以上の並列ツール呼び出しのバッチでは、この機能と `ui.compactMode: true` を組み合わせることで最も見やすいトランスクリプトになります。コンパクトビューはバッチ全体を1つのラベル付き行（`✓  Read txt files  · 4 tools`）に折り畳み、ツールの各行とその後のサマリーを並べて表示しません。詳細は `Ctrl+O` で1キー操作でアクセスできます。

```json
{
  "fastModel": "qwen3-coder-flash",
  "ui": {
    "compactMode": true
  },
  "experimental": {
    "emitToolUseSummaries": true
  }
}
```

フルモード（デフォルト）では、サマリーはツールグループの下に末尾の `● <label>` 行として表示されます。大規模または異種バッチには便利ですが、小さな同種バッチ（例：`Read × 3`）では、ラベルが表示されているツール行の言い換えに見えることがあります。これが通常のワークフローに当てはまる場合は、上記のようにコンパクトモードをオンにするか、`experimental.emitToolUseSummaries: false` でサマリーを完全に無効にしてください。

## モニタリング

サマリーモデルの使用量は `/stats` 出力のfast modelトークン合計に表示され、プロンプトサジェストや他のバックグラウンドタスクと区別できるよう `prompt_id` は `tool_use_summary_generation` になっています。

## データフローとプライバシー

サマリー呼び出しでは、各成功ツールの名前、切り詰めた `args`、切り詰めた結果（各フィールドは300文字でキャップ）を **fast model** に送信します。また、アシスタントの直近のテキストの最初の200文字を意図プレフィックスとして送信します。

fast modelがメインセッションモデルと同じプロバイダー/認証で設定されている場合、データはメインセッションがすでに使用している同じ境界を通じて流れます（信頼スコープの変更なし）。**別のプロバイダー**のfast modelを設定した場合、ツールの入出力（`read_file` で読み込んだファイルの内容、シェル呼び出しのコマンド出力、MCPツールで取得した値を含む可能性があります）がサマリー化プロンプトの一部としてそのプロバイダーに送信されます。これはメインセッション単独よりも厳密に広いデータ共有スコープです。

ワークフロー上これが重要な場合、2つのクリーンな選択肢があります：

- `fastModel` をメインセッションと同じプロバイダーのモデルに設定し、サマリー呼び出しが新しい認証/データ境界を越えないようにする。
- `experimental.emitToolUseSummaries: false`（または `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=0`）で機能を完全に無効にする。

フィールドあたり300文字のキャップでリスクを制限していますが、排除はできません。キャップウィンドウ内のツール出力で発見されたシークレットは依然として送信される可能性があります。fast modelのデータ境界はメインモデルと同様に扱ってください。

## コスト

対象となるツールバッチごとにfast modelを1回呼び出します。入力は小さな固定システムプロンプトに加え、切り詰めたツールの入出力（各フィールドは300文字でキャップ）です。出力は短い1行（100文字でキャップ、通常20トークン以下）です。典型的なfast modelでは1バッチあたり約$0.001です。

追加コストを避けたい場合は、`experimental.emitToolUseSummaries: false` または `QWEN_CODE_EMIT_TOOL_USE_SUMMARIES=0` で機能をオフにしてください。
