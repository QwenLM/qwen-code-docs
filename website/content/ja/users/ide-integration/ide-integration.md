# IDE 統合

Qwen Code は、よりシームレスでコンテキストを意識した体験を提供するために、IDE と統合できます。この統合により、CLI がワークスペースをよりよく理解できるようになり、エディタ内でのネイティブな差分表示（diffing）といった強力な機能が可能になります。

現在対応している IDE は、[Visual Studio Code](https://code.visualstudio.com/) および VS Code 拡張機能をサポートする他のエディタのみです。他のエディタへの対応を実装するには、[IDE Companion Extension Spec](../ide-integration/ide-companion-spec) を参照してください。

## 機能

- **ワークスペースのコンテキスト:** CLI は自動的にワークスペースを認識し、より関連性が高く正確な応答を提供します。このコンテキストには以下の情報が含まれます:
  - ワークスペース内で**最近アクセスした上位 10 個のファイル**。
  - 現在のカーソル位置。
  - 選択中のテキスト（最大 16KB まで；それより長い選択範囲は切り捨てられます）。

- **ネイティブな差分表示:** Qwen がコードの変更を提案すると、IDE のネイティブな差分ビューアーで変更内容を直接確認できます。これにより、提案された変更をレビュー、編集、および承認または却下する作業をシームレスに実行できます。

- **VS Code コマンド:** VS Code のコマンドパレット（`Cmd+Shift+P` または `Ctrl+Shift+P`）から、Qwen Code の機能に直接アクセスできます:
  - `Qwen Code: Run`: 統合ターミナルで新しい Qwen Code セッションを開始します。
  - `Qwen Code: Accept Diff`: アクティブな差分エディター内の変更を適用します。
  - `Qwen Code: Close Diff Editor`: 変更を却下し、アクティブな差分エディターを閉じます。
  - `Qwen Code: View Third-Party Notices`: 拡張機能のサードパーティ通知を表示します。

## インストールとセットアップ

IDE 統合を設定する方法は以下の 3 つがあります。

### 1. 自動ナッジ（推奨）

サポート対象のエディター内で Qwen Code を実行すると、環境が自動的に検出され、接続のためのプロンプトが表示されます。「はい」と回答すると、必要なセットアップ（補助拡張機能のインストールおよび接続の有効化）が自動的に実行されます。

### 2. CLI からの手動インストール

以前にプロンプトを無視した場合、または拡張機能を手動でインストールしたい場合は、Qwen Code 内で以下のコマンドを実行できます。

```
/ide install
```

このコマンドにより、使用中の IDE に対応する正しい拡張機能が検索され、インストールされます。

### 3. マーケットプレイスからの手動インストール

拡張機能をマーケットプレイスから直接インストールすることもできます。

- **Visual Studio Code の場合：** [VS Code マーケットプレイス](https://marketplace.visualstudio.com/items?itemName=qwenlm.qwen-code-vscode-ide-companion) からインストールしてください。
- **VS Code のフォーク版の場合：** VS Code のフォーク版をサポートするため、この拡張機能は [Open VSX レジストリ](https://open-vsx.org/extension/qwenlm/qwen-code-vscode-ide-companion) にも公開されています。お使いのエディタの指示に従って、このレジストリから拡張機能をインストールしてください。

> NOTE:
> 「Qwen Code Companion」拡張機能は、検索結果の下部に表示されることがあります。すぐに見つからない場合は、画面を下にスクロールするか、「新規公開順」で並べ替えてみてください。
>
> 拡張機能を手動でインストールした後は、CLI で `/ide enable` を実行して統合機能を有効化する必要があります。

## 使い方

### 有効化と無効化

CLI 内から IDE 統合を制御できます。

- IDE への接続を有効化するには、以下を実行します：
  ```
  /ide enable
  ```
- 接続を無効化するには、以下を実行します：
  ```
  /ide disable
  ```

有効化すると、Qwen Code は自動的に IDE コンパニオン拡張機能に接続しようと試みます。

### ステータスの確認

接続ステータスを確認し、CLI が IDE から受信したコンテキストを表示するには、以下を実行します：

```
/ide status
```

接続されている場合、このコマンドは接続先の IDE 名と、認識している最近開かれたファイルの一覧を表示します。

（注：ファイル一覧はワークスペース内の直近 10 件のアクセス済みファイルに制限されており、ローカルディスク上のファイルのみが含まれます。）

### 差分の操作

Qwen モデルにファイルの変更を依頼すると、エディター内で差分ビューを直接開くことができます。

**差分を適用するには**、以下のいずれかの操作を行ってください：

- 差分エディターのタイトルバーにある **チェックマークアイコン** をクリックします。
- ファイルを保存します（例：`Cmd+S` または `Ctrl+S`）。
- コマンドパレットを開き、「**Qwen Code: Accept Diff**」を実行します。
- CLI で確認が表示された際に `yes` と応答します。

**差分を拒否するには**、以下のいずれかの操作を行ってください：

- 差分エディターのタイトルバーにある **「×」アイコン** をクリックします。
- 差分エディタータブを閉じます。
- コマンドパレットを開き、「**Qwen Code: Close Diff Editor**」を実行します。
- CLI で確認が表示された際に `no` と応答します。

また、差分ビュー内で **提案された変更を直接編集してから適用することもできます**。

CLI で「はい、常に許可する」を選択した場合、変更は IDE 上に表示されなくなり、自動的に適用されます。

## サンドボックス環境での使用

サンドボックス内で Qwen Code を使用する場合、以下の点に注意してください。

- **macOS の場合：** IDE 統合には、IDE コンパニオン拡張機能と通信するためのネットワークアクセスが必要です。ネットワークアクセスを許可する Seatbelt プロファイルを使用する必要があります。
- **Docker コンテナ内での実行の場合：** Qwen Code を Docker（または Podman）コンテナ内で実行しても、IDE 統合はホストマシン上で実行中の VS Code 拡張機能に接続できます。CLI は `host.docker.internal` 上で IDE サーバーを自動的に検出するよう設定されています。通常、特別な設定は必要ありませんが、コンテナからホストへの接続を Docker のネットワーキング設定が許可していることを確認する必要がある場合があります。

## トラブルシューティング

IDE 統合で問題が発生した場合、以下に代表的なエラーメッセージとその解決方法を示します。

### 接続エラー

- **メッセージ:** `🔴 切断されました: [IDE 名] の IDE コンパニオン拡張機能に接続できませんでした。拡張機能が実行中であることを確認し、ターミナルを再起動してください。拡張機能をインストールするには、`/ide install` を実行してください。`
  - **原因:** Qwen Code が IDE への接続に必要な環境変数 (`QWEN_CODE_IDE_WORKSPACE_PATH` または `QWEN_CODE_IDE_SERVER_PORT`) を見つけられませんでした。これは通常、IDE コンパニオン拡張機能が実行されていないか、正しく初期化されていないことを意味します。
  - **解決方法:**
    1.  IDE に **Qwen Code Companion** 拡張機能がインストールされ、有効化されていることを確認してください。
    2.  正しい環境変数が反映されるよう、IDE 内で新しいターミナルウィンドウを開いてください。

- **メッセージ:** `🔴 切断されました: IDE 接続エラー。予期せず接続が失われました。`/ide enable` を実行して再接続してください。`
  - **原因:** IDE コンパニオンへの接続が失われました。
  - **解決方法:** 再接続を試すために `/ide enable` を実行してください。問題が継続する場合は、新しいターミナルウィンドウを開くか、IDE を再起動してください。

### 設定エラー

- **メッセージ:** `🔴 切断されました: ディレクトリが一致していません。Qwen Code は、[IDE 名] で開いているワークスペースとは異なる場所で実行されています。CLI は、プロジェクトのルートフォルダーと同じディレクトリから実行してください。`
  - **原因:** CLI の現在の作業ディレクトリが、IDE で開いているフォルダーまたはワークスペースの外部にあります。
  - **解決方法:** IDE で開いているディレクトリに `cd` で移動し、CLI を再起動してください。

- **メッセージ:** `🔴 切断されました: この機能を使用するには、[IDE 名] でワークスペースフォルダーを開いてから再度お試しください。`
  - **原因:** IDE で開いているワークスペースがありません。
  - **解決方法:** IDE でワークスペースを開き、CLI を再起動してください。

### 一般的エラー

- **メッセージ:** `IDE 統合は、現在の環境ではサポートされていません。この機能を利用するには、Qwen Code を以下の対応 IDE のいずれかで実行してください：[IDE の一覧]`
  - **原因:** Qwen Code を、対応していない IDE 以外の端末または環境で実行しています。
  - **解決策:** VS Code などの対応 IDE の統合ターミナルから Qwen Code を実行してください。

- **メッセージ:** `この IDE 用のインストーラーは利用できません。マーケットプレイスから「Qwen Code Companion」拡張機能を手動でインストールしてください。`
  - **原因:** `/ide install` コマンドを実行しましたが、CLI にはお使いの IDE 用の自動インストーラーが用意されていません。
  - **解決策:** IDE の拡張機能マーケットプレイスを開き、「Qwen Code Companion」を検索して、手動でインストールしてください。
