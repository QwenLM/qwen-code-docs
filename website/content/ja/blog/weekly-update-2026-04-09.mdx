今週は機能バージョン **v0.14.0** と **v0.14.2**、バグ修正バージョン **v0.13.2** と **v0.14.1** の計4バージョンをリリースしました。

v0.14.0 は大型アップデートです：

- **Channels システムにより Qwen Code はターミナルに限定されなくなり**、Telegram、WeChat、DingTalk などのプラットフォームからリモート操作が可能に；
- Cron 定期タスクにより AI がスケジュールに従って自動的に繰り返し作業を実行；
- Qwen 3.6 Plus モデルが正式リリース。v0.14.1 ではフォローアップ提案、詳細/簡潔モード切り替えなどの体験改善を実施。
- v0.14.2 では `/plan` 計画モードコマンド、thinking block のターン間保持、適応型出力トークンのアップグレード（デフォルト 8K + 自動 64K アップグレード）、組み込みバグ修正ワークフローとデバッグスキルを追加。

## ✨ 新機能

### Channels：DingTalk、Telegram、WeChat から Qwen Code をリモート操作

Qwen Code はもうターミナルだけのツールではありません。Channels システムはプラグイン方式で複数のプラットフォームに対応しており、現在 Telegram、WeChat、DingTalk をサポートしています。スマートフォンから Bot にメッセージを送信して、サーバー上でタスクを実行させることができます。

**こんなことができます：**

- 外出先から Telegram で Qwen Code にメッセージを送り、スクリプトの実行やログの確認を依頼
- DingTalk グループでボットを @メンションして、チームのコード関連タスクを処理
- WeChat でさっとメッセージを送って、開発環境をリモート操作

PR [#2628](https://github.com/QwenLM/qwen-code/pull/2628) を参照

![](https://gw.alicdn.com/imgextra/i4/O1CN01fWI9Ow1NUwXa0kIJR_!!6000000001574-0-tps-4032-2565.jpg)

### Cron 定期タスク：AI にスケジュール通りに自動作業させる

新しい Cron ツールにより、現在のセッション内で定期的な繰り返しタスクを設定できます。AI は設定したスケジュールに従って自動的に実行するため、監視する必要はありません。

**こんなことができます：**

- 30分ごとにテストが通っているか自動チェック
- 毎朝自動的に最新コードをプルしてビルドを実行
- ログファイルを定期的に監視し、異常を検出したら自動通知

PR [#2731](https://github.com/QwenLM/qwen-code/pull/2731) を参照

<video class="mt-5" poster="https://img.alicdn.com/imgextra/i2/6000000002950/O1CN018Sepbk1Xf9Wjb2uR5_!!6000000002950-0-tbvideo.jpg" src="https://cloud.video.taobao.com/vod/VN0xFz-zPV1vvFzIuT6S3HUTFUX36x_JtySMTzAJYpE.mp4" controls></video>

**有効化方法**

`~/.qwen/settings.json` に以下を追加：

```json
{
	"experimental":{
		"cron": true
	}
}
```

### Qwen 3.6 Plus モデルリリース

Qwen 3.6 Plus が Qwen Code で正式に利用可能になりました。無料で使用できます。プログラミングシーンでのパフォーマンスは GPT-5 や Claude 3.7 Sonnet に匹敵し、特に中国語理解とロングコンテキスト処理で顕著な優位性があります。Alibaba Cloud ModelStudio Coding Plan も利用可能です。

**こんなことができます：**

- Qwen Code で最新の Qwen 3.6 Plus モデルを直接使用
- 無料で1日1,000回の呼び出し、100万トークンのコンテキスト
- 中国語プログラミングシーンでのより良い体験

### フォローアップ提案：AI がタスク完了後に次のステップを教えてくれる

タスク完了後、AI が自動的に2〜3個のフォローアップ提案を提示します。「次に何をすればいいか」を自分で考える必要はなく、クリックするだけで続行できます。

**こんなことができます：**

- コンポーネントを書いた後、AI が「ユニットテストを追加しますか？」と提案 — クリックするだけで開始
- バグを修正した後、AI が「似たような箇所をチェックしますか？」と提案 — 考える手間を省略
- 初心者が次に何をすべかわからない時、フォローアップ提案がナビゲーションの役割を果たす

PR [#2525](https://github.com/QwenLM/qwen-code/pull/2525) を参照

![](https://gw.alicdn.com/imgextra/i3/O1CN01bduOSo1HBoIgKUkbq_!!6000000000720-2-tps-2208-1202.png)

**有効化方法**

`~/.qwen/settings.json` に以下を追加：

```json
{
	"ui":{
		"enableFollowupSuggestions": true
	}
}
```

より速い提案レスポンスのために、`/model --fast` で軽量モデルをバックグラウンド提案専用に指定できます。

### サブエージェントのクロスモデル選択：タスクごとに異なるモデルを使用

サブエージェントがメインエージェントと異なるモデルを選択できるようになりました。メインタスクには強力なモデルで品質を確保し、サブタスクには軽量モデルで高速化 — 柔軟な組み合わせでトークンを節約しつつ品質を維持。

**こんなことができます：**

- メインタスクに Qwen 3.6 Plus、サブタスクに軽量モデルを使用し、重要なタスクの品質を維持しながらトークン消費を削減
- 異なるタイプのサブタスクに最適なモデルを自動的に割り当て

PR [#2698](https://github.com/QwenLM/qwen-code/pull/2698) を参照

<video class="mt-5" poster="https://img.alicdn.com/imgextra/i4/6000000000621/O1CN01uFmNW81GST5dMLGJA_!!6000000000621-0-tbvideo.jpg" src="https://cloud.video.taobao.com/vod/gWur3ruz89gz8-LdUper69IQkHXcQNSgFs1HgKpllNs.mp4" controls></video>

**使い方**

異なるモデルを指定するサブエージェント Skill ファイルを作成：

```markdown
---
name: test-cross-provider
description: クロスプロバイダーの Agent テスト
model: openai:gpt-4o
---

あなたはテストアシスタントです。使用しているモデル名を含めて、簡単に自己紹介してください。
```

その後、会話で Qwen Code にこのサブエージェントを呼び出すよう指示します。

### Ctrl+O 詳細/簡潔モード切り替え：ワンキーで出力の詳細度を変更

Ctrl+O を押すと verbose（詳細）モードと compact（簡潔）モードを切り替えられます。デバッグ時は詳細出力を確認し、日常使用では簡潔版に切り替え — 設定変更不要。

**こんなことができます：**

- デバッグ時に verbose モードに切り替えて、完全な実行プロセスを確認
- 日常使用では compact モードに切り替えて、すっきりしたインターフェースに
- いつでも切り替え可能、再起動や設定ファイルの編集は不要

PR [#2770](https://github.com/QwenLM/qwen-code/pull/2770) を参照

### /plan コマンド：行動する前に計画を立てる

新しい `/plan` コマンドで、ワンコマンドで計画モードに入れます。AI がまず完全な実行計画を提示し、確認後に実行を開始します。複雑なタスクに最適 — まず全体像を把握してから、段階的に進めましょう。

**こんなことができます：**

- 大規模モジュールのリファクタリング前に、AI に変更が必要なすべてのファイルとステップをリストアップさせる
- マルチステップタスクではまず計画を確認し、AI が脱線してやり直しになるのを防止
- チーム協業時に、まず計画を生成して同僚に確認してもらってから実行

PR [#2921](https://github.com/QwenLM/qwen-code/pull/2921) を参照

![](https://gw.alicdn.com/imgextra/i4/O1CN01wm82SI1tw1fL4l7DE_!!6000000005965-2-tps-2840-1310.png)

### 適応型出力トークン：レスポンスが途中で切れなくなる

出力トークンはデフォルト 8K で、AI がレスポンスの切り捨てを検出すると自動的に 64K にアップグレードしてリトライします。パラメータを手動で調整する必要はなく、AI が必要なレスポンス空間を自動判断します。

**こんなことができます：**

- 長いファイル生成が途中で切れなくなり、完全なコンテンツを一度に出力
- 複雑なコード生成でセグメントを繋ぎ合わせる必要なし
- max_tokens を手動設定する必要なし — AI が自動適応

PR [#2898](https://github.com/QwenLM/qwen-code/pull/2898) を参照

### Thinking Block のターン間保持

AI の思考プロセス（thinking block）が会話のターンをまたいで保持されるようになり、アイドル時に自動クリーンアップされます。マルチターン会話で AI が以前の推論チェーンをより良く継続でき、より一貫性のあるレスポンスを提供します。

PR [#2897](https://github.com/QwenLM/qwen-code/pull/2897) を参照
