こんな状況に陥ったことはありませんか——

半年以上かけて、納得のいくオープンソースプロジェクトを深夜まで作り上げました。GitHub のコードはきれいに整理され、README も何版も書きました。しかし、ソーシャルメディアで宣伝する段階になると、**何かが足りない**と感じてしまいます。

静止画のスクリーンショット？味気なさすぎる。テキストでの紹介？誰も最後まで読む耐心がない。デモビデオを作る？剪映（CapCut）を開いては閉じ、結局「まあいいか、今度こそは」とため息をつく。

私が知っているある大学生の開発者の友人は、昨年かなり面白いフロントエンドコンポーネントライブラリを作りましたが、スター数は 20〜30 個で頭打ちになりました。彼は私に愚痴りました。「自分のプロジェクトが劣らないことは知っているけど、どうやって他の人に劣らないことを知ってもらえばいいのかわからない」と。

この言葉は痛烈ですが、確かに多くの開発者の現実の处境です。

## ひと言で、ビデオが完成

最近 Qwen Code を使っていて、とても面白いスキルを見つけました——**[oss-styles](https://github.com/heimanba/oss-video-skill)**。

簡単に言うと、Qwen Code にひと言伝えるだけです：

```
このスキル https://github.com/QwenLM/qwen-code-examples/blob/main/skills/oss-styles/SKILL.md に基づいて
リポジトリ https://github.com/QwenLM/qwen-code のデモビデオを生成してください
```

すると、自動的に GitHub からプロジェクト情報を取得し、技術スタック、統計データ、コントリビューター情報を分析し、最後に**映画のようなプロモーションビデオ**を生成します。

このプロセス全体で、あなたは以下をする必要はありません：

- ❌ 剪映や Premiere を開く
- ❌ スクリプトを書いたり、絵コンテを考えたりする
- ❌ 素材を探したり、音楽を選んだりする
- ❌ アニメーションを調整したり、タイムラインを合わせたりする

AI がこれらすべてをしてくれます。

## 完成ビデオはどうなる？

Qwen Code プロジェクトで試してみたところ、生成されたビデオは約 30〜40 秒で、テンポが良く、ビジュアルが統一されています。Twitter、小紅書（Xiaohongshu）、即刻（Jike）に直接投稿したり、プロジェクトの GitHub README の冒頭に配置したりできます。

## 誰が使えるのか？

考えてみたところ、このスキルは少なくとも以下の側面で特に実用的です：

### 1. 個人開発者のコールドスタート

サイドプロジェクトを立ち上げたばかりで、Product Hunt や V2EX で宣伝したい場合。プロフェッショナルな製品ビデオは、乾いたテキストと画像の紹介よりも記憶に残りやすいです。

### 2. 大学生のオープンソース課題/卒業制作

多くのコンピュータサイエンス専攻の学生には、オープンソースプロジェクトの課題があります。以前は GitHub リンクを提出するだけでしたが、今はビデオを添付して、発表や答辩で再生できます。先生や他の学生の印象点が大幅に上がります（ps、本当に非常に役立ちます）。

### 3. テックブロガーのコンテンツ素材

テックブログを書いたり、テックビデオを作成したりして、特定のオープンソースプロジェクトを紹介したい場合、oss-video を直接使用してプロジェクト紹介クリップを生成し、自分のコンテンツに挿入できます。画面録画や編集の手間が省けます。

### 4. 企業のオープンソースプロジェクトの対外宣伝

多くの企業が内部ツールをオープンソース化しますが、運用が追いつかないことがよくあります。このスキルを使えば、開発チーム自身が迅速に宣伝資料を作成でき、デザイナーのスケジュール待ちをする必要がありません。

### 5. ハッカソン/技術コンテストのプロジェクト展示

時間は緊迫し、タスクは重いですが、デモビデオは不可欠です。ひと言でビデオを生成し、浮いた時間をコーディングやデモの調整に充てられます。

## どうやって使うのか？

すでに Qwen Code をお持ちの場合、このスキルのインストールは簡単です：

```bash
# プロジェクトディレクトリに移動
cd your-project

# Qwen Code を起動
qwen

# その後、Qwen Code に直接伝える
このスキル https://github.com/QwenLM/qwen-code-examples/blob/main/skills/oss-styles/SKILL.md に基づいて
リポジトリ <あなたのリポジトリ URL> のデモビデオを生成してください"
```

AI は自動的に：

1. GitHub API からプロジェクト情報を取得
2. プロジェクトの公式サイトを訪問し、ロゴとブランドカラーを取得
3. プロジェクトの位置づけと技術スタックを分析
4. 5 つのシーンのビデオクリップを生成
5. 最終ビデオを合成

スタイルを調整したい場合——例えば配色を変更する、文案を修正する、アニメーションのテンポを調整するなど——AI に直接伝えることもできます。すべてのコードはオープンソースであり、生成された Remotion プロジェクトは自由に修正できます。

<Callout>
**オープンソースの力**：すべてのコードはオープンソースであり、生成された Remotion プロジェクトを自由に修正・拡張して、自分だけのビデオスタイルを作り上げることができます。
</Callout>

## ささやかな感慨

正直に言うと、このスキルを見たとき、私の最初の反応はこうでした：**オープンソースプロジェクトの「表紙」の敷居が、ついに下がった**と。

以前は、大企業のオープンソースプロジェクトだけが、精巧なプロモーションビデオを作る予算を持っていました。個人開発者や学生プロジェクトは、コードそのもので語るしかありませんでした。しかし、コードは自ら語ることはできません。特にこの情報過多の時代においては。

今や、ひと言でプロフェッショナルレベルのプロモーションビデオを生成できます。これは何を意味するのでしょうか？

それは、**良いプロジェクトが見られやすくなる**ことを意味します。

それは、**技術力がよりよく伝わる**ことを意味します。

それは、**オープンソースコミュニティがより活気で、より多様になる**ことを意味します。

私はこれで良いと思います。
