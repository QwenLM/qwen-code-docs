# Web Fetch ツール (`web_fetch`)

このドキュメントでは、Qwen Code 用の `web_fetch` ツールについて説明します。

## 概要

`web_fetch` を使用すると、指定された URL からコンテンツを取得し、AI モデルで処理できます。このツールは URL とプロンプトを入力として受け取り、URL のコンテンツを取得して HTML を Markdown に変換した後、小型かつ高速なモデルを用いてプロンプトに基づいてコンテンツを処理します。

### 引数

`web_fetch` は以下の 2 つの引数を取ります：

- `url`（文字列、必須）：コンテンツを取得する URL。`http://` または `https://` で始まる、完全な有効な URL である必要があります。
- `prompt`（文字列、必須）：ページのコンテンツから抽出したい情報を記述するプロンプト。

## `web_fetch` を Qwen Code で使用する方法

Qwen Code で `web_fetch` を使用するには、URL と、その URL から抽出したい内容を記述したプロンプトを指定します。このツールは、URL の取得を行う前に確認を要求します。確認後、ツールはコンテンツを直接取得し、AI モデルで処理します。

このツールは、HTML を自動的にテキストに変換し、GitHub の blob URL（raw URL に変換）を処理し、セキュリティのため HTTP URL を HTTPS にアップグレードします。

使用方法：

```
web_fetch(url="https://example.com", prompt="この記事の要点を要約してください")
```

## `web_fetch` の使用例

単一の記事を要約する：

```
web_fetch(url="https://example.com/news/latest", prompt="この記事の主な要点を要約できますか？")
```

特定の情報を抽出する：

```
web_fetch(url="https://arxiv.org/abs/2401.0001", prompt="この論文で述べられている主要な知見と手法は何ですか？")
```

GitHub のドキュメントを分析する：

```
web_fetch(url="https://github.com/QwenLM/Qwen/blob/main/README.md", prompt="インストール手順と主な機能は何ですか？")
```

## 重要な注意点

- **単一 URL の処理:** `web_fetch` は一度に 1 つの URL を処理します。複数の URL を分析するには、ツールを個別に呼び出してください。
- **URL 形式:** このツールは HTTP URL を自動的に HTTPS にアップグレードし、GitHub blob URL を生データ（raw）形式に変換して、より良いコンテンツへのアクセスを実現します。
- **コンテンツ処理:** このツールはコンテンツを直接取得し、AI モデルで処理して HTML を読みやすいテキスト形式に変換します。
- **出力品質:** 出力の品質は、プロンプト内の指示の明確さに依存します。
- **MCP ツール:** MCP が提供する Web 取得ツール（名前が `"mcp__"` で始まるもの）が利用可能な場合は、制限が少ない可能性があるため、そちらを優先して使用してください。
