# Qwen Code のツール

Qwen Code には、モデルがローカル環境と対話し、情報にアクセスし、アクションを実行するための組み込みツールが含まれています。これらのツールは CLI の機能を拡張し、テキスト生成にとどまらず、幅広いタスクをサポートします。

## Qwen Code ツールの概要

Qwen Code においてツールとは、モデルが実行を要求できる特定の関数またはモジュールです。たとえば「`my_document.txt` の内容を要約して」と依頼した場合、モデルはそのファイルを読む必要があると判断し、`read_file` ツールの実行を要求します。

コアコンポーネント（`packages/core`）がこれらのツールを管理し、ツールの定義（スキーマ）をモデルに提示し、要求に応じて実行し、結果をモデルに返してユーザー向けのレスポンスを生成します。

ツールが提供する主な機能は次のとおりです。

- **ローカル情報へのアクセス:** ツールを使用することで、モデルはローカルファイルシステムにアクセスし、ファイルの内容を読み取ったり、ディレクトリを一覧表示したりできます。
- **コマンドの実行:** `run_shell_command` などのツールを使用して、モデルはシェルコマンドを実行できます（適切な安全対策とユーザー確認を経て）。
- **Web との連携:** ツールは URL からコンテンツを取得できます。
- **アクションの実行:** ツールはファイルの変更、新規ファイルの作成、その他のシステム操作を実行できます（通常はセーフガードが設けられています）。
- **回答の根拠付け:** ツールを使用してリアルタイムまたは特定のローカルデータを取得することで、実際のコンテキストに基づいた、より正確で適切な回答が得られます。

## Qwen Code ツールの使い方

Qwen Code のツールを使用するには、CLI にプロンプトを入力します。処理の流れは次のとおりです。

1.  CLI にプロンプトを入力します。
2.  CLI はプロンプトをコアに送信します。
3.  コアは、プロンプトと会話履歴とともに、利用可能なツールの一覧とその説明・スキーマを設定済みのモデル API に送信します。
4.  モデルはリクエストを分析します。ツールが必要と判断した場合、レスポンスには特定のパラメータを持つツールの実行要求が含まれます。
5.  コアはそのツール要求を受け取り、検証し、（機密性の高い操作の場合はユーザー確認を経て）ツールを実行します。
6.  ツールの出力がモデルに返されます。
7.  モデルはツールの出力を使用して最終的な回答を生成し、コアを経由して CLI に返され、画面に表示されます。

CLI では、ツールが呼び出されているかどうか、また成功したか失敗したかを示すメッセージが表示されます。

## セキュリティと確認

ファイルシステムの変更やコマンド実行が可能なツール（`write_file`、`edit`、`run_shell_command` など）は、特に安全性を考慮して設計されています。Qwen Code は通常、次の措置を取ります。

- **確認の要求:** 機密性の高い操作を実行する前に、実行しようとしているアクションを表示してユーザーに確認を求めます。
- **サンドボックスの活用:** すべてのツールはサンドボックスによって制限されます（[Qwen Code のサンドボックス](./sandbox.md) を参照）。サンドボックス環境で動作する場合、使用したいツール（MCP サーバーを含む）はサンドボックス環境の _内部_ で利用可能である必要があります。たとえば `npx` で MCP サーバーを実行するには、`npx` 実行ファイルがサンドボックスの Docker イメージ内にインストールされているか、`sandbox-exec` 環境で利用可能である必要があります。

ツールの実行を許可する前に、確認プロンプトを必ず注意深く確認してください。

## Qwen Code のツールについてさらに詳しく

Qwen Code の組み込みツールは、大きく以下のカテゴリに分類されます。

- **[ファイルシステムツール](./file-system.md):** ファイルやディレクトリの操作（読み取り、書き込み、一覧表示、検索など）に使用します。
- **[シェルツール](./shell.md)（`run_shell_command`）:** シェルコマンドの実行に使用します。
- **[モニターツール](./monitor.md)（`monitor`）:** 長時間実行されるシェルコマンドを実行し、出力をバックグラウンドタスクの通知としてストリームします。
- **[Web フェッチツール](./web-fetch.md)（`web_fetch`）:** URL からコンテンツを取得します。
- **[Todo 書き込みツール](./todo-write.md)（`todo_write`）:** コーディングセッション中に構造化されたタスクリストを作成・管理します。
- **[エージェントツール](./task.md)（`agent`）:** 複雑なタスクを専門のサブエージェントに委譲します。
- **[プランモード終了ツール](./exit-plan-mode.md)（`exit_plan_mode`）:** プランモードを終了して実装に移行します。

さらに、以下の機能も組み込まれています。

- **[MCP サーバー](./mcp-server.md)**: MCP サーバーは、モデルとローカル環境または API などの外部サービスをつなぐブリッジとして機能します。
  - **[MCP ユーザーガイド](../../users/features/mcp.md)**: Qwen Code から MCP サーバーを設定・管理します
  - **[MCP 経由の Web 検索](./web-search.md)**: MCP を通じて Web 検索サービス（Bailian、Tavily、GLM）に接続します
- **[サンドボックス](./sandbox.md)**: サンドボックスはモデルとその変更をあなたの環境から分離し、潜在的なリスクを軽減します。
