# 複数ファイルの読み取り (`read_many_files`)

> [!note]
>
> `read_many_files` はかつてスタンドアロンのツールとして公開されていましたが、内部ユーティリティ関数にリファクタリングされました。モデルが直接呼び出すことはなくなり、代わりに `read_file`、`glob`、`grep_search` ツールが個別および複数ファイルの読み取りをカバーしています。以下の情報は参照用として残しています。

## 概要

`read_many_files` は、パスまたはグロブパターンで指定された複数のファイルからコンテンツを読み取ります。ファイルの種類によって動作が異なります。

- テキストファイルの場合、このツールはそれらのコンテンツを単一の文字列に結合します。
- 画像（PNG、JPEG など）、PDF、音声（MP3、WAV）、動画（MP4、MOV）ファイルの場合、名前または拡張子で明示的に指定されたものに限り、base64 エンコードデータとして読み取り返します。

`read_many_files` は、コードベースの概要把握、特定機能の実装箇所の特定、ドキュメントのレビュー、複数の設定ファイルからのコンテキスト収集といったタスクに活用できます。

**注意:** `read_many_files` は指定されたパスまたはグロブパターンに従ってファイルを検索します。`"/docs"` のようなディレクトリパスは空の結果を返します。対象ファイルを特定するには、`"/docs/*"` や `"/docs/*.md"` のようなパターンが必要です。

### 引数

`read_many_files` は以下の引数を取ります。

- `paths` (list[string]、必須): ツールのターゲットディレクトリからの相対パスまたはグロブパターンの配列（例: `["src/**/*.ts"]`、`["README.md", "docs/*", "assets/logo.png"]`）。
- `exclude` (list[string]、任意): 除外するファイル/ディレクトリのグロブパターン（例: `["**/*.log", "temp/"]`）。`useDefaultExcludes` が true の場合、デフォルトの除外パターンに追加されます。
- `include` (list[string]、任意): 追加で含めるグロブパターン。`paths` とマージされます（例: `["*.test.ts"]` で広範に除外されたテストファイルを追加する、または `["images/*.jpg"]` で特定の画像タイプを含める）。
- `recursive` (boolean、任意): 再帰的に検索するかどうか。主にグロブパターン内の `**` によって制御されます。デフォルトは `true`。
- `useDefaultExcludes` (boolean、任意): デフォルトの除外パターン（例: `node_modules`、`.git`、画像/PDF 以外のバイナリファイル）を適用するかどうか。デフォルトは `true`。
- `respect_git_ignore` (boolean、任意): ファイル検索時に .gitignore パターンを尊重するかどうか。デフォルトは true。

## Qwen Code で `read_many_files` を使う方法

`read_many_files` は、`exclude` パターンとデフォルトの除外設定（有効な場合）を尊重しながら、指定された `paths` および `include` パターンに一致するファイルを検索します。

- テキストファイルの場合: 一致した各ファイルのコンテンツを読み取り（画像/PDF として明示的に指定されていないバイナリファイルはスキップを試みます）、各ファイルのコンテンツの間に `--- {filePath} ---` というセパレータを挿入しながら単一の文字列に結合します。デフォルトエンコーディングは UTF-8 です。
- 最後のファイルの後に `--- End of content ---` が挿入されます。
- 画像および PDF ファイルの場合: 名前または拡張子で明示的に指定された場合（例: `paths: ["logo.png"]` または `include: ["*.pdf"]`）、ファイルを読み取り、その内容を base64 エンコード文字列として返します。
- その他のバイナリファイル（一般的な画像/PDF タイプに一致せず、明示的に指定もされていないもの）は、初期コンテンツ内のヌルバイトを確認することで検出しスキップを試みます。

使用方法:

```
read_many_files(paths=["Your files or paths here."], include=["Additional files to include."], exclude=["Files to exclude."], recursive=False, useDefaultExcludes=false, respect_git_ignore=true)
```

## `read_many_files` の使用例

`src` ディレクトリ内のすべての TypeScript ファイルを読み取る:

```
read_many_files(paths=["src/**/*.ts"])
```

メインの README、`docs` ディレクトリ内のすべての Markdown ファイル、特定のロゴ画像を読み取り、特定のファイルを除外する:

```
read_many_files(paths=["README.md", "docs/**/*.md", "assets/logo.png"], exclude=["docs/OLD_README.md"])
```

すべての JavaScript ファイルを読み取りながら、テストファイルと `images` フォルダ内のすべての JPEG を明示的に含める:

```
read_many_files(paths=["**/*.js"], include=["**/*.test.js", "images/**/*.jpg"], useDefaultExcludes=False)
```

## 重要な注意事項

- **バイナリファイルの扱い:**
  - **画像/PDF/音声/動画ファイル:** このツールは一般的な画像タイプ（PNG、JPEG など）、PDF、音声（mp3、wav）、動画（mp4、mov）ファイルを読み取り、base64 エンコードデータとして返すことができます。これらのファイルは `paths` または `include` パターンで明示的に指定する必要があります（例: `video.mp4` のような正確なファイル名、または `*.mov` のようなパターン）。
  - **その他のバイナリファイル:** ツールは初期コンテンツ内のヌルバイトを検査することで、その他のバイナリファイルを検出してスキップを試みます。これらのファイルは出力から除外されます。
- **パフォーマンス:** 非常に多数のファイルや非常に大きな個別ファイルを読み取ると、リソースを大量に消費する可能性があります。
- **パスの指定:** パスとグロブパターンがツールのターゲットディレクトリからの相対パスとして正しく指定されていることを確認してください。画像/PDF ファイルの場合は、パターンがそれらを含めるのに十分に具体的であることを確認してください。
- **デフォルト除外:** デフォルトの除外パターン（`node_modules`、`.git` など）に注意し、オーバーライドが必要な場合は `useDefaultExcludes=False` を使用してください。ただし、慎重に行ってください。
