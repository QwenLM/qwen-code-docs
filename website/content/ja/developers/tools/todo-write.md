# Todo 書き込みツール (`todo_write`)

このドキュメントでは、Qwen Code 向けの `todo_write` ツールについて説明します。

## 概要

`todo_write` を使用して、現在のコーディングセッション向けに構造化されたタスクリストを作成および管理します。このツールは、AI アシスタントが進捗を追跡し、複雑なタスクを整理するのを支援し、実行中の作業内容を可視化します。

### 引数

`todo_write` は 1 つの引数を取ります：

- `todos`（配列、必須）：Todo 項目からなる配列。各項目には以下のプロパティが含まれます：
  - `content`（文字列、必須）：タスクの説明。
  - `status`（文字列、必須）：現在のステータス（`pending`、`in_progress`、または `completed`）。
  - `activeForm`（文字列、必須）：現在進行形で記述された実行中の作業内容（例：「テストを実行中」、「プロジェクトをビルド中」）。

## `todo_write` を Qwen Code で使用する方法

AI アシスタントは、複雑で複数ステップからなるタスクを処理する際に、このツールを自動的に使用します。明示的に要求する必要はありませんが、リクエストに対する計画的なアプローチを確認したい場合は、アシスタントに ToDo リストの作成を依頼できます。

このツールは ToDo リストをホームディレクトリ（`~/.qwen/todos/`）内にセッション固有のファイルとして保存するため、各コーディングセッションは独自のタスクリストを保持します。

## AI がこのツールを使用するタイミング

アシスタントは以下のケースで `todo_write` を使用します：

- 複数のステップを要する複雑なタスク
- 複数のコンポーネントを含む機能実装
- 複数のファイルにまたがるリファクタリング操作
- 3 つ以上の明確に区別されるアクションを伴う作業

一方、単純な単一ステップのタスクや、純粋に情報提供を目的としたリクエストに対しては、このツールは使用されません。

### `todo_write` の使用例

機能実装計画の作成：

```
todo_write(todos=[
  {
    "content": "ユーザー設定モデルを作成する",
    "status": "pending",
    "activeForm": "ユーザー設定モデルを作成しています"
  },
  {
    "content": "設定用の API エンドポイントを追加する",
    "status": "pending",
    "activeForm": "設定用の API エンドポイントを追加しています"
  },
  {
    "content": "フロントエンドコンポーネントを実装する",
    "status": "pending",
    "activeForm": "フロントエンドコンポーネントを実装しています"
  }
])
```

## 重要な注意点

- **自動利用:** AI アシスタントは、複雑なタスク実行中に todo リストを自動的に管理します。
- **進捗の可視化:** 作業が進むにつれて、todo リストがリアルタイムで更新されます。
- **セッションの分離:** 各コーディングセッションには独自の todo リストがあり、他のセッションと干渉しません。
