# Qwen Code の実行とデプロイ

このドキュメントでは、Qwen Code の実行方法と、Qwen Code が採用するデプロイアーキテクチャについて説明します。

## Qwen Code の実行

Qwen Code を実行する方法はいくつかあります。使用目的に応じて、適切な方法を選択してください。

---

### 1. 標準インストール（一般ユーザー向け推奨）

エンドユーザーが Qwen Code をインストールする場合の推奨方法です。NPM レジストリから Qwen Code パッケージをダウンロードします。

- **グローバルインストール:**

  ```bash
  npm install -g @qwen-code/qwen-code
  ```

  その後、任意のディレクトリから CLI を実行できます:

  ```bash
  qwen
  ```

- **NPX による実行:**

  ```bash
  # グローバルインストールなしで、NPM 上の最新版を直接実行
  npx @qwen-code/qwen-code
  ```

### 2. サンドボックス（Docker/Podman）内での実行

セキュリティおよび隔離を確保するため、Qwen Code をコンテナ内で実行できます。これは、副作用を引き起こす可能性のあるツールを CLI が実行する際のデフォルト方法です。

- **レジストリからの直接実行：**  
  公開されているサンドボックスイメージを直接実行できます。これは、Docker のみが利用可能で CLI を実行したい環境に適しています。
  ```bash
  # 公開されているサンドボックスイメージを実行
  docker run --rm -it ghcr.io/qwenlm/qwen-code:0.0.11
  ```
- **`--sandbox` フラグを使用する方法：**  
  Qwen Code をローカルにインストール済み（上記の標準的なインストール手順による）の場合、CLI にサンドボックスコンテナ内で実行するよう指示できます。
  ```bash
  qwen --sandbox -y -p "ここにプロンプトを入力"
  ```

### 3. ソースコードから実行する（Qwen Code のコントリビューターに推奨）

このプロジェクトのコントリビューターは、CLI をソースコードから直接実行したいと考えるでしょう。

- **開発モード：**  
  この方法ではホットリロードが有効になり、積極的な開発に便利です。  
  ```bash
  # リポジトリのルートディレクトリから実行
  npm run start
  ```
- **本番環境に近いモード（リンク済みパッケージ）：**  
  この方法では、ローカルのパッケージをグローバルな `node_modules` にリンクすることで、グローバルインストールを模倣します。これは、本番ワークフロー内でローカルビルドをテストする際に便利です。

  ```bash
  # ローカルの cli パッケージをグローバルな node_modules にリンク
  npm link packages/cli

  # これで `qwen` コマンドを使ってローカル版を実行できます
  qwen
  ```

---

### 4. GitHub から最新の Qwen Code コミットを実行する

GitHub リポジトリから、最新のコミット済みバージョンの Qwen Code を直接実行できます。これは、まだ開発中の機能をテストする際に便利です。

```bash

# GitHub の main ブランチから CLI を直接実行する
npx https://github.com/QwenLM/qwen-code

## デプロイアーキテクチャ

上記で説明した実行方法は、以下のアーキテクチャコンポーネントおよびプロセスによって実現されています。

**NPM パッケージ**

Qwen Code プロジェクトはモノレポであり、コアパッケージを NPM レジストリに公開しています。

- `@qwen-code/qwen-code-core`: バックエンド。ロジック処理およびツール実行を担当します。
- `@qwen-code/qwen-code`: ユーザー向けのフロントエンドです。

これらのパッケージは、標準的なインストール手順を実行する場合や、ソースコードから Qwen Code を直接実行する場合に使用されます。

**ビルドおよびパッケージングプロセス**

配布チャネルに応じて、2 種類の異なるビルドプロセスが使用されます。

- **NPM への公開:** NPM レジストリへ公開する際には、`@qwen-code/qwen-code-core` および `@qwen-code/qwen-code` の TypeScript ソースコードを、TypeScript コンパイラー (`tsc`) を用いて標準 JavaScript にトランスパイルします。生成された `dist/` ディレクトリの内容が、NPM パッケージとして公開されます。これは、TypeScript ライブラリにおいて標準的な手法です。

- **GitHub 経由での `npx` 実行:** GitHub から最新版の Qwen Code を直接実行する場合、`package.json` 内の `prepare` スクリプトによって異なるプロセスが起動します。このスクリプトでは `esbuild` を使用し、アプリケーション全体およびその依存関係を単一の自己完結型 JavaScript ファイルにバンドルします。このバンドルはユーザーのマシン上でオンザフライで生成され、リポジトリにはコミットされません。

**Docker サンドボックスイメージ**

Docker を使用した実行方法は、`qwen-code-sandbox` コンテナイメージによってサポートされています。このイメージはコンテナレジストリに公開されており、Qwen Code のグローバルインストール済みバージョンを含んでいます。

## リリースプロセス

リリースプロセスは GitHub Actions を使用して自動化されています。リリースワークフローでは、以下の操作が実行されます。

1.  `tsc` を使用して NPM パッケージをビルドします。
2.  NPM パッケージをアーティファクトレジストリに公開します。
3.  バンドル済みのアセットを含む GitHub リリースを作成します。
