## 概要

コミット前に誰かにコードを見てもらいたいけど、同僚は忙しくて手が空かない？`/review` を使うだけ — AIがコード品質をチェックし、潜在的な問題を見つけ、改善を提案します。単純なlintチェックではなく、経験豊富な同僚のように、ロジック、命名、エッジケースの処理を徹底的にレビューします。

## これでできること

- **コミット前レビュー**：AIにコードを通してもらい、問題を早期発見 — Code Reviewでの驚きはもうありません
- **ソロ開発の第二の目**：低レベルなエラーを減らし、コード品質を向上
- **ベストプラクティスを学ぶ**：AIは「ここに問題がある」だけでなく、なぜそのアプローチが良いのかを説明します

## 手順

<Steps>

### レビューするファイルを開く

Qwen Codeでレビューしたいコードファイルまたはプロジェクトディレクトリを開きます。コードが保存されていることを確認してください。

### /reviewコマンドを入力

チャットで `/review` と入力します。Qwen Codeが自動的に現在のファイルまたは最近のコード変更を分析します。

```bash
/review
```

### レビュー結果を確認

AIがコード品質、ロジックの正確性、命名規約、エッジケースの処理など、複数の観点からレビューフィードバックを提供し、具体的な改善提案を添えます。

</Steps>

<Callout type="info">
`/review` は単純なlintチェックとは異なります — コードロジックを深く理解し、潜在的なバグ、パフォーマンスの問題、設計上の欠陥を特定します。
</Callout>
