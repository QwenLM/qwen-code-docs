# Terminology

This document defines key terminology used in the project and provides translation guidelines to maintain documentation consistency across languages.

## Critical Rules

### Markdown Tags - DO NOT TRANSLATE

**Never translate** Markdown alert tags. Always use English:

| Correct ✅       | Incorrect ❌ |
| ---------------- | ------------ |
| `> [!note]`      | `> [!注意]`  |
| `> [!tip]`       | `> [!提示]`  |
| `> [!important]` | `> [!重要]`  |
| `> [!warning]`   | `> [!警告]`  |
| `> [!caution]`   | `> [!谨慎]`  |

## Translation Style Guide

- **Code-related terms**: Keep in English, e.g., `skill`, `track`, `todo`
- **Technical acronyms**: Preserve as-is, e.g., CLI, API, MCP, LLM
- **Verbs**: Use concise verbs, e.g., "run" instead of "execute and run"
- **Sentence structure**: Use active voice, avoid verbose expressions
- **Markdown syntax elements**: Keep alert tags, code block language identifiers in English
