Аутентификация

Qwen Code поддерживает два метода аутентификации. Выберите тот, который соответствует способу запуска CLI:

  • Qwen OAuth (рекомендуется): войдите в свою учетную запись qwen.ai в браузере.
  • Совместимый с OpenAI API: используйте ключ API (OpenAI или любого совместимого с OpenAI поставщика / конечной точки).

Вариант 1: Qwen OAuth (рекомендуется и бесплатно) 👍

Используйте этот вариант, если вы хотите получить самую простую настройку и используете модели Qwen.

  • Как это работает: при первом запуске Qwen Code открывает страницу входа в браузере. После завершения входа учетные данные кэшируются локально, поэтому вам обычно больше не придется входить повторно.
  • Требования: аккаунт qwen.ai + доступ к интернету (по крайней мере для первого входа).
  • Преимущества: нет необходимости управлять API-ключами, автоматическое обновление учетных данных.
  • Стоимость и квоты: бесплатно, с квотой 60 запросов в минуту и 1000 запросов в день.

Запустите CLI и следуйте инструкциям в браузере:

qwen

Вариант 2: Совместимый с OpenAI API (API-ключ)

Используйте этот вариант, если вы хотите использовать модели OpenAI или любого поставщика, который предоставляет совместимый с OpenAI API (например, OpenAI, Azure OpenAI, OpenRouter, ModelScope, Alibaba Cloud Bailian или собственный совместимый эндпоинт).

Рекомендуется: План кодирования (на основе подписки) 🚀

Используйте это, если вы хотите предсказуемые расходы с более высокими квотами использования для модели qwen3-coder-plus.

Important

План кодирования доступен только для пользователей в материковом Китае (регион Пекин).

  • Принцип работы: подпишитесь на План кодирования с фиксированной ежемесячной платой, затем настройте Qwen Code на использование специальной конечной точки и вашего API-ключа подписки.
  • Требования: активная подписка на План кодирования от Alibaba Cloud Bailian .
  • Преимущества: более высокие квоты использования, предсказуемые ежемесячные расходы, доступ к последней модели qwen3-coder-plus.
  • Стоимость и квоты: зависит от плана (см. таблицу ниже).

Быстрая настройка для тарифного плана Coding Plan

При выборе опции совместимости с OpenAI в CLI введите следующие значения:

  • Ключ API: sk-sp-xxxxx
  • Базовый URL: https://coding.dashscope.aliyuncs.com/v1
  • Модель: qwen3-coder-plus

Примечание: Ключи API для тарифного плана Coding Plan имеют формат sk-sp-xxxxx, который отличается от стандартных ключей API Alibaba Cloud.

Настройка через переменные окружения

Установите эти переменные окружения для использования Coding Plan:

export OPENAI_API_KEY="ваш-ключ-api-coding-plan"  # Формат: sk-sp-xxxxx
export OPENAI_BASE_URL="https://coding.dashscope.aliyuncs.com/v1"
export OPENAI_MODEL="qwen3-coder-plus"

Для получения дополнительной информации о Coding Plan, включая варианты подписки и устранение неполадок, см. полную документацию по Coding Plan .

Другие совместимые с OpenAI провайдеры

Если вы используете другие провайдеры (OpenAI, Azure, локальные LLM и т.д.), используйте следующие методы настройки.

Настройка через аргументы командной строки

 
# Только ключ API
qwen-code --openai-api-key "ваш-ключ-api-здесь"
 
# Пользовательский базовый URL (совместимая с OpenAI конечная точка)
qwen-code --openai-api-key "ваш-ключ-api-здесь" --openai-base-url "https://ваша-конечная-точка.com/v1"
 
# Пользовательская модель
qwen-code --openai-api-key "ваш-ключ-api-здесь" --model "gpt-4o-mini"

Настройка через переменные окружения

Вы можете установить их в профиле вашей оболочки, в CI или в файле .env:

export OPENAI_API_KEY="ваш-ключ-api-здесь"
export OPENAI_BASE_URL="https://api.openai.com/v1"  # опционально
export OPENAI_MODEL="gpt-4o"                        # опционально

Сохранение переменных окружения с помощью .env / .qwen/.env

Qwen Code будет автоматически загружать переменные окружения из первого найденного файла .env (переменные не объединяются из нескольких файлов).

Порядок поиска:

  1. Из текущей директории, поднимаясь вверх к /:
    1. .qwen/.env
    2. .env
  2. Если ничего не найдено, используется домашняя директория:
    • ~/.qwen/.env
    • ~/.env

Рекомендуется использовать .qwen/.env, чтобы изолировать переменные Qwen Code от других инструментов. Некоторые переменные (например, DEBUG и DEBUG_MODE) исключаются из проектных файлов .env, чтобы не мешать работе qwen-code.

Примеры:

 
# Проектные настройки (рекомендуется)
mkdir -p .qwen
cat >> .qwen/.env <<'EOF'
OPENAI_API_KEY="ваш-ключ-api"
OPENAI_BASE_URL="https://api-inference.modelscope.cn/v1"
OPENAI_MODEL="Qwen/Qwen3-Coder-480B-A35B-Instruct"
EOF
 
# Пользовательские настройки (доступны везде)
mkdir -p ~/.qwen
cat >> ~/.qwen/.env <<'EOF'
OPENAI_API_KEY="ваш-ключ-api"
OPENAI_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
OPENAI_MODEL="qwen3-coder-plus"
EOF

Переключение метода аутентификации (без перезапуска)

В интерфейсе Qwen Code выполните:

/auth

Автономные / безголовые среды (CI, SSH, контейнеры)

В автономном терминале вы обычно не можете завершить процесс входа через браузер OAuth. Используйте метод API, совместимый с OpenAI, через переменные окружения:

  • Установите хотя бы OPENAI_API_KEY.
  • При необходимости установите OPENAI_BASE_URL и OPENAI_MODEL.

Если ни одна из этих переменных не установлена в автономной сессии, Qwen Code завершит работу с ошибкой.

Примечания по безопасности

  • Не сохраняйте ключи API в системе контроля версий.
  • Предпочтительно использовать .qwen/.env для хранения локальных секретов проекта (и не добавляйте его в git).
  • Считайте вывод терминала конфиденциальным, если он отображает учетные данные для проверки.
