Qwen Code Еженедельник: Channels мультиплатформенный доступ, Cron запланированные задачи, режим /plan, запуск Qwen 3.6 Plus

Qwen Team
2026-04-08

На этой неделе мы выпустили две функциональные версии v0.14.0 и v0.14.2, а также две версии с исправлениями v0.13.2 и v0.14.1.

v0.14.0 — это крупное обновление:

  • Система Channels освобождает Qwen Code от терминала, позволяя удалённо управлять через Telegram, WeChat, DingTalk и другие платформы;
  • Запланированные задачи Cron позволяют ИИ автоматически выполнять повторяющуюся работу по расписанию;
  • Модель Qwen 3.6 Plus официально запущена. v0.14.1 приносит предложения по дальнейшим действиям, переключение режима подробности и другие улучшения.
  • v0.14.2 добавляет команду /plan в режиме планирования, сохранение блоков размышлений между ходами, обновление адаптивных выходных токенов (по умолчанию 8K + автоматическое повышение до 64K), а также встроенный рабочий процесс исправления ошибок и навыки отладки.

Спасибо новым участникам этой недели @chinesepowered, @pic4xiu, @YingchaoX, @euxaristia, @kulikrch, @nsalvacao, @mj4444ru, @chiga0 🎉

✨ Новые возможности

Channels: Управляйте Qwen Code удалённо через DingTalk, Telegram, WeChat

Qwen Code больше не ограничен терминалом. Система Channels поддерживает подключение к нескольким платформам через плагины — в настоящее время поддерживаются Telegram, WeChat и DingTalk. Вы можете отправить сообщение боту с телефона и заставить его выполнять задачи на вашем сервере.

Что вы можете сделать:

  • Находясь вне дома, отправьте сообщение Qwen Code через Telegram, чтобы запустить скрипт или проверить логи
  • Упомяните @бота в группе DingTalk, чтобы он помог команде с задачами, связанными с кодом
  • Отправьте быстрое сообщение в WeChat, чтобы удалённо управлять средой разработки

См. PR #2628 

Запланированные задачи Cron: Пусть ИИ работает по расписанию

Новый инструмент Cron позволяет настраивать запланированные повторяющиеся задачи в текущей сессии. ИИ будет автоматически выполнять их по заданному расписанию без необходимости наблюдения.

Что вы можете сделать:

  • Автоматически проверять прохождение тестов каждые 30 минут
  • Автоматически получать последний код и запускать сборку каждое утро
  • Мониторить файлы логов по расписанию и уведомлять при обнаружении аномалий

См. PR #2731 

Как включить?

Добавьте в ~/.qwen/settings.json:

{
	"experimental":{
		"cron": true
	}
}

Запуск модели Qwen 3.6 Plus

Qwen 3.6 Plus официально доступен в Qwen Code, бесплатно. Его производительность в программировании сопоставима с GPT-5 и Claude 3.7 Sonnet, с заметными преимуществами в понимании китайского языка и обработке длинного контекста. План Coding от Alibaba Cloud ModelStudio также доступен.

Что вы можете сделать:

  • Использовать новейшую модель Qwen 3.6 Plus непосредственно в Qwen Code
  • Бесплатно 1 000 вызовов/день, контекст 1 миллион токенов
  • Лучший опыт для сценариев программирования на китайском языке

См. PR #2820 

Предложения по дальнейшим действиям: ИИ подскажет, что делать дальше

После завершения задачи ИИ автоматически предлагает 2-3 варианта дальнейших действий. Не нужно думать «что делать дальше» — просто нажмите, чтобы продолжить.

Что вы можете сделать:

  • После написания компонента ИИ предлагает «Хотите добавить юнит-тесты?» — просто нажмите, чтобы начать
  • После исправления бага ИИ предлагает «Хотите проверить похожие места?» — экономит ваше время на размышления
  • Когда новички не знают, что делать дальше, предложения по дальнейшим действиям служат навигацией

См. PR #2525 

Как включить?

Добавьте в ~/.qwen/settings.json:

{
	"ui":{
		"enableFollowupSuggestions": true
	}
}

Для более быстрого времени отклика предложений можно использовать /model --fast, чтобы указать лёгкую модель для фоновых предложений.

Кросс-модельный выбор для субагентов: Разные модели для разных задач

Субагенты теперь могут использовать модели, отличные от основного агента. Используйте мощную модель для основной задачи для обеспечения качества, а лёгкую модель для подзадач для ускорения — гибкие комбинации, экономящие токены без потери качества.

Что вы можете сделать:

  • Использовать Qwen 3.6 Plus для основной задачи и лёгкую модель для подзадач, снижая потребление токенов при сохранении качества критических задач
  • Разные типы подзадач автоматически получают наиболее подходящую модель

См. PR #2698 

Как начать?

Создайте файл Skill субагента с указанием другой модели:

---
name: test-cross-provider
description: Тест кросс-провайдерного Agent
model: openai:gpt-4o
---
 
Вы тестовый ассистент. Пожалуйста, кратко представьтесь, включая название используемой модели.

Затем скажите Qwen Code вызвать этого субагента в разговоре.

Переключение подробности Ctrl+O: Измените уровень детализации одной клавишей

Нажмите Ctrl+O для переключения между режимами verbose (подробный) и compact (краткий). Просматривайте подробный вывод при отладке, переключайтесь в компактный режим для повседневного использования — без изменения конфигурации.

Что вы можете сделать:

  • Переключиться в режим verbose при отладке, чтобы видеть полный процесс выполнения
  • Переключиться в режим compact для повседневного использования для более чистого интерфейса
  • Переключаться в любое время без перезапуска или редактирования файлов конфигурации

См. PR #2770 

Команда /plan: Планируйте перед действием

Новая команда /plan позволяет войти в режим планирования одной командой. ИИ сначала предоставит полный план выполнения, и вы подтвердите перед началом выполнения. Идеально для сложных задач — сначала увидьте полную картину, затем продвигайтесь шаг за шагом.

Что вы можете сделать:

  • Перед рефакторингом большого модуля попросите ИИ перечислить все файлы и шаги, которые нужно изменить
  • Сначала просмотрите план для многошаговых задач, чтобы избежать отклонения ИИ и необходимости начинать заново
  • В командной работе сначала сгенерируйте план для подтверждения коллегами, затем выполняйте

См. PR #2921 

Адаптивные выходные токены: Ответы больше не обрезаются

Выходные токены по умолчанию 8K. Когда ИИ обнаруживает, что ответ обрезается, он автоматически повышает до 64K и повторяет попытку. Не нужно вручную настраивать параметры — ИИ сам определяет, сколько пространства для ответа ему нужно.

Что вы можете сделать:

  • Генерация длинных файлов больше не обрезается — полное содержимое за один вывод
  • Генерация сложного кода без необходимости склеивать сегменты
  • Не нужно вручную настраивать max_tokens — ИИ адаптируется автоматически

См. PR #2898 

Сохранение блоков размышлений между ходами

Процесс размышления ИИ (thinking block) теперь может сохраняться между ходами разговора с автоматической очисткой во время простоя. В многоходовых разговорах ИИ может лучше продолжать предыдущую цепочку рассуждений для более связных ответов.

См. PR #2897
